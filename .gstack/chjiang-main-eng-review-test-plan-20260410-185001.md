## Key Tests to Add (scripts/test.sh)

### 1. Version bump integration test
- Scaffold test skill → run skill-version.sh test-skill patch
- Verify: SKILL.md frontmatter version = 0.2.0
- Verify: catalog version = 0.2.0
- Verify: CHANGELOG.md has [0.2.0] entry
- Cleanup: restore original state

### 2. Ship integration test
- Scaffold test skill → fill DESIGN.md → run skill-version.sh → skill-ship.sh
- Verify: git tag exists (test-skill-v0.2.0)
- Verify: commit message matches pattern
- Cleanup: delete tag, reset commit

### 3. Resume scenario test
- Run skill-design.sh → interrupt → re-run /skill-author
- Verify: DESIGN.md exists, scaffold step is skipped

## Critical Paths
- End-to-end: `/skill-author my-new-skill` produces all artifacts + git tag
- Resume: partially-completed skill can be completed on second invocation
- Error: invalid skill name produces user-friendly error, not raw shell trace

## Edge Cases
- Skill name with underscores (should fail with clear message)
- Skill name that collides with git tag namespace
- Running /skill-author when staged changes exist
- Running ship when tag already exists
