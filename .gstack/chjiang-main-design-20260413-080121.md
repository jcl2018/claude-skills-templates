## Problem Statement

F000001_workflow_alpha has 3 user stories and 4 tasks. The work is essentially done: S000001 (4-phase router) and S000002 (template consolidation) are fully implemented, S000003 (structural completeness) shipped via PR #22 and #24, and only T000004 (close-out) remains open. The three-story decomposition was useful during development but now creates maintenance overhead: `/docs check` flags S000001 missing its entire doc triplet, S000002 has 0 tasks (INCOMPLETE), and keeping 3 separate PRD/ARCHITECTURE/TEST-SPEC sets in sync for completed work is pure cost.

## What Makes This Cool

One clean hierarchy instead of a messy three-story tree. The `/docs check` output goes from a wall of DRIFT/MISSING/INCOMPLETE findings to mostly PASS. Less ceremony, same history preserved in git.

## Constraints

- Must satisfy hierarchy rules: feature >= 1 user-story, user-story >= 1 task
- Should preserve the strongest documentation content (S000003's docs are the most detailed)
- Should absorb acceptance criteria from all three stories
- Must work on main branch (no feature branch needed for tracking files)

## Premises

1. The three user stories were useful decomposition during development but are now overhead
2. A single consolidated story can represent all the work that was done
3. One task is sufficient to cover the implementation
4. This is safe to do on main (tracking files only, no code changes)
5. The consolidated items should absorb the strongest content from each existing item

## Approaches Considered

### Approach A: Adopt & Merge (reuse S000001 + T000001)

Rename S000001_four_phase to S000001_workflow_implementation. Copy S000003's doc triplet (PRD, ARCHITECTURE, TEST-SPEC, milestones) as the base. Merge acceptance criteria from all 3 stories into S000001's tracker. Collapse all tasks into T000001_implement_workflow. Delete S000002, S000003 and their children. Update F000001 tracker.

Effort: M. Risk: Low. Preserves original ID lineage.

### Approach B: Clean Slate (fresh IDs)

Create S000004_workflow_implementation with synthesized docs. Create T000005_implement_workflow. Delete all old S/T directories. Write new docs from scratch.

Effort: L. Risk: Low. Cleanest result but more writing.

## Recommended Approach

**Approach A: Adopt & Merge.** S000003's docs are already strong. Rewriting from scratch gains little when the implementation is done. Using `git mv` for doc files preserves rename history.
