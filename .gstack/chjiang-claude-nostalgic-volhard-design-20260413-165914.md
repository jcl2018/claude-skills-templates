## Problem Statement

F000003_company_spec_system was scaffolded before the 3-phase lifecycle and the "features get tracker only" rule existed. It has 3 user stories (S000003, S000004, S000005), each with a task, plus a feature-level doc triplet that violates `artifact-manifests.json`. All 9 trackers use the old 4-phase lifecycle. The content is good but the structure is wrong. This feature is an active work item, so it needs to match the current template system exactly and serve as an exemplary reference for how the system works.

## What Makes This Cool

F000001 is the closed reference. F000003 becomes the active reference. Together they show the full lifecycle: "here's what a feature looks like while you're building it, and here's what it looks like when it ships." The fresh rewrite means zero legacy artifacts, zero format debt, zero stitching.

## Scope Clarification

This restructures tracking docs only (work-items/F000003_*). The company-workflow skill, templates, and registry are untouched. The feature is not being cancelled, only how it is tracked.

## Constraints

- Must satisfy hierarchy rules: feature >= 1 user-story, user-story >= 1 task
- Must use 3-phase lifecycle (Track, Implement, Ship) matching current templates
- Feature gets tracker only (no feature-level doc triplet)
- User-story gets full doc triplet (PRD, ARCHITECTURE, TEST-SPEC, milestones)
- Task gets tracker + test-plan
- Must preserve real design decisions and insights (journal entries from all 3 stories)
- S000003 (template registry) implementation is complete. S000004 (enforcement) and S000005 (scaffolding) are not started. Status must reflect this accurately.

## Premises

1. Delete feature-level doc triplet (F000003_PRD, F000003_ARCHITECTURE, F000003_TEST-SPEC, F000003_milestones)
2. Convert all trackers from 4-phase to 3-phase lifecycle
3. Consolidate 3 user stories into 1, using S000003 directory as the base
4. Consolidate 3 tasks into 1, using T000002 directory as the base
5. Rewrite all docs from scratch (fresh start) using current templates as the format guide
6. Preserve completion state: template registry work is done, enforcement/scaffolding is remaining
7. Preserve journal entries with consolidation context: add a note "Originally split into 3 stories (S000003-S000005), consolidated 2026-04-13" before old entries. Keep technology decisions (subfolder namespacing, validation in skill, userstory spelling). Drop-scope decisions that reference the now-reversed split only if they add no other value.

## Approaches Considered

### Approach A: Stitch & Convert
Append content from S000004/S000005 into S000003 docs. Convert trackers to 3-phase.
Effort: S. Completeness: 5/10. Fast but reads as stitched.

### Approach B: Synthesize & Rewrite
Use S000003 docs as seed, rewrite to cover all 3 areas coherently.
Effort: M. Completeness: 9/10. Good balance of effort and quality.

### Approach C: Fresh Start (CHOSEN)
Delete all docs, recreate from templates with new synthesized content.
Effort: L. Completeness: 10/10. Cleanest possible, zero legacy.

## Recommended Approach

Approach C: Fresh Start. Every doc rewritten from scratch using current templates as the format reference. Content synthesized from the best of all 3 stories, but written as one coherent feature narrative.
