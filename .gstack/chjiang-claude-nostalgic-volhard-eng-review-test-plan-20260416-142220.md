This is a CLI/test-infrastructure change. No HTTP routes. The "user surface" is:

- `/company-workflow validate` (consumer skill in deployed repos like ai-content)

## Key Interactions to Verify

### Round-trip runner (NEW: scripts/test-roundtrip.sh)
1. **Skill discovery** — runner walks `skills/*/`, processes only skills with both `contract.json` and `*-artifact-manifests.json`. `system-health` (no contract, no manifest) is skipped silently.
2. **Per-skill, per-type round-trip** — for each tracker type in the manifest, scaffold the template into a temp dir, substitute placeholders deterministically (FEATURE_ID=F999999, NAME=test, dates=2026-01-01, etc.), exec the skill's validator subcommand against the scaffolded file, assert zero violations.
3. **Subcommand mapping** — `personal-workflow` uses `check`; `company-workflow` uses `validate`. Hardcoded mapping inside the runner.
4. **Failure differentiation** — TEMPLATE_BUG (scaffold fails: malformed template, missing placeholder mapping) vs DRIFT (validator returns violations) appear as distinct exit codes / messages.
5. **Cleanup** — temp dirs removed via `trap '... rm -rf ...' EXIT` even on failure.

### Company-workflow validator (existing /company-workflow validate)
1. **Frontmatter check** — `workflow_type` becomes required (or recommended per pre-step grep result against ai-content); `url` becomes recommended.
2. **Section order** — `Acceptance Criteria` and `Reproduction Steps` accepted between Lifecycle and Todos via `order_skip_absent: true` + `type_specific_optional`.
3. **Artifact completeness** — feature now requires `tracker + feature-summary + milestones`; user-story unchanged at 5 artifacts; defect/task/review unchanged.
4. **Legacy tracker tolerance** — existing ai-content trackers with feature-scope PRD/ARCH/TEST-SPEC do not produce false-positive violations.

## Edge Cases

### Round-trip runner
- Skill present with contract.json but missing manifest (or vice versa) — skip, do not fail
- Manifest declares a tracker type whose template file doesn't exist — TEMPLATE_BUG
- Template has a placeholder the substituter doesn't know about — TEMPLATE_BUG (caller error)
- Two skills having templates with the same filename — runner isolates per-skill via per-skill temp dirs
- Validator subcommand is missing or not executable — TEMPLATE_BUG with clear message
- Validator returns >1 violation — DRIFT, count surfaced

### Company-workflow validator
- Tracker file with no `url` field — passes (recommended, not required)
- Tracker file with no `workflow_type` — depends on pre-step grep policy; either passes (recommended) or fails (required)
- feature dir with legacy PRD/ARCH/TEST-SPEC files — passes (validator only requires the new 3-artifact set; extras are not flagged)
- feature dir missing `feature-summary.md` — fails with clear missing-artifact message
- defect tracker with `Reproduction Steps` between Lifecycle and Todos — passes (was passing accidentally; now passes intentionally)
