## Problem Statement

The CJ_goal_* skill family has a casing inconsistency introduced by F000027's two-verb refactor (v5.0.x, just shipped). After F000027:

- **Lowercase (2 skills, new F000027 verbs):** `cj_goal_feature`, `cj_goal_defect`
- **Uppercase (2 skills, older):** `CJ_goal_investigate`, `CJ_goal_todo_fix`

Both halves are end-to-end goal orchestrators with identical semantic role (they share the "goal" family token because they share an end-state goal pattern: ship a PR, ship a fix, drain a TODO). The casing mismatch is purely cosmetic but visible across every routing surface: `rules/skill-routing.md`, `CLAUDE.md`, `doc/PHILOSOPHY.md`, `doc/ARCHITECTURE.md`, TODOS.md rows that reference them, work-items/ trackers, and the operator's muscle memory.

Fresh readers parse the mix as a real defect because the rest of the CJ_* family is uniformly uppercase (9 of 11 skills: `CJ_personal-workflow`, `CJ_system-health`, `CJ_scaffold-work-item`, `CJ_implement-from-spec`, `CJ_qa-work-item`, `CJ_personal-pipeline`, `CJ_suggest`, `CJ_goal_investigate`, `CJ_goal_todo_fix`, `CJ_improve-queue`, plus the `CJ_goal_run`/`CJ_goal_auto` deprecation shims). The 2 lowercase F000027 verbs are the outliers.

The user originally framed this as "remove the goal token, it's not used" — prior conversation surfaced that the goal token IS used as a family signal (orchestrator vs single-phase utility), and the real defect is casing only. This design lands the casing fix; the goal token stays.

## What Makes This Cool

This is the lowest-risk "cleanup-PR-as-statement" in the workbench: one rename, one shim pair, zero functional change, but it eliminates the single visible inconsistency that fresh readers (and future-you) will spend cycles parsing. It also dogfoods the F000027 deprecation-shim pattern a SECOND time, validating that the pattern is reusable (not bespoke to its first use). And the shim sunset naturally bundles into the v6.0.0 milestone that's already retiring `CJ_goal_run` + `CJ_goal_auto` — so we don't create a new sunset wave, we just add two more rows to an existing one.

## Constraints

- **macOS case-insensitive APFS — TWO constraints, not one.** First: `git mv skills/cj_goal_feature skills/CJ_goal_feature` is a no-op on case-insensitive volumes (git sees both as the same path); use a two-step rename via temp name (`git mv lower TMP && git mv TMP UPPER`), the standard git-on-APFS workaround. Second and more important: the lowercase deprecation shim CANNOT live at `skills/cj_goal_feature/` because that path case-collides with the newly-canonical `skills/CJ_goal_feature/` (same inode on APFS — two directories with names differing only in case can't coexist). The shim must live at a different path.
- **Deprecation shim location: `deprecated/{name}/` (this PR introduces the convention's first actual use).** CLAUDE.md's "Deprecated skills convention" section AND `deprecated/README.md` both state deprecated skills should live under `deprecated/{name}/` with the catalog's `files` field pointing there. F000027 did NOT follow this — its `CJ_goal_run` + `CJ_goal_auto` shims live at `skills/CJ_goal_run/` and `skills/CJ_goal_auto/` (status=deprecated in catalog, but physically at `skills/`). This PR establishes the documented convention as actual practice by placing the new lowercase shims under `deprecated/cj_goal_feature/SKILL.md` and `deprecated/cj_goal_defect/SKILL.md`. Migrating the F000027 shims to match is deferred (see Open Questions #6) — they get removed entirely at v6.0.0 sunset, so a mid-life migration would be pure churn.
- **Shim shape mirrors F000027 actual content (not a minimal stub).** The F000027 shim at `skills/CJ_goal_run/SKILL.md` is ~40 lines: YAML frontmatter (name, description, version, allowed-tools: [Skill]), `## Deprecation Banner` section with the literal one-line banner to print, `## Routing` section with the Skill-tool invocation instruction. Mirror this shape exactly — flip frontmatter `name` to `cj_goal_feature`/`cj_goal_defect`, banner + routing target the new uppercase canonical.
- **Catalog discipline.** `skills-catalog.json` keeps 4 active CJ_goal_* entries + 4 deprecated shim entries (CJ_goal_run, CJ_goal_auto + the 2 new lowercase shims). Validate.sh's catalog/filesystem cross-check fires per-entry; both renamed dirs + both shim dirs must exist before commit.
- **No upstream-consumer churn.** Per memory `feedback_workbench_scope`: do not surface downstream-consumer (portfolio/exploration repos) work as loose ends from this repo. Workbench-only scope.
- **Routing-surface freshness.** Skill-routing.md is the source of truth deployed globally to `~/.claude/rules/skill-routing.md`. Every routing example using `/cj_goal_feature` or `/cj_goal_defect` needs to flip to `/CJ_goal_feature` / `/CJ_goal_defect`. The shims keep the lowercase routes working for in-flight pipelines.
- **Memory file references.** Per `MEMORY.md` index, several memory files reference these names (`project_cj_goal_run_nested_subagent_wall`, `project_cj_personal_pipeline_task_type_qa_halt`, `project_investigate_retire_candidate`, etc.). The names are quoted as historical artifacts in some, as active routing in others. Active-routing references need updating; historical-artifact references stay verbatim.
- **No commit-message rewrites.** F000027 + the recent v5.0.x history names these skills in lowercase. Do not rewrite git history. New commits use the new uppercase names; old commits keep their wording.

## Premises

All five confirmed at D1:

1. The "goal" token in CJ_goal_* names is a load-bearing family signal (end-to-end orchestrator vs single-phase utility). Keep it.
2. The real defect: `cj_goal_feature` / `cj_goal_defect` (lowercase, F000027) are inconsistent with `CJ_goal_investigate` / `CJ_goal_todo_fix` (uppercase, older).
3. Direction of fix: lowercase → UPPERCASE (taxonomy weight: 9 of 11 CJ_* skills already uppercase).
4. Deprecation shims required, sunset matching F000027 v6.0.0 schedule. Pattern precedent: `CJ_goal_run` / `CJ_goal_auto` shims from F000027.
5. Workbench-only scope; no downstream consumers.

## Approaches Considered

### Approach A: Full rename + shims now (chosen)

Two-step `git mv` on case-insensitive macOS APFS, create lowercase deprecation shims under `deprecated/`, update catalog (4 entries: 2 renames + 2 new deprecated), update cross-references in routing/CLAUDE/PHILOSOPHY/ARCHITECTURE/README, run validate.sh + test.sh. Ships as one self-contained PR. Sunset of the 2 new shims naturally bundles into the existing F000027 v6.0.0 cleanup wave.

**Pros**: Self-contained PR; immediate visible cleanup; reuses tested F000027 shim pattern; doesn't create a new sunset wave.

**Cons**: Adds 2 more shim entries to the catalog until v6.0.0.

### Approach B: Defer to F000027 v6.0.0 sunset bundle

Add a TODOS row P3/M: "When sunsetting CJ_goal_run/auto, also flip cj_goal_feature/defect to uppercase in the same PR." No code change now.

**Pros**: Zero churn this week; lets F000027 dust settle.

**Cons**: Cosmetic inconsistency persists for weeks/months; risk of being forgotten (TODOS rows that depend on a future bundled change have worse hit rates per CLAUDE.md hygiene conventions); defers a fix the user already asked for.

Rejected: the user explicitly asked for the fix now, and a P3/M TODO row that depends on a future bundle is exactly the kind of row that gets stale and re-picked by `/loop /CJ_goal_todo_fix` (per CLAUDE.md edge case 2).

### Approach C: Codify casing as intentional (no rename)

Document a convention: "lowercase `cj_goal_*` = F000027 verb-style front doors; uppercase `CJ_*` = pipeline phases, utilities, validators." Single doc patch.

**Pros**: Zero code churn; respects F000027's lowercase-verb design intent.

**Cons**: Weakly differentiated convention — `CJ_goal_investigate` and `CJ_goal_todo_fix` are ALSO end-to-end orchestrators, so "verbs are lowercase" doesn't hold. Fresh readers will still parse the mix as inconsistency. Doesn't address the user's flagged discomfort.

Rejected: the convention claim doesn't survive contact with the existing taxonomy.

## Recommended Approach

**Approach A**, full rename + shims now. Detailed plan:

### Step 1: Two-step git mv (case-insensitive FS workaround)

Standard git-on-case-insensitive-FS pattern: rename via a temporary name, then rename to the target case. Verify each step's exit code; if APFS exhibits a stale-index edge case, run `git update-index --refresh` between steps. This is NOT documented as having been used previously in this repo; the prior F000021 family rename did direct R-renames between distinct names (CJ_run → CJ_goal_run), which doesn't trigger the case-collision case at all.

```bash
git mv skills/cj_goal_feature skills/CJ_goal_feature_TMP
git mv skills/CJ_goal_feature_TMP skills/CJ_goal_feature
git mv skills/cj_goal_defect skills/CJ_goal_defect_TMP
git mv skills/CJ_goal_defect_TMP skills/CJ_goal_defect
```

### Step 2: Update SKILL.md self-references

In `skills/CJ_goal_feature/SKILL.md`:
- Frontmatter `name:` field flips lowercase → uppercase
- All in-body self-references (`/cj_goal_feature` → `/CJ_goal_feature`) flip
- Path resolution block (the `_REPO_ROOT/skills/cj_goal_feature/pipeline.md` checks) flips
- Same for `skills/CJ_goal_defect/SKILL.md`

### Step 3: Update pipeline.md self-references

Same edits in `skills/CJ_goal_feature/pipeline.md` + `skills/CJ_goal_defect/pipeline.md`. Resume state file paths (`.cj-goal-feature/`, `.cj-goal-defect/`) stay lowercase — these are runtime state dirs the operator never sees; renaming them would break in-flight resume state for any open pipeline.

### Step 4: Create deprecation shims at `deprecated/{name}/SKILL.md`

Mirror the F000027 shim shape at `skills/CJ_goal_run/SKILL.md` (~40 lines: frontmatter + Deprecation Banner section + Routing section). New file `deprecated/cj_goal_feature/SKILL.md`:

```markdown
---
name: cj_goal_feature
description: "DEPRECATED ALIAS (casing-fix follow-up to F000027, sunsets v6.0.0) — superseded by uppercase /CJ_goal_feature for family-name consistency with CJ_goal_investigate / CJ_goal_todo_fix. This thin alias prints a one-line deprecation banner then routes to /CJ_goal_feature. Removal is deferred to v6.0.0; in-flight pipelines finish under the lowercase name via skills-deploy install --include-deprecated."
version: 5.0.12
allowed-tools:
  - Skill
---

## Deprecation Banner

This skill is a thin alias retained for backwards-compatible muscle memory. The
casing-fix refactor (v5.0.12) flipped the F000027 verbs from lowercase to
uppercase for consistency with the rest of the CJ_* family (CJ_personal-workflow,
CJ_system-health, CJ_goal_investigate, CJ_goal_todo_fix, etc.).

Print this banner before doing anything else:

```
[DEPRECATED] /cj_goal_feature is deprecated; use /CJ_goal_feature instead. Sunsets in v6.0.0.
See CHANGELOG.md (v5.0.12 casing-fix) for the rationale and migration guidance.
```

## Routing

Delegate to `/CJ_goal_feature` with the same arguments. Use the Skill tool to invoke the canonical verb:

```
Skill: CJ_goal_feature, args: "$@"
```

`/CJ_goal_feature` (`skills/CJ_goal_feature/SKILL.md`) runs the full feature pipeline (worktree → `/office-hours` inline → scaffold/impl/qa leaf subagents → `/ship` → STOP at the PR). This shim does NOT create its own worktree — the canonical skill does that.
```

`deprecated/cj_goal_defect/SKILL.md` is the same shape with name/description/banner/routing flipped to `cj_goal_defect` → `/CJ_goal_defect`.

The shim location (`deprecated/`) follows CLAUDE.md's "Deprecated skills convention" — first actual use of that documented pattern in this repo (F000027's `CJ_goal_run` + `CJ_goal_auto` shims are at `skills/` for historical reasons; their migration to `deprecated/` is deferred to v6.0.0 sunset, see Open Questions #6).
