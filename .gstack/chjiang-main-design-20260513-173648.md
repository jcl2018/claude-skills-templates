touch different sections of run.md.

Effort: S each (pipeline.md ~50 LOC; run.md Branch(f) ~80 LOC; run.md Branch(b) ~120 LOC)

### Approach B: S036 first, then S039+S037 combined
S039 and S037 in one PR since both modify run.md.
Rejected: combined diff harder to review; blast radius if one has a bug.

## Key Implementation Notes

**S000036 (pipeline.md):**
- Flag: `--work-item-dir <path>` added to arg parsing at Step 1
- New variable: `WORK_ITEM_DIR_OVERRIDE`
- DESIGN_DOC validation block gated on `[ -z "$WORK_ITEM_DIR_OVERRIDE" ]`
- Branch (e) at Step 2: a **new** labeled branch to be inserted into `pipeline.md`
  (does not exist today). When override is set, skip footer search + Phase 1, jump to Step 4
- Step 4 sub-step 1 (footer check) skipped in this mode
- Telemetry Step 9.1 adds `"work_item_dir_mode": true` field

**S000039 (run.md Branch(f)):**
- Inserted after Branch(g) at Step 1.0, before Step 1.1's design-doc branches
- Reads TRACKER gate strings verbatim (from tracker-user-story.md template)
- Six MODE values: `impl_qa_ship`, `qa_ship`, `ship`, `open_pr`, `already_shipped`, `pr_unknown_state`
- PR-state check: `gh pr view $PR_URL --json state -q .state` with graceful UNKNOWN fallback
- Branch(g) dispatch → Branch(f): Branch(g) sets `ARG` to the selected work-item dir, then falls through to Branch(f)

**S000037 (run.md Branch(b)):**
- Replaces the halt message block wholesale
- Per-child sequence: fetch + branch-off-main + copy scaffold files + commit → dispatch pipeline subagent → /ship → /land-and-deploy → restore to feature branch
- Resume guard: check `gh pr list --state merged` for each child before dispatching
- State file extended: CHILDREN_TOTAL, CHILDREN_DONE, CHILDREN_FAILED, CHILD_PR_URLS
- Failure: halt loop, set END_STATE=halted_at_pipeline, restore to feature branch, print remaining children
- **Known fragile surface:** git worktree state after a failed `/land-and-deploy` mid-child
  can leave the tree in an unexpected state. The restore step (`git checkout FEATURE_BRANCH`)
  itself can fail if the worktree is detached-HEAD. S000037's TEST-SPEC must include a
  test row for "child pipeline fails mid-flight: repo left on feature branch, no half-committed state."
  The SPEC is the contract; this doc flags it as the highest-risk implementation path.
