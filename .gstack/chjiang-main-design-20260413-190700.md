## Problem Statement

The `doc-milestones.md` artifact is mapped as a required artifact for the `user-story` type in both `artifact-manifests.json` and `rules/work-items.md`. Milestones track the delivery timeline of an entire feature (which contains multiple stories and tasks), not individual user stories. This is a misplacement in the artifact type mapping.

## What Makes This Cool

This is the repo's first defect (D000001). It validates the defect workflow template end-to-end: tracker + RCA + test-plan artifacts, the `fix/` branch naming convention, and the 3-phase lifecycle for defects.

## Constraints

- Only `artifact-manifests.json` and `rules/work-items.md` need changes
- The `doc-milestones.md` template itself is correct and unchanged
- No existing work items have a `milestones.md` file, so no migration needed
- F000001 and F000002 will be flagged as missing `milestones.md` by `/docs check` after the fix (expected, correct behavior)

## Premises

1. **Milestones belong at the feature level.** A feature has multiple stories with multiple tasks. Milestones track the feature's overall delivery timeline. Confirmed.
2. **Manifest + rules change only.** The template file is fine. Confirmed.
3. **No migration needed.** No existing milestones.md files exist in work-items/. Confirmed.

## Approaches Considered

### Approach A: Move milestones to feature (required)
Move `milestones` from `user-story.required` to `feature.required` in the manifest. Update `rules/work-items.md` to match. Every feature must have a milestones doc, enforced by `/docs check`.

Effort: S | Risk: Low
Pros: Enforced consistency, every feature gets delivery timeline tracking.
Cons: Existing features flagged as missing milestones (correct behavior).

### Approach B: Move milestones to feature (optional)
Move `milestones` from `user-story.required` to `feature.optional`. No enforcement.

Effort: S | Risk: Low
Pros: No /docs check noise on small features.
Cons: Optional means it might never get created. Less consistency.

## Recommended Approach

**Approach A (required).** If a feature is small enough to skip milestones, it's probably a task, not a feature. Milestones at the feature level is the whole point of the fix.

### Files to change

1. **`artifact-manifests.json`**
   - Remove `{"artifact": "milestones", "template": "doc-milestones.md", "filename": "milestones.md"}` from `types.user-story.required` array
   - Add same entry to `types.feature.required` array, after the tracker entry

   Before (feature):
   ```json
   "feature": {
     "required": [
       {"artifact": "tracker", "template": "tracker-feature.md", "filename": "TRACKER.md"}
     ]
   }
   ```
   After (feature):
   ```json
   "feature": {
     "required": [
       {"artifact": "tracker", "template": "tracker-feature.md", "filename": "TRACKER.md"},
       {"artifact": "milestones", "template": "doc-milestones.md", "filename": "milestones.md"}
     ]
   }
   ```

   Before (user-story required, 5 entries):
   ```json
   {"artifact": "tracker", ...}, {"artifact": "prd", ...}, {"artifact": "architecture", ...},
   {"artifact": "test-spec", ...}, {"artifact": "milestones", ...}
   ```
   After (user-story required, 4 entries — remove only the milestones entry; leave tracker, prd, architecture, and test-spec unchanged):
   ```json
   {"artifact": "tracker", ...}, {"artifact": "prd", ...}, {"artifact": "architecture", ...},
   {"artifact": "test-spec", ...}
   ```

2. **`rules/work-items.md`**
   - Change feature line from `- **feature**: tracker-feature.md` to `- **feature**: tracker-feature.md + doc-milestones.md`
   - Change user-story line from `...+ doc-TEST-SPEC.md + doc-milestones.md` to `...+ doc-TEST-SPEC.md` (remove milestones)
