## Problem Statement

F000005 (v1.2.0, shipped 2026-05-02) made `skills-deploy install` skip `company-workflow` on clean machines, but kept all its source files at `skills/company-workflow/` and `templates/company-workflow/`. The reason: `work-copilot/` byte-mirrors 7 things from those paths (validate.sh Error check 10's MIRROR_SPECS array), and deletion would break the bundle.

End state today: `skills/` contains `company-workflow/` (deprecated), `personal-workflow/` (active), `system-health/` (active). The deprecated entry is visually indistinguishable from the active ones — `status: deprecated` lives only in the catalog JSON. Opening the folder reads as "deployable skills + one deprecated holdout" instead of "deployable skills only."

The user's reaction on revisiting after the ship: "I still see company-workflow related skill/doc/templates in this folder."

## What Makes This Cleaner

`skills/` becomes a curated catalog of *deployable* skills only. `deprecated/company-workflow/` becomes the explicit upstream-truth source for the work-copilot byte-mirror — the path itself names the lifecycle state. Future deprecations (if any) join the same parent. Future bundles (work-cursor, etc.) can still mirror from the same source.

## Constraints

- `validate.sh` Error check 10's MIRROR_SPECS byte-identity guarantee MUST still hold post-move. We are moving the source paths, not changing the invariant.
- `skills-deploy install --include-deprecated` MUST still install company-workflow successfully (escape hatch from F000005, must not regress).
- `./scripts/test.sh` MUST pass post-move with no behavioral changes — only path replacements.
- No SKILL.md content changes (frontmatter, body) as part of this refactor — pure relocation.

## Premises

1. `skills/` is the catalog of deployable skills. Deprecated entries don't belong there.
2. `templates/{skill}/` follows the same logic — templates of a deprecated skill move with the skill.
3. `deprecated/` is a top-level peer of `skills/`, `templates/`, `work-copilot/`, named for the lifecycle state. Future deprecated skills join the same parent.
4. Consumer scripts (`skills-deploy`, `validate.sh`) should derive paths from the catalog's `files[]` / `templates[]` arrays instead of hardcoding `skills/{name}/` — the principled refactor that makes future moves trivial.

## Approaches Considered

### Approach A: Catalog-driven paths + relocate (recommended)
Move files. Update `skills-catalog.json` `files[]` and `templates[]` to point at `deprecated/company-workflow/...`. Refactor `skills-deploy` (line 260, line 278) and `validate.sh` (line 30 catalog walker) so they derive paths from the catalog instead of hardcoding `skills/{name}/`. Update `validate.sh` Error check 10's MIRROR_SPECS to new paths. Update `test.sh` to use a `COMPANY_PATH` constant defined once at the top.

- Effort: M (multi-hour, mostly mechanical).
- Risk: Med — touches validate.sh's catalog walker, skills-deploy's path derivation, test.sh's ~40 hardcoded refs.
- Reuses: Catalog `files[]` and `templates[]` are already populated; we're making more of the code respect them as source of truth.

### Approach B: Move files + special-case deprecated path lookup
Move files. Leave most of `skills-deploy` and `validate.sh` alone — add a small "if status == deprecated, look in deprecated/" branch wherever paths are derived from `skills/{name}/`.

- Effort: S (smaller diff).
- Risk: Med — adds a special case that future contributors will trip on; doesn't make future moves easier.
- Reuses: Existing hardcoded structure, with a conditional patch.

### Approach C: Move files, run tests, fix breakage as discovered
Move files. Run `./scripts/test.sh`, see what breaks, patch one error at a time.

- Effort: Unknown (depends on hidden assumptions in test fixtures).
- Risk: High — incomplete coverage; CI may pass on main but break in a clean-target install or a fixture-only test path.
- Reuses: Nothing; reactive.

**RECOMMENDATION:** A — the catalog-driven refactor is the principled fix and makes future relocations trivial. B trades short-term diff size for long-term maintenance burden. C is too reactive for a refactor with mirror-invariant verification.
