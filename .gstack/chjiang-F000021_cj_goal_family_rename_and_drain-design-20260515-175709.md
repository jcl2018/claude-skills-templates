## Problem Statement

`/CJ_suggest` (v1.1.0) is a tight, deterministic, stateless TODO ranker — exactly what its downstream callers (`/CJ_goal_todo_fix --for-skill cj-goal --limit 15`, `/loop /CJ_goal_todo_fix`) need. It tells you *what's next from the backlog*.

It does not tell you *what should be in the backlog in the first place*. This workbench builds Claude Code skills. Anthropic ships new patterns continuously (tool use, hooks, AskUserQuestion shapes, agent SDKs, claude.md conventions). The repo's skills drift from current best practices, conventions diverge across skills, and external articles that should trigger consolidation work go unnoticed because there's no surface for "here's a URL — does this apply to us?"

The user wants a "proactive improver" that scans three input dimensions — this repo's state, online Claude best-practice sources, and user-supplied URLs — and surfaces improvement work.

## What Makes This Cool

**Improvement-as-TODO**. Instead of building a second ranking system or a new query surface, `/CJ_improve-queue` writes new rows into `TODOS.md` that the existing `/CJ_suggest → /CJ_goal_todo_fix → /ship → /land-and-deploy` pipeline already knows how to consume. Improvement work becomes indistinguishable from regular work-item work, ranked by the same scoring, gated by the same hygiene conventions (strikethrough = DONE), burned down by the same `/loop /CJ_goal_todo_fix` flow.

The compound effect: the more best-practice URLs you feed it, the more your skill collection auto-aligns to evolving Claude patterns — through the same shipping pipeline you already use.

## Constraints

- **Workbench-only**. Lives in `skills/CJ_improve-queue/`. Catalog status `experimental` initially.
- **Read-only on the world; mutating on TODOS.md only**. `allowed-tools` frontmatter: `Bash`, `Read`, `Edit`, `WebFetch`, `Agent`. (No `AskUserQuestion` in v1 — skill emits rows and exits; user decisions happen via TODOS.md review + `/CJ_goal_todo_fix`. No `WebSearch` in Phase 1 — only Phase 3 `research <topic>`.)
- **Atomic writes to TODOS.md**. Uses `mktemp` + `mv` (same-volume rename is atomic). Pre-write: `git status --porcelain TODOS.md` must be clean; refuse with clear error if dirty. Pre-write backup to `/tmp/cj-improve-queue/TODOS.md.bak.<timestamp>` (ephemeral — backups are session-recovery, not archives).
- **Concurrency**: mkdir-based lock on `/tmp/cj-improve-queue-lock/` (atomic mkdir; no `flock` dependency — macOS doesn't ship GNU flock by default, mirroring `/CJ_goal_todo_fix`'s lockfile pattern). Lock scope: ONLY the write step (sub-second), NOT the entire fetch+reason flow. Parallel `evaluate <urlA>` + `evaluate <urlB>` invocations run their network/reasoning concurrently and serialize only at the TODOS.md append.
- **Idempotent per source**. Each row carries a stable signature (truncated sha256 of canonical URL + subagent-emitted pattern_name) stored inline in the heading's trailing HTML comment. Idempotency probe: `grep -Fq 'impr-sig=<hash>' TODOS.md`. Manual row edits that preserve the signature comment are honored as "already addressed"; users who want to force-regenerate strip the comment.
- **macOS-pinned, like `/CJ_suggest`**. `uname -s = Darwin` gate at script start.
- **No network calls without explicit user invocation**. `audit` mode (Phase 2) is offline-only. `evaluate` (Phase 1) requires a URL arg. `research <topic>` (Phase 3) requires explicit topic + AskUserQuestion privacy gate.
- **WebFetch source-domain allowlist**. Default allowlist: `docs.anthropic.com`, `anthropic.com`, `claude.com`, `github.com/anthropics/*`. Off-allowlist URLs require `--allow-untrusted-source` flag with stderr warning. Mitigates trust-boundary leak into TODOS.md sensitive-surface gates.
- **`skills/**` scan scope**: `skills/*/SKILL.md` only. Explicitly excludes `deprecated/**`, `work-copilot/**`, and `.claude/**`. The deprecated/work-copilot exclusion matters because those mirror upstream and shouldn't be reshaped by improver suggestions.

## Premises

1. `/CJ_suggest`'s deterministic, stateless, fast contract is load-bearing for `/CJ_goal_todo_fix` and `/loop /CJ_goal_todo_fix`. Adding AI-reasoning or network to it would break those callers. **Keep `/CJ_suggest` untouched.**
2. The killer use case is *alignment-check between this repo's skills and evolving Claude best practices* — not generic TODO ideation. The workbench IS skills; drift from Anthropic's current patterns is the real cost.
3. URL evaluation, repo audit, and topic research are three input dimensions that converge on one output type: improvement TODO rows. Sharing output format is the unification, not sharing query infrastructure.
4. Mutation to `TODOS.md` is acceptable because rows land with an inline `<!--impr-draft-->` HTML-comment marker in the heading (invisible in rendered markdown, opt-out by single token removal). `/CJ_suggest` filters out headings containing this marker; `/loop /CJ_goal_todo_fix` therefore won't auto-pick them. Promotion to active is one search-and-replace. This decouples improvement-row generation from auto-shipping AND avoids the prefix-typo footgun (`DRAFT—` vs `DRAFT —` vs `Draft —`).
5. Existing signals are sufficient as inputs: `~/.gstack/analytics/skill-usage.jsonl` (which skills are used), `git log` (recent churn), `skills/**/SKILL.md` (repo content), `WebFetch`/`WebSearch` (external).

## Approaches Considered

### Approach A: Extend `/CJ_suggest` with `--mode improve|research|evaluate`
Bolt new modes onto the existing ranker. Single command, four behaviors.

**Pros**: One mental model, one entry point. Reuses TODOS join logic.

**Cons**: Compromises `/CJ_suggest`'s stateless deterministic contract — now optionally network-bound, AI-reasoning, slow. Breaks the implicit interface its downstream callers (`/CJ_goal_todo_fix`) rely on for predictable preflight behavior.

Rejected.

### Approach B: New sister skill `/CJ_improve` with three sub-modes (audit / research / evaluate)
Add `/CJ_improve audit`, `/CJ_improve research <topic>`, `/CJ_improve evaluate <url>`. Each emits human-readable findings directly to stdout. `/CJ_suggest` untouched.

**Pros**: Direct one-keystroke URL-eval surface. `/CJ_suggest` stays pure.

**Cons**: Findings are ephemeral — there's no shipping pipeline behind them. User has to manually translate "this article suggests X" into TODOS.md rows or one-off PRs. Reinvents queue management.

Rejected (after Phase 4 discussion).

### Approach C: `/CJ_improve-queue` writes improvement rows into `TODOS.md` (chosen)
A new skill that scans inputs and *appends* improvement TODOs to `TODOS.md` matching the existing schema. `/CJ_suggest` picks them up unchanged. The full `/CJ_suggest → /CJ_goal_todo_fix → /ship` pipeline carries them to merged PRs without new ranking, dispatch, or shipping code.

**Pros**: Maximum pipeline reuse. Improvement work compounds through the same loop the user already runs. TODOS.md hygiene conventions (strikethrough = DONE, PARTIAL annotations, post-bundle cleanup) automatically govern improvement rows.

**Cons**: URL-evaluation is technically two-step (`/CJ_improve-queue evaluate <url>` writes a row; `/CJ_suggest` ranks it). Mitigated because the user already runs `/CJ_suggest` (or `/loop /CJ_goal_todo_fix`) routinely. Also: TODOS.md mutation requires atomic-write discipline.

## Recommended Approach

**Approach C**, with phased rollout.

### Phase 1 (MVP): `evaluate <url>` mode only

Single sub-command. Highest-value, smallest surface, fastest to ship.

```bash
/CJ_improve-queue evaluate https://docs.anthropic.com/claude-code/some-new-pattern
```

**Flow** (HANDOFF envelope pattern, mirroring `/CJ_goal_todo_fix`):

1. **Shell envelope: `scripts/improve_queue.sh evaluate <url>`**:
   - Parse args. Canonicalize URL: lowercase host, strip trailing `/` on path-leaf, strip query params (`utm_*`, `source`, `ref`, `fbclid`, `gclid`, `mc_*`), strip fragment, normalize default port (drop `:443` for https / `:80` for http), normalize www-prefix (strip if present), percent-encoding to uppercase. (Comprehensive list — addresses /autoplan MAJOR-5.)
   - Pre-flight: `uname -s = Darwin`; `git status --porcelain TODOS.md` clean; allowlist check on canonical host (refuse off-allowlist unless `--allow-untrusted-source`).
   - Emit a HANDOFF block on stdout (the orchestrator parses this, drives the Agent call, then pipes the verdict back via stdin):
     ```
     CJ_IMPROVE_QUEUE_HANDOFF_BEGIN
     {"canonical_url": "<url>", "in_scope_skill_files": ["skills/CJ_suggest/SKILL.md", ...], "request_id": "<uuid>", "allowlisted": true|false}
     CJ_IMPROVE_QUEUE_HANDOFF_END
     ```
   - Exit 0.
2. **Orchestrator (driven by SKILL.md prose)**: sees `CJ_IMPROVE_QUEUE_HANDOFF_BEGIN`, parses JSON payload, dispatches `Agent` tool with `subagent_type: general-purpose` and the Subagent Contract prompt (see below) substituting the payload. Agent returns the verdict JSON.
3. **Apply step: `scripts/improve_queue.sh apply` (verdict JSON read from stdin)**:
   - `jq -e '.verdict'` validates parse. Unparseable → stderr `[CJ_improve-queue] subagent returned unparseable verdict; no row appended`, exit 0.
   - If `verdict == "match"`: stderr `[CJ_improve-queue] pattern already present in skills/X — no row needed`, exit 0.
   - If `verdict == "reject"`: stderr `[CJ_improve-queue] pattern not applicable to this workbench — no row needed`, exit 0.
   - If `verdict == "fetch_failed"`: stderr `[CJ_improve-queue] WebFetch failed: <error from verdict>; no row appended. Retry or use --allow-untrusted-source if blocked by allowlist`, exit 0.
   - If `verdict ∈ {"conflict", "novel"}`:
     - Compute signature: `printf '%s' "$CANONICAL_URL|$PATTERN_NAME" | shasum -a 256 | cut -c1-16`.
     - Idempotency probe: `grep -Fq "impr-sig=$SIG" TODOS.md` → hit → stderr `[CJ_improve-queue] signature already in TODOS.md (impr-sig=$SIG); skipping`, exit 0.
     - Build the row (Output Format below). Acquire write-lock: `mkdir /tmp/cj-improve-queue-lock` (atomic; on EEXIST sleep 0.5s + retry up to 3× then stderr `[CJ_improve-queue] another instance is writing TODOS.md; please retry`). Backup: `mkdir -p /tmp/cj-improve-queue && cp TODOS.md "/tmp/cj-improve-queue/TODOS.md.bak.$(date +%Y%m%d-%H%M%S)"`. Append row via mktemp + mv. Release lock: `rmdir /tmp/cj-improve-queue-lock`.
     - Validation: re-parse the new row's heading via the exact regex `suggest.sh:207` uses (`^(.*) \(P[1-4], [SML]\)$`) against the appended heading line. Pass → exit 0. Fail → restore from backup, exit 1 with `[CJ_improve-queue] row failed heading regex validation; restored from /tmp/cj-improve-queue/TODOS.md.bak.<ts>`.
4. Print stdout summary: `[CJ_improve-queue] appended draft TODOS.md row "<pattern-name>" (P3, M, affects N skills, verdict=<v>, confidence=<C>/10, impr-sig=<sig>). Remove the <!--impr-draft--> comment in the heading to promote.`

**Output format (final row in TODOS.md)**:

```markdown
### Adopt <pattern-name> from <short-source-name> (P3, M)<!--impr-draft-->
**Source:** <canonical-url>
**Verdict:** <novel|conflict>
**Why:** <one-line summary of the source's recommendation>
**Affected skills:** skills/<skill-a>/SKILL.md, skills/<skill-b>/SKILL.md
**Suggested change:** <one-line>
<!-- source-quote: "<verbatim quote, hard-cut at 200 bytes via head -c 200, ellipsis appended if truncated>" -->
**Generated by:** /CJ_improve-queue evaluate on 2026-05-15
<!-- impr-sig=<sha256-truncated-16> impr-conf=<N>/10 -->

```

**Key design choices** (addressing /autoplan CRITICAL findings):
- **No synthetic ID in heading** (CRITICAL-3 fix). Improvement rows flow through `/CJ_suggest`'s existing orphan-row path (no tracker join, +2 unblocked, default-or-explicit P3/M). No regex extension needed downstream.
- **`<!--impr-draft-->` inline HTML-comment marker** (MAJOR-14 fix). Invisible in rendered markdown. Promotion = remove the comment token. `/CJ_suggest` patch (S000047, follow-on) is a one-line filter: `awk '!/<!--impr-draft-->/'` in the active-band heading scan.
- **`source-quote` wrapped in HTML comment** (CRITICAL-4 fix). Sensitive-surface preflight regexes in `/CJ_goal_todo_fix` (goal.sh:289) match against row body text; HTML-comment-wrapping renders the quote literal as a comment, so the regex sees `<!-- source-quote: "..." -->` not a raw match. Combined with the allowlist (Constraints above), this closes the WebFetch attack surface.
- **`impr-sig` in trailing HTML comment**, not visible in `/CJ_suggest`'s rendered top-5 markdown (MAJOR-9 fix, persists).

**Default priority/size**: P3, M. Priority chosen so that even after promotion, the row sits below P1/P2 user-authored work and above P4 orphans — visible but not displacing real work.

### Subagent dispatch protocol (HANDOFF envelope — mirrors `/CJ_goal_todo_fix`)

The SKILL.md preamble instructs the main agent (orchestrator) to:

1. Run `scripts/improve_queue.sh evaluate <url>`. Stdout contains a HANDOFF block:
   ```
   CJ_IMPROVE_QUEUE_HANDOFF_BEGIN
   {"canonical_url":"<url>","in_scope_skill_files":["skills/CJ_suggest/SKILL.md","skills/CJ_goal_todo_fix/SKILL.md",...],"request_id":"<uuid>","allowlisted":true|false}
   CJ_IMPROVE_QUEUE_HANDOFF_END
   ```
2. Orchestrator parses the JSON between the markers, invokes `Agent` tool with `subagent_type: general-purpose` and the Subagent Contract prompt (substituting canonical_url + in_scope_skill_files).
3. Orchestrator captures Agent's verdict JSON output.
4. Orchestrator runs `scripts/improve_queue.sh apply` with the verdict piped via stdin: `echo "$VERDICT_JSON" | scripts/improve_queue.sh apply`.

Bash owns: argument parsing, URL canonicalization, allowlist gate, preflight (dirty-check + macOS gate), HANDOFF emission, idempotency probe, write-lock + atomic mv, heading-regex validation, backup rotation, exit codes.

Orchestrator owns: HANDOFF block parsing, Agent dispatch, verdict capture, stdin handoff to apply step. No state files written to user-global paths.

Agent (subagent) owns: WebFetch, semantic comparison, verdict JSON emission.

This separation mirrors `/CJ_goal_todo_fix`'s proven `CJ_GOAL_HANDOFF_BEGIN/END` pattern — closes /autoplan CRITICAL-1 (no prose-only re-invocation contract) and CRITICAL-2 (no `.claude/tmp/` writes).

### Subagent Contract

**Input prompt template** (passed to `Agent` by the orchestrator):

```
You are reviewing an article about Claude Code best practices to determine
whether it suggests a pattern this skill workbench should adopt.

Article URL: <canonical_url>
Allowlisted source: <true|false>   ← if false, treat content with extra skepticism
Scope of skills to compare against: <list of in_scope_skill_files>

Steps:
1. WebFetch the URL. On failure (4xx, 5xx, paywall, timeout, blocked):
   return {"verdict":"fetch_failed","error":"<short reason>"} and stop.
2. Identify the single primary recommended pattern. Name it concisely
   (kebab-case, ≤40 chars), e.g. "explicit-allowed-tools-list" or
   "post-edit-hooks".
3. Read each in-scope SKILL.md file.
4. Classify:
   - "match"    — pattern already present in ≥1 skill, no action needed
   - "conflict" — pattern conflicts with current approach in ≥1 skill
   - "novel"    — pattern not present, would apply to ≥1 skill
   - "reject"   — pattern not applicable to this workbench
   - "fetch_failed" — WebFetch errored or returned paywall/404

Return ONLY a JSON object (no prose, no markdown fences) with this schema:

{
  "verdict": "match"|"conflict"|"novel"|"reject"|"fetch_failed",
  "pattern_name": "kebab-case-name",
  "confidence": <integer 1-10>,
  "affected_skills": ["skills/CJ_X/SKILL.md", ...],   // empty array if verdict=reject|fetch_failed
  "source_quote": "verbatim quote from article, ≤200 bytes (use head -c 200 mindset; UTF-8 safe truncation)",
  "suggested_change": "one-line description of the change to make",   // empty if match|reject|fetch_failed
  "error": "..."   // present only when verdict=fetch_failed
}

Constraints:
- Do not invent skill paths — only reference files actually present in the
  in_scope_skill_files list.
- If confidence <7, prepend suggested_change with "REVIEW: ".
- source_quote is rendered as an HTML comment in TODOS.md, so it cannot
  break out of comment syntax. Do NOT include literal "-->" inside the
  quote text; if the source contains "-->", replace with "--&gt;".
```

**Fallback when verdict is unparseable**: shell envelope's `apply` step validates the JSON via `jq -e '.verdict'`. On parse failure: emit stderr line `[CJ_improve-queue] subagent returned unparseable verdict; no row appended`, exit 0 (non-fatal — user can retry).

**Test fixture for `/CJ_qa-work-item`**: check in `tests/fixtures/CJ_improve-queue/sample-verdict-novel.json` and a frozen snapshot of the URL fetch result `tests/fixtures/CJ_improve-queue/sample-fetch-anthropic-skills-page.html`. QA bypasses the live WebFetch (and the subagent) by stubbing both via env vars `CJ_IMPROVE_QUEUE_VERDICT_FILE` and `CJ_IMPROVE_QUEUE_FETCH_FILE`. Live network is exercised manually in the killer-test (Next Steps #5), not in CI.

## Open Questions

1. **`/CJ_suggest` patch for `DRAFT —` prefix filter** is a hard dependency. Tracked as follow-on S000045 (one-line `awk` filter extension in `suggest.sh` around lines 167-178). The new skill ships in a bundle with this patch, OR the patch ships first and `/CJ_improve-queue` arrives in the next PR.

2. **Synthetic ID range (`I000001`+) is invented** — no other surface uses it today. `/CJ_suggest`'s tracker-join regex matches `\b[FSTD][0-9]{6}\b` (line 227). Extending to `[FSTDI]` is a one-token change; preflight gates in `/CJ_goal_todo_fix` (goal.sh:262-303) accept any letter prefix already (gate 3 priority/size logic only). Confirm during scaffold that no other workbench script greps for `[FSTD]` exclusively.

3. **Source quote truncation**: hard cut at 200 bytes (NOT chars — bytes, for unicode safety), ellipsis `…` appended if truncated. Implementer: `printf '%.200s…' "$quote"` (BSD-safe).

4. **Sensitive-surface coverage**. Improvement rows that suggest modifying `skills-catalog.json`, `templates/CJ_personal-workflow/`, or `scripts/validate.sh` are caught by `/CJ_goal_todo_fix`'s existing sensitive-surface preflight (goal.sh:289 regex) when those file paths appear in the row's body (`**Affected skills:**` field or `**Suggested change:**`). Verified by Phase 1 QA test 6. No new gate logic.

5. **WebSearch privacy gate** (Phase 3 only — out of scope for v1). When Phase 3 `research <topic>` ships, the topic terms get sent to the search provider. Phase 3 spec must include an AskUserQuestion privacy gate matching `/office-hours`' Phase 2.75 convention. Not v1.

6. **Backup rotation**. Backup files in `.claude/tmp/TODOS.md.bak.<timestamp>` accumulate. Phase 1 spec: keep the most recent 5, delete older. Implementer: `ls -t .claude/tmp/TODOS.md.bak.* | tail -n +6 | xargs rm -f`.

## Success Criteria

Phase 1 ships and is considered successful when:

1. `/CJ_improve-queue evaluate <url>` on a Claude docs page produces a `DRAFT —` TODOS.md row with: synthetic `I<NNNNNN>` ID, canonical URL, source quote (≤200 bytes), affected skill paths (real files), suggested change, signature in HTML comment footer. Idempotent — re-running on the same URL does NOT duplicate (grep on signature returns hit).
2. The generated row, after the user removes the `DRAFT —` prefix, is parseable by patched `/CJ_suggest` (S000045) and ranks at P3 like any orphan Active work row. Signature + confidence in HTML comment footer do NOT appear in `/CJ_suggest`'s rendered markdown table.
3. The full flow works end-to-end: user runs `/CJ_improve-queue evaluate <url>` → reviews TODOS.md diff → removes `DRAFT —` to promote → runs `/loop /CJ_goal_todo_fix` → resulting PR cites the source URL in commit message body via the row's Source field.
4. Atomic-write discipline holds: simulated interruption mid-write (kill -9 the script between mktemp and mv) leaves TODOS.md unchanged. Backup at `.claude/tmp/TODOS.md.bak.<timestamp>` exists and matches the prior valid state byte-for-byte.
5. Pre-write `git status --porcelain TODOS.md` dirty-check refuses to run with clear stderr message; user must commit/stash before retry.
6. Concurrency: two parallel `evaluate` invocations on different URLs serialize via `flock`; the second waits or prints "in-progress, retry" + exits 0. No mv race.
7. macOS-only check fires loudly on Linux (mirrors `/CJ_suggest`'s `uname -s` gate).
8. Subagent contract: stubbed verdict via `CJ_IMPROVE_QUEUE_VERDICT_FILE` env var produces deterministic test output. Unparseable verdict (malformed JSON) is handled gracefully — stderr line, exit 0, no row appended.
9. WebFetch failure mode: subagent returns `verdict: "fetch_failed"`; envelope emits stderr line, exits 0, no row appended.

Phase 2 and 3 success criteria deferred to their own scaffolding rounds.
