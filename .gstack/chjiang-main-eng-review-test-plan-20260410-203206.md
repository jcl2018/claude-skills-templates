## Test Implementation Plan

All tests in `scripts/test-deploy.sh`. Each test uses a temp directory for
`SKILLS_TARGET` and `REPO_SOURCE` to isolate from the real environment.

### Environment Override Pattern
```bash
SKILLS_TARGET=$(mktemp -d)  # replaces ~/.claude/skills/
MANIFEST_PATH="$SKILLS_TARGET/.skills-templates.json"  # replaces ~/.claude/.skills-templates.json
```

`skills-deploy` should accept env vars to override target paths:
- `SKILLS_DEPLOY_TARGET` — default `~/.claude/skills`
- `SKILLS_DEPLOY_MANIFEST` — default `~/.claude/.skills-templates.json`

### Priority Tests (must ship with v1)
1. Install all — symlinks created for all 9 skills
2. Install with deps — work auto-installs 4 deps
3. Multi-file skill — align-feature-contract gets all .md files
4. Remove requires args — no-arg blocked
5. Remove checks reverse deps
6. Migration with identical files — converts
7. Migration with modified files — warns and skips
8. Doctor on healthy state — passes
9. Doctor on broken state — reports issues

### Deferred Tests
- setup.sh clone behavior (requires network mock)
