## Problem Statement

`scripts/skills-update-check` hardcodes `origin/main` for fetch + show. Users on a fork that pulls from `upstream/main` (with `origin/main` either absent or stale) get no upgrade nudge — `git fetch origin main` fails silently, the version comparison short-circuits, no banner emits.

Captured in TODOS.md:8 (P3, S). Small + well-scoped — a good first real run for /personal-pipeline.

## Recommended Approach

Add an `upstream/main` fallback before the fetch. If `origin/main` doesn't exist as a tracking branch, try `upstream/main`. Same comparison logic; different remote name.

## Acceptance Criteria

- A user on a fork with `upstream/main` but no `origin/main` sees the upgrade banner when a newer collection version is available.
- Existing `origin/main`-only users see no behavior change.
- Fallback is silent when both remotes are absent (don't spam errors).
