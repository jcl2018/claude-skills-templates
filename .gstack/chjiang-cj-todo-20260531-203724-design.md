## Problem Statement

`/CJ_goal_investigate` is a defect-to-shipped-fix orchestrator (catalog v1.1.0,
`skills/CJ_goal_investigate/`). It was the F000027 plan's open-fate sibling
to `/CJ_goal_defect` — F000027 deprecated `/CJ_goal_run` + `/CJ_goal_auto`,
kept `/CJ_goal_todo_fix` + `/CJ_personal-pipeline`, and left investigate's
fate "to decide once defect proves out."

As of 2026-05-31, defect has proven out and investigate is dead code:

| Skill | Last 9 days | Lifetime telemetry |
|---|---|---|
| `/CJ_goal_defect` | 5 runs, 1 green ship (D000026 / v5.0.14 / PR #184) | 5 runs |
| `/CJ_goal_investigate` | 0 runs | 4 runs (last on 2026-05-21) |

TODOS:37 has tracked the retirement since 2026-05-21 with a gate ("`/CJ_goal_defect`
earns ≥1–2 real green ships first"). The gate is met. TODOS:37 prescribes
the route: `/CJ_goal_todo_fix "retire CJ_goal_investigate"`. That route halted
at the sensitive-surface preflight (skill-work always hits this; cannot
auto-proceed under bash). The user re-routed via `/CJ_goal_feature` for proper
scaffolding.

## What Makes This Cool

Closes F000027's "collapse to two intent-named verbs" thesis to its final
shape. After this PR, the workbench has exactly two `CJ_goal_*` canonicals
(`feature`, `defect`), one TODOS-bridge (`todo_fix`), one internal pipeline
engine (`personal-pipeline`), and four deprecation shims (`CJ_goal_run`,
`CJ_goal_auto`, `cj_goal_feature`, `cj_goal_defect`) — soon five with
`CJ_goal_investigate`. The skill-routing surface gets simpler; the v6.0.0
sunset wave (TODOS:47) becomes a single coordinated PR that removes all five
shims together.

## Constraints

- **PR-stop semantics required** ([[project_workbench_auto_deploy_unsafe]]). Sensitive-surface change touching catalog + manifest + multiple SKILL.md files. /ship Gate #2 stays human.
- **Implementer will hit a sensitive-surface AUQ at `/CJ_implement-from-spec`** (catalog edit, SKILL.md rewrite). Propose-and-confirm is the expected disposition — NOT a halt. The /CJ_personal-pipeline AUQ surfaces once at orchestrator level; auto-confirm or surface to operator.
- **D-id args MUST be rejected by the shim.** Bare `D-id` forwarding to `/CJ_goal_defect` would slug it as a description and mint a new D-id — corrupting work-item tracking.
- **No code change to `/CJ_goal_defect`** — it stays exactly as it is. Retirement is one-sided.
- **No fix for F000031 case-collision in this PR** — separate defect, separate scope (Premise P4).
- **Workbench-only scope** ([[feedback_workbench_scope]]). Downstream consumers (portfolio repo PR #138) are out of scope.
- **Two NAMED audit surfaces under doc/** (workbench convention): `doc/PHILOSOPHY.md` AND `doc/ARCHITECTURE.md`. Both must be audited for /CJ_goal_investigate drift, NOT just PHILOSOPHY.md. `/document-release` Step 2 will flag any leak; better to catch it in this PR.

## Approaches Considered

### Approach A: F000031-pattern relocation with archive
Move `skills/CJ_goal_investigate/ → deprecated/CJ_goal_investigate/` via `git
mv` (preserves pipeline.md + scripts/ as archival reference under deprecated/).
Overwrite SKILL.md at the new location with the shim (banner + routing to
`/CJ_goal_defect` with D-id rejection). Catalog `files` array TRIMMED to a
single path: `["deprecated/CJ_goal_investigate/SKILL.md"]` — matching the
`cj_goal_feature` + `cj_goal_defect` shim catalog entries (the archival
pipeline.md + scripts/ live in the dir but are NOT catalog-registered,
NOT deployed). Status: deprecated, description refresh to the run/auto
deprecation banner pattern. Touch all six doc/routing surfaces (CLAUDE.md,
rules/skill-routing.md, doc/PHILOSOPHY.md, doc/ARCHITECTURE.md, README.md
via generate-readme.sh, TODOS:47 update + TODOS:81 close + CHANGELOG).
VERSION via check-version-queue.sh.

Effort: M (human: ~30 min / CC: ~10 min).
Risk: Low — preserves history; matches current workbench Deprecated convention.

### Approach B: S000060-pattern in-place
Leave dir at `skills/CJ_goal_investigate/`. Overwrite SKILL.md with shim;
pipeline.md + scripts/ stay in place (unused). Catalog `files` stays the same;
only `status` flips to deprecated. Same routing/CLAUDE.md/tombstone/VERSION
updates as A.

Effort: S (human: ~20 min / CC: ~7 min).
Risk: Low — but creates new documented inconsistency with CLAUDE.md "Deprecated
skills convention" (deprecated skills are supposed to live at `deprecated/`).

### Approach C: F000031-pattern relocation + aggressive strip
Same as A, but delete `pipeline.md + scripts/` under
`deprecated/CJ_goal_investigate/`. Only the shim SKILL.md survives. Old
pipeline recoverable from git history if needed.

Effort: M (human: ~30 min / CC: ~10 min).
Risk: Low-Medium — slightly destructive; recovery slower than just changing
one path.

## Recommended Approach

**Approach A — F000031 relocation + archive.** Matches current workbench
convention (no precedent ambiguity for future maintainers). Preserves the old
pipeline.md + scripts/ under `deprecated/CJ_goal_investigate/` as archival
reference — quick restore if a downstream consumer (e.g. portfolio repo)
surfaces a need. Larger diff than B, but the diff is structural moves (cheap
to review) plus standard catalog/doc edits.
