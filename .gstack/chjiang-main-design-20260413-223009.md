## Problem Statement

Five format inconsistencies in the work item system:

1. **No type subfolders.** All work items live flat under `work-items/`. Defects, features, tasks, and stories should be grouped: `work-items/defects/`, `work-items/features/`.
2. **Inconsistent ID prefix on artifacts.** Features are mostly ID-prefixed (`F000001_TRACKER.md`, `S000001_PRD.md`) but `milestones.md` is not. Defects have no prefixes at all: `TRACKER.md`, `RCA.md`, `test-plan.md`. All artifacts should be prefixed: `D000001_TRACKER.md`, `F000001_milestones.md`, etc.
3. **Phase 2 gate text too prescriptive.** Defect tracker template says "Fix committed with regression test". Not every fix has an automated regression test. Should be "Fix committed".
4. **D000001 tracker not closed.** Fix shipped in #28 but `status: active`. Should be closed.
5. **D000001 test-plan not closed.** All test cases still "Pending", status "Draft". Fix verified and merged.

## What Makes This Cool

This is the system debugging itself. The first defect (D000001) exposed format bugs in the defect workflow. Fixing these makes every future work item structurally consistent from the start.

## Constraints

- Must update `rules/work-items.md` (repo + global at `~/.claude/rules/work-items.md`)
- Must update `artifact-manifests.json` filename fields to `{ITEM_ID}_` prefix pattern and update placement values from `"root"` to type subfolder names
- Must update `templates/tracker-defect.md` Phase 2 gate text
- Must migrate all existing work items to new directory structure
- Must close out D000001 tracker and test-plan
- Must update `/docs check` placement validation in `skills/docs/check.md` — currently enforces depth-1 for `"root"` placement, needs to handle type-subfolder intermediate directory
- Standalone user-stories and tasks (not under a feature) must have a defined location — either `work-items/stories/`, `work-items/tasks/`, or remain flat at `work-items/`

## Premises

1. **Type subfolders are the right organizational move.** Groups items by type at top level, reduces clutter as the repo grows. Confirmed.
2. **All artifacts should be ID-prefixed consistently.** Features already do this. Defects should too. Confirmed.
3. **"Fix committed with regression test" is too prescriptive.** Remove "with regression test". Confirmed.
4. **D000001 must be closed.** Fix shipped in #28. Tracker and test-plan should reflect this. Confirmed.
5. **This is two actions: close D000001 + create/implement D000002.** Confirmed.

## Approaches Considered

### Approach A: Forward-fix only
Update rules, manifest, and defect template for future items. Don't touch existing work items.

Effort: S | Risk: Low
Pros: Smallest diff, no risk of breaking references.
Cons: Leaves D000001 and F000001/F000002 in old format. Mixed conventions. Completeness: 4/10.

### Approach B: Full migration (chosen)
Fix all templates/rules + migrate every existing work item to new structure + rename artifacts + close D000001.

Effort: M-L | Risk: Medium (placement logic in /docs check needs updating)
Pros: 100% consistency. No legacy format. Clean starting point.
Cons: More files touched. /docs check placement validation must be updated. Effort is higher than just git mv due to check.md rework.
Reuses: git mv for moves, validate.sh to catch regressions. Completeness: 9/10.

## Recommended Approach

**Approach B: Full migration.** Zero legacy inconsistency is worth the extra work. The effort is more than just git mv — `/docs check` placement logic needs updating across multiple sections (placement validation, stray detection, tree rendering, graph paths, report). Roughly 1-2 hours including check.md rework and verification.

### Implementation plan

**Part 1: Close D000001**
1. Update `work-items/D000001_milestones_artifact_placement/TRACKER.md`: set `status: closed`, check Phase 1 and Phase 2 gates, add PR link, add log entry
2. Update `work-items/D000001_milestones_artifact_placement/test-plan.md`: set `status: Complete`, mark all 10 test cases as "Pass", check verification steps, fill environments table
3. Update `work-items/D000001_milestones_artifact_placement/RCA.md`: already complete, no changes needed

**Part 2: Update templates and rules**
4. `templates/tracker-defect.md` line 42: change `Fix committed with regression test` to `Fix committed`
5. `rules/work-items.md`: update directory structure to show type subfolders:
   ```
   work-items/
     features/{slug}/
     defects/{slug}/
     (user-stories and tasks nest inside their parent feature)
   ```
   Standalone user-stories/tasks (not under a feature) are not expected in this repo. If created, they nest inside a feature per the hierarchy rules. No separate `stories/` or `tasks/` subfolders needed.
6. `artifact-manifests.json`:
   - Keep `filename` values as base names (`"TRACKER.md"`, `"RCA.md"`, etc.). The `{ITEM_ID}_` prefix is a scaffolding-time convention applied by the rules in `work-items.md`, not encoded in the manifest. This avoids changing `/docs check` filename normalization logic (Step 8 of check.md strips ID prefixes to match against base names — that continues to work as-is).
   - Update `placement` section: `"feature": "features"`, `"defect": "defects"` (was `"root"` for both). `"user-story": "feature"` and `"task": "user-story"` remain unchanged (they nest inside parents).
7. Sync global rules: `~/.claude/rules/work-items.md`

**Part 3: Migrate existing work items**
8. Create type subfolders: `work-items/features/`, `work-items/defects/`
9. Move features:
   - `git mv work-items/F000001_workflow_alpha/ work-items/features/F000001_workflow_alpha/`
   - `git mv work-items/F000002_system_health_v1/ work-items/features/F000002_system_health_v1/`
   - Rename unprefixed artifact: `git mv milestones.md F000001_milestones.md` (inside F000001)
10. Move defects:
    - `git mv work-items/D000001_milestones_artifact_placement/ work-items/defects/D000001_milestones_artifact_placement/`
11. Rename D000001 artifacts with ID prefix:
    - `git mv TRACKER.md D000001_TRACKER.md`
    - `git mv RCA.md D000001_RCA.md`
    - `git mv test-plan.md D000001_test-plan.md`

**Part 4: Scaffold D000002**
12. Create `work-items/defects/D000002_workitem_format_consistency/` with:
    - `D000002_TRACKER.md` (from tracker-defect.md template, updated gate text)
    - `D000002_RCA.md` (from doc-RCA.md template)
    - `D000002_test-plan.md` (from doc-test-plan.md template)

**Part 5: Update /docs check placement logic**
13. `skills/docs/check.md`: update placement validation to handle type subfolders. Specific sections affected:
    - Step 15c (placement check): currently enforces `"root"` = direct child of `work-items/`. Must now resolve `"features"` and `"defects"` placement values as valid type-subfolder containers. A defect at `work-items/defects/D000001_*/` should pass placement check when `placement.defect = "defects"`.
    - Step 15d (stray directory detection): must allowlist `features/` and `defects/` directories under `work-items/` as valid containers (they have no TRACKER.md themselves).
    - Step 16b (tree report): path rendering must account for the extra nesting level.
    - Step 17 (graph artifact): `path` fields in the JSON graph change to include the type subfolder.
    - Step 19 (report): inherits all path changes from above.

**Part 6: Verify**

16. Verify tree report renders correctly with new structure
