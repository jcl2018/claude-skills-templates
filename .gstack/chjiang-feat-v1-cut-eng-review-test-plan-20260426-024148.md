## Edge Cases

- **OS junk in mirror dirs**: `find -type f` plus `*.md` glob in the
  MIRROR_SPECS pattern keeps `.DS_Store` (macOS) and `Thumbs.db`
  (Windows) out of the check. Verified by including a synthetic
  `.DS_Store` in a tmpdir test case and asserting the check ignores it.
- **CRLF normalization** (existing test, lines 1480-1497) — keep passing
  after bundle expansion. Mutate a NEW dir file to CRLF, doctor still
  reports clean (sha256_file's text-file branch handles this).
- **Empty mirror dir** — if source dir exists but has zero files, the
  check should warn "no files found" rather than silently pass.
- **install on target with pre-existing `.github/reference/`** — current
  installer policy from F000004 v1 Risks-table row 4 (refuse to clobber
  non-bundle files without `--overwrite`) extends to the new dirs
  unchanged; no new test needed for this since install logic itself is
  unchanged. Documented in S000010_ARCHITECTURE.

## Critical Paths

- **Bundle sync end-to-end**: PR opens → CI runs `validate.sh` → Error
  check 10 fails on any drift across the 7 MIRROR_SPECS entries → blocks
  merge. This is the primary safety net the realignment is built on.
- **Bundle install end-to-end**: developer runs `python
  scripts/copilot-deploy.py install <target>` on the work box → all
  bundle artifacts land under `<target>/.github/` → doctor reports
  clean → developer uses Copilot Chat with full bundle context.
- **Drift fix regression**: `valid-feature-dir/TRACKER.md` content is
  changed from bundle-version to source-version. **Critical** — verify
  `/personal-workflow check` and `/company-workflow validate` still
  consume the updated fixture without breakage. The fixture is itself
  consumed by the validators it tests.
