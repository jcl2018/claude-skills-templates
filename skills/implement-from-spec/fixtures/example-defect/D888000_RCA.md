## Symptom

The fixture's expected greeting file `output/fixed.txt` is missing. Any consumer expecting the canonical content `Hello from defect fix\n` cannot find it.

This is a synthetic fixture; the symptom is "file simply doesn't exist."

## Reproduction Steps

1. Inspect `skills/implement-from-spec/fixtures/example-defect/output/`.
2. **Observe:** `fixed.txt` is absent.

**Environment:** synthetic; not OS-dependent. The fixture's canonical state is "file absent" (cleanup script removes it after each dogfood run).

## Root Cause

**Root cause:** The file `output/fixed.txt` has never been written; the fixture's canonical state has it absent. This is intentional fixture state — the dogfood produces the file, and cleanup removes it.

**Location:** `skills/implement-from-spec/fixtures/example-defect/output/fixed.txt`

## Affected Components

| Component | File/Module | Impact |

| Greeting output | `skills/implement-from-spec/fixtures/example-defect/output/fixed.txt` | File must be created with exact content per test-plan |

## Fix Description

Write `output/fixed.txt` (relative to the fixture dir) with content `Hello from defect fix\n` (one line, trailing newline). One Write call, no Read needed (file is NEW).
