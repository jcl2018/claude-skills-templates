# Fixture: example-design-doc

Happy-path full-pipeline run.

## What it tests

Single keystroke `/personal-pipeline <design-doc>` produces a green
end_state on a clean-slate design doc:
- Step 2 branch (d) — clean slate
- Step 3 — Phase 1 scaffold subagent succeeds
- Step 4 — post-scaffold gate green; user approves shape
- Step 5 — Phase 2 implement subagent succeeds (no sensitive surface, no taste forks)
- Step 6 — post-implement gate green
- Step 7 — Phase 3 qa subagent succeeds
- Step 8 — post-QA gate green
- Step 9 — telemetry written, end_state=green

## Setup

1. Create a synthetic design doc under `~/.gstack/projects/`:

   ```bash
   PROJECTS=~/.gstack/projects/jcl2018-claude-skills-templates
   FIXTURE_DOC="$PROJECTS/fixture-example-design-$(date +%s).md"
   cat > "$FIXTURE_DOC" <<'EOF'
   # Design: Add a no-op marker file under tests/

   Generated by /office-hours on 2026-05-09 (FIXTURE — not a real design)
   Branch: feat/personal-pipeline
   Status: APPROVED
   Mode: Builder

   ## Problem Statement
   Need a no-op file at `tests/.fixture-marker` to verify the orchestrator pipeline.

   ## Recommended Approach
   Touch a single file `tests/.fixture-marker` with content "fixture".

   ## Components Affected
   | Component | Repo | Change Type | Description |
   |-----------|------|------------|-------------|
   | tests/.fixture-marker | claude-skills-templates | New | One-line file |

   ## Open Questions
   None.

   ## Success Criteria
   - File exists at tests/.fixture-marker
   EOF
   ```

2. Verify your branch state (orchestrator runs in `feat/*` per F000010 pattern).

## Invocation

```bash
/personal-pipeline "$FIXTURE_DOC"
```

## Expected outcome

- `~/.gstack/analytics/personal-pipeline.jsonl` gets one new line with `end_state: green`
- A new work-item dir under `work-items/tasks/` (or wherever the type-detection lands) with all required artifacts
- `tests/.fixture-marker` file exists with content `fixture`
- Tracker journal contains `[orchestrator]`, `[impl-pass]`, and `[qa-pass]` entries
- Final summary in chat shows `PIPELINE COMPLETE: end_state=green`

## Cleanup

```bash
rm -f tests/.fixture-marker "$FIXTURE_DOC"
rm -rf work-items/tasks/<the-fixture-work-item-dir>
# Optionally remove the telemetry line via tail/head; the orchestrator's
# sunset check counts these lines, so leaving them in distorts the trip-wire.
```
