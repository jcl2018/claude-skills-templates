# Fixture: regression-partial-write-halt

Step 2 branch (c) — partial-write recovery halt.

## What it tests

Orchestrator detects scaffold-crashed-mid-write state (work-item dir exists
and references the design doc, but design-doc footer is absent — implying
scaffold halted between its Step 9 boundary check and Step 12 footer-write).
Without this branch, re-running would dispatch Phase 1 and risk a duplicate
dir via the TODOS.md:26 idempotency hole in `/scaffold-work-item`.

## Setup

1. Pick or create a synthetic design doc:

   ```bash
   PROJECTS=~/.gstack/projects/jcl2018-claude-skills-templates
   FIXTURE_DOC="$PROJECTS/fixture-partial-write-$(date +%s).md"
   cat > "$FIXTURE_DOC" <<'EOF'
   # Design: Partial-write fixture (do not implement)

   Generated by /office-hours on 2026-05-09 (FIXTURE)
   Branch: feat/personal-pipeline
   Status: APPROVED
   Mode: Builder

   ## Problem Statement
   Synthetic for fixture; do not implement.

   ## Recommended Approach
   No-op (this fixture should halt at Step 2 branch (c)).
   EOF
   ```

2. Hand-scaffold a partial work-item dir that references this design doc:

   ```bash
   FIXTURE_DIR=work-items/tasks/personal-pipeline-fixture/T999998_partial_write_fixture
   mkdir -p "$FIXTURE_DIR"
   cat > "$FIXTURE_DIR/T999998_TRACKER.md" <<EOF
   ---
   name: "Partial-write fixture"
   type: task
   id: "T999998"
   status: active
   created: "2026-05-09"
   updated: "2026-05-09"
   parent: ""
   repo: "$(git rev-parse --show-toplevel)"
   branch: "feat/personal-pipeline"
   blocked_by: ""
   ---
   <!-- Source design: $FIXTURE_DOC -->
   ## Lifecycle
   ### Phase 1: Track
   <!-- intentionally minimal -->
   EOF
   # NOTE: Intentionally missing test-plan.md — that's what makes this "partial"
   ```

3. Confirm the design doc has NO footer:

   ```bash
   grep -c "Status: SCAFFOLDED" "$FIXTURE_DOC"  # should print 0
   ```

## Invocation

```bash
/personal-pipeline "$FIXTURE_DOC"
```

## Expected outcome

- Step 2 takes branch (c). Stdout shows: "Halt: scaffold appears to have crashed between its Step 9 boundary check and Step 12 footer-write. Partial dir(s) reference this design doc: <FIXTURE_DIR>/T999998_TRACKER.md"
- AskUserQuestion appears with three options (delete partial / hand-write footer / abort)
- No Phase 1 subagent dispatch happens
- Telemetry: end_state=user_aborted (assuming you Cancel) or no telemetry line if Halt-before-write semantics apply

## Cleanup

```bash
rm -f "$FIXTURE_DOC"
rm -rf work-items/tasks/personal-pipeline-fixture/
```
