## Problem Statement

Provide a one-shot greeting file at a known path with exact content. Used as a fixture target for `/CJ_implement-from-spec`.

## Mental Model

```
SPEC says:
  path → skills/CJ_implement-from-spec/fixtures/example-user-story/output/greeting.txt
  content → Hello from /CJ_implement-from-spec\n

/CJ_implement-from-spec writes the file → file exists with exact content
```

## Acceptance Criteria

### Story #1: Single file written [core]

```
GIVEN  the fixture directory has no prior `output/greeting.txt`
WHEN   /CJ_implement-from-spec runs on this user-story
THEN   `output/greeting.txt` exists relative to the fixture dir
       AND its content equals "Hello from /CJ_implement-from-spec\n" exactly
```

### Story #2: Trivial-mode classification [resilience]

```
GIVEN  the SPEC's Components Affected lists 1 file (greeting.txt) — no sensitive surface
WHEN   /CJ_implement-from-spec is invoked WITHOUT --auto
THEN   Step 6.5 classifies TRIVIAL=true
       AND if --auto is passed, MODE=auto and the propose preview is skipped
       AND if --auto is omitted, MODE=propose and the preview fires
```

## Architecture

```
SPEC.md
   │
   ▼
parse Components Affected
   │  greeting.txt (NEW)
   ▼
plan: one Write call, target path known, content known
   │
   ▼
mode resolution: TRIVIAL=true, SENSITIVE=false → --auto allowed
   │
   ▼
Write greeting.txt
   │
   ▼
update tracker
   │
   ▼
boundary check
```

### Components Affected

| Component | Repo | Change Type | Description |
|-----------|------|------------|-------------|
| `skills/CJ_implement-from-spec/fixtures/example-user-story/output/greeting.txt` | claude-skills-templates | New | Greeting file with exact content per Story #1 |

### Data Flow

1. /CJ_implement-from-spec reads SPEC's Components Affected
2. Plans 1 Write operation (no Read needed — file is NEW)
3. (If --auto AND TRIVIAL AND not SENSITIVE): write directly
4. (Else): preview + AUQ; on Apply, write
5. Write `Hello from /CJ_implement-from-spec\n` to the target path
6. Update tracker

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| File location | Inside fixture dir (`output/greeting.txt`) | `/tmp/greeting.txt` | Fixture self-containment; cleanup is `rm output/greeting.txt` not a global tmpfile |
| Content shape | Exact one-line greeting | Multi-line / templated | Easier byte-equality check |
