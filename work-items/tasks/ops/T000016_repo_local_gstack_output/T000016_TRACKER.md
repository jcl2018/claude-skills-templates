## Insights

<!-- Non-obvious findings worth remembering. Things that surprised you,
     patterns discovered, or context that future readers will need. -->

- **gstack honors `GSTACK_HOME` only partially.** `gstack-paths` resolves it correctly, but ~30 SKILL.md files hardcode `~/.gstack/projects/$SLUG/...` in their bash blocks instead of using `${GSTACK_HOME:-$HOME/.gstack}`. Setting `GSTACK_HOME=<repo>/.gstack` alone leaves you in a split-output state. The symlink works around this by redirecting at the filesystem layer where every code path lands.
- **Symlinking the project dir is verified safe.** `grep -rn '! -L' ~/.claude/skills/gstack/` returns ~20 hits, ALL targeting `.claude/skills/gstack` (vendored-install detection in skill preambles), zero on `~/.gstack/projects/$SLUG/`. Re-run this grep after `gstack-upgrade` to catch any new checks added upstream.
- **Symlink target must be the MAIN repo, not a worktree.** Worktrees share the slug; the symlink at `~/.gstack/projects/<slug>` can only point to one physical location. Resolving via `git rev-parse --path-format=absolute --git-common-dir` then `dirname` works from either main or worktree. Worktrees writing gstack output land in main's working tree — `git status` from the worktree won't show those files. Commit from main, or `git -C <main-repo> add .gstack/`.
- **Two parallel design surfaces now.** `.gstack/` (lateral/exploratory: /office-hours, /plan-ceo-review, /context-save) and `work-items/` (structured per-feature: TRACKER, DESIGN, ROADMAP, etc.). Different lifecycles, different surfaces — keep them parallel, don't try to merge. Document split in CLAUDE.md.
- **Brain-sync coexists.** gstack's `.brain*` writes target `~/.gstack/.git/`, not `~/.gstack/projects/<slug>/.git/`. The symlink at `~/.gstack/projects/<slug>` doesn't interact with brain-sync. Also: `artifacts_sync_mode: off` for this user, so brain-sync isn't even running.
