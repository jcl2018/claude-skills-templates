## Problem Statement

Workbench owners running `/personal-pipeline` repeatedly (back-to-back work
items, SPEC-heavy work with multiple Tradeoff rows, sensitive-surface SPECs)
hit 5+ AskUserQuestion gates per run. The orchestrator removed the manual
scaffolding/implement/QA labor, but the human-decision labor stayed. Each
pipeline run is 5-30 minutes wall-clock with N inline interruptions.

The user is the workbench owner (jcl2018) running /personal-pipeline against
their own TODOS.md backlog. They want one keystroke from design doc to
"ready to /ship" review, with close-call decisions surfaced at one terminal
gate instead of inline.

## Mental Model

`--auto` is a mode flag on /personal-pipeline. Mode dispatch happens at
Step 1 (input validation). When `$AUTO_MODE=true`, every existing AUQ is
wrapped: classify (Mechanical / Taste / User Challenge sub-types), apply
6 principles, log to `$DECISION_LOG`, continue. Some classifications halt
the pipeline (Halt-at-Gate User Challenge); the rest accumulate into a
single Step 8.5 final approval gate. Manual mode (`$AUTO_MODE=false`,
default) is byte-identical to v1.13.0.

```
[design doc] → [Step 1 parse --auto]
                    │
                    ├── manual: existing v1.13.0 path (byte-identical)
                    │
                    └── auto: existing 9 steps with auto-decision wrapper
                              ├── Mechanical → log silent, continue
                              ├── Taste → log + recommend, continue, surface at 8.5
                              ├── User Challenge — Approve-with-surfacing
                              │      → auto-pick "approve" forward, log, surface at 8.5
                              ├── User Challenge — Halt-at-Gate
                              │      → log, halt at gate, end_state=halted_at_gate
                              ├── Boundary check red → halt-regardless (no log)
                              └── Subagent crash → halt-regardless (no log)
                              [Step 8.5 final gate, AUQ Approve | Abort]
```

## Acceptance Criteria

### Story #1: Manual mode unchanged [core]

```
GIVEN /personal-pipeline v1.13.0 baseline behavior
WHEN  user invokes `/personal-pipeline <design-doc-path>` without --auto
THEN  same Bash blocks execute, same subagent prompts dispatch, same AUQs
      fire at Steps 4/5.2/5.3/6/8/9 as v1.13.0; spot-check diff vs baseline
      shows manual code path is unchanged; all auto-mode logic gated on
      $AUTO_MODE=true.
```

### Story #2: Auto mode green path [core]

```
GIVEN a green-path SPEC (no Tradeoff rows, no sensitive surface, QA green)
WHEN  user invokes `/personal-pipeline --auto <design-doc-path>`
THEN  pipeline runs scaffold/implement/QA without per-gate AUQs; reaches
      Step 8.5 empty-state short-circuit (writes [auto-pipeline-clean] to
      tracker journal, skips AUQ); exits with end_state=green.
```

### Story #3: Sensitive-surface surfaced at 8.5 [core]

```
GIVEN a SPEC whose Components Affected table includes skills-catalog.json
WHEN  user invokes `/personal-pipeline --auto <design-doc-path>`
THEN  Step 5.2 auto-picks "approve" forward (subagent proceeds with
      catalog change); logs `user_challenge_approved` line to $DECISION_LOG
      with files_affected=["skills-catalog.json", ...], reasoning, run_id;
      Step 8.5 surfaces the decision in the AUQ's user_challenge_approved
      list with full context block.
```

### Story #4: Boundary check red halts auto [resilience]

```
GIVEN /personal-workflow check returns red on the freshly-scaffolded dir
WHEN  user invokes `/personal-pipeline --auto <design-doc-path>`
THEN  auto mode halts at the boundary check (Step 4); no $DECISION_LOG
      write for this gate; tracker journal gets [gate-red] entry;
      telemetry written with end_state=halted_at_gate; mode=auto.
```

### Story #5: Subagent crash halts auto [resilience]

```
GIVEN a Phase 1 scaffold subagent that produces no RESULT line (crash)
WHEN  user invokes `/personal-pipeline --auto <design-doc-path>`
THEN  auto mode halts; no $DECISION_LOG write; tracker journal gets
      [subagent-crash] entry; telemetry written with
      end_state=subagent_crashed; mode=auto.
```

### Story #5b: Halt-at-Gate User Challenge logs before halting [observability]

```
GIVEN Step 8 post-QA red (E2E ambiguous)
WHEN  user invokes `/personal-pipeline --auto <design-doc-path>` and Phase 3
      QA produces a red RESULT
THEN  auto mode writes a user_challenge_halt line to $DECISION_LOG with
      gate_id="post-qa", recommendation="abort", reasoning, files_affected,
      run_id; then halts; tracker journal [gate-red]; telemetry
      end_state=halted_at_gate.
```

### Story #6: Step 8.5 summary format [observability]

```
GIVEN $DECISION_LOG has 3 mechanical, 2 taste, 1 user_challenge_approved
      lines for this run_id
WHEN  Step 8.5 fires
THEN  AUQ summary shows D-numbered header, project/branch line, ELI10,
      Stakes, Recommendation, Note (kind-difference), Decision summary
      with $N=3 mechanical (collapsed count), $M=2 taste (full list with
      reasoning), $K=1 user_challenge_approved (full context block per
      gstack format), Log path; options A/B with ≥40-char ✅/❌ Pros/Cons
      and Net line.
```

### Story #7: Empty-state short-circuit [usability]

```
GIVEN $DECISION_LOG has only mechanical entries (no taste, no
      user_challenge_approved) for this run_id
WHEN  Step 8.5 fires
THEN  no AUQ is shown; tracker journal gets [auto-pipeline-clean] entry;
      pipeline continues to Step 9; end_state=green.
```

### Story #8: Abort prints what to revert [usability]

```
GIVEN Step 8.5 fires with $K=1 user_challenge_approved (catalog change)
WHEN  user picks Abort + show what to revert
THEN  printed output groups files_affected by gate ("post-implement Phase 2
      via user_challenge_approved at gate sensitive-surface-catalog:
      skills-catalog.json, README.md"); end_state=user_aborted; pipeline
      state preserved (work-item dir + repo files NOT auto-reverted; user
      runs `git restore` themselves).
```

### Story #9: Telemetry mode field [core]

```
GIVEN any pipeline run completes (any end_state)
WHEN  Step 9.1 writes telemetry to ~/.gstack/analytics/personal-pipeline.jsonl
THEN  the JSON line includes a "mode" field with value "auto" or "manual"
      matching the invocation flag.
```

### Story #10: Sunset trip-wire pooled [resilience]

```
GIVEN telemetry contains a mix of auto and manual mode lines, totaling 6+
      lines for this skill
WHEN  invocation 6 (or every 5 thereafter) fires Step 9.2 sunset checkpoint
THEN  the trip-wire counts halted_at_gate across both modes; AUQ surfaces
      the prior 5 lines (mode-mixed); recommendation honors the existing
      threshold logic.
```

### Story #11: Multi-classification synthetic test [observability]

```
GIVEN a synthetic SPEC at fixtures/regression-auto-multi-classification/
      with 1 sensitive surface (skills-catalog.json) + 2 Tradeoff taste-fork
      rows + a passing QA path
WHEN  user invokes `/personal-pipeline --auto <fixture-design-doc>`
THEN  Step 8.5 summary shows $N≥1 mechanical, $M=2 taste, $K=1
      user_challenge_approved; counts match `jq -c "select(.run_id ==
      \"$RUN_ID\")" $DECISION_LOG | jq -s 'group_by(.classification) |
      map({(.[0].classification): length}) | add'` output.
```

## Architecture

```
                   ┌─────────────────────────────────────────────┐
                   │ Step 1: parse --auto, set $AUTO_MODE,        │
                   │         set $DECISION_LOG (constant path)    │
                   └─────────────────────┬───────────────────────┘
                                         │
                  $AUTO_MODE=false       │       $AUTO_MODE=true
              ┌──────────────────────────┴──────────────────────────┐
              │                                                     │
              ▼                                                     ▼
    [v1.13.0 manual path]                  ┌────────────────────────────────────┐
    [unchanged Steps 2-9]                  │ Auto Mode Overlay (top of pipeline)│
                                           │  - 6 principles (P1-P5 + new P6)    │
                                           │  - Decision classification          │
                                           │  - $DECISION_LOG schema             │
                                           │  - Halt-regardless exceptions       │
                                           └─────────────┬──────────────────────┘
                                                         │
                                  ┌──────────────────────┴──────────────────────┐
                                  │ Each existing AUQ wraps:                    │
                                  │   classify → apply 6 principles → log →     │
                                  │   continue (or halt for User Challenge —    │
                                  │   Halt-at-Gate)                             │
                                  └──────────────────────┬──────────────────────┘
                                                         │
                                                         ▼
                                  ┌──────────────────────────────────────────────┐
                                  │ Step 8.5: Final Approval Gate (auto only)    │
                                  │  - Filter $DECISION_LOG by run_id            │
                                  │  - Empty-state? short-circuit to Step 9       │
                                  │  - Otherwise: render gstack-format AUQ        │
                                  │  - Approve / Abort+show-what-to-revert        │
                                  └──────────────────────┬──────────────────────┘
                                                         │
                                                         ▼
                                  ┌──────────────────────────────────────────────┐
                                  │ Step 9: telemetry (mode: auto|manual) +       │
                                  │         sunset checkpoint (pooled across      │
                                  │         modes)                                │
                                  └──────────────────────────────────────────────┘
```

### Data Flow

1. User invokes `/personal-pipeline [--auto] <design-doc-path>`.
2. Step 1 parses positional args; sets `$AUTO_MODE=true|false`; sets `$DECISION_LOG=~/.gstack/analytics/personal-pipeline-auto-decisions.jsonl`; `mkdir -p ~/.gstack/analytics`.
3. Manual mode (`$AUTO_MODE=false`): execute v1.13.0 pipeline path unchanged.
4. Auto mode (`$AUTO_MODE=true`): execute Steps 2-8 with each AUQ wrapped per the per-gate classification table (in DESIGN.md and pipeline.md Auto Mode Overlay).
5. Each wrapped AUQ:
   a. Classify the decision per the table (Mechanical / Taste / User Challenge — Approve-with-surfacing / User Challenge — Halt-at-Gate).
   b. Apply 6 principles to compute the decision and reasoning.
   c. Append a JSON line to `$DECISION_LOG` (jq-emitted, JSON-safe; tagged with `run_id`).
   d. Continue (Mechanical / Taste / Approve-with-surfacing) or halt (Halt-at-Gate).
6. Halt-regardless exceptions (boundary check red, subagent crash) skip `$DECISION_LOG` entirely; tracker journal + telemetry only.
7. Step 8.5 (auto only): filter `$DECISION_LOG` by `run_id`; empty-state → short-circuit to Step 9; otherwise → render gstack-format AUQ with full Pros/Cons; user picks Approve or Abort.
8. Step 9: telemetry write includes `mode: auto|manual`; sunset checkpoint counts both modes pooled.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Implementation shape | --auto flag on existing skill | New /personal-pipeline-auto skill | Premise 1; subagent-wraps-skill broken (S000026) |
| Implementation shape | --auto flag on existing skill | Per-user gstack-config opt-in | Surprise factor on sensitive surfaces; conflicts with F000014 per-invocation-consent intent |
| P6 principle | Bias toward halt-on-doubt | /autoplan's Bias toward action | Higher blast radius (code-mutating pipeline vs plan-review) |
| User Challenge categorization | Split into Approve-with-surfacing vs Halt-at-Gate | Single User Challenge class | Conflated "needs human attention" with "must halt now"; broke the single-final-gate contract |
| Step 8.5 Reject UX | Abort + manual revert | Reject specific decisions + programmatic rollback | Programmatic rollback across mid-pipeline edits is fragile; user does `git restore` |
| `$DECISION_LOG` location | `~/.gstack/analytics/personal-pipeline-auto-decisions.jsonl` (single shared) | Per-run `~/.gstack/projects/$SLUG/personal-pipeline-auto-$RUN_ID.jsonl` | Matches telemetry single-file pattern; no rotation needed; run_id-tagged |
| `$DECISION_LOG` retention | Single shared file, no rotation in v1 | Keep last 20 runs, prune older | YAGNI; revisit if file exceeds 1MB |
| Step 9 sunset checkpoint | Always-AUQ in both modes | Auto-decide in auto mode | Per-invocation visible by design — sunset is the one place auto mode breaks its own contract by design |
