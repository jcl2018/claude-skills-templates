## Problem Statement

The workbench needs a deployable `/personal-pipeline` skill that orchestrates
the 3 existing pipeline skills (scaffold-work-item, implement-from-spec,
qa-work-item) per F000014's design. The skill author is chjiang. The pain
today is three-keystroke ergonomics + uneven trust in upstream skill self-checks
(known idempotency hole in scaffold via TODOS.md:26).

## Mental Model

The orchestrator is a thin coordinator: it does NOT re-implement scaffold/implement/qa logic; it dispatches them as fresh-context Agent subagents and runs independent gates between phases. The orchestrator's own context stays tiny (~5K tokens); each subagent gets a small prompt (≤500 tokens) and returns a short verdict (≤200 tokens, ending with `RESULT: <key>=<value>`).

**Two adjustments locked by the S000026 spike (2026-05-09):**
- **AUQ pre-collection at orchestrator.** AskUserQuestion is unreachable inside Agent subagents in Claude Code 2.1.91 (not in the deferred-tools list). The orchestrator scans the SPEC's Components Affected for sensitive-surface paths AND Tradeoffs for taste-fork rows BEFORE dispatching the implement subagent, AUQs the human up front, then dispatches the subagent with answers threaded into the prompt tail. The subagent never attempts AUQ.
- **Lenient RESULT-line parsing.** Subagents reliably produce RESULT-line content but inconsistently format it (2/5 strict hits in the spike; failures wrapped in markdown blockquotes or code fences). Parser shape: `grep -E 'RESULT: [A-Z_]+=' "$output" | tail -1 | sed 's/^[> ]\+//;s/```//g'`. Strips `>` blockquote prefixes and code fences; finds the line wherever it lands.

```
operator: /personal-pipeline <design-doc-path>
                ↓
   orchestrator skill (Claude reads pipeline.md, follows steps)
                ↓
   Step 1: input validation
   Step 2: pre-scaffold idempotency check (4 branches)
   Step 3: Phase 1 — scaffold subagent (Agent tool, fresh context)
   Step 4: post-scaffold gate
   Step 5: Phase 2 — implement subagent
   Step 6: post-implement gate
   Step 7: Phase 3 — qa subagent
   Step 8: post-qa gate
   Step 9: final summary + telemetry write to ~/.gstack/analytics/personal-pipeline.jsonl
                ↓
   end-state: green | halted_at_gate | user_aborted | subagent_crashed
```

## Acceptance Criteria

### Story #1: SKILL.md + frontmatter [core]

```
GIVEN  the skill is authored and `skills-deploy install` has been run
WHEN   I run `./scripts/validate.sh`
THEN   it exits 0 with no errors mentioning personal-pipeline
AND    `~/.claude/skills/personal-pipeline/SKILL.md` exists with the workbench's frontmatter conventions (name, description, version, allowed-tools)
```

### Story #2: pipeline.md orchestration [core]

```
GIVEN  pipeline.md authored per F000014_DESIGN's 9-step plan
WHEN   I invoke /personal-pipeline on a fresh design-doc that has no Status: SCAFFOLDED footer
THEN   Step 2 takes branch (d) clean-slate, dispatches Phase 1, post-scaffold gate runs, AskUserQuestion confirms shape, Phase 2 runs, post-implement gate runs, Phase 3 runs, post-qa gate runs, telemetry line written
AND    final summary prints WORK_ITEM_DIR and suggests /ship
```

### Story #3: Pre-scaffold idempotency regression [core]

```
GIVEN  F000010's design doc already has a Status: SCAFFOLDED → work-items/features/personal-workflow/F000010_pipeline_skills/ footer
WHEN   I invoke /personal-pipeline on F000010's design doc
THEN   Step 2 takes branch (a), reuses the existing dir, skips Phase 1, runs only post-scaffold gate (which passes), and proceeds to Phase 2
```

### Story #4: Partial-write halt regression [core]

```
GIVEN  a partial scaffold (work-item dir exists with TRACKER but missing DESIGN; design doc has NO footer)
WHEN   I invoke /personal-pipeline on that design doc
THEN   Step 2 takes branch (c), detects the partial dir via tracker grep, halts with manual-cleanup AUQ
AND    no Phase 1 subagent dispatch happens
```

### Story #5: Post-implement gate regression [core]

```
GIVEN  a synthetic implement output that breaks `scripts/validate.sh` (e.g., catalog entry missing required fields)
WHEN   I invoke /personal-pipeline on the corresponding design and reach the post-implement gate
THEN   the orchestrator captures the validate.sh failure to the tracker journal as `[gate-red]`
AND    AskUserQuestion surfaces with abort/fix/override options
AND    end_state in telemetry is halted_at_gate
```

### Story #6: Telemetry [core]

```
GIVEN  the orchestrator has been invoked at least once
WHEN   I tail ~/.gstack/analytics/personal-pipeline.jsonl
THEN   each invocation produced exactly one line with {run_id, design_doc, end_state}
AND    end_state ∈ {green, halted_at_gate, user_aborted, subagent_crashed}
```

### Story #7: Subagent token budgets [resilience]

```
GIVEN  a real pipeline run
WHEN   I inspect the dispatched subagent prompts and returns
THEN   each prompt is ≤500 tokens and each return is ≤200 tokens
```

├── pipeline.md        # 9-step orchestration; the meaty logic
└── fixtures/
    ├── example-design-doc/                       # happy path

    ├── regression-partial-write-halt/            # branch (c)

skills-catalog.json

          source="local", depends.skills=["scaffold-work-item","implement-from-spec","qa-work-item","personal-workflow"]

└── one line per invocation: {run_id, design_doc, end_state}

### Data Flow

1. User invokes `/personal-pipeline <design-doc-path>`. Claude Code loads SKILL.md.
2. SKILL.md preamble runs path resolution; routes to pipeline.md.
3. Step 1: validates input is a design doc under `~/.gstack/projects/`.
4. Step 2: reads the design-doc footer; routes to one of 4 branches. Branch (a/d) → continue; branches (b/c) → halt with AUQ.
5. Step 3: dispatches Agent subagent with `subagent_type: general-purpose`. Subagent invokes `/scaffold-work-item`. Returns `RESULT: WORK_ITEM_DIR=<path>`.
6. Step 4: orchestrator runs `/personal-workflow check`, footer-write-back confirm, multi-story-feature halt branch, AskUserQuestion approve.
7. Step 5: orchestrator scans SPEC for sensitive-surface paths (catalog/manifests/validators) AND Tradeoffs rows with multiple alternatives. AUQs the human up front for any potential decision points (one AUQ per group). Then dispatches the implement subagent with answers threaded into the prompt tail. The subagent runs in `--auto`-equivalent mode (no AUQ attempts; AUQ tool is unreachable in subagent context per S000026 spike). On unexpected escalation the subagent emits `RESULT: ESCALATION_NEEDED=<reason>` for the orchestrator's pre-scan miss path.
8. Step 6: orchestrator runs check + validate.sh.
9. Step 7: dispatches qa subagent. Returns smoke/E2E/Phase2-gates triplet.
10. Step 8: orchestrator parses tracker journal entries; halt on red.
11. Step 9: orchestrator writes summary + telemetry line.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Skill structure | SKILL.md + pipeline.md (two files) | Single SKILL.md (everything inline) | Matches F000010 children's pattern; SKILL.md as compact entry point keeps the slash-command preamble fast |
| Catalog status | experimental | active | Sunset criterion is built into the skill; experimental signals "may be deleted at run 6" |
| Subagent type | general-purpose | custom (e.g., scaffold-runner) | Custom adds infra without clear v1 benefit; revisit if tool-access lockdown becomes load-bearing |
| Telemetry destination | `~/.gstack/analytics/personal-pipeline.jsonl` | tracker journal entries | Analytics is the right surface for cross-run trip-wires; tracker journal is per-work-item |
| Fixture count | 1 happy + 3 regression | 1 happy + N exhaustive | 3 regression maps to F000014's 3 design decisions (idempotency / partial-write / post-implement gate); more is YAGNI for v1 |
| Token budgets | ≤500 prompt, ≤200 return | unbounded | Premise that orchestrator-context-stays-tiny gates the cost story; verifiable on first real run |
| Phase 2 AUQ model | Pre-collect at orchestrator (scan SPEC, AUQ human up front, thread answers into subagent prompt) | Subagent reports `RESULT: AUQ_NEEDED=...` and orchestrator re-AUQs | Locked by S000026 spike: AskUserQuestion is unreachable inside Agent subagents in Claude Code 2.1.91. The original "subagent reports up" pattern can't work because the subagent has no AUQ tool to invoke. Orchestrator-side pre-collection is the only viable shape. |
| RESULT-line parser | Lenient (`grep RESULT: \| tail -1 \| sed strip blockquote+fence`) | Strict (`grep -E '^RESULT: '` against last line only) | Locked by S000026 spike leg (b): subagents wrap RESULT in markdown blockquotes / code fences 60% of trials. Content is reliably present; format is not. Lenient parser caught all 5 spike trials in retro analysis. |
