## Problem Statement

After S000014 lands the new templates + manifest + validator, all 13 historical work items (5 features + 8 user-stories) still match the OLD artifact shape (feature-summary + milestones for features; PRD + ARCHITECTURE + TEST-SPEC for user-stories). Without migration, every `/personal-workflow check` run will emit walls of `[MISSING] ROADMAP.md` and `[MISSING] SPEC.md` and `[MISSING] DESIGN.md` findings — making the validator unusable in practice.

This story sweeps all 13 historical items + F000008 itself to the new shape in the same PR, eliminating the broken intermediate state. After this story lands, `/personal-workflow check work-items/` returns zero findings.

## Mental Model

The migration is mechanical but content-bearing — we're not just renaming files, we're consolidating their content into fewer files:

- **For each feature:** `feature-summary.md` (Scope, Success Criteria, Constituent User-Stories, Out-of-Scope) + `milestones.md` (timeline table + dependency graph) → `ROADMAP.md` (everything from feature-summary + Delivery Timeline section absorbing milestones, with `### Delivery History` sub-section for any historical merge-date entries).
- **For each user-story:** `PRD.md` (Problem, Mental Model, User Stories with P0/P1/P2 tables, Acceptance Criteria, Success Metrics, Out of Scope, Assumptions) + `ARCHITECTURE.md` (Overview, Architecture diagram, Components, Data Flow, API Changes, Dependencies, Risk Assessment, Design Decisions) → `SPEC.md` (PRD's content + ARCHITECTURE's content, with `## Requirements` containing the preserved `### P0/P1/P2` sub-sections so check.md Step 18 keeps working).
- **DESIGN.md backfill:** 8 user-stories that didn't previously have DESIGN.md get a 5-line stub per design Open Question 1 default. If a matching `~/.gstack/projects/` design doc happens to exist for a specific user-story, distill it instead — but don't block on the search.

F000008 self-migrates as part of this sweep — same treatment as the 13 historicals. Order matters: S000014 must land first (so new templates/manifest exist); then S000015 sweeps everything (including F000008) to the new shape.

## Acceptance Criteria

### Story #1: All 5 historical features migrated [core]

```
GIVEN S000014 has landed (new templates + manifest in place)
WHEN  migration of all 5 historical features completes
THEN  for each feature ID in {F000001, F000002, F000004, F000005, F000006}:
        - {ID}_ROADMAP.md exists
        - {ID}_feature-summary.md does NOT exist
        - {ID}_milestones.md does NOT exist
        - {ID}_DESIGN.md exists with `Roadmap:` cross-link (not `Milestones:`)
        - {ID}_TRACKER.md unchanged
```

### Story #2: All 8 historical user-stories migrated [core]

```
GIVEN S000014 has landed
WHEN  migration of all 8 historical user-stories completes
THEN  for each user-story ID in {S000001, S000006-S000010, S000012, S000013}:
        - {ID}_SPEC.md exists with `## Requirements` containing `### P0/P1/P2` sub-sections (where the original PRD had them)
        - {ID}_DESIGN.md exists (stub or distilled)
        - {ID}_PRD.md does NOT exist
        - {ID}_ARCHITECTURE.md does NOT exist
        - {ID}_TEST-SPEC.md frontmatter has `spec: SPEC.md` (not `prd:` + `architecture:`)
```

### Story #3: F000008 self-migrated [core]

```
GIVEN S000014 has landed
WHEN  F000008's self-migration is complete
THEN  work-items/features/personal-workflow/F000008_tracker_recut/ contains:
        - F000008_TRACKER.md
        - F000008_DESIGN.md (with `Roadmap:` link)
        - F000008_ROADMAP.md
        AND no feature-summary or milestones
AND   each child user-story dir (S000014_*, S000015_*, S000016_*) contains:
        - S{ID}_TRACKER.md
        - S{ID}_DESIGN.md (stub or condensed from parent design)
        - S{ID}_SPEC.md
        - S{ID}_TEST-SPEC.md
        AND no PRD or ARCHITECTURE
```

### Story #4: Step 18 traceability still passes [core]

```
GIVEN a migrated user-story that originally had P0 stories with AC mappings (e.g., S000012)
WHEN  I run `/personal-workflow check work-items/.../S000012_deprecated_status_semantics/`
THEN  Step 18 reports `[PASS] All P0 stories have TEST-SPEC coverage`
AND   no [UNTESTED] findings for stories that previously had coverage
```

### Story #5: Full work-items/ tree passes validation [core]

```
GIVEN all migrations (5 features + 8 user-stories + F000008 self) are complete
WHEN  I run `/personal-workflow check`  (no path arg)
THEN  report shows:
        - 0 [MISSING] findings
        - 0 [DRIFT] findings
        - 0 [EXTRA] findings (unless a non-required artifact was kept; advisory only)
        - 0 [UNTESTED] findings
        - tree walk completes without errors
```
