## Problem Statement

Beyond the templates + manifest + check.md changes (S000014) and the historical work-item migration (S000015), the artifact-set re-cut also touches a long tail of supporting surfaces: example docs that show "what a filled-in PRD looks like" (now should show SPEC), fixtures that the validator's tests consume (assertion sets change), and repo-level docs that name the artifact set in their narrative (CONTRIBUTING, PHILOSOPHY, template-registry, scripts).

Without this story, six concrete surfaces stay stale and either (a) confuse new contributors who follow examples that don't match the templates, (b) break tests that consume fixtures with wrong expected-MISSING assertions, or (c) silently propagate stale artifact names to anyone who reads CONTRIBUTING / PHILOSOPHY / template-registry. The third failure mode is especially risky — `template-registry.json` is read by external consumers (e.g., `/personal-workflow` deployment in other repos via `skills-deploy`) so a stale `doc_types` array would break downstream tooling.

## Mental Model

Three subsurface clusters, processed in this order to minimize coupling:

1. **Examples** (`skills/personal-workflow/examples/`) — visible to humans only. 8 files affected: 4 deleted, 2 created, 2 rewritten. Updates after S000014 lands.
2. **Fixtures** (`skills/personal-workflow/fixtures/`) — consumed by `scripts/test.sh` for validator regression tests. ~6 files audited; valid-feature-dir actually migrated; test.sh assertions for invalid-missing-artifact-dir updated.
3. **Repo-level surfaces** — read by humans (CONTRIBUTING, PHILOSOPHY) or external tooling (template-registry, skills-catalog) or CI (scripts/test.sh, scripts/test-deploy.sh). 6 files affected.

The `scripts/test-deploy.sh` canary template swap (doc-PRD.md → doc-RCA.md, 19 refs) is the trickiest mechanical change because of the line 414 path bug — sed alone won't produce a working test.

## Acceptance Criteria

### Story #1: Examples aligned with new templates [core]

```
GIVEN S000014 has shipped (new templates exist)
WHEN  this story's example sweep is complete
THEN  ls skills/personal-workflow/examples/ shows:
        - example-doc-SPEC.md (new)
        - example-doc-ROADMAP.md (new)
        - example-doc-DESIGN.md (kept)
        - example-doc-RCA.md (kept)
        - example-doc-test-plan.md (kept)
        - example-doc-TEST-SPEC.md (kept; updated by v1.4.0)
        - example-tracker-feature.md (rewritten)
        - example-tracker-user-story.md (rewritten)
        - example-tracker-task.md (kept)
        - example-tracker-defect.md (kept)
AND   none of: example-doc-PRD.md, example-doc-ARCHITECTURE.md, example-doc-feature-summary.md, example-doc-milestones.md
```

### Story #2: Fixtures consistent with new manifest [core]

```
GIVEN S000014 has shipped
WHEN  fixture sweep + test.sh assertion update is complete
THEN  `./scripts/test.sh` returns zero failures
AND   skills/personal-workflow/fixtures/valid-feature-dir/ contains the new artifact set
AND   skills/personal-workflow/fixtures/invalid-missing-artifact-dir/ assertion in test.sh expects {DESIGN, ROADMAP} as MISSING (not {feature-summary, DESIGN, milestones})
```

### Story #3: template-registry.json declares new doc_types [core]

```
GIVEN this story's registry update is complete
WHEN  I run: jq '.sets["personal-workflow"]' template-registry.json
THEN  output shows version: "3.0.0"
AND   doc_types: ["design", "spec", "roadmap", "test-spec", "rca", "test-plan"]
AND   tracker_types unchanged: ["feature", "defect", "task", "user-story"]
```

### Story #4: test-deploy.sh works post-canary-swap [core]

```
GIVEN sed substitution complete + line 414 manual fix complete
WHEN  I run `./scripts/test-deploy.sh`
THEN  all tests pass (or skip cleanly if dependencies absent)
AND   no "file not found" errors reference doc-PRD.md
AND   no "file not found" errors reference $REPO_ROOT/templates/doc-RCA.md (correct path is templates/personal-workflow/doc-RCA.md)
```

### Story #5: Narrative consistency in CONTRIBUTING + PHILOSOPHY [core]

```
GIVEN this story's CONTRIBUTING + PHILOSOPHY edits are complete
WHEN  I read CONTRIBUTING.md:44-45 and PHILOSOPHY.md:25,42,43
THEN  CONTRIBUTING.md table shows doc-SPEC.md and doc-ROADMAP.md (not doc-PRD or doc-ARCHITECTURE)
AND   PHILOSOPHY.md prose references SPEC/ROADMAP (not PRD/ARCHITECTURE/feature-summary/milestones) for active personal-workflow concepts
```
