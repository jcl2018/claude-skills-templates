## Problem Statement

The author wants one front door to build a feature from a one-line topic: run the single interactive design phase (office-hours), then silently scaffold/implement/QA and stop at a `/ship` PR for human review. It must avoid the nested-subagent wall (`run → personal-pipeline → scaffold/impl/qa`) and must NOT auto-deploy, because the handoff-gate denylist blocks exactly the skill surfaces every feature touches.

## Mental Model

A flat orchestrator with one interactive phase up front and a hard stop at the PR. office-hours and `/ship` run inline (office-hours because it AUQs; `/ship` because the PR is the review). Scaffold/impl/qa run as silent depth-≤2 leaf subagents. A resume state file records phase boundaries (phase + HEAD SHA + PR number) so re-invocation can validate-before-skip rather than trusting a coarse flag.

## Acceptance Criteria

<!-- Given/When/Then format. One block per requirement or logical group. -->

### Story #1: Happy path to PR-stop [core]

```
GIVEN a clean main checkout
WHEN  /cj_goal_feature "<topic>" runs and office-hours emits an APPROVED design doc
THEN  scaffold/impl/qa run as silent leaf subagents and /ship opens a PR, then the pipeline STOPS at the PR
```

### Story #2: Zero AUQ after approval [usability]

```
GIVEN office-hours has emitted an APPROVED design doc
WHEN  the silent build and /ship run
THEN  no AUQ fires between the office-hours approval gate and the opened PR (the /ship diff-review AUQ is suppressed)
```

### Story #3: No auto-deploy [core]

```
GIVEN any /cj_goal_feature run
WHEN  the pipeline reaches /ship
THEN  it opens a PR and STOPS — no autoplan ran and no auto-merge/deploy occurs
```

### Story #4: Validate-before-skip resume [resilience]

```
GIVEN a halted run with last_completed_phase + recorded HEAD SHA + PR number
WHEN  /cj_goal_feature is re-invoked and the recorded SHA is NOT an ancestor of (or equal to) current HEAD, or the open PR no longer resolves to OPEN
THEN  the affected phase restarts rather than the pipeline trusting the flag and skipping ahead
```

### Story #5: office-hours resume on recorded path [resilience]

```
GIVEN last_completed_phase = office-hours and a recorded doc path
WHEN  /cj_goal_feature is re-invoked
THEN  it re-locates the doc by the recorded path, re-confirms Status: APPROVED, and proceeds without re-running office-hours on the unchanged doc
```

### Story #6: Halt taxonomy + telemetry [observability]

```
GIVEN any halt during the feature pipeline
WHEN  the pipeline stops
THEN  it emits a family halt end-state with next_action= / resume_cmd= / pr_url= (when a PR exists) and writes one JSONL line to ~/.gstack/analytics/CJ_goal_feature.jsonl
```

## Architecture

<!-- High-level system design for this story. -->

```
/cj_goal_feature "<topic>"   (orchestrator, top level)
   │  cj-worktree-init.sh --caller feature  (S000057)
   ▼
/office-hours   [INLINE — interactive; emits APPROVED design doc]
   │   ↳ not APPROVED / abandoned → HALT (next_action=resume /office-hours)
   ▼
capture doc path → resume state file: last_completed_phase + HEAD SHA + PR#
   │
   ▼  silent, no AUQ
scaffold (leaf subagent) → implement (leaf subagent) → qa (leaf subagent)
   │
   ▼
/ship   [INLINE — creates PR; diff-review AUQ suppressed]
   │
   ▼
STOP at PR   (human reviews + merges on GitHub; /land-and-deploy is a separate human step)
   │
   ▼
telemetry (cj-goal-common.sh; ~/.gstack/analytics/CJ_goal_feature.jsonl)
```

### Data Flow

<!-- How does data move through the system for the primary use case? -->

1. Author runs `/cj_goal_feature "<topic>"`; orchestrator creates a worktree (`--caller feature`).
2. `/office-hours` runs inline; on Approve it emits a design doc whose path is recorded in the resume state file with the current HEAD SHA.
3. scaffold → impl → qa run as silent leaf subagents; each phase boundary updates `last_completed_phase` + HEAD SHA.
4. `/ship` runs inline with the diff-review AUQ suppressed, opens a PR, records the PR number, and the pipeline STOPS.
5. On re-invocation, the orchestrator validates the recorded SHA (ancestor-of/equal-to HEAD) and PR (OPEN) before resuming; office-hours resume re-locates the doc by recorded path + re-confirms APPROVED.
6. `cj-goal-common.sh` writes one telemetry JSONL line per run.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| End state | PR-stop (no auto-merge/deploy) | Auto-merge small diffs | Auto-deploy of skill-work is unsafe-by-construction (denylist blocks the skill surfaces every feature touches). |
| Architecture gate | Human PR review (no autoplan) | Run autoplan before build | With auto-deploy gone, the PR review is the gate; "autoplan only on the auto-deploy branch" was incoherent. |
| Resume granularity | `last_completed_phase` + per-phase HEAD SHA + PR#, validate-before-skip | A/S/P/M coarse flag | The flag model was too lossy and could skip into a later phase on stale state. |
| office-hours doc recovery | Recorded path + APPROVED re-confirm | Blind newest-glob | The glob is fragile across parallel runs; a machine-readable pointer is a deferred follow-up. |
