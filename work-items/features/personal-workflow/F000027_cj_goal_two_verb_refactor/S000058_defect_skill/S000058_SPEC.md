## Problem Statement

The author has no single front door that turns a plain bug description (with no pre-existing defect dir) into a root-caused, shipped fix. `/CJ_goal_investigate` assumes a scaffolded defect and sits inside the five-orchestrator family being collapsed. `/cj_goal_defect "<bug>"` must scaffold the report, enforce the Iron-Law via `/investigate`, and ship on the proven human-gated tail.

## Mental Model

A flat, defect-first pipeline that reshapes investigate v1.1 (~80% reuse): scaffold a throwaway bug report → root-cause (Iron-Law) → promote to a real defect dir → QA → human-ship → deploy. The orchestrator dispatches `/investigate` and `/CJ_qa-work-item` as depth-≤2 subagents; `/ship` and `/land-and-deploy` run inline. Common deterministic bits come from `cj-goal-common.sh` (S000057).

## Acceptance Criteria

<!-- Given/When/Then format. One block per requirement or logical group. -->

### Story #1: No-doc bug-report scaffolding [core]

```
GIVEN no pre-existing defect dir for the bug
WHEN  /cj_goal_defect "<bug description>" runs
THEN  a bug report is scaffolded at .inbox/<slug>/DRAFT.md
```

### Story #2: Iron-Law via /investigate [core]

```
GIVEN the scaffolded .inbox draft
WHEN  /investigate runs as an Agent subagent and returns no root cause (or DONE_WITH_CONCERNS)
THEN  the pipeline HALTS, nothing is promoted or shipped, and a resume_cmd is emitted
```

### Story #3: Promotion after root cause [core]

```
GIVEN /investigate returns a populated root cause (Iron-Law satisfied)
WHEN  the pipeline continues
THEN  RCA + test-plan are written and .inbox/<slug>/ is promoted to work-items/defects/.../D000NNN_<slug>/
```

### Story #4: Human-gated deploy tail [integration]

```
GIVEN a promoted defect dir that passed /CJ_qa-work-item
WHEN  /ship reaches Gate #2 (human diff review) and the human approves
THEN  /land-and-deploy --suppress-readiness-gate runs and the tracker journal + telemetry are written
```

### Story #5: Inherited halt taxonomy [resilience]

```
GIVEN any halt condition during the defect pipeline
WHEN  the pipeline stops
THEN  it emits investigate v1.1's halt end-state with next_action= / resume_cmd= / pr_url= (when a PR exists)
```

## Architecture

<!-- High-level system design for this story. -->

```
/cj_goal_defect "<bug>"   (orchestrator, top level)
   │  cj-worktree-init.sh --caller defect  (S000057)
   ▼
scaffold .inbox/<slug>/DRAFT.md
   │
   ▼
/investigate   ──[Agent subagent, sentinel JSON, Iron-Law]──►  root cause? ──no──► HALT
   │ yes
   ▼
write RCA + test-plan  →  promote .inbox → work-items/defects/.../D000NNN_<slug>/
   │
   ▼
/CJ_qa-work-item  (leaf subagent)
   │
   ▼
/ship  (inline, human Gate #2)  →  /land-and-deploy --suppress-readiness-gate  (inline)
   │
   ▼
tracker journal + telemetry (cj-goal-common.sh; ~/.gstack/analytics/CJ_goal_defect.jsonl)
```

### Data Flow

<!-- How does data move through the system for the primary use case? -->

1. Author runs `/cj_goal_defect "<bug>"`; orchestrator creates a worktree (`--caller defect`).
2. A bug report is scaffolded at `.inbox/<slug>/DRAFT.md`.
3. `/investigate` runs as an Agent subagent; sentinel-wrapped JSON carries the verdict; Iron-Law gates promotion.
4. On root cause: RCA + test-plan written; `.inbox` promoted to a `D000NNN_<slug>/` defect dir.
5. `/CJ_qa-work-item` runs the test-plan; `/ship` (Gate #2) then `/land-and-deploy --suppress-readiness-gate` run inline.
6. `cj-goal-common.sh` writes the tracker journal entry + one telemetry JSONL line.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Build approach | Reshape investigate v1.1's flat pipeline (~80% reuse) | Wrap `/CJ_goal_investigate` | Wrapping re-inherits the nested-subagent wall; a flat reshape keeps depth ≤ 2. |
| Bug-report home before root cause | `.inbox/<slug>/DRAFT.md`, promote after Iron-Law | Scaffold directly into `work-items/defects/` | Keeps `work-items/defects/` free of un-investigated stubs; promotion-after-root-cause is the clean boundary. |
| Deploy gate | Human `/ship` Gate #2 then deploy | Auto-merge/deploy | Matches current investigate; human diff review is the autonomy ceiling for fixes. |
