## Problem Statement

After the two verbs land, `/CJ_goal_run` and `/CJ_goal_auto` are redundant front doors that still appear in routing rules and consumer habits. They must be deprecated with a banner, a route to `/cj_goal_feature`, and a sunset date — without breaking in-flight pipelines and without disturbing the orthogonal `/CJ_goal_todo_fix` + `/CJ_personal-pipeline` drain utility.

## Mental Model

A documentation-and-catalog change plus two thin alias shims. The catalog is the source of truth for skill paths; flipping `status` to `deprecated` (and keeping the entries installable via `--include-deprecated`) drives `skills-deploy`/`validate.sh`/`test.sh`. Routing is edited in two files (`rules/skill-routing.md` + `CLAUDE.md`). No new orchestration logic.

## Acceptance Criteria

<!-- Given/When/Then format. One block per requirement or logical group. -->

### Story #1: Alias shims with sunset [core]

```
GIVEN the deprecated /CJ_goal_run (and /CJ_goal_auto) skill
WHEN  it is invoked
THEN  it prints a one-line deprecation banner naming the sunset (next major, e.g. v6.0.0) and routes to /cj_goal_feature
```

### Story #2: Catalog state [integration]

```
GIVEN skills-catalog.json after this story
WHEN  validate.sh runs
THEN  cj_goal_feature + cj_goal_defect are status=experimental, CJ_goal_run + CJ_goal_auto are status=deprecated, and skills-deploy install --include-deprecated still installs the deprecated pair
```

### Story #3: Routing updated [usability]

```
GIVEN rules/skill-routing.md and CLAUDE.md after this story
WHEN  a reader looks up "build a feature" or "fix a bug"
THEN  the routing points at /cj_goal_feature and /cj_goal_defect respectively, and no longer recommends /CJ_goal_run or /CJ_goal_auto as the primary door
```

### Story #4: Kept skills unaffected [resilience]

```
GIVEN the deprecation changes
WHEN  /CJ_goal_todo_fix or /CJ_personal-pipeline (or /schedule, /loop) is invoked
THEN  it behaves exactly as before the change
```

### Story #5: Repo stays green [core]

```
GIVEN all catalog, routing, and shim edits
WHEN  validate.sh and test.sh run
THEN  both pass
```

## Architecture

<!-- High-level system design for this story. -->

```
skills/CJ_goal_run/SKILL.md   ──►  [banner] ──► route to /cj_goal_feature   (alias shim)
skills/CJ_goal_auto/SKILL.md  ──►  [banner] ──► route to /cj_goal_feature   (alias shim)

skills-catalog.json
   ├─ cj_goal_feature   : experimental   (new)
   ├─ cj_goal_defect    : experimental   (new)
   ├─ CJ_goal_run       : deprecated     (installable via --include-deprecated)
   └─ CJ_goal_auto      : deprecated     (installable via --include-deprecated)

routing:  rules/skill-routing.md  +  CLAUDE.md   ──►  point at /cj_goal_feature + /cj_goal_defect
kept:     /CJ_goal_todo_fix , /CJ_personal-pipeline   (unchanged)
```

### Components Affected

| Component | Repo | Change Type | Description |
|-----------|------|------------|-------------|
| `skills/CJ_goal_run/SKILL.md` | claude-skills-templates | Modified | Becomes a hard alias shim (banner → `/cj_goal_feature`) + sunset; possibly relocated under `deprecated/` per convention |
| `skills/CJ_goal_auto/SKILL.md` | claude-skills-templates | Modified | Same as above |
| `skills-catalog.json` | claude-skills-templates | Modified | 2 new `experimental` entries; flip `CJ_goal_run` + `CJ_goal_auto` to `deprecated` |
| `rules/skill-routing.md` | claude-skills-templates | Modified | Route "build a feature"/"fix a bug" to the two verbs |
| `CLAUDE.md` | claude-skills-templates | Modified | Mirror the routing update |
| `README.md` | claude-skills-templates | Modified | Regenerate from catalog if needed |

### Data Flow

<!-- How does data move through the system for the primary use case? -->

1. A caller invokes `/CJ_goal_run` (or `/CJ_goal_auto`); the shim prints a banner and routes to `/cj_goal_feature`.
2. `skills-deploy install` reads the catalog; deprecated entries are skipped (WARN) unless `--include-deprecated` is passed, in which case they install for in-flight use.
3. `validate.sh` checks catalog ↔ filesystem (status enum, paths via `dirname(files[0])`); `test.sh` runs the full suite.
4. Routing lookups in both files now resolve "build a feature"/"fix a bug" to the new verbs.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Removal strategy | Hard alias shim + sunset date | Remove `run`/`auto` immediately | Immediate removal breaks in-flight pipelines + muscle memory; the alias+sunset pattern is already proven (`CJ_run`). |
| Scope of deprecation | Only `run` + `auto` | Also deprecate `/CJ_personal-pipeline` now | personal-pipeline is still todo_fix's engine; deprecating it requires the deferred todo_fix migration first. |
| Installability of deprecated skills | Keep installable via `--include-deprecated` | Drop from install entirely | In-flight items must finish under the old skills (in-flight migration). |
