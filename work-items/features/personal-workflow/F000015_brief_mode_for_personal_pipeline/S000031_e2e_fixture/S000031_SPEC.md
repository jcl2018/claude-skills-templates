## Problem Statement

S000030 ships a manually-smokeable change. Without a durable fixture, regressions in brief-mode (especially structural breakage from special-character handling, telemetry mode field drift, and parser-default regression) land silently. This story creates the fixture and wires it into `scripts/test.sh`.

## Mental Model

```
[scripts/test.sh]
        |
        v
[fixture: skills/personal-pipeline/fixtures/brief-mode/]
        |
        +-- run /personal-pipeline --brief "<text-with-special-chars>" --type defect
        |        |
        |        v
        |   [Step 0a synthesizes stub]
        |        |
        |        v
        |   [scaffold subagent]
        |        |
        |        v
        |   [implement subagent]
        |        |
        |        v
        |   [qa subagent]
        |        |
        |        v
        |   [telemetry write]
        |
        +-- assert: stub file well-formed (parser-clean)
        +-- assert: pipeline ran green end-to-end
        +-- assert: telemetry line contains `mode: "brief"`
        +-- assert: legacy telemetry (no mode field) parses as `manual`
        +-- cleanup: remove stub + work-item dir + telemetry line
```

## Acceptance Criteria

### Story #1: fixture exists [core]

```
GIVEN  the repo at HEAD after S000031 ships
WHEN   listing `skills/personal-pipeline/fixtures/`
THEN   a `brief-mode/` subdirectory exists
```

### Story #2: special-char brief text [core]

```
GIVEN  the fixture's brief-text source
WHEN   inspected
THEN   the brief text contains at least one backtick character AND at least one `## ` line within the brief text
```

### Story #3: pipeline runs green [core]

```
GIVEN  the fixture is invoked via `scripts/test.sh` (or directly)
WHEN   the brief-mode pipeline completes
THEN   the final RESULT line is a green / pass status (PASS, GREEN, or equivalent matching the existing pipeline's success signal)
```

### Story #4: telemetry mode field [core]

```
GIVEN  the fixture has run to completion
WHEN   the most-recent line of `~/.gstack/analytics/personal-pipeline.jsonl` is parsed
THEN   the JSON object's `mode` key has value `"brief"`
```

### Story #5: legacy telemetry parses as manual [observability]

```
GIVEN  a synthetic legacy telemetry line (no `mode` key) is appended in fixture setup
WHEN   the sunset-checkpoint parser reads it
THEN   the parsed object has `mode = "manual"`
```

### Story #6: idempotent fixture [resilience]

```
GIVEN  the fixture has been run once (and cleaned up)
WHEN   the fixture is run a second time in the same invocation of `scripts/test.sh`
THEN   no leftover state from run #1 affects run #2; both runs are green
```

### Story #7: fixture wired into test.sh [integration]

```
GIVEN  the repo at HEAD
WHEN   `scripts/test.sh` is invoked with no special flags
THEN   the brief-mode fixture is among the cases run (visible in test.sh output)
```

## Architecture

```
+----------------------------------+
| skills/personal-pipeline/        |
|   fixtures/                      |
|     brief-mode/                  |
|       brief-text.txt   ← contains backtick + ## Header
|       expected-stub.md ← golden file (post-synthesis)
|       run.sh          ← invokes pipeline, asserts, cleans up
|       legacy-telemetry-line.json ← regression input
+----------------------------------+
                |
                v
+----------------------------------+
| scripts/test.sh                  |
|   - existing test cases          |
|   - NEW: invoke fixture          |
+----------------------------------+
                |
                v
+----------------------------------+
| /personal-pipeline --brief "..." |
|     --type defect                |
| (fully exercised end-to-end)     |
+----------------------------------+
                |
                v
+----------------------------------+

| ~/.gstack/analytics/.jsonl       |

```

### Components Affected

| Component | Repo | Change Type | Description |
|-----------|------|------------|-------------|
| `skills/personal-pipeline/fixtures/brief-mode/` | claude-skills-templates | New | Brief-mode fixture (brief text + run script + assertions + golden file + legacy line) |
| `scripts/test.sh` | claude-skills-templates | Modified | Wire fixture into standard run |

### Data Flow

1. `scripts/test.sh` invokes fixture's `run.sh`.
2. `run.sh` reads `brief-text.txt` (containing intentional backtick + `## Header`).
3. `run.sh` invokes `/personal-pipeline --brief "$(cat brief-text.txt)" --type defect`.
4. Pipeline runs Step 0a (synthesizes stub), Steps 1+ (scaffold, implement, qa), telemetry write.
5. `run.sh` asserts the synthesized stub matches `expected-stub.md` modulo timestamps + path tokens.
6. `run.sh` asserts the final RESULT line is green.
7. `run.sh` asserts the most-recent telemetry line has `mode: "brief"`.
8. `run.sh` appends `legacy-telemetry-line.json` (no `mode` field) and asserts the parser reads `mode = "manual"`.
9. `run.sh` cleans up: removes the synthesized stub, the work-item dir, and the telemetry lines added by the fixture.
10. Fixture exits 0 on all assertions PASS, non-zero on any FAIL.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Fixture organization | Single combined fixture covering all four checks | Four separate fixtures (one per assertion) | Lower overhead; logical end-to-end is naturally one invocation. Separate fixtures duplicate setup/teardown without proportional payoff. |
| Special-char content | Backtick + `## Header` line in brief text | Just one or the other | Both are common Markdown structures; covering both in one brief is cheap. |
| Mock vs real target | Mock target work-item (or trivially-cleanable real one) | Run against a real shipping change | Fixture must be idempotent and self-cleaning; coupling to real shipping work risks fixture-induced flakiness. |
| Concurrent race | Not exercised | Add a parallel-invocation case | Accepted risk per parent F000015; testing it requires locking primitives that don't exist yet. |
| Golden-file matching | Modulo timestamps + path tokens | Exact-match golden | Timestamps + paths are environment-dependent; matching modulo those is the correctness boundary. |
| Test runtime impact | Acceptable; fixture invokes 3 subagents | Skip in default CI | Brief-mode is a load-bearing feature; CI should exercise it by default. If runtime becomes a problem, opt-out via flag — not opt-in. |
