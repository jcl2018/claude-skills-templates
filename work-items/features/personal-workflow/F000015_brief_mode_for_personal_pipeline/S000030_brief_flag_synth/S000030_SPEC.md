## Problem Statement

A user wants to type `/personal-pipeline --brief "<paragraph>" --type defect` and have the orchestrator synthesize a real stub design doc, then continue through the existing scaffold/implement/qa flow unchanged. Today, `/personal-pipeline` rejects any invocation without a positional design-doc path. This spec defines the SKILL.md and pipeline.md changes that close that gap.

## Mental Model

```
[user invokes]
   /personal-pipeline --brief "<text>" --type {task|defect}
         |
         v
   [Step 0a: NEW — flag parsing + validation + synth]
         |
         +-- validate flags → on error: emit prescribed message, exit clean
         |
         +-- synthesize stub at ~/.gstack/projects/{slug}/{user}-{branch}-design-{ts}-brief[-N].md
         |
         v
   [Step 1+: EXISTING — pre-scaffold idempotency, dispatch, gates, telemetry]
         |
         v
   [Telemetry write: mode field added (manual|auto|brief|brief+auto)]
```

When `--brief` is absent, Step 0a is a no-op pass-through. Existing Step 1+ behavior is untouched.

## Acceptance Criteria

### Story #1: Step 0a parses flags [core]

```
GIVEN  the user invokes `/personal-pipeline --brief "<text>" --type defect`
WHEN   Step 0a runs
THEN   the orchestrator captures BRIEF_TEXT and TYPE and proceeds toward synthesis (does not look for a positional path)
```

### Story #2: Step 0a validates flag combination [core]

```
GIVEN  any of: `--brief` without `--type`; `--brief` with empty text; `--brief` with text > 2000 chars; `--brief` AND a positional path; `--brief --type feature`; `--brief --type user-story`
WHEN   Step 0a runs
THEN   the corresponding prescribed error message is emitted verbatim and the orchestrator exits clean (no synthesis, no work-item write)
```

### Story #3: Step 0a synthesizes stub on disk [core]

```
GIVEN  a valid `--brief "<text>" --type defect` invocation
WHEN   Step 0a runs
THEN   a file at `~/.gstack/projects/{slug}/{user}-{branch}-design-{YYYYMMDD-HHMMSS}-brief.md` exists with the stub template populated and brief text wrapped in a fenced verbatim block
```

### Story #4: Filename collision applies suffix `-2` [core]

```
GIVEN  a target filename already exists at the planned path
WHEN   Step 0a synthesizes
THEN   the suffix `-2` is appended before `.md`; if `-2` exists, `-3`; etc. The un-suffixed filename is the implicit `-1` slot.
```

### Story #5: --brief absent is byte-identical to current behavior [core]

```
GIVEN  any existing `/personal-pipeline <path>` or `/personal-pipeline <path> --auto` invocation
WHEN   the change ships
THEN   captured stdout and resulting filesystem state match the pre-change run byte-for-byte
```

### Story #6: Telemetry mode field [core]

```
GIVEN  any pipeline invocation
WHEN   Step 12 (or equivalent) writes telemetry to ~/.gstack/analytics/personal-pipeline.jsonl
THEN   the JSON object contains a `mode` key with one of `manual`, `auto`, `brief`, `brief+auto`
```

### Story #7: Sunset-parser default [observability]

```
GIVEN  a telemetry line without the `mode` field (pre-change format)
WHEN   the sunset-checkpoint parser reads it
THEN   the parsed object has `mode = "manual"` (default)
```

### Story #8: Six prescribed error messages [usability]

```
GIVEN  any of the six error conditions from F000015_DESIGN.md "Error Handling table additions"
WHEN   the user invokes the failing combination
THEN   stderr (or stdout, matching the existing message channel) contains the exact prescribed message string
```

### Data Flow

1. User invokes `/personal-pipeline --brief "<text>" --type defect`.
2. pipeline.md Step 0a fires.
3. Step 0a parses `--brief`, `--type`, optional positional path.
4. Step 0a validates: positional path absent; `--type` ∈ {task, defect}; brief text non-empty after whitespace trim; brief text length ≤ 2000.
5. On any validation failure, emit prescribed message; exit clean.
6. Step 0a computes filename: `{user}-{branch-with-slashes-as-dash}-design-{$(date +%Y%m%d-%H%M%S)}-brief.md`.
7. If file exists, increment suffix: `-2`, `-3`, …
8. Step 0a writes the stub template (from F000015_DESIGN's "Synthesized stub design doc template") with:
   - `{title-slug}` derived from brief text (lowercase, non-alphanum→`-`, trim, ≤60 chars; fallback `brief-{type}` if empty).
   - `{date}`, `{branch}`, `{owner/repo}` from environment.
   - Brief text inside ` ```text ... ``` ` block.
9. Step 0a sets `DESIGN_DOC_PATH` to the synthesized file.
10. Pipeline continues into existing Step 1 (pre-scaffold idempotency); synthesized stub by construction has no SCAFFOLDED footer → clean-slate branch.
11. End-of-pipeline telemetry write includes `mode` field: `brief` if `--brief` set; `brief+auto` if also `--auto`; otherwise `manual` or `auto`.
12. Sunset-checkpoint parser (line ~204) reads `mode`, defaults to `"manual"` if absent.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Where Step 0a lives | Before existing Step 1 in pipeline.md | Inside Step 1 as a sub-branch | Existing Step 1 logic stays untouched. Synthesized stubs land in clean-slate branch by construction. Easier to revert Step 0a if it regresses. |
| Filename time component | local-time `date +%Y%m%d-%H%M%S` matching existing convention | UTC | Match what manual /office-hours docs already use; consistency over correctness for a filename. |
| Collision-suffix start | `-2` (un-suffixed = implicit `-1`) | `-1` written explicitly on first collision | Reserves `-1` as a no-op alias; explicit grammar prevents drift. Documented in F000015_DESIGN. |
| Brief text encoding | Fenced ` ```text ... ``` ` block | Backslash-escape special chars | Fenced block is structurally safe; backslash-escape leaves edge cases. |
| Telemetry mode | Additive field (default `manual`) | Replace existing `auto: true/false` | Additive preserves backward compat; existing consumers don't break. |
| `--type user-story` in v1 | Reject with v1.1-deferred message | Accept | Premise 4 narrowing in spec-review iteration 1; user-story-as-brief evaluated against real usage data, not pre-judged. |
