## Problem Statement

After `/implement-from-spec` writes code per a user-story SPEC, the user today manually runs smoke tests (e.g., `scripts/test.sh`) and walks E2E by hand before `/ship`. The May 5 design kept E2E manual deliberately. Today's design changes that: a QA engineer subagent runs E2E autonomously by reading TEST-SPEC and using whatever tools it needs (Bash, Read, Diff). The skill `/qa-work-item` orchestrates: smoke first (cheap script), then E2E (subagent), writes findings to the tracker, and gates Phase 2 → Phase 3 lifecycle transition.

This generalizes E2E beyond pre-scripted harnesses. A TEST-SPEC saying "verify the validator catches missing frontmatter" can be verified by a subagent that reads the validator code, constructs a malformed input, runs the validator, and checks the output — without any pre-scripted automation.

## Acceptance Criteria

### Story #1: Smoke + E2E run [core]

```
GIVEN  a user-story dir with Phase 2 implementation gates checked
       AND TEST-SPEC.md has populated Smoke Tests + E2E Tests tables
WHEN   /qa-work-item <user-story-dir> is invoked
THEN   smoke runs first (each Smoke Test row's Script/Command executed)
       AND E2E runs second (QA engineer subagent invoked with TEST-SPEC path)
       AND findings written to tracker (journal + ## QA Run section)
       AND Phase 2 gates updated if both green
```

### Story #2: Subagent short report [core]

```
GIVEN  the QA engineer subagent has finished E2E
WHEN   the subagent returns control to the parent skill
THEN   subagent's response is ≤ 200 tokens (1-2 sentences + file pointers)
       AND detailed findings are written by the subagent to the tracker
       AND the parent skill does NOT request "describe everything you did"
```

### Story #3: Smoke red short-circuit [resilience]

```
GIVEN  a TEST-SPEC where smoke test S2 fails
WHEN   /qa-work-item runs smoke
THEN   smoke failure recorded in tracker
       AND E2E subagent NOT invoked
       AND skill exits early with clear "smoke red, fix before E2E" message
       AND AskUserQuestion to re-run after fix
```

### Story #4: Subagent timeout [resilience]

```
GIVEN  the QA engineer subagent runs longer than 5 minutes
WHEN   the timeout fires
THEN   skill writes timeout entry to tracker journal
       AND AskUserQuestion: "Subagent timed out. Re-run / Skip E2E / Abort?"
       AND user's answer determines next step
       AND skill does NOT auto-retry
```

### Story #5: Idempotency [resilience]

```
GIVEN  a user-story with Phase 2 gates already green from a prior /qa-work-item run
WHEN   /qa-work-item is re-invoked
THEN   skill detects "already QA'd" via tracker state + boundary check
       AND prints "S000017 already QA'd green; nothing to do"
       AND exits without spawning subagent
```

### Story #6: Decision gate on red [usability]

```
GIVEN  the QA engineer subagent finds 1 red E2E criterion
WHEN   subagent returns
THEN   AskUserQuestion: "QA found 1 red finding: <one-line summary>. Review and fix? Yes / Skip / Defer"
       AND user's answer determines next step
       AND green path (no red, no ambiguous) is silent — skill exits without AUQ
```

### Story #7: Boundary check refuses on incomplete impl [resilience]

```
GIVEN  a user-story whose Phase 2 implementation gates are NOT met (e.g., "Acceptance criteria verified" unchecked)
WHEN   /qa-work-item is invoked
THEN   /personal-workflow check at start surfaces the gate gap
       AND skill refuses to run with message "Phase 2 incomplete; run /implement-from-spec first"
```

## Architecture

```
+----------------+
| /qa-work-item  |
+----------------+
       |
       | 1. validate input (user-story dir, not feature)
       |
       v
   +-------------+
   | Boundary    |   /personal-workflow check at START (1.3A)
   | check IN    |   refuse if Phase 2 implementation gates not met
   +-------------+
       |
       | 2. read TEST-SPEC.md
       |
       v
   +-------------+
   | Run smoke   |
   |             |   for each Smoke Test row:
   |             |     execute Script/Command
   |             |     capture exit code + stdout/stderr
   |             |     write entry to tracker journal
   +-------------+
       |
       | smoke green? ─── no ──────────────────┐
       |                                       │
       v                                       │
   +-------------+                             │
   | Spawn QA    |                             │
   | subagent    |   Agent tool, fresh context │
   |             |   prompt template:          │
   |             |     [STABLE]  "you are a QA │  short-circuit
   |             |                engineer..." │  exit, AUQ to
   |             |     [VARIABLE] "<TEST-SPEC  │  re-run after
   |             |                 path>, ..." │  fix

   +-------------+                             │
       |                                       │
       | subagent return: 1-2 sentences        │

       v                                       │
   +-------------+                             │
   | Process     |                             │
```

### Data Flow

1. User runs `/qa-work-item <user-story-dir>`
2. Skill validates input is user-story dir; on feature dir, lists children + AUQ
3. Skill runs `/personal-workflow check` at start; refuses if Phase 2 implementation gates not met
4. Skill reads TEST-SPEC.md
5. Skill executes smoke tests (each row's Script/Command); captures results, writes journal entries
6. Smoke red? Short-circuit: write smoke summary, AUQ to re-run after fix
7. Smoke green? Spawn QA engineer subagent (Agent tool, fresh context, 5-min timeout)
8. Subagent reads TEST-SPEC, runs verification (whatever tools needed), writes findings to tracker, returns 1-2 sentence summary
9. Skill parses summary: all green → silent transition of Phase 2 gates; any red/ambiguous → AUQ
10. Skill runs `/personal-workflow check` at end; AUQ on violations
11. Skill exits

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| E2E mechanism | QA engineer subagent | Pre-scripted bash E2E only | User invention; generalizes beyond scripts; works for any TEST-SPEC |
| Smoke vs E2E order | Smoke first, E2E only on green | Run both regardless | Cheap test first; saves tokens on broken impls |
| Subagent return | 1-2 sentences + file pointers | Full structured findings returned | Premise 1: short reports keep parent context small; details written to tracker by subagent |
| Timeout | Hard 5-min cap, AUQ on timeout | Auto-retry with backoff | Bounded latency; user decides on retry |
| Decision gate | AUQ only on red/ambiguous | AUQ always (review every result) | Autoplan-style: silent on green, intervene on red |
| Prompt placement | Hardcoded in SKILL.md (v1) | Separate prompts/ file or TEST-SPEC frontmatter | Source design Open Q1; v1 hardcode for simplicity, extract if reuse demands |
| Multiple subagents | Single QA engineer subagent | Two subagents (QA + adversarial) | Defer to v2 P2; one subagent earns its cost first |
