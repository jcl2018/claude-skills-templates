## Problem Statement

The CJ_goal_* orchestrators (`/CJ_goal_run`, `/CJ_goal_todo_fix`) currently run on
the current branch, polluting the main checkout and blocking parallel sessions.
This story implements the auto-worktree-default behavior described in the parent
feature design (F000025_DESIGN.md). User: chjiang (workbench operator); pain:
17+ active worktrees show the user's working pattern, but these orchestrators
are the one gap that doesn't honor it.

## Mental Model

Three layers, top-down:

1. **Helper** (`scripts/cj-worktree-init.sh`): bash script that detects worktree
   state, parses args, optionally creates a worktree, emits single-line JSON on
   stdout. Pure function over (cwd, git state, args) → JSON.
2. **Caller preambles** (`skills/CJ_goal_*/SKILL.md`): bash block that resolves
   the helper via `.skills-templates.json`'s `source` field, invokes it, parses
   the JSON with `jq -r '.field'`, and `cd`s on `state ∈ {created, detected}`.
   Runs BEFORE the existing Path Resolution block.
3. **Drain loop**
   (`skills/CJ_goal_todo_fix/scripts/drain-one-todo.sh`): per-iteration call to
   the helper with `--force-create --quiet`, resolved via BASH_SOURCE relative
   path. Each drained TODO gets its own worktree+branch+PR.

## Acceptance Criteria

### Story #1: Helper exists and emits valid JSON [core]

```
GIVEN scripts/cj-worktree-init.sh is shipped + executable
WHEN  bash scripts/cj-worktree-init.sh --caller run --dry-run is invoked from main
THEN  stdout is exactly one line of valid JSON parseable as {"state":"created","path":"...","branch":"cj-run-...","note":"..."}
AND   exit code is 0
```

### Story #2: /CJ_goal_run auto-worktree on main [core]

```
GIVEN current branch is main + workbench is clean
WHEN  /CJ_goal_run <design-doc-path> is invoked
THEN  a new worktree is created at .claude/worktrees/cj-run-<ts>-<pid>/ on branch cj-run-<ts>-<pid>
AND   the orchestrator cds into the worktree before any code-mutating phase
AND   the main checkout remains clean (`git status` post-run shows no new untracked files)
```

### Story #3: /CJ_goal_todo_fix single-TODO auto-worktree on main [core]

```
GIVEN current branch is main + workbench is clean
WHEN  /CJ_goal_todo_fix T000045 is invoked
THEN  a new worktree is created at .claude/worktrees/cj-todo-<ts>-<pid>/ on branch cj-todo-<ts>-<pid>
AND   the orchestrator cds into the worktree before scaffolding the T-task
```

### Story #4: Drain mode creates per-iteration worktrees [core]

```
GIVEN current branch is main + workbench is clean
WHEN  /CJ_goal_todo_fix --max-drain 3 is invoked AND drains 3 TODOs
THEN  three distinct worktrees exist at .claude/worktrees/cj-todo-<ts>-<pid>{N}/ (one per drained TODO)
AND   each has its own branch + PR
```

### Story #5: Already-in-worktree detection [resilience]

```
GIVEN cwd is inside an existing worktree at .claude/worktrees/<name>/
WHEN  /CJ_goal_run <doc> is invoked
THEN  helper emits state="detected", path=<cwd>
AND   the orchestrator runs in-place
AND   [worktree] already in <name> is echoed (gated on QUIET=0)
```

### Story #6: --no-worktree opt-out [usability]

```
GIVEN current branch is main + workbench is clean
WHEN  /CJ_goal_run --no-worktree <doc> is invoked
THEN  helper emits state="opted_out", path=<cwd>
AND   no worktree is created; orchestrator runs on current branch
```

### Story #7: --quiet gates echo [usability]

```
GIVEN /CJ_goal_todo_fix --quiet --max-drain 3 is invoked
WHEN  helper runs per iteration
THEN  no [worktree] line is printed to stdout
AND   JSON output stays internal to drain-one-todo.sh
```

### Story #8: Dirty-checkout halt [resilience]

```
GIVEN current branch is main AND workbench has uncommitted edits
WHEN  /CJ_goal_run <doc> is invoked (NOT --quiet)
THEN  helper emits state="failed", note="dirty checkout: stash/commit or pass --no-worktree"
AND   preamble exits 1 with [worktree] ERROR: <note>
```

### Story #9: No-arg invocation skips helper [resilience]

```
GIVEN current branch is main
WHEN  /CJ_goal_run is invoked with no args (Branch g auto-resume)
THEN  the [ $# -gt 0 ] guard short-circuits the helper invocation
AND   orchestrator runs on current branch (today's behavior preserved)
```

### Story #10: Visible WARN when helper unreachable [resilience]

```
GIVEN ~/.claude/.skills-templates.json is missing OR helper path is non-executable
WHEN  /CJ_goal_run <doc> is invoked
THEN  preamble emits [worktree] WARN: helper unreachable; running on current branch
AND   orchestrator continues on current branch
```

### Story #11: test.sh regression assertion [core]

```
GIVEN preamble block in /CJ_goal_run/SKILL.md is deleted
WHEN  scripts/test.sh is run
THEN  the test fails with a clear message naming the missing block
```

### Story #12: 5-case helper test [core]

```
GIVEN tests/cj-worktree-init.test.sh exists + executable
WHEN  bash tests/cj-worktree-init.test.sh is run
THEN  all 5 cases pass:
        - on-main creates a worktree
        - in-worktree detects + no-ops
        - --no-worktree opts out
        - --force-create overrides in-worktree detection
        - dirty-check halts (interactive) / skips (--quiet)
```

### Story #13: TODOS row for deferred work [core]

```
GIVEN TODOS.md is checked into the repo
WHEN  grep -i 'cj_goal_investigate' TODOS.md is run
THEN  a deferred row matches: "Add worktree-default preamble to /CJ_goal_investigate (P3, S)"
```

### Story #14: CLAUDE.md note [usability]

```
GIVEN CLAUDE.md is checked into the repo
WHEN  grep -i 'auto-worktree' CLAUDE.md is run
THEN  a one-line note exists, e.g. "/CJ_goal_run + /CJ_goal_todo_fix auto-worktree on main; opt out with --no-worktree"
```

## Architecture

```
                        +---------------------------+
                        | scripts/cj-worktree-init  |  <-- shared bash helper
                        |   .sh (~60 lines)         |       (JSON output)
                        +-------------+-------------+
                                      ^
                  +-------------------+--------------------+
                  |                   |                    |
        +---------+---------+ +-------+--------+ +---------+----------+
        | skills/CJ_goal_   | | skills/CJ_goal_| | skills/CJ_goal_    |
        | run/SKILL.md      | | todo_fix/      | | todo_fix/scripts/  |
        | (preamble)        | | SKILL.md       | | drain-one-todo.sh  |
        |                   | | (preamble)     | | (per-iteration)    |
        | --caller run      | | --caller todo  | | --force-create     |
        +-------------------+ +----------------+ +--------------------+
```

### Data Flow

1. User invokes `/CJ_goal_run <design-doc>` from `main` branch.
2. SKILL.md preamble's `[ $# -gt 0 ]` guard passes (arg present).
3. Preamble reads `~/.claude/.skills-templates.json`'s `source` field, locates `$source/scripts/cj-worktree-init.sh`.
4. Preamble invokes helper with `--caller run "$@"` (forwards args so `--no-worktree`, `--quiet`, etc. propagate).
5. Helper parses args, runs `git rev-parse --show-toplevel`, detects in-worktree (`--git-dir != --git-common-dir`), detects on-main, runs dirty-check.
6. On clean main: helper composes `name="cj-run-${ts}-${pid}"`, runs `git worktree add` against `<toplevel>/.claude/worktrees/${name}`, retries once on collision.
7. Helper emits `{"state":"created","path":"<abs>","branch":"...","note":"created ..."}` on stdout, exit 0.
8. Preamble parses with `jq -r '.state' / '.path' / '.note'`, `cd "$_WT_PATH"`, echoes `[worktree] $note` if `QUIET != 1`.
9. Existing Path Resolution block runs against the worktree; subsequent phases run there.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Output transport | JSON parsed with jq | KEY=VALUE + eval | Both Eng reviewers flagged eval-injection; JSON is safe by construction |
| Helper placement | BEFORE Path Resolution block | After (with re-resolution) | Helper-first avoids stale `_REPO_ROOT` after cd; simpler |
| Drain mode | One worktree per drained TODO (a) | One worktree per session (b) | /ship Gate #2 enforces one PR per TODO; (b) would force all PRs onto the same branch and break /ship |
| Branch name | `cj-{type}-{YYYYMMDD-HHMMSS}-{PID}` | `cj-{type}-{YYYYMMDD-HHMMSS}` | PID suffix prevents same-second collision in drain mode |
| Helper resolution (drain loop) | BASH_SOURCE relative path | `.skills-templates.json` manifest read | DRY — script already lives inside workbench tree; second resolution layer is redundant |
