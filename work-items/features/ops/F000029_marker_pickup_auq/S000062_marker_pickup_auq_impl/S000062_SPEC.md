## Problem Statement

The workbench operator runs `/cj_goal_feature "new thing"` (or `/cj_goal_defect` / `/CJ_goal_investigate`). F000028's git hook may have dropped a doc-sync marker at `~/.gstack/doc-sync-pending/<repo-slug>.json` since the operator's last pull. Today: nothing reads the marker. The operator must remember to run `/document-release` periodically, and they don't. Docs drift silently from the merged main state.

The story: each of the 3 cj_goal orchestrator preambles must detect the marker, surface an AUQ at the right moment (right before starting the next workbench task), and let the operator decide in one keystroke whether to invoke `/document-release` now, snooze, or skip. The marker is then cleared (Y path) or suppressed (snooze/skip paths) so it stops accumulating.

Who: the workbench operator (single user: chjiang). Pain today: silent doc drift + no inline reminder to fix it. After this story: the AUQ surfaces at exactly the moment work begins, with one-keystroke resolution.

## Mental Model

Three layers:

1. **Detection script (`scripts/skills-doc-sync-check`)**: stateless probe. Reads the marker + cache, emits `DOC_SYNC_PENDING <marker-path>` to stdout on a hit, silent otherwise. Also handles 3 subcommands (`--snooze`/`--skip`/`--resolved`) that update cache state. Mirrors `scripts/skills-update-check` line-for-line in structure.
2. **Preamble bash block (~4 lines, 3 places)**: calls the script in each of the 3 cj_goal SKILL.md preambles. Captures stdout; passes it through to the orchestrator. Identical block across all three modulo skill-name comment.
3. **AUQ-instruction prose block (~30 lines, 3 places, identical)**: tells the orchestrator how to react when the bash output contains `DOC_SYNC_PENDING`. Includes branch-detection logic, copy-paste AUQ template (4 options A/B/C/D), per-option follow-through (especially the A path's auto-commit step), error-handling for non-green `/document-release`.

### Story #1: New script exists, executable, structurally mirrors update-check [core]

```
GIVEN  the repo on `cj-feat-20260530-222955-29095`
WHEN   I run `ls -la scripts/skills-doc-sync-check && head -20 scripts/skills-doc-sync-check`
THEN   the file exists with the executable bit set

AND    `shellcheck scripts/skills-doc-sync-check` returns 0
```

### Story #2: Default invocation emits on hit, silent otherwise [core]

```
GIVEN  a fresh `~/.gstack/doc-sync-pending/<slug>.json` for the current repo
AND    no cache state (no `~/.gstack/doc-sync-cache.json` or empty)
WHEN   I run `bash scripts/skills-doc-sync-check`
THEN   stdout is exactly `DOC_SYNC_PENDING ~/.gstack/doc-sync-pending/<slug>.json` (path expanded)
AND    exit code is 0
GIVEN  no marker file
WHEN   I run `bash scripts/skills-doc-sync-check`
THEN   stdout is empty AND exit code is 0
```

### Story #3: --snooze writes cache atomically [core]

```
GIVEN  a fresh marker exists
WHEN   I run `bash scripts/skills-doc-sync-check --snooze 24`
THEN   `~/.gstack/doc-sync-cache.json` contains `snooze_until` field set to (now + 86400) seconds
AND    a subsequent default invocation is silent for 24h
AND    after the snooze expires, the AUQ surfaces again
```

### Story #4: --skip suppresses by head_sha [core]

```
GIVEN  a fresh marker with head_sha = "abc123"
WHEN   I run `bash scripts/skills-doc-sync-check --skip abc123`
THEN   `~/.gstack/doc-sync-cache.json` contains `skip_head_sha: "abc123"`
AND    subsequent default invocations are silent while the marker still has head_sha = "abc123"
AND    a new marker (different head_sha, e.g. "def456") re-fires the AUQ
```

### Story #5: --resolved closes the loop, idempotent [core]

```
GIVEN  a marker exists AND cache contains snooze_until + skip_head_sha
WHEN   I run `bash scripts/skills-doc-sync-check --resolved`
THEN   the marker file is deleted
AND    the cache no longer contains snooze_until or skip_head_sha
AND    a subsequent default invocation is silent
WHEN   I run `bash scripts/skills-doc-sync-check --resolved` AGAIN (marker already absent)
THEN   exit code is 0 (silent success, no error message)
```

### Story #6: Stale head_sha self-cleans [resilience]

```
GIVEN  a marker exists with head_sha = "0000000000000000000000000000000000000000" (unreachable)
WHEN   I run `bash scripts/skills-doc-sync-check`
THEN   the marker file is deleted
AND    stdout is empty AND exit code is 0
```

### Story #7: Corrupted marker JSON self-cleans [resilience]

### Data Flow

<!-- How does data move through the system for the primary use case?
     Step-by-step, component to component. -->

1. Operator runs `/cj_goal_feature "topic"` at repo root on main.
2. SKILL.md preamble (existing) executes `skills-update-check` (silent if no upgrade).
3. SKILL.md preamble (NEW) executes `skills-doc-sync-check`:
   a. Script computes `_REPO_SLUG=$(basename $(git rev-parse --show-toplevel))`.
   b. Reads `~/.gstack/doc-sync-pending/${_REPO_SLUG}.json` — if absent, exit 0 silent.
   c. Reads `~/.gstack/doc-sync-cache.json` — checks `snooze_until` (clock-based), `skip_head_sha` (SHA-equality).
   d. Reads marker's `head_sha` — checks reachability via `git rev-parse --verify`. Stale → silent self-delete.
   e. If none of (b-d) suppress, emits `DOC_SYNC_PENDING <marker-path>` to stdout.
4. Preamble captures the output; if `DOC_SYNC_PENDING` present, passes to orchestrator.
5. Orchestrator follows AUQ-instruction prose:
   a. Reads marker fields via `jq -r '.head_sha, .main_moved_at, .changed_files'`.
   b. Detects current branch via `git symbolic-ref --short HEAD`.
   c. Surfaces AUQ with branch-aware recommendation (Y on main; Snooze 1h on feature branch).
6. Operator picks an option:
   - **A on main**: orchestrator invokes `/document-release` via Skill tool → on green, auto-commit doc changes → calls `skills-doc-sync-check --resolved` → continues pipeline.
   - **A on feature branch**: orchestrator prints warning, falls back to B (snooze 1h).
   - **B/C**: calls `skills-doc-sync-check --snooze N` → continues pipeline.
   - **D**: calls `skills-doc-sync-check --skip <head_sha>` → continues pipeline.
7. Pipeline proceeds to Step 1.9 isolation gate (clean tree post-auto-commit, or unchanged tree on B/C/D).

## Tradeoffs

<!-- Design decisions made in this spec, with rejected alternatives and rationale.
     Future readers need to know why this path over the others. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Where the AUQ logic lives | SKILL.md prose (per-preamble instruction block) | Inside the script (script emits the AUQ directly) | Scripts can't AUQ — they have no Skill tool / AskUserQuestion access. The script's job is "detect," the orchestrator's job is "decide." Splitting is structural, not stylistic. |
| Whether to abstract the 3 preamble blocks into a helper | Verbatim 4-line block in each SKILL.md | Source a shared `scripts/cj-doc-sync-preamble.sh` helper | The bash block is already a thin shim around a script call; abstracting adds a second indirection layer. Reviewer-flagged P1 — accepted as duplication cost is one `diff` check at QA time. |
| Cache file location | `~/.gstack/doc-sync-cache.json` (user-global) | `<repo>/.git/doc-sync-cache.json` (per-repo) | Marker file is already user-global (`~/.gstack/doc-sync-pending/<slug>.json` per F000028); cache should match. Easier operator inspection: `ls ~/.gstack/` shows everything. |
| Snooze granularity | Global (single `snooze_until`) | Per-marker (`{<head_sha>: snooze_until}` map) | Simpler. Operator hitting Snooze means "stop bugging me for a while," not "stop bugging me about this specific head_sha." Revisit if operator hits the case. |
| Branch-aware AUQ option ordering | SKILL.md prose detects branch + reorders | Script outputs a branch hint; orchestrator passes through | Keeps script's single job ("is there a marker?") clean. Branch detection is one `git symbolic-ref` call in the orchestrator; not worth adding a `--detect-branch` flag to the script. |
