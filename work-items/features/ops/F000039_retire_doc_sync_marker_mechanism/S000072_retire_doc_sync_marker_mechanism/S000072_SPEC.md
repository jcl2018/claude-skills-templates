## Problem Statement

The maintainer of this workbench (operator/agent running `/CJ_goal_feature` +
`/CJ_goal_defect`) is shown a `DOC_SYNC_PENDING` AUQ asking them to run
`/document-release` for documentation drift that has ALREADY been folded into the
same PR by F000036's inline Step 5.5 doc-sync. The marker→AUQ loop is dead weight:
the post-merge/post-rewrite hooks write markers, `scripts/skills-doc-sync-check`
reads them, the preamble AUQ surfaces them — all for a path that no longer exists
in practice. The operator flagged it as obsolete during a v6.0.8 run. This story
removes the entire F000028/F000029 mechanism and documents the one narrow
(non-`/ship`, non-orchestrator) main-move path that becomes manually-recovered.

## Mental Model

"doc-sync" names TWO mechanisms; this story touches ONLY the first:

```
  DIES (retire)                              LIVES (do NOT touch)
  ------------------------------------       ------------------------------------
  post-merge Section 3 doc-sync trigger      /CJ_document-release skill
  post-rewrite hook (doc-sync only)          ### Step 5.5: Doc-sync prose (×3 pipeline.md)
  scripts/skills-doc-sync-check              [doc-sync-red] / [doc-sync-non-doc-write] halts
  DOC_SYNC_PENDING preamble AUQ (×2)         cj-document-release.json (F000037)
  ~/.gstack/doc-sync-pending/*.json          scripts/cj-document-release-config.sh (F000037 parser)
  ~/.gstack/doc-sync-cache.json              tests/cj-goal-doc-sync-wiring.test.sh
```

Every Components-Affected row below is tagged DELETE / EDIT (DIES side) or
PRESERVE (LIVES side). The retirement is "complete" only when both completeness
greps (TEST-SPEC) return zero live references — a passing test suite alone is
insufficient.

## Tradeoffs

<!-- Design decisions, rejected alternatives, rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| How much to remove | Approach A: full delete + document the narrow gap | Approach C: drop only the AUQ | C leaves dead code (the script), orphaned state-file writers (the hooks), and markers still written every merge — incomplete retirement. |
| The rare non-/ship path | Document it as manually-recoverable | Approach B: lightweight post-merge reminder | B keeps a hook alive for a rare path; a stateless echo is easy to ignore and re-introduces a maintenance surface. |
| The F000037 config parser | KEEP, fix 2 comments | Delete with the rest | It is the live F000037 `cj-document-release.json` parser; only its comments reference the deleted script. |
| Story decomposition | Single atomic story | Multiple task children | The work is one cohesive de-referencing change, not independently parallelizable sub-units. |
