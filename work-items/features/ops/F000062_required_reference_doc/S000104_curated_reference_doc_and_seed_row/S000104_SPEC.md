## Problem Statement

A contributor (human or agent) building this workbench has no governed home for
the external references that shaped it (Anthropic/Claude Code docs, gstack,
Keep-a-Changelog, shellcheck, the GitHub CLI, etc.) and must rediscover the
canonical sources each time. This story adds a REQUIRED general-tier
`docs/reference.md` — a curated, contract-audited shelf of those references — and
wires it into the doc contract so it cannot drift unaudited.

## Mental Model

The change has two halves bound by one invariant. **The content half:** a curated
`docs/reference.md` (grouped categories, grep-grounded entries, one-line whys, no
work-item IDs). **The contract half:** the doc is declared in the registry as a
`section: common` / `audit_class: human-doc` doc with no `front_table`, and that
declaration must land byte-identically in the three seed copies. **The invariant:**
3-way seed byte-identity (`doc-spec.sh --seed` == `spec/doc-spec.md` ==
`templates/doc-spec-common.md`) — every seed edit (the row, the prose-table row,
the `eleven`→`twelve` count) lands in all three or the build is broken. The
generated `docs/doc-general.md` view is regenerated from the merged registry;
`validate.sh` is NOT touched because its checks read the registry.

```
                    seed (byte-identical x3)
   scripts/doc-spec.sh heredoc ─┐
   templates/doc-spec-common.md ─┼─ row + table-row + "twelve"  ──cmp──> equal
   spec/doc-spec.md ────────────┘
            │ (merged registry, parsed by doc-spec.sh)
            ▼
   docs/doc-general.md  (regenerated view; Check 23 diff-clean)
   docs/reference.md    (the curated human-doc itself; Check 19 no-IDs)
   spec/doc-spec-custom.md  "twelve"   CLAUDE.md ~L539  +docs/reference.md
```

## Acceptance Criteria

<!-- Given/When/Then format. One block per requirement. -->

### Story #1: Curated reference.md exists [core]

```
GIVEN the repo with no docs/reference.md
WHEN  the build writes the curated doc
THEN  docs/reference.md exists with a short intro line + grouped category
      sections, each entry `- [Name](url) — one-line why`, every entry
      grep-grounded in CLAUDE.md / scripts/ / CHANGELOG.md / docs/ / .github/,
      and `grep -E '[FSTD][0-9]{6}' docs/reference.md` returns nothing
```

### Story #2: Registry row byte-identical in 3 seed copies [core]

```
GIVEN the three seed copies (scripts/doc-spec.sh heredoc, templates/doc-spec-common.md, spec/doc-spec.md)
WHEN  the docs/reference.md registry entry is added after docs/architecture.md
THEN  each copy carries an identical block with section: common,
      audit_class: human-doc, NO front_table, and the shape-not-links
      requirement string; doc-spec.sh --validate exits 0 (OK schema_version=1)
```

### Story #3: Human-docs prose-table row in 3 seed copies [core]

```
GIVEN the seed's "## The doc contract" → Human docs table in all 3 copies
WHEN  the `| docs/reference.md | Curated external references ... |` row is added
THEN  all 3 copies show the row directly under the docs/architecture.md row
```

### Story #4: Count edit swept everywhere [core]

```
GIVEN every place that states the general-doc count
WHEN  the count is updated
THEN  `eleven` → `twelve` in the 3 seed copies and spec/doc-spec-custom.md,
      AND CLAUDE.md's human-docs parenthetical (~L539) includes docs/reference.md
```

### Story #5: View regenerated, Check 23 green [core]

```
GIVEN the merged registry now declares docs/reference.md
WHEN  scripts/generate-doc-views.sh runs
THEN  docs/doc-general.md lists 12 common docs incl. docs/reference.md,
      AND validate.sh Check 23 (regen-to-temp + diff) is clean
```

### Story #6: 3-way seed byte-identity holds [resilience]

```
GIVEN the row + table-row + count edits applied to all 3 seed copies
WHEN  `bash scripts/doc-spec.sh --seed | cmp - spec/doc-spec.md` AND
      `cmp spec/doc-spec.md templates/doc-spec-common.md` run
THEN  both report no differences (exit 0)
```

### Story #7: Config test 8b growth-safe + lists reference.md [observability]

```
GIVEN config test 8b in tests/cj-document-release-config.test.sh (inclusion-based)
WHEN  the seed declares 12 docs and the test runs
THEN  8b passes (it asserts the seed INCLUDES core docs, not an exact count),
      AND it now also asserts docs/reference.md is among the declared seed docs
```

### Story #8: QA dogfoods the three-stage audit [usability]

```
GIVEN this build's QA phase
WHEN  the Step 8.6c /CJ_doc_audit runs
THEN  docs/reference.md is judged `satisfies` against its requirement (Stage 2)
      and `no-drift` against the implementation (Stage 3)
```

## Architecture

<!-- High-level design for this story. -->

```
EDIT (mirror x3, byte-identical):
  scripts/doc-spec.sh        ── heredoc: + registry row (after architecture)
  templates/doc-spec-common.md                + Human-docs table row
  spec/doc-spec.md                            + "eleven" -> "twelve"
                  │
                  ├── spec/doc-spec-custom.md  : "eleven" -> "twelve"
                  └── CLAUDE.md (~L539)        : parenthetical += docs/reference.md

CREATE:
  docs/reference.md          ── curated grouped link shelf (grep-grounded)

REGENERATE:
  scripts/generate-doc-views.sh ──> docs/doc-general.md (12 docs)

VERIFY (no source edit):
  doc-spec.sh --seed | cmp -      (3-way byte identity)
  doc-spec.sh --validate / --list-declared / --check-on-disk
  validate.sh (Checks 15/15a/17/19/20/23 read the registry — auto-cover)
  test.sh (config test 8b)
```

### Data Flow

<!-- How the contract resolves the new doc, end to end. -->

1. The three seed copies carry the new registry row identically; `doc-spec.sh` reads `spec/doc-spec.md` (general) merged with `spec/doc-spec-custom.md` (overlay).
2. `doc-spec.sh --list-declared` / `--check-on-disk` surface `docs/reference.md`; `validate.sh` Checks 15/15a/17/19/20 read that merged list and cover the doc with no validate.sh edit.
3. `generate-doc-views.sh` renders the `section: common` rows into `docs/doc-general.md`; Check 23 regenerates into a temp dir and diffs — drift is a hard error.
4. QA Step 8.6c runs `/CJ_doc_audit`: Stage 2 quotes the doc's `requirement` and judges `satisfies`; Stage 3 cross-walks the doc against the live tree and reports `no-drift`.

## Tradeoffs

<!-- Design decisions, rejected alternatives, rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Tier | General seed (`section: common`) | Custom-overlay row in `spec/doc-spec-custom.md` | "Required everywhere" — the operator chose the contract-wide option; the rule means the same in every adopting repo (D15.1). |
| v1 content | Curated, grep-grounded entries | Empty stub | A useful doc from day one; honesty enforced by grep-grounding (D15.2). |
| Front table | Omit `front_table` | Require a leading summary table | reference.md is a link shelf, not a principle/workflow index — Check 20 should not apply. |
| Requirement string | Asserts SHAPE + PURPOSE | Enumerate specific links | Links churn; the doc must not self-stale on a single add/remove. |
| validate.sh | Leave untouched | Add a reference.md-specific check | Registry-reading checks already cover any declared doc; an extra check is redundant. |
