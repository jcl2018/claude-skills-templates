## Problem Statement

`/CJ_personal-pipeline` is an `experimental`, internal-only orchestrator that chains
scaffold→implement→qa as fresh-context subagents and returns a `PIPELINE_END_STATE`
contract. F000027 already flattened `/CJ_goal_feature` and `/CJ_goal_defect` off it,
leaving `/CJ_goal_todo_fix` as its **only** live caller — single-TODO mode
(`todo_fix.sh:870` → `DISPATCH_CHAIN=/CJ_personal-pipeline --work-item-dir …
--suppress-final-gate`) and drain mode (`drain-one-todo.sh`). The maintainer
(workbench developer) wants the indirection layer gone: flatten todo_fix to dispatch
impl→qa leaf subagents directly (the proven feature/defect pattern), then delete
`/CJ_personal-pipeline` and clean every live-surface reference. The pain today is a
near-vestigial middle layer — one consumer, a stale contract (its docs still name the
retired `/CJ_goal_run`), and a duplicate of dispatch logic the other orchestrators
already inline.

## Mental Model

The change has three phases, applied in order. **(1) Flatten** the two todo_fix dispatch
sites (single-TODO + drain) to call `/CJ_implement-from-spec` → `/CJ_qa-work-item` as
leaf Agent subagents — this is `CJ_goal_feature` Steps 3.2-3.3 (both Agent-tool,
silent/no-AUQ, halt-on-red between), minus the scaffold step (todo_fix already scaffolds
the T-task dir in pure bash at `todo_fix.sh:608-693`, which mirrors
personal-pipeline's `--work-item-dir` = "skip scaffold, run impl→qa" mode at
`pipeline.md:158`). **(2) Delete** `skills/CJ_personal-pipeline/` and rewrite the
catalog `depends.skills`. **(3) Sweep** ~18 live-surface reference files, the trickiest
being `validate.sh` Check 12 (a guard-token grep on the now-deleted `pipeline.md`),
which MUST be reconciled together with `test.sh`'s parallel reference in the same change.

```
BEFORE:  /CJ_goal_todo_fix ──► /CJ_personal-pipeline ──► [scaffold] ──► impl ──► qa   (depth 3 conceptually; nested-subagent wall)
AFTER:   /CJ_goal_todo_fix ──► (bash scaffold) ──► /CJ_implement-from-spec ──► /CJ_qa-work-item   (depth ≤ 2, leaf subagents)
                                                   └── halt-on-red between phases ──┘
```

### Data Flow

<!-- How does control move through the orchestrator for the primary use case? -->

1. Operator invokes `/CJ_goal_todo_fix <T-ID>` (single-TODO) or `--max-drain N` (drain).
2. todo_fix preamble (single-TODO) or `drain-one-todo.sh:255` (drain) establishes the per-TODO worktree — UNCHANGED by this story.
3. `todo_fix.sh` scaffolds the T-task dir in pure bash (lines 608-693) — UNCHANGED.
4. The orchestrator dispatches `/CJ_implement-from-spec` as a leaf Agent subagent (was: `/CJ_personal-pipeline --work-item-dir …`).
5. On impl RESULT green, the orchestrator dispatches `/CJ_qa-work-item` as a leaf Agent subagent; halt-on-red between (`halted_at_impl` / `halted_at_qa`).
6. On qa green, control returns to the existing `/ship` → `/land-and-deploy` tail (Gate #2 still fires per drained PR — unchanged).

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Delete vs orphan | Delete `/CJ_personal-pipeline` in the same PR (Approach A) | Flatten now, delete in a follow-up (B) | B leaves a zero-caller experimental skill + a second PR; A leaves the tree in its final state, matching every prior front-door retirement |
| Flatten vs re-consolidate | Flatten todo_fix to leaf subagents (removes the layer) | Re-route all 3 orchestrators back through personal-pipeline (C) | C re-introduces the F000027 nested-subagent wall and reverts a deliberate decision |
| Flag handling | Drop `--suppress-final-gate` | Translate it to a leaf-subagent flag | It is a personal-pipeline-only AUQ-suppression concept; the leaf subagents have no such gate, so there is nothing to translate |
| Check 12 handling | Remove the whole block + reconcile test.sh together | Remove only the grep line / leave test.sh for a follow-up | Check 12 greps the deleted `pipeline.md`; a half-removal goes red. The validate.sh↔test.sh pairing is a known blind spot (F000032/F000034/F000035). |
