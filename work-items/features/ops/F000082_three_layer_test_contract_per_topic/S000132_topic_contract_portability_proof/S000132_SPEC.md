## Problem Statement

A maintainer of this workbench cannot today assert that a test *topic* is covered
at all three verification layers, nor that the local layer carries both a
deterministic and an agentic test. The `categories:` axis (F000078) classifies
each test by category × layer + mode, but there is no notion of a *topic* spanning
rows, and no enforcement that a topic reaches every layer. Concretely, the
portability topic has a deterministic local test whose agentic counterpart is
deferred — so a green suite can hide an inert version-notification preamble. This
story builds the contract (topic axis + enrollment + hard Check) plus the agentic
mechanism (a repo-neutral sandbox lib) and proves it on portability.

## Mental Model

The change adds ONE new axis (`topic`) on top of the existing category × layer +
mode model, plus an ENROLLMENT list that opts a topic into a hard contract:

```
categories: row  =  name × category × layer × mode × tier × command × doc × purpose  +  [NEW] topic
                                                                                            |
topic_contracts: [portability]  ── enrolls ──> the hard Check applies to this topic only
                                                                                            |
                                                                                            v
   enrolled topic MUST reach:  CI-push  +  CI-nightly  +  (local-hook ∩ deterministic)  +  (local-hook ∩ agentic)
```

Deterministic vs agentic is drawn by *where it can run*: agentic lives at
local-hook (the machine has Claude), which keeps model spend out of CI. Because
`mode:agentic ⇒ tier≠free`, the agentic row is present-in-CI-but-never-executed;
the hard Check proves DECLARATION (CI-safe), the executor proves BEHAVIOR
(local-only).

## Acceptance Criteria

<!-- Given/When/Then. Each #n header matches the # column in Requirements. -->

### Story #1: topic axis parses across the six sites [core]

```
GIVEN every categories: row now carries a topic: field (9th column) and all 12 existing rows are backfilled
WHEN  I run test-spec.sh --validate and test-spec.sh --list-categories
THEN  --validate passes (no field-count/enum error) and --list-categories emits a topic for all 12 rows
```

### Story #2: enrollment parses [core]

```
GIVEN spec/test-spec-custom.md declares topic_contracts: [portability]
WHEN  the engine parses the overlay
THEN  portability reads as enrolled and unenrolled topics are unaffected
```

### Story #3: the hard Check catches an under-covered enrolled topic [resilience]

```
GIVEN portability is enrolled and its agentic row/test/doc is removed
WHEN  I run test-spec.sh --check-topic-contract
THEN  it exits non-zero with a finding naming the missing local-hook+agentic coverage; restoring the row makes it exit 0
```

### Story #4: the Check is wired CI-safely + negatively tested [integration]

```
GIVEN validate.sh calls --check-topic-contract as a new hard Check and scripts/test.sh has the paired negative test
WHEN  the per-PR gate runs on this repo
THEN  validate.sh is green, spends zero model tokens, and the negative test plants→fails→restores→passes invoking ONLY the targeted engine (no whole-validate re-run)
```

### Story #5: the agentic-sandbox lib exists + deterministic helpers smoke [core]

```
GIVEN scripts/lib/agentic-sandbox.sh (POSIX+LF)
WHEN  I run its deterministic-helper smoke (neutral sandbox creation + git init --bare tagged upstream via SKILLS_UPDATE_REMOTE_URL, no git shim)
THEN  the helpers succeed with no model spend
```

### Story #6: the portability agentic test proves the nudge reaches a human [observability]

```
GIVEN tests/portability-version-agentic.test.sh
WHEN  it runs without CJ_E2E_LOCAL=1 + a verified claude login
THEN  it SKIPs cleanly (exit 0, no model spend);
AND   WHEN run locally with a verified login it drives claude --print (--model sonnet, budget $1.00) and PASSES iff its {surfaced_nudge, evidence} verdict shows the agent relayed the upgrade nudge
```

### Story #7: the cj_test_ verbs run + audit the row [usability]

```
GIVEN the new categories row portability-version-agentic (infra / local-hook / agentic / local-only / topic: portability)
WHEN  I run /CJ_test_run portability-version-agentic --e2e (and separately a default free run) and /CJ_test_audit
THEN  --e2e runs it, the default free run SKIPs it (tier-not-selected), and /CJ_test_audit Stage 1 reports it wired; a --topic portability selector runs every portability row
```

### Story #8: docs + contract prose green, grandfather recorded [integration]

```
GIVEN the front-door doc (three sections) + docs/tests/index.md + spec/doc-spec-custom.md + spec/test-spec.md prose (+ byte-identical _emit_seed) + CLAUDE.md updates + grandfather TODOs
WHEN  I run validate.sh
THEN  Checks 24/26/27/28 + doc-spec --check-on-disk pass and the seed-identity test passes
```

### Data Flow

<!-- How data moves through the system for the primary use case. -->

1. The maintainer edits `spec/test-spec-custom.md`: backfills `topic:` on 12 rows, adds `topic_contracts: [portability]`, and adds the `portability-version-agentic` row.
2. `test-spec.sh --check-topic-contract` reads the merged registry, resolves portability's rows, and asserts the four required (layer, mode) coverage points + front-door docs exist.
3. `validate.sh` calls the engine as a hard Check on every PR (declaration-only → CI-safe, zero model spend).
4. Locally, `/CJ_test_run portability-version-agentic --e2e` runs `tests/portability-version-agentic.test.sh`, which builds a neutral sandbox + a bare upstream tagged `v<newer>` via `SKILLS_UPDATE_REMOTE_URL`, runs the update-check preamble through `claude --print`, and extracts `{surfaced_nudge, evidence}` — PASS iff the agent surfaced the nudge.

## Tradeoffs

<!-- Design decisions with rejected alternatives and rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Enforcement strength | First-class `topic:` + a HARD Check | Advisory mode-aware NOTE (Approach A) | The operator wants the contract to bite, not to emit another NOTE. |
| Landing safety | Per-topic enrollment (`topic_contracts:`) as an opt-in seam | "Every local-hook topic needs both modes" (implicit, global) | A global rule reds the build immediately (workflow topics are agentic-only today); enrollment grandfathers them. |
| Coverage bar for enrolled topics | All three layers AND local both-modes | Local both-modes only | Portability already has CI-push + CI-nightly, so requiring all three costs nothing and is the stronger guarantee (OQ2 resolved). |
| Neutral-upstream mechanism | Reuse `SKILLS_UPDATE_REMOTE_URL` (tagged bare repo) | PATH-prepend a `git` shim | A `.sh` git shim is fragile on Windows Git Bash and reinvents an existing hook. |
| Lib surface | The 3 helpers the first consumer uses | A fuller speculative helper set | YAGNI; the `e2e-local`/`eval.sh` migrations that justify more are fast-follows. |
