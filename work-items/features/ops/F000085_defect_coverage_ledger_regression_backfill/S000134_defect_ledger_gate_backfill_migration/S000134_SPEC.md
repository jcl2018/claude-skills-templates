## Problem Statement

The workbench maintainer — and the `/CJ_test_audit` + `/CJ_test_run` verbs
acting on their behalf — cannot answer "is defect X still protected, and by
what?" without archaeology. 38 defect dirs exist under `work-items/defects/**`;
the `categories:` axis carries zero `regression` rows; proof is scattered
(dedicated test files, inline `scripts/test.sh` D-blocks, shared suites, or
nothing) and nothing links a defect to its proof. An inventory agent
demonstrably cited two nonexistent test files with high confidence — undeclared
proof is folklore. The operator also plans to remove agentic tests later; an
undeclared ledger could be silently orphaned by that purge.

## Mental Model

Two stages inside one story, ledger-first. The LEDGER (Stage 1) is a new
declarative axis + a deterministic gate: every defect dir gets exactly one row
with one of three closed dispositions, and the engine proves each row live on
every validate run. The MIGRATION (Stage 2) is legibility on top: pure
dedicated drills move into the contract's `tests/regression/<layer>/` folder
shape and become named, runnable-by-name `categories:` rows. Commits 1–3 are
the ledger, commits 4–6 the migration; a migration red descopes to ledger-only
because the ledger's MIGRATE rows are valid `covered-by-anchor` rows until the
final flip.

```
disposition: covered-by        -> a named regression categories row (deterministic)
disposition: covered-by-anchor -> source + anchor, anchor greps live (shared proof)
disposition: waived            -> reason (gaps: "gap — ..." + TODOS row)
```

### Data Flow

<!-- Primary use case: "is defect X still protected, and by what?" -->

1. Maintainer (or `validate.sh` Check 32 / `/CJ_test_audit` Stage 1) invokes `bash scripts/test-spec.sh --check-defect-coverage`.
2. The engine resolves the merged registry (spec/-then-root), gates on registry + `defect_coverage:` axis + `work-items/defects/` presence — any absent ⇒ named `inactive` SKIP, exit 0.
3. Forward pass: enumerate `work-items/defects/**/D??????_*` dirs; each must map to exactly one ledger row (keyed by full relative path).
4. Reverse pass per row: dir exists; `covered-by` → the named `categories:` row exists and is `mode: deterministic` (agentic ⇒ FINDING); `covered-by-anchor` → anchor greps live in source; `waived` → non-empty reason.
5. Engine prints `defect coverage: dirs=<N> rows=<N> findings=<M>`; Check 32 hard-fails on findings; `/CJ_test_run --category regression` separately executes the named regression rows.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Linkage home | `defect_coverage:` overlay axis | Per-test docs or a new root file | Check 19 bans D-IDs in human-docs; the overlay is operational tier and already carries the sibling axes; no dual-write |
| Row key | Full dir path relative to `work-items/defects/` | Bare D-ID | D000021 is duplicated across two component dirs — bare IDs ambiguous |
| Inline D-block battery | One coarse units row, referenced via `covered-by-anchor` | Per-defect extraction (Approach C) | Extraction destabilizes a proven battery and mints junk rows; rejected by the cold read |
| Packaging | Single-PR Full, ledger-first commits | Two PRs ledger-first (cold-read recommendation) | Operator chose the complete end state; commit ordering preserves descope-on-red |
| Sweep recursion | Token-grammar change (rel-path tokens, both sites) | Naive glob recursion | Basename tokens + the `scripts/test.sh` source pin make naive recursion break token matching |
| Moved files' units rows | `anchor:` updated, `source:` stays `scripts/test.sh` | `source:` pointed at the test file | A self-referential source would self-satisfy the forward grep (engine comment ~:1603-1609) |
| Verification failures | Default `waived: "gap — …"` + TODOS row | Author gap drills in-PR | Keeps the PR bounded; exception only for ≤30-line no-fixture shape-guards, cap 3 |
| Tier enforcement | Mode gate only (`covered-by` ⇒ deterministic) | Separate tier check in the engine | The contract-wide `mode: agentic ⇒ tier ≠ free` rule + the mode gate already pin the tier |
