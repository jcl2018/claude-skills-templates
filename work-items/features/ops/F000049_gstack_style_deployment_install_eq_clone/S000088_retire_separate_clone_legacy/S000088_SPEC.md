## Problem Statement

F000049 S4 is the subtractive close-out: "drop `.source` + retire the worktree /
`post-land-sync` machinery + flip `--bundle` to default." The de-risking finding
(the live manifest's `source` already equals the dev checkout; `install_mode` is
`null`) shows install==clone is reachable **in place** — no relocation. So S4
becomes a staged cleanup: (1) **declare** the default install install==clone
(`install_mode: in-place`), (2) **drop** the now-redundant runtime `.source`
reach-backs from the skill preambles (the `_cj-shared` deposit from S1 + the
repo-local tier already cover every invocation path), and (3) **reframe** (not
delete) the sync helpers to the in-place checkout. The literal roadmap verbs
over-promise a full deletion that is partly UNSAFE: `gh pr merge` lands code
REMOTELY, so the in-place checkout still needs a post-merge `git pull` —
`post-land-sync` / `--phase sync` are kept, reframed.

## Mental Model

```
Before:  default install records source=<checkout> but never DECLARES install==clone;
         4 orchestrators resolve cj-goal-common.sh repo-local → _cj-shared → .source (legacy);
         10 skills read manifest.source to locate skills-update-check.
S4:      default install stamps install_mode: in-place + bundle_path=<checkout>   (declare)
         4 orchestrators: drop the .source tier → repo-local → _cj-shared          (no exec reach)
         10 skills: update-check resolves from _cj-shared                          (no nudge reach)
         post-land-sync / --phase sync / cj-goal-common: reframed to the in-place checkout (pull KEPT)
         --bundle: retained as the fresh-consumer bootstrap; the DEFAULT is install==clone-in-place
KEEP:    the cj-feat-* worktree flow (D2 — the parallel-build isolation primitive)
```

## Acceptance Criteria

### Story #1: default install declares install==clone-in-place [core]

```
GIVEN a default `skills-deploy install` (no --bundle) from a workbench checkout
WHEN  the install finishes
THEN  the manifest carries install_mode: "in-place" AND bundle_path equal to source (the checkout)
```

### Story #2: orchestrators resolve shared scripts with no .source tier [core]

```
GIVEN the 4 orchestrator SKILL.md preambles
WHEN  their cj-goal-common.sh resolution is inspected
THEN  it is 2-tier (repo-local → _cj-shared) with NO `.source` elif branch
```

### Story #3: no skill reads manifest.source at runtime [core]

```
GIVEN the 10 skills that previously read manifest.source for the update-check
WHEN  their preambles are inspected
THEN  the update-check resolves skills-update-check from _cj-shared with NO `jq -r '.source'` read
```

### Story #4: family audit-clean + mode visible [integration]

```
GIVEN the S4 changes
WHEN  the portability audit runs over the 4 orchestrators AND `skills-deploy bundle-status` runs on an in-place install
THEN  the family is local-only/standalone with no `.source`-reachback PREAMBLE finding AND status reports install_mode: in-place
```

### Story #5: documented + dev-flow-safe (sync reframed, not deleted) [usability]

```
GIVEN the S4 changes
WHEN  a maintainer reads the docs AND runs post-land-sync / --phase sync after a remote merge
THEN  the install==clone-in-place model is documented AND the sync helpers still pull the in-place checkout (reframed, functional)
```

## Architecture

```
  skills-deploy install            (default — no --bundle)
        │  resolve main_toplevel (the checkout you ran from)
        ▼
  manifest write:  + install_mode: "in-place", bundle_path: <main_toplevel>     (declare install==clone)
        │
        ▼
  SKILL.md preambles (the runtime de-coupling)
     · 4 orchestrators:  cj-goal-common.sh  →  repo-local → _cj-shared           (drop .source elif)
     · 10 skills:        update-check       →  $HOME/.claude/_cj-shared/scripts  (drop manifest.source read)
        │
        ▼
  audit:  cj-portability-audit — fewer .source-reachback notes; family stays local-only
  status: bundle-status — recognizes install_mode: in-place
  reframe (NOT delete): post-land-sync / --phase sync / cj-goal-common — operate on the in-place checkout
```

### Data Flow

1. The default install resolves `main_toplevel` (the checkout it ran from) and writes `source` + the new `install_mode: in-place` + `bundle_path`.
2. A skill preamble resolves its shared scripts repo-local → `_cj-shared` (orchestrators) or the update-check from `_cj-shared` (all 10) — no `manifest.source` read.
3. `skills-update-check` (the script) still reads `manifest.source` internally = the in-place checkout — maintainer tooling, not a skill reach-back; out of the AC's "skill" scope.
4. `post-land-sync` / `--phase sync` pull the in-place checkout after a remote merge (reframed, retained).

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Bundle location (D1) | In-place: declare the existing checkout install==clone | Relocate every dev env to `~/.claude/skills/cj-workbench` (D1-A) | `source` already == the checkout; relocation is needless disruption for a single-dev workbench. Same no-reach-back end-state. |
| "Flip `--bundle` to default" | The DEFAULT install IS install==clone-in-place (declared); `--bundle` retained as the fresh-consumer bootstrap | Make `--bundle` (relocate) the literal default | Forcing relocation on developers contradicts D1-B; the default already symlinks from the checkout it ran from |
| "Retire `post-land-sync` / `--phase sync`" | REFRAME to the in-place checkout; keep the functional pull | Delete them | `gh pr merge` is a REMOTE merge — the in-place checkout still needs a post-merge `git pull`; deleting breaks the dev-flow refresh |
| Worktrees (D2) | KEEP the `cj-feat-*` worktree flow | Retire worktrees as "separate-clone machinery" | Worktrees are the parallel-build isolation primitive (created INSIDE the checkout); "develop in place" replaces the `.source` reach-back, not worktrees |
| Update-check repoint | `_cj-shared`-only (2-line, git-free, uniform sed across 10) | repo-local-first for the nudge too | The passive nudge needs no repo-local precedence; the deposited copy tracks the checkout and is lower-risk to swap uniformly |
