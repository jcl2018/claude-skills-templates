## Problem Statement

The two F000027-introduced skills (`cj_goal_feature`, `cj_goal_defect`) ship in lowercase while the rest of the CJ_* skill family (9 of 11 skills, including the other two `CJ_goal_*` orchestrators) is uppercase. Fresh readers parse the mix as a real defect; the user (a workbench operator who has to interact with the routing surfaces daily) experiences low-grade cognitive friction every time their eye lands on the inconsistency. This story implements the casing-only fix: rename the dirs, create lowercase deprecation shims, update all active-routing references, ship at v5.0.12.

## Mental Model

The implementation has three nested loops:

1. **Outer loop — five logical groups** (per the parent DESIGN's Shape table): rename, self-reference flips, shim creation, catalog edits, cross-reference flip.
2. **Middle loop — per-skill (feature vs defect)**: every group except catalog runs twice, once per renamed skill.
3. **Inner loop — per-file**: within each skill, multiple files (SKILL.md + pipeline.md + shim) get edited.

The S000060 regression test (`scripts/test.sh` lines 1044-1049) is the gating fact: it asserts the F000027 shim at `skills/CJ_goal_run/SKILL.md` references the canonical via `grep -qE '/cj_goal_feature'`. To keep `test.sh` green during the rename, the regex flip + the F000027 shim cross-reference flip (Step 6's last sub-bullet) + the catalog `depends.skills` flip (Step 5's dep-chain fix) all need to land in the same commit as the dir rename.

### Story #1: Rename dirs to uppercase [core]

```
GIVEN  `skills/cj_goal_feature/` and `skills/cj_goal_defect/` exist (lowercase)
WHEN   the implementer runs `git mv skills/cj_goal_feature skills/CJ_goal_feature_TMP && git mv skills/CJ_goal_feature_TMP skills/CJ_goal_feature` (and same for defect)
THEN   `ls skills/ | grep -i goal` shows `CJ_goal_feature CJ_goal_defect CJ_goal_run CJ_goal_auto CJ_goal_investigate CJ_goal_todo_fix` (uppercase only); old lowercase paths no longer resolve
```

### Story #2: Flip SKILL.md self-references [core]

```
GIVEN  `skills/CJ_goal_feature/SKILL.md` and `skills/CJ_goal_defect/SKILL.md` exist with lowercase frontmatter + in-body references
WHEN   the implementer flips frontmatter `name:` field + all `/cj_goal_feature` / `/cj_goal_defect` in-body mentions + path-resolution-block references to uppercase
THEN   `grep -n 'cj_goal_feature\|cj_goal_defect' skills/CJ_goal_feature/SKILL.md skills/CJ_goal_defect/SKILL.md` returns ZERO active-routing matches (any remaining lowercase mention is annotated as historical/runtime)
```

### Story #3: Flip pipeline.md self-references; keep resume state dirs lowercase [core]

```
GIVEN  `skills/CJ_goal_feature/pipeline.md` and `skills/CJ_goal_defect/pipeline.md` exist with lowercase in-body references
WHEN   the implementer flips in-body skill references to uppercase BUT preserves `.cj-goal-feature/` / `.cj-goal-defect/` runtime state dir references
THEN   `grep -n 'cj_goal_feature' skills/CJ_goal_feature/pipeline.md` returns only `.cj-goal-feature/` (runtime state path); no active-routing matches
```

### Story #4: Create lowercase deprecation shims under deprecated/ [usability]

```
GIVEN  `deprecated/cj_goal_feature/` and `deprecated/cj_goal_defect/` do NOT exist (or are empty)
WHEN   the implementer creates `deprecated/cj_goal_feature/SKILL.md` and `deprecated/cj_goal_defect/SKILL.md` with the F000027-shim-shape content (YAML frontmatter with name=lowercase + description + version=5.0.12 + allowed-tools=[Skill]; `## Deprecation Banner` section; `## Routing` section instructing Skill-tool invocation of the uppercase canonical)
THEN   both files exist, are ~40 lines each, and pass the structural check that frontmatter `name` matches the lowercase dir basename
```

### Story #5: Update skills-catalog.json (6 edits) [core]

```
GIVEN  `skills-catalog.json` has 2 active entries `cj_goal_feature` + `cj_goal_defect` (lowercase, files under `skills/`) and 2 deprecated entries `CJ_goal_run` + `CJ_goal_auto` (with `depends.skills: ["cj_goal_feature"]`)
WHEN   the implementer flips the 2 active entries (name + files paths to uppercase), adds 2 new deprecated entries (lowercase name, files under `deprecated/`), and flips the 2 F000027 shim entries' `depends.skills` to `["CJ_goal_feature"]`
THEN   `./scripts/validate.sh` exits 0; catalog now has 4 active CJ_goal_* + 4 deprecated CJ_goal_* / cj_goal_* entries (8 total in the family)
```

### Story #6: Flip F000027 shim cross-references to uppercase canonical [core]

```
GIVEN  `skills/CJ_goal_run/SKILL.md` and `skills/CJ_goal_auto/SKILL.md` reference `/cj_goal_feature` (lowercase) in description, Deprecation Banner, and Routing
WHEN   the implementer flips all 3 sites in each file to `/CJ_goal_feature` (uppercase)
THEN   the deprecation chain is one hop: `/CJ_goal_run` → banner → `/CJ_goal_feature` (canonical), NOT `/CJ_goal_run` → banner → `/cj_goal_feature` (deprecated shim) → banner → `/CJ_goal_feature`
```

### Story #7: Flip all active-routing references across docs/scripts/tests [core]

```
GIVEN  routing references to `/cj_goal_feature` and `/cj_goal_defect` exist in rules/skill-routing.md, CLAUDE.md, doc/PHILOSOPHY.md, doc/ARCHITECTURE.md, scripts/cj-goal-common.sh, tests/cj-goal-feature-smoke.test.sh, tests/cj-worktree-init.test.sh

### Story #8: Update scripts/test.sh S000060 regression test [core]

```
GIVEN  `scripts/test.sh` lines 1044-1049 contain `grep -qE '/cj_goal_feature'` asserting the F000027 shim references the lowercase canonical

### Data Flow

<!-- How does data move through the system for the primary use case?
     Step-by-step, component to component. -->

Primary use case = operator invokes `/CJ_goal_feature "<topic>"` (canonical) or `/cj_goal_feature "<topic>"` (deprecated shim).

**Canonical path** (after this PR):
1. Operator types `/CJ_goal_feature "<topic>"` in Claude Code.
2. Claude Code resolves to `~/.claude/skills/CJ_goal_feature/SKILL.md` (deployed via `skills-deploy install`).
3. SKILL.md preamble + pipeline.md execute the feature pipeline (worktree → office-hours → scaffold/impl/qa subagents → /ship → PR-stop).
4. No deprecation banner.

**Shim path** (after this PR):
1. Operator types `/cj_goal_feature "<topic>"` (old muscle memory).
2. Claude Code resolves to `~/.claude/skills/cj_goal_feature/SKILL.md` (deployed via `skills-deploy install --include-deprecated`).
3. Shim prints one-line deprecation banner.
4. Shim invokes Skill tool with `CJ_goal_feature` + same args.
5. Pipeline runs identically to the canonical path.

**Validation flow** (during the PR being shipped):
1. Pre-commit hook runs `./scripts/validate.sh` → catalog/filesystem cross-check → must exit 0.
2. Pre-commit hook runs `./scripts/test.sh` → lines 1044-1049 S000060 regression test runs `grep -qE '/CJ_goal_feature' skills/CJ_goal_run/SKILL.md` → must match (because Step 6 flipped the F000027 shim cross-reference).
3. If either exits non-zero, commit is blocked; implementer fixes and re-runs.

## Tradeoffs

<!-- Design decisions made in this spec, with rejected alternatives and rationale.
     Future readers need to know why this path over the others. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Story decomposition shape | Single atomic story (S000064 carries all 10 steps) | Multi-story (e.g., "rename skills" + "create shims" + "flip references" + "update tests") | S000060 regression test couples the rename + the shim cross-ref flip + the test-regex flip. Any decomposition leaves an intermediate PR red. Atomic = the only shape that keeps the pre-commit hook passing throughout. |
| git mv approach | Two-step via `_TMP` intermediate | Direct `git mv lower UPPER` | APFS is case-insensitive: direct rename is a no-op (git sees both as the same path). Temp-name forces git to record an R-rename. |
| Shim placement | `deprecated/cj_goal_feature/` + `deprecated/cj_goal_defect/` | `skills/cj_goal_feature/` + `skills/cj_goal_defect/` (mirror F000027 placement) | (a) APFS case-collision blocks coexistence with the new uppercase canonical at `skills/CJ_goal_feature/`. (b) CLAUDE.md "Deprecated skills convention" documents `deprecated/` placement; this PR is the first actual use. |
| README handling | Auto-regenerate via `./scripts/generate-readme.sh` | Hand-edit | README is generated from catalog; hand-edits drift on next regen. |
| Memory file handling | Excluded from PR diff (operator-local follow-up after merge) | Edit in this PR | Memory files are operator-local state per `feedback_workbench_scope`. Editing in PR diff leaks personal state + doesn't apply on other operators. |
| Version literal in shim frontmatter | Bake `5.0.12` literal with `check-version-queue.sh` preflight escape hatch | Use a `{VERSION}` placeholder substitution at install time | The shim frontmatter `version` field is file content, not runtime state. Substitution would need an install-time pipeline that doesn't exist. Preflight + hand-edit (~5 sec) handles the rare collision. |
