## Problem Statement

The workbench operator (sole user, this repo) needs the v6.0.0 sunset wave executed: 5 deprecated alias-shim skill directories removed, their catalog entries dropped, the deprecation infrastructure that supports them (status enum value, `deprecated/` dir, `--include-deprecated` flag, F000030 audit convention, tombstone doc sections) retired, dead tests deleted, live tests updated, hygiene sweeps run, VERSION + CHANGELOG bumped, README regenerated — all as one atomic squash-merge PR with title `v6.0.0 feat: F000035 sunset deprecated shims + deprecation infrastructure (full nuke)`.

The pain today: 5 dead shim entry points pollute the skill catalog and confuse intent. The infrastructure that supports them exists only for backward-compat with multi-operator scenarios that don't exist in this solo project. Validate.sh Check 9b's closed-enum still names `deprecated` even though nothing remains tagged that way after the shims go. The doc surfaces carry tombstones that are no longer needed once the pattern itself retires.

## Mental Model

Two layers, one commit:

```
Layer 1: Live skill surface
  - Delete 5 skill dirs                          (skills/CJ_goal_run/, skills/CJ_goal_auto/, deprecated/{3 entries})
  - Filter 5 catalog entries                     (skills-catalog.json via jq)
  - Delete deprecated/ tree                      (rm -rf deprecated/)

Layer 2: Supporting infrastructure
  - validate.sh Check 9b enum                    (close enum to {active, experimental})
  - skills-deploy --include-deprecated           (remove 6 sites)
  - generate-readme.sh DEPRECATED_COUNT block    (drop entirely)
  - CLAUDE.md (4 surgeries)                      (skill-routing paragraphs, deprecated-skills subsection, retired-skill drift check, auto-worktree bullet)
  - doc/PHILOSOPHY.md ## Retired skills          (delete section + cross-refs)
  - doc/ARCHITECTURE.md ## Deprecation tombstones + inline mentions
  - rules/skill-routing.md ## Deprecated front doors
  - Tests: delete 2 + delete tests/eval/CJ_goal_run/ + inspect-update 2
  - TODOS.md sweep, memory cleanup, VERSION, CHANGELOG, README regen

Ordering: All 15 edits are independent on disk. Stage once at the end via git add -A.
         The pre-commit hook runs validate.sh against the fully-staged tree;
         either green (commit) or surgical fix-and-restage.
```

### Data Flow

1. Operator on feature branch `cj-feat-20260602-010655-sunset` (already at `chore: WIP hygiene` commit `006ffe3`).
2. Execute Layer 1 deletes: `rm -rf skills/CJ_goal_run/ skills/CJ_goal_auto/ deprecated/`.
3. Filter catalog: `jq '[.[] | select(.status != "deprecated")]' skills-catalog.json > tmp && mv tmp skills-catalog.json`.
4. Edit `scripts/validate.sh` Check 9b — close enum to `{active, experimental}`, update fail messages + comment.
5. Edit `scripts/skills-deploy` — remove 6 `--include-deprecated` sites (arg parse, gate `if`, INFO labels, help text).
6. Edit `scripts/generate-readme.sh` — drop `DEPRECATED_COUNT` line + `if > 0` block.
7. Apply 4 CLAUDE.md surgeries via Edit tool.
8. Apply doc surface edits: PHILOSOPHY.md (delete `## Retired skills` + cross-refs), ARCHITECTURE.md (delete `## Deprecation tombstones` + rewrite inline mentions), skill-routing.md (delete `## Deprecated front doors`).
9. Test cleanup: `rm tests/cj-goal-investigate-shim.test.sh tests/cj-goal-investigate-did-allocator.test.sh; rm -rf tests/eval/CJ_goal_run/`. Inspect + update 2 remaining live test files.
10. TODOS.md sweep + memory deletion + MEMORY.md index update.
11. VERSION 5.0.19 → 6.0.0; CHANGELOG entry.
12. Regen README: `./scripts/generate-readme.sh`.
13. `git add -A && git commit -m "v6.0.0 feat: F000035 sunset deprecated shims + deprecation infrastructure (full nuke)"` — pre-commit runs validate.sh; either green or surgical fix-and-restage.
14. `/ship` opens PR; no auto-merge.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Commit shape | Single atomic commit | 3-5 sub-commits (Layer 1 / Layer 2 / docs / tests / version) | Check 9b enum edit can't ship safely independently of catalog filter — would leave validate.sh red. Squash-merge collapses anyway. |
| validate.sh Check 13/14/15 predicates | Leave as `status != "deprecated"` | Drop the predicate (filters nothing now) | Robust to future re-introduction of enum value; one fewer downstream change. Minimal cost. |
| `generate-readme.sh` cleanup | Drop DEPRECATED_COUNT block entirely | Leave the block (filters to 0; produces no output) | Dead code lingers; harder to read; "the simpler script is the cleaner end-state" per design Constraint #10. |
| Tombstone preservation | None — full delete | Leave one breadcrumb in PHILOSOPHY.md `## Retired skills` | CHANGELOG + git history form the audit trail. The tombstone section itself was infrastructure being retired; keeping it is internally inconsistent. |
| Memory cleanup | Delete `project_investigate_retire_candidate.md` | Leave it as historical context | The memory's own body says "delete after v6.0.0 sunset wave" — follow its instruction. |
| `tests/eval/` parent | Conditional delete (P2) | Always delete / never delete | Depends on whether other eval children exist post-Phase 2; can't know until checked. |
| Auto-merge | PR-stop | `/land-and-deploy` invocation | Skill-surface change hits the cj-handoff-gate denylist per memory `project_workbench_auto_deploy_unsafe`. Operator merges manually. |
