## Problem Statement

A user with an Anthropic best-practices URL in hand wants a one-keystroke way to discover whether the pattern is already adopted in this workbench's skills, and if not, where it should apply. Today, this requires manually opening the article, scanning every `skills/*/SKILL.md` for the pattern, deciding which skills are affected, and writing a TODOS.md row by hand. Phase 1 ships the bash + orchestrator + subagent triple that automates this — emitting an inline-marked draft TODOS.md row that flows through the existing `/CJ_suggest -> /CJ_goal_todo_fix -> /ship` pipeline once the user removes the marker.

The user is: the workbench owner (chjiang) running `/CJ_improve-queue evaluate <url>` ad-hoc when an interesting article surfaces. The pain today: silent drift between Anthropic's evolving patterns and the workbench's skills, with no surface to bridge.

## Mental Model

Three actors, each owning a layer:

```
+----------------------+  HANDOFF block  +-----------------+  Agent prompt  +-------------+
| improve_queue.sh     | --------------> | Orchestrator    | -------------> | Subagent    |
| (evaluate-prepare)   |                 | (main agent;    |                | (general-   |
|                      | <-- verdict --- | SKILL.md prose) | <-- JSON ----- | purpose)    |
|                      |  on stdin       |                 |                |             |
| improve_queue.sh     |                 +-----------------+                +-------------+
| (apply)              |
+----------------------+
```

- **Shell envelope** (bash): owns deterministic I/O, gates, locking, file mutation. Cannot be tricked by network content because it never sees network content.
- **Orchestrator** (main agent, driven by SKILL.md prose): owns the HANDOFF protocol — parses the block, dispatches the Agent tool, captures verdict JSON, pipes to `apply`.
- **Subagent** (general-purpose, fresh context): owns network + reasoning. Receives canonical URL + in-scope skill files; emits verdict JSON.

Each layer is independently testable: shell via direct invocation with stub `CJ_IMPROVE_QUEUE_VERDICT_FILE`; orchestrator via SKILL.md prose review; subagent via Subagent Contract prompt template review.

## Acceptance Criteria

<!-- Given/When/Then format. One block per requirement or logical group.
     Each #n header should match the # column in the Requirements tables. -->

### Story #1: Single inline-comment-marked row appended [core]

```
GIVEN /CJ_improve-queue evaluate https://docs.anthropic.com/claude-code/some-page
  AND TODOS.md is clean (no uncommitted changes)
  AND the URL is on the allowlist
  AND subagent returns verdict: "novel" with pattern_name and source_quote
WHEN the script runs end-to-end
THEN a single new row appears at the bottom of TODOS.md with heading shape
     `### Adopt <pattern-name> from <short-source-name> (P3, M)<!--impr-draft-->`
  AND the row body contains `**Source:** <canonical-url>`, `**Verdict:** novel`,
      `**Affected skills:** ...`, `**Suggested change:** ...`,
      `<!-- source-quote: "..." -->`, and a trailing
      `<!-- impr-sig=<16-char-hex> impr-conf=<N>/10 -->` footer.
  AND the appended row passes `^(.*) \(P[1-4], [SML]\)$` heading regex validation.
```

### Story #2: URL canonicalization [resilience]

```
GIVEN /CJ_improve-queue evaluate "https://www.docs.anthropic.com:443/some-page/?utm_source=x&fbclid=y#section"
WHEN canonicalization runs
THEN the canonical form is "https://docs.anthropic.com/some-page" (www stripped,
     default port dropped, query params stripped, fragment stripped, trailing
     slash stripped on path leaf, host lowercased).
  AND re-running with any equivalent URL form is a NO-OP (signature hit).
```

### Story #3: Source-domain allowlist [security]

```
GIVEN /CJ_improve-queue evaluate https://random-blog.example.com/article
  AND --allow-untrusted-source flag is NOT set
WHEN the script runs
THEN the script exits non-zero with stderr message naming the host and the
     override flag ("source 'random-blog.example.com' is not on the allowlist;
     pass --allow-untrusted-source to override").
  AND TODOS.md is untouched.
  AND no HANDOFF block is emitted.
```

### Story #4: Inline `<!--impr-draft-->` marker [usability]

```
GIVEN a freshly written row from /CJ_improve-queue evaluate
WHEN the row is viewed in rendered markdown
THEN the `<!--impr-draft-->` marker is invisible.
  AND `grep -c '<!--impr-draft-->' TODOS.md` returns the count of marked rows.
  AND removing the literal marker token (e.g., via `sed -i '' 's/<!--impr-draft-->//g'`)
      promotes the row in a single op.
```

### Story #5: Atomic-write discipline [resilience]

```
GIVEN /CJ_improve-queue evaluate is running mid-write
WHEN the process is killed (kill -9) between mktemp and mv
THEN TODOS.md is byte-identical to its pre-run state.
  AND /tmp/cj-improve-queue/TODOS.md.bak.<timestamp> contains the prior valid state byte-for-byte.
```

### Story #6: Pre-write dirty-check [security]

```
GIVEN TODOS.md has uncommitted changes
WHEN /CJ_improve-queue evaluate <url> is invoked
THEN the script exits non-zero with stderr message
     ("TODOS.md has uncommitted changes — commit or stash before retry").
  AND no HANDOFF block is emitted.
  AND TODOS.md is untouched.
```

### Story #7: Concurrency serialization [resilience]

```
GIVEN two parallel /CJ_improve-queue evaluate invocations on different URLs
  AND both reach the write step at approximately the same time
WHEN they contend for /tmp/cj-improve-queue-lock/
THEN the first acquires the lock (mkdir succeeds) and writes its row.
  AND the second retries (up to 3 × 0.5s); on persistent contention it exits 0
      with stderr "another instance is writing TODOS.md; please retry".
  AND no mv race occurs (mv is atomic; lock serializes the mktemp -> mv window).
```

### Story #8: macOS-only gate [usability]

```
GIVEN /CJ_improve-queue evaluate <url> on a Linux box
WHEN the script starts
THEN it exits non-zero with stderr message naming the gate
     ("macOS-only skill (uname -s = Darwin required)").
```

### Story #9: Subagent contract test (stubbed verdict) [observability]

```
GIVEN CJ_IMPROVE_QUEUE_VERDICT_FILE=tests/fixtures/CJ_improve-queue/sample-verdict-novel.json
  AND /CJ_improve-queue evaluate <url>
WHEN the script runs
THEN the WebFetch + reasoning step is skipped (stub verdict consumed).
  AND a row matching the stub verdict appears in TODOS.md.
  AND test fixtures sample-verdict-{novel,conflict,fetch-failed,malformed}.json each
      produce their documented stderr/stdout/exit-code/TODOS.md behavior deterministically.
```

### Story #10: WebFetch failure [resilience]

```
GIVEN subagent returns verdict: "fetch_failed" with error: "404 Not Found"
WHEN /CJ_improve-queue evaluate <url> runs
THEN stderr line: "[CJ_improve-queue] WebFetch failed: 404 Not Found; no row appended.".
  AND exit 0.
  AND TODOS.md is untouched.
```

### Story #11: HANDOFF envelope [core]

```
GIVEN /CJ_improve-queue evaluate-prepare <url>
WHEN the script runs
THEN stdout contains exactly one block of the form
     "CJ_IMPROVE_QUEUE_HANDOFF_BEGIN\n<single JSON line>\nCJ_IMPROVE_QUEUE_HANDOFF_END".
  AND the JSON has keys canonical_url, in_scope_skill_files (array), request_id (uuid), allowlisted (bool).
  AND the orchestrator (SKILL.md prose) instructs the main agent to parse this block,
      dispatch Agent with the Subagent Contract prompt, and pipe the verdict JSON to
      `improve_queue.sh apply` via stdin.
```

### Story #12: Signature-based idempotency [observability]

```
GIVEN /CJ_improve-queue evaluate <url> ran once and wrote a row with impr-sig=<SIG>
WHEN /CJ_improve-queue evaluate <url-equivalent> runs (same canonical URL, same pattern_name)
THEN signature = sha256(canonical_url + "|" + pattern_name) truncated to 16 chars matches.
  AND grep -Fq "impr-sig=$SIG" TODOS.md returns hit.
  AND the script exits 0 with stderr note "[CJ_improve-queue] signature already in TODOS.md (impr-sig=$SIG); skipping".
  AND TODOS.md is untouched.
```

### Story #13: Backup rotation [resilience]

```
GIVEN 6 previous /CJ_improve-queue evaluate runs created /tmp/cj-improve-queue/TODOS.md.bak.<timestamps>
WHEN a 7th run completes
THEN /tmp/cj-improve-queue/ contains exactly 5 most-recent backups (oldest deleted).
  AND `ls -t /tmp/cj-improve-queue/TODOS.md.bak.* | tail -n +6 | xargs rm -f` is the rotation rule.
```

## Architecture

<!-- High-level system design for this story. -->

```
 user CLI
  v
  /CJ_improve-queue evaluate <url>
  v
+-------------------------------------------+
| skills/CJ_improve-queue/SKILL.md          |
| (orchestrator prose; main agent reads)    |
|                                           |
| Step 1: bash scripts/improve_queue.sh     |
|         evaluate-prepare <url>            |
|         -> stdout has HANDOFF block       |
|                                           |
| Step 2: parse HANDOFF block JSON          |
|                                           |
| Step 3: dispatch Agent tool with          |
|         Subagent Contract prompt          |
|         (substitute canonical_url +       |
|          in_scope_skill_files)            |
|         -> Agent returns verdict JSON     |
|                                           |
| Step 4: echo $VERDICT_JSON |              |
|         bash scripts/improve_queue.sh     |
|         apply                             |
+-------------------------------------------+
                v
+-------------------------------------------+
| scripts/improve_queue.sh apply (stdin)    |
| ----------------------------------------- |
| 1. jq -e '.verdict' -> validate parse     |
| 2. branch on verdict:                     |
|    - match/reject: stderr note, exit 0    |
|    - fetch_failed: stderr note, exit 0    |
|    - conflict/novel:                      |
|      a. compute signature                 |
|      b. idempotency probe (grep -Fq)      |
|      c. mkdir /tmp/cj-improve-queue-lock  |
|         (retry 3 × 0.5s)                  |
|      d. backup TODOS.md to /tmp/...       |
|      e. build row (heading + body)        |
|      f. mktemp + append + mv (atomic)     |
|      g. heading-regex validation          |
|         (fail -> restore backup, exit 1)  |
|      h. rmdir lock                        |
|      i. rotate backups (keep last 5)      |
| 3. stdout summary line                    |
+-------------------------------------------+
                v
            TODOS.md (one new row appended)
```

### Data Flow

<!-- How does data move through the system for the primary use case? -->

1. User invokes `/CJ_improve-queue evaluate https://docs.anthropic.com/claude-code/<page>`.
2. SKILL.md prose: main agent invokes `bash scripts/improve_queue.sh evaluate-prepare <url>`.
3. Script canonicalizes URL, checks allowlist (host on default list -> allowlisted=true), runs preflight gates (macOS, TODOS.md clean). On fail: stderr message, exit non-zero, no HANDOFF.
4. Script emits HANDOFF block on stdout containing `{"canonical_url": "...", "in_scope_skill_files": [...], "request_id": "<uuid>", "allowlisted": true}`. Exit 0.
5. Main agent (SKILL.md prose) parses HANDOFF block JSON.
6. Main agent dispatches Agent tool with the Subagent Contract prompt, substituting canonical_url + in_scope_skill_files.
7. Subagent runs WebFetch on canonical_url; on failure emits `{"verdict":"fetch_failed","error":"..."}`. On success: reads each in-scope SKILL.md, classifies the pattern, emits verdict JSON.
8. Main agent captures Agent's stdout, extracts the JSON object.
9. Main agent invokes `echo "$VERDICT_JSON" | bash scripts/improve_queue.sh apply`.
10. `apply` reads stdin, `jq -e '.verdict'` validates parse; branches on verdict value (match/reject/fetch_failed exit cleanly; conflict/novel proceed to write).
11. For conflict/novel: compute signature, idempotency probe (skip on hit), acquire lock, backup TODOS.md, build row, atomic mv, validate heading, release lock, rotate backups.
12. Stdout summary line.

## Tradeoffs

<!-- Design decisions made in this spec, with rejected alternatives and rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Subagent dispatch contract | HANDOFF envelope (BEGIN/END block on stdout) | Prose-only re-invocation: bash writes `.claude/tmp/CJ_improve-queue.request.json`, exits 0, SKILL.md prose instructs main agent to call Agent and re-invoke `improve_queue.sh apply <verdict-file>` | Prose-only contract is the least-reliable surface; model skipping the re-invoke leaves orphan request.json with no error. HANDOFF mirrors `/CJ_goal_todo_fix`'s proven pattern. |
| Temp dir location | `/tmp/cj-improve-queue/` | `.claude/tmp/` | `~/.claude/` is user's global config — sacred. Backups SHOULD be ephemeral; `/tmp/` is natural home. Matches `/CJ_goal_todo_fix`'s precedent. |
| Synthetic ID | None (drop `I<NNNNNN>` proposal) | Synthetic `I<NNNNNN>` ID in heading | Avoids broadening `[FSTD][0-9]{6}` regex across validator + scaffolder + drain paths. Orphan rows have P3/M default rank that suits improvement rows. |
| WebFetch trust | Allowlist + HTML-comment-wrap source_quote | Trust WebFetch output unconditionally | Layered defense: allowlist (default-on) + comment-wrap (neutralizes regex-injection into `/CJ_goal_todo_fix` sensitive-surface gates). Closes /autoplan CRITICAL-4. |
| Draft marker | `<!--impr-draft-->` inline HTML comment | `DRAFT — ` heading prefix | Invisible in rendered markdown; promotion = single-token removal. Typo-resistant (vs `DRAFT—` no-space defeating filter). |
| Lock mechanism | mkdir-based on `/tmp/cj-improve-queue-lock/` | `flock` | macOS doesn't ship GNU flock by default; mkdir is atomic + dependency-free. Matches `/CJ_goal_todo_fix`'s precedent. |
| Lock scope | Write step only (mktemp -> mv -> validate -> rmdir) | Full fetch + reason + apply | Sub-second lock window vs 10-30s; parallel evaluations on different URLs run concurrently except at the write moment. |
| Verdict input to apply | stdin | File path arg | Avoids per-run state file; consistent with `/CJ_goal_todo_fix`'s precedent (no temp-file writes under sacred dirs). |
| Signature length | 16 chars (64 bits) | Full sha256 (64 chars) | 64 bits is enough collision space for any TODOS.md size; full sha256 in a heading is visually noisy. |
| Backup retention | Last 5 | Last 10 / unlimited | Restoring from backup older than last 5 is vanishingly rare; bounded retention keeps `/tmp/` clean. |
