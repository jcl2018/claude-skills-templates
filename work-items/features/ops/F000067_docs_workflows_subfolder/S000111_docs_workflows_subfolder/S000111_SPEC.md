## Problem Statement

A workbench maintainer reading or editing `docs/workflow.md` faces an 863-line /
56KB file that mixes a top-level overview with deep per-workflow reference detail
(ASCII charts + 4-bullet Touches for four orchestrators, plus utilities and audit
sections). The single file scales poorly, and the doc contract
(`spec/doc-spec.md` + `validate.sh` Checks 15a/15b/19) has no notion of a
per-workflow subfolder. The maintainer needs the deep detail factored into a
`docs/workflows/` subfolder, `workflow.md` reduced to a pure index, and the doc
contract taught the two-level structure so the split is enforced — portably,
across every repo that adopts the contract.

## Mental Model

The change has two halves that must land together:

1. **Content move (verbatim):** six dense sections of `docs/workflow.md` move into
   six `docs/workflows/*.md` files; `workflow.md` becomes a pure index.
2. **Contract teach (registry-gated portable mandate):** the engine
   (`doc-spec.sh`), `validate.sh`, the portable seed, the overlay, and the test
   contract all learn that `docs/workflows/` is required + non-empty in an
   adopting repo, that each `docs/workflows/*.md` is a declared human-doc, and
   that the overview index must still name every workflow (no-vanish).

The mandate fires only when the registry is present (`REGISTRY=absent` ⇒ skip),
so it never affects an unrelated repo.

## Acceptance Criteria

<!-- Given/When/Then. -->

### Story #1: Verbatim split + pure index [usability]

```
GIVEN docs/workflow.md with the six dense sections
WHEN  the split is performed
THEN  six docs/workflows/*.md files exist with the moved content verbatim
AND   docs/workflow.md is reduced to a ~80–120-line index that links all six
AND   no prose is lost in the move (the moved sections appear verbatim in their new files)
```

### Story #2: 3-way byte-identical seed teach [core]

```
GIVEN the portable doc-spec seed in three copies
WHEN  the docs/workflow.md Requirement is reworded and the docs/workflows/ mandate prose is added
THEN  spec/doc-spec.md == templates/doc-spec-common.md == the doc-spec.sh --seed heredoc (byte-identical)
AND   tests/cj-document-release-config.test.sh no-drift passes
```

### Story #3: Engine mandate + recursion [core]

```
GIVEN a repo with the registry present and a non-empty docs/workflows/
WHEN  doc-spec.sh --check-on-disk runs
THEN  the workflows-subfolder check reports PASS
AND   the recursed orphan scan requires every docs/workflows/*.md to be declared (else FINDING: stage1/orphans)
AND   --list-human-docs surfaces the docs/workflows/*.md paths
```

### Story #4: Registry-gated mandate [resilience]

```
GIVEN a temp-dir repo with no doc-spec registry (REGISTRY=absent)
WHEN  doc-spec.sh --check-on-disk runs
THEN  it reports REGISTRY=absent and exits 0 (the workflows-subfolder mandate does not fire)
```

### Story #5: Overlay rows + validator retarget + no-vanish [core]

```
GIVEN the 6 new docs/workflows/*.md files and spec/doc-spec-custom.md
WHEN  validate.sh runs
THEN  the 6 overlay rows are declared and Check 15a (recursed) finds no orphans
AND   Check 15b verifies each CJ_goal_* orchestrator's docs/workflows/<name>.md has its chart + 4-bullet Touches
AND   Check 15c verifies docs/workflow.md links each orchestrator's subfolder file (no-vanish)
AND   Check 19 finds no work-item IDs in the moved human-doc content
```

### Story #6: Test contract + suite green [observability]

```
GIVEN the new engine check, recursion, and Checks 15b/15c
WHEN  the test contract and suite are updated
THEN  spec/test-spec-custom.md declares units rows for each new surface
AND   tests/doc-spec-overlay.test.sh covers the new engine check + recursion
AND   the scripts/test.sh zzz-test-scaffold integration fixture exercises the new validate checks
AND   scripts/validate.sh and scripts/test.sh both run green
```

### Data Flow

1. The portable seed (`doc-spec.sh --seed` heredoc == `templates/doc-spec-common.md` == `spec/doc-spec.md`) is the general tier.
2. `doc-spec.sh` merges the general tier with `spec/doc-spec-custom.md` (overlay) into one registry.
3. `--check-on-disk` runs the deterministic conformance checks against the disk, now including the `workflows-subfolder` mandate + the recursed orphan scan.
4. `validate.sh` (Checks 15a recursed / 15b retargeted / 15c new / 16 / 17 / 19), `/CJ_document-release`, and `/CJ_doc_audit` consume the merged registry + the engine output.
5. `spec/test-spec-custom.md` units rows feed `validate.sh` Check 24's coverage cross-check; the tests exercise the new engine + validator surfaces.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Where the mandate lives | Portable seed + engine (Approach C) | Overlay-only (recommended A) | Every adopting repo inherits the mandate; the split becomes a uniform contract, not a workbench quirk. |
| How much to split | Full split — `workflow.md` → pure index | Orchestrators-only | The utilities + audits sections are also dense; a partial split leaves `workflow.md` still large and mixed. |
| Mandate strength | Mandate (required + non-empty) | Describe + permit | A permissive contract would let an adopting repo skip the subfolder; the user chose uniform enforcement. |
| No-vanish guarantee | New Check 15c | Rely on Check 15b alone | Retargeting 15b to the subfolder removes the guarantee that the overview still names every workflow; 15c restores it. |
