## Problem Statement

The workbench maintainer classifies every test through the test contract
(`spec/test-spec.md` + `spec/test-spec-custom.md`, driven by `test-spec.sh` +
`test-run.sh`), but today the `categories:` axis `{workflow, CI-push, CI-nightly}`
conflates a semantic *kind* (`workflow`) with a run *cadence* (`CI-push`/`CI-nightly`), and
the general `layers[]` map `{local-hook, ci, pipeline-gate, ratchet}` lumps all CI into one
`ci` blob and treats the `ratchet` monotonic-guard *property* as a *location*. The
maintainer cannot answer "what kind of test is this, and when does it fire?" from one place,
and the wrong cut already produced a half-built artifact — `tests/{workflow, CI-push,
CI-nightly}/` folders carrying only `.gitkeep`, zero tests migrated. This story gives every
test a precise, self-describing home by splitting the muddle into two orthogonal axes plus a
per-test mode attribute, and ships the workflow category populated.

## Mental Model

Three coordinates on a test (two closed-enum axes + one required attribute), orthogonal to
the pre-existing cost `tier` and the `ratchet: true` flag:

```
              layer (WHERE/WHEN it runs)
              CI-push   CI-nightly   pipeline-gate   local-hook
category   +----------+------------+---------------+-----------+
(KIND)     |          |            |               |           |
workflow   | portab-  | portab-    |      —        | e2e-local |
           | smoke    | deploy;    |               |           |
           |          | goal-feat- |               |           |
           |          | eval;      |               |           |
           |          | doc-sync   |               |           |
regression | (29 flat tests today; migration DEFERRED)         |
infra      | validate | windows-   |   (gates)     |           |
           | suite    | deploy     |               |           |
           | test-dep |            |               |           |
           +----------+------------+---------------+-----------+

mode ∈ {deterministic, agentic}   (required per categories: row; agentic ⇒ tier ≠ free)
tier ∈ {free, paid, local-only}   (pre-existing, orthogonal — NOT the layer)
ratchet: true                     (pre-existing flag — no longer a layer)
```

Physical home: `tests/<category>/<layer>/<name>.test.sh` (or a `command`-only row for script
invocations). Docs: `docs/tests/<category>/<layer>/<name>.md` (front-door sections), indexed
by `docs/tests/index.md`. The general `layers[]` and the category definition live in the
portable seed tier (`spec/test-spec.md`), inherited by consumer repos.

## Acceptance Criteria

<!-- Given/When/Then. Each #n header matches the # column in Requirements. -->

### Story #1: Two-axis + mode enums validate [core]

```
GIVEN spec/test-spec-custom.md declares categories/units/layers with the new vocabulary
WHEN  I run `scripts/test-spec.sh --validate`
THEN  it exits 0 with the category enum {workflow, regression, infra}, the layer enum
      {CI-push, CI-nightly, pipeline-gate, local-hook} (units subset {local-hook, CI-push,
      CI-nightly}), and it errors on any out-of-enum category/layer/mode value
```

### Story #2: General tier byte-identical to seed [core]

```
GIVEN spec/test-spec.md carries the new layers[] + category definition + mode attribute
WHEN  the 3-way-lockstep seed-identity test runs (parse comment, human table, --seed heredoc)
THEN  `test-spec.sh --seed` is byte-identical to spec/test-spec.md and the test passes
```

### Story #3: All 83 layer:ci rows re-mapped by trigger [core]

```
GIVEN the overlay's 83 units: rows previously recorded layer: ci
WHEN  --validate and Check 24 run
THEN  each row's layer is CI-push (trigger pr-ci/push-main/pre-commit) or CI-nightly
      (trigger nightly), and no units: row references a layer absent from layers[]
```

### Story #4: mode required + agentic⇒¬free cross-check [core]

```
GIVEN a categories: row omits mode, or declares mode: agentic with tier: free
WHEN  I run `scripts/test-spec.sh --validate`
THEN  it errors (mode is required; agentic requires tier ∈ {paid, local-only})
```

### Story #5: list + structure surface the contract honestly [usability]

```
GIVEN the adopted two-axis contract with 29 un-migrated flat tests
WHEN  I run `--list-categories` and `--check-structure`
THEN  --list-categories prints the TSV `name category layer mode command tier doc purpose`,
      --check-structure validates the 2-deep tests/<category>/<layer>/ structure, exempts
      command-only rows from check (b), reports the 29 flat tests as advisory backfill
      findings, and exits 0
```

### Story #6: selection by category/layer/name [usability]

```
GIVEN the adopted contract
WHEN  I run `test-run.sh --category workflow`, `--layer CI-nightly`, `--category workflow
      --layer CI-nightly`, or a bare test NAME
THEN  each selects the correct test set, a default (no --evals/--e2e/--all) run stays
      free-tier and spends no model tokens
```

### Story #7: workflow category ships populated with front-door docs [core]

```
GIVEN the four workflow tests are declared
WHEN  I inspect tests/workflow/<layer>/ and docs/tests/
THEN  portability-smoke (CI-push), portability-deploy (CI-nightly), goal-feature-eval
      (CI-nightly), and doc-sync (CI-nightly) each exist as a file row or command row with a
      front-door doc (## What it is / ## How to run / ## Explanation) listed in
      docs/tests/index.md
```

### Story #8: gates stay green after re-categorization [resilience]

```
GIVEN rows are re-categorized (moving their generated doc paths)
WHEN  validate.sh runs
THEN  Check 15/15a finds no orphan docs, Check 26 render is fresh, Check 27 workflow docs are
      fresh, and Check 28's reused workflow-cj-goal-feature-runs behavior resolves to
      CJ_goal_feature — all green
```

### Story #9: seed-docs seeds the 2-deep front-door path [core]

```
GIVEN a declared category test with no doc on disk
WHEN  I run `test-spec.sh --seed-docs`
THEN  it idempotently creates docs/tests/<category>/<layer>/<name>.md carrying the three
      front-door section headings and never moves any test script
```

### Story #10: portable script edits [resilience]

```
GIVEN the edited test-spec.sh / test-run.sh
WHEN  scripts/windows-smoke.sh and shellcheck run
THEN  they pass — edits are LF, any new jq call goes through the CR-strip jq() wrapper, and
      date arithmetic uses the portable probe idiom
```

## Architecture

<!-- High-level design. -->

```
              spec/test-spec.md  (GENERAL / seed tier)
              +-- layers[]  {CI-push, CI-nightly, pipeline-gate, local-hook}
              +-- categories definition  {workflow, regression, infra}
              +-- mode attribute  {deterministic, agentic}
                        |  (byte-identical, 3-way lockstep)
                        v
   scripts/test-spec.sh --seed  ==  spec/test-spec.md
        |  parses + merges
        v
   spec/test-spec-custom.md  (OVERLAY)
     +-- units:       83 rows, layer re-mapped ci -> CI-push | CI-nightly by trigger
     +-- categories:  7 rows re-categorized + 4 workflow tests, each + layer + mode
     +-- behaviors:   + workflow-doc-audit-runs (level: integration) + coverage link
                        |
        +---------------+----------------+
        v                                v
   test-spec.sh                     test-run.sh
   --validate (Check 24)            --category {workflow,regression,infra}
   --list-categories (TSV)          --layer {CI-push,CI-nightly,pipeline-gate,local-hook}
   --check-structure (2-deep,       --category+--layer composition
      command-only check-(b) exempt) single-NAME selection
   --seed-docs (2-deep path)        default = free tier (no model spend)
   --render-docs (Check 26)
        |
        v
   docs/tests/<category>/<layer>/<name>.md  (front-door: What/How/Explanation)
   docs/tests/index.md  (INDEX)
   tests/<category>/<layer>/<name>.test.sh  (physical; workflow populated, regression DEFERRED)
```

### Data Flow

1. The maintainer edits `spec/test-spec.md` (general) — `layers[]`, category definition, `mode` — keeping it byte-identical across the three lockstep locations.
2. `test-spec.sh --seed` re-emits that exact general file; the 3-way-lockstep seed-identity test compares them.
3. `spec/test-spec-custom.md` (overlay) declares `units:` (83 rows, `layer` re-mapped by trigger), `categories:` (7 re-categorized + 4 workflow, each with `layer`+`mode`), and the new behavior; `test-spec.sh` merges general + overlay.
4. `--validate` enum-checks category/layer/units-layer + the `agentic⇒¬free` cross-check; `validate.sh` Check 24 calls it (HARD).
5. `--render-docs` regenerates `docs/tests/**` + `docs/test-catalog.md`; `--seed-docs` seeds any missing 2-deep front-door stub; Check 26 diffs the render for freshness; Check 15/15a checks doc-spec orphans; Check 28 re-checks workflow coverage.
6. `test-run.sh --category`/`--layer`/NAME maps the selection to each row's `command` via the extended `--list-categories` and runs the chosen tier (default free).

## Tradeoffs

<!-- Decisions made, rejected alternatives, rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| How much lands in this PR | Model + engines + populated workflow category (Approach A) | Model + engines only, workflow category empty (Approach B) | B repeats the exact half-built-scaffold mistake that motivated the feature. |
| Flat-test migration timing | Defer the 29-flat-test migration to a tracked backfill | Also migrate all 29 flat tests now (Approach C) | C compounds the migration into the 83-row re-map = one unreviewable XL/High-risk diff; the repo's "land additively" philosophy makes it its own increment. |
| Layer flip timing | Full re-map now (all 83 rows) | Stage the risky 83-row re-map | Staging carries two layer vocabularies simultaneously; one bigger coherent diff avoids dual-vocabulary debt (operator's call, Premise 6). |
| `ratchet`'s place | A `ratchet: true` flag (already exists) | Keep `ratchet` as a layer | `ratchet` is a monotonic-guard property, never a location; demoting leaves the layer set clean at four. |
| Third category | Add an honest `infra` bucket | Overload `regression` with validator/suite/deploy checks | The operator chose to grow the model by one honest bucket rather than stretch a word. |
| Layer on category rows | Descriptive metadata (kept consistent by hand) | Make the row's `layer` drive CI scheduling | Real cron/trigger stays in `.github/workflows/*.yml`; binding them would duplicate scheduling truth (documented drift risk, review fix #9). |
