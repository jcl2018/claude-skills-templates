## Problem Statement

`/CJ_run` invokes `/land-and-deploy` without any flag to suppress the
readiness-gate AUQ. That AUQ fires on every invocation regardless of state.
Branch(f) `open_pr` mode does not auto-continue into `/land-and-deploy` —
it prints a pointer and exits 0, breaking the "let it run to the end" intent.

This story closes both gaps on the workbench side. The gstack-side flag
itself is owned by the user in a separate PR.

## Mental Model

```
   /CJ_run <design-doc>                    /CJ_run <work-item-dir>
            │                                       │
            v                                       v
   ┌──────────────────────┐              ┌────────────────────────┐
   │ Phase 1-3 (existing) │              │ Branch(f) phase detect │
   └──────────────────────┘              └────────────────────────┘
            │                                       │
            v                          (open_pr mode resolved)
   ┌──────────────────────┐                         │
   │ Step 5 — /ship       │                         v
   └──────────────────────┘              ┌────────────────────────┐
            │                            │ Inline PR_NUM parse    │
            v                            │ (duplicated from Step 5)│
   ┌──────────────────────────────────┐  └────────────────────────┘
   │ Step 5 — /land-and-deploy        │             │
   │   --suppress-readiness-gate  ◀───┼─────────────┘
   │   #<PR_NUM>                      │
   └──────────────────────────────────┘
```

Both paths converge on the same /land-and-deploy invocation with the flag.

## Acceptance Criteria

### Story #1: Step 5 passes the flag [core]

```
GIVEN /CJ_run is in Phase 4, /ship completed green, PR_URL is set
WHEN  Step 5 invokes /land-and-deploy via the Skill tool
THEN  the invocation prose passes `--suppress-readiness-gate`
AND   passes the literal PR_NUM (e.g. `#42`) when PR_NUM is non-empty
```

### Story #2: Branch(f) open_pr auto-dispatches [core]

```
GIVEN a work-item with PR URL in TRACKER frontmatter AND `gh pr view` returns state=OPEN
WHEN  Branch(f) resolves MODE=open_pr
THEN  the handler row instructs the orchestrator to invoke /land-and-deploy --suppress-readiness-gate #<PR_NUM>
AND   NOT to print-and-exit 0
```

### Story #3: PR_NUM parsing in Branch(f) [core]

```
GIVEN PR_URL like "https://github.com/jcl2018/claude-skills-templates/pull/42"
WHEN  Branch(f) open_pr handler parses PR_NUM
THEN  the inline-duplicated block from Step 5 (try ${PR_URL##*/}; fall back to `gh pr list --head ...`) sets PR_NUM=42
AND   the invocation prose threads the literal #42 into /land-and-deploy
```

### Story #4: Version bumped [core]

```
GIVEN skills/CJ_run/SKILL.md frontmatter
WHEN  the implementation is complete
THEN  the `version:` field reads `0.5.0` (was `0.4.0`)
```

### Story #5: CHANGELOG entry [core]

```
GIVEN CHANGELOG.md
WHEN  the implementation is complete
THEN  a new entry exists for the upcoming collection version (e.g. [3.4.0])
AND   the entry mentions: (a) Step 5 pass-through of --suppress-readiness-gate, (b) Branch(f) open_pr auto-dispatch, (c) forward-compat note (safe no-op until gstack lands)
```

### Story #6: SKILL.md description + Phase 4 entry [core]

```
GIVEN skills/CJ_run/SKILL.md
WHEN  the implementation is complete
THEN  the `description` frontmatter mentions the suppression behavior
AND   the Phase 4 line in the "## Pipeline shape" section reads similar to "Phase 4 — /land-and-deploy (Skill, inline, --suppress-readiness-gate) — merge PR, verify deploy. AUQ-free on green; alerts on red."
```

### Story #7: Forward-compat safe no-op [resilience]

```
GIVEN gstack /land-and-deploy does NOT recognize --suppress-readiness-gate yet
WHEN  /CJ_run invokes it with the flag

AND   /land-and-deploy proceeds with today's behavior (legacy AUQ fires)
AND   no error surfaces to /CJ_run
```

### Story #8: Flag is greppable in run.md [observability]

```
GIVEN skills/CJ_run/run.md

```

```
AND   no [DRIFT]/[MISSING] findings related to CJ_run version, catalog, or templates
```

### Data Flow

1. User invokes `/CJ_run <design-doc>` (or `<work-item-dir>` resolving to `open_pr` mode).
2. Pipeline reaches Phase 4 (Step 5) or Branch(f) `open_pr` handler.
3. PR_NUM resolves via inline parsing (try `${PR_URL##*/}`; fall back to `gh pr list --head "$CURRENT_BRANCH"`).
4. /CJ_run invokes /land-and-deploy via the Skill tool with `--suppress-readiness-gate #<PR_NUM>`.
5. /land-and-deploy (post-gstack-PR) skips its Step 3.5e readiness AUQ on green; halts on red as before.
6. /land-and-deploy (pre-gstack-PR) ignores the unknown flag; legacy AUQ fires; behavior identical to today.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| PR_NUM parsing in Branch(f) | inline duplicate of Step 5 block | extract a /CJ_run-internal helper function | ~15 lines duplicated is cheaper than introducing the abstraction; one duplication is fine, three+ would justify the helper. (User taste decision.) |
| Flag-passing conditional | unconditional under /CJ_run | env-var or `--no-suppress-gate` opt-out | /CJ_run is the only caller; opt-in semantics are owned at the wrapper level; an opt-out would just be ceremony. |
| Branch(f) `open_pr` behavior change | print + auto-dispatch | print + exit 0 (status quo) | "Let it run to the end" intent. Print preserves the audit trail; auto-dispatch closes the resume-from-PR-open gap. |
| Version bump | 0.4.0 → 0.5.0 (minor) | 0.4.1 (patch) | Behavior change is user-visible (Branch(f) no longer exit-0); minor bump is honest. |
