## Problem Statement

A workbench user with a small idea or no design preference wants to hand a one-line description to Claude and get a deployed change back. Today the friction is producing an APPROVED design doc through `/office-hours` — disproportionate for small features. The user wants exactly one human touchpoint per run (the autoplan GATE #1 final-approval AUQ) AND auto-merged GATE #2 for changes that are *provably* small, denylist-clean, and Phase-2-green; otherwise the path degrades to the existing human-gated flow. Every gate is fail-closed.

## Mental Model

`/CJ_goal_auto` is a thin orchestrator over the existing `/CJ_goal_run` pipeline. It adds:

```
Stage 0 (worktree + capability) → Stage 0.5 (classifier) → Stage 1 (workbench-owned generator)
  → Stage 1.5 (fail-closed doc gate) → Stage 2 (/CJ_goal_run --handoff --no-drain)
  → [autoplan GATE #1 = always human] → [Phase 3 /ship = PR-prep]
  → scripts/cj-handoff-gate.sh (orchestrator-owned GATE #2)
  → [Phase 4 /land-and-deploy = merge] → Stage 3 (Deploy + audit-log)
```

`--handoff` (alias `--auto-merge-small-diffs`) is a per-invocation opt-in; the default `/CJ_goal_auto "<idea>"` shape is human-gated at GATE #2.

## Acceptance Criteria

### Story #1: Three explicit CLI shapes + deprecated alias [usability]

```
GIVEN  the operator runs `/CJ_goal_auto`
WHEN   they pass no flag with an idea, `--auto-merge-small-diffs`, or `--dry-run`
THEN   the resolved mode is unambiguous and `--handoff` is accepted as a deprecated alias
```

### Story #2: Resolved-mode echo [usability]

```
GIVEN  any shape of invocation
WHEN   Stage 0 begins
THEN   stderr prints `mode=...` with the resolved flags before any work happens
```

### Story #3: Stage 0 worktree + preflight + capability check [core]

```
GIVEN  the operator runs `/CJ_goal_auto` from `main` with no `--no-worktree`
WHEN   Stage 0 runs
THEN   a per-run-unique worktree `cj-auto-{YYYYMMDD-HHMMSS}-{PID}` is created (Conductor sessions no-op),
       `check-version-queue.sh` runs and halts on collision with the next-free slot,
       AND the `--handoff` sentinel is grep'd on the resolved `CJ_goal_run/run.md`; absent → fail closed
```

### Story #4–#5: Stage 0.5 classifier + jsonl log [core, observability]

```
GIVEN  Stage 0 passes
WHEN   Stage 0.5 evaluates the one-liner
THEN   the verdict is one of {small-unambiguous, needs-human-taste, too-big};
       only `small-unambiguous` proceeds; the orchestrator (not the subagent)
       appends `{ts, one_liner, verdict, reason}` to `~/.gstack/analytics/cj-goal-auto-classifier.jsonl`;
       non-`small-unambiguous` halts with a structured stop block + manual route
```

### Story #6–#7: Stage 1 generator + Stage 1.5 fail-closed gate [core, resilience]

```
GIVEN  Stage 0.5 returns `small-unambiguous`
WHEN   Stage 1 runs the workbench-owned generator
THEN   a design doc is written to `~/.gstack/projects/<slug>/<user>-<branch>-design-<datetime>.md`
       containing `Problem Statement`, `Premises`, `Recommended Approach`, `Success Criteria`,
       `Distribution Plan`, and `Status: APPROVED` (all non-empty);
       AND Stage 1.5 asserts: file exists at computed path AND `Status: APPROVED` present
       AND every required section non-empty; ANY miss → abort + Stage 2 is never invoked
```

### Story #8: Stage 2 invocation [core]

```
GIVEN  Stage 1.5 passes
WHEN   Stage 2 runs
THEN   `/CJ_goal_run <doc> --handoff --no-drain` is invoked unchanged
```

### Story #9–#12: GATE #2 helper conditions [security, resilience]

```
GIVEN  the helper is invoked after Phase 3 (`/ship` created the PR) and before Phase 4
WHEN   the helper runs
THEN   it: `git fetch origin main`; pins `BASE=$(git merge-base origin/main HEAD)`;
       runs `git diff --no-renames --raw -z $BASE HEAD` for denylist matching
       (fails closed on R/C/T on EITHER old or new path AND on any new symlink mode 120000);
       counts added lines via `git diff --numstat $BASE HEAD | awk '{a+=$1} END{print a+0}'` ≤ 120;
       counts files via `git diff --name-only $BASE HEAD | wc -l` ≤ 5;
       verifies Phase-2 markers `PIPELINE_END_STATE=green` AND `SMOKE=pass` AND `E2E=pass`
       AND all `PHASE2_GATES` checked;
       exit 0 = proceed to merge, non-zero = halt for human (fail-closed unknown state)
```

### Story #13–#14: Audit receipt + `--audit` mode [observability, usability]

```
GIVEN  any run (including dry-run and halted runs)
WHEN   the run completes
THEN   the orchestrator appends a JSON line to `~/.gstack/analytics/CJ_goal_auto.jsonl`
       containing classifier verdict, doc path, work-item dir, PR URL, pinned BASE SHA,
       changed files, added lines, denylist result, Phase-2 markers, gate result, resume_cmd;
       AND `--audit`/`--list-handoffs` prints the last 10 entries (capped) in human-readable form
```

### Story #15: Structured halt contract [usability]

```
GIVEN  any halt path
WHEN   the orchestrator surfaces the halt
THEN   it prints a plain-English stop block + `next_action=` + `resume_cmd=` (copy-paste)
       + `pr_url=`/`work_item_dir=` when applicable;
       AND a GATE #2 demotion specifically names which condition tripped (denylisted path / count vs cap / Phase-2 marker)
       and prints "PR #N is created and review-ready; `gh pr diff N`, merge manually if good"
```

### Story #16: `--dry-run` [usability]

```
GIVEN  `/CJ_goal_auto --dry-run "<idea>"`
WHEN   Stage 0+0.5 run
THEN   classifier verdict + reason, would-create paths, sentinel presence,
       and GATE #2 caps are printed; ZERO writes anywhere; Stage 1 is NEVER invoked
```

### Story #17: Informed GATE #1 [usability]

```
GIVEN  Stage 2 reaches the autoplan final-approval AUQ
WHEN   the AUQ is about to fire
THEN   the generated doc's `Problem Statement` and `Recommended Approach` are printed first
       so the operator can read what they are approving
```

### Story #18: Every-run retro AUQ [observability]

```
GIVEN  the operator has used `--auto-merge-small-diffs`
WHEN   the count of auto-merged runs is in {1,2,3,4,5}
THEN   one retro AUQ fires after the run listing the diff(s) ("confirm none needed review");
       after run 5, cadence relaxes to every-5th
```

### Story #19: `/CJ_goal_run` wiring [core]

```
GIVEN  the bootstrap PR
WHEN   it modifies `skills/CJ_goal_run/run.md`
THEN   `--handoff` (alias `--auto-merge-small-diffs`) + `--no-drain` + co-located support sentinel
       are wired at the post-`/ship` / pre-`/land-and-deploy` point;
       NOT in `SKILL.md` and NOT in `/ship`; test 9 asserts sentinel within N lines of gate call
```

### Story #20: Deploy + PR body audit line [core]

```
GIVEN  the gate auto-approved
WHEN   Phase 4 runs
THEN   `/land-and-deploy --suppress-readiness-gate` performs the merge;
       AND the PR body contains `auto-merged under handoff: N files / M lines / QA <markers>` + pinned BASE SHA
```

### Data Flow

Primary use case — `/CJ_goal_auto --auto-merge-small-diffs "<small idea>"`:

1. Operator runs the command from `main` (or inside a Conductor session).
2. Stage 0: cj-worktree-init.sh creates `cj-auto-{ts}-{pid}` (or no-ops in Conductor); check-version-queue.sh confirms VERSION slot is free; sentinel grep on resolved `CJ_goal_run/run.md` confirms `--handoff` is supported (fail-closed otherwise).
3. Stage 0: orchestrator echoes resolved mode to stderr.
4. Stage 0.5: classifier subagent reads ONLY the one-liner, returns `{verdict, reason}`. Orchestrator appends to `~/.gstack/analytics/cj-goal-auto-classifier.jsonl`.
5. If verdict ≠ `small-unambiguous`: structured halt + manual route + audit receipt write. STOP.
6. Stage 1: orchestrator computes `~/.gstack/projects/<slug>/<user>-<branch>-design-<datetime>.md` and writes the design doc from the fixed template (sections: `Problem Statement`, `Premises`, `Recommended Approach`, `Success Criteria`, `Distribution Plan`, `Status: APPROVED`).
7. Stage 1.5: orchestrator asserts file exists + `Status: APPROVED` + all required sections non-empty. ANY miss → abort + receipt + manual route. STOP. Stage 2 NEVER invoked.
8. Stage 2: orchestrator invokes `/CJ_goal_run <doc> --handoff --no-drain`.
9. Inside `/CJ_goal_run`: autoplan runs → prints generated doc's Problem Statement + Recommended Approach → fires final-approval AUQ (always human; v1 mandatory touchpoint).
10. Operator approves at GATE #1 → scaffold/impl/qa run autonomously (reversible — a branch).
11. Phase 3: `/ship` creates the PR (F000021-allowed PR-prep).
12. `/CJ_goal_run` invokes `scripts/cj-handoff-gate.sh` with the pinned BASE SHA.
13. Helper: `git fetch origin main` → `BASE=merge-base origin/main HEAD` → denylist matching → file/line counts → Phase-2 marker check. Exit 0 OR non-zero.
14. Exit 0: `/CJ_goal_run` proceeds to Phase 4 `/land-and-deploy --suppress-readiness-gate`. PR body line appended with audit + pinned BASE SHA.
15. Exit non-zero: halt for human; structured stop block + `next_action=` + `resume_cmd=` + `pr_url=` + named tripped condition.
16. Stage 3: orchestrator appends per-run audit receipt to `~/.gstack/analytics/CJ_goal_auto.jsonl`. If this is the operator's run 1–5 OR every-5th: retro AUQ fires.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Stage 1 source | Workbench-owned fixed-template generator | Headless office-hours (`SPAWNED_SESSION`) | Not a real contract — env-var-gated only, no extension point for the classifier taxonomy or doc-write refusal. Recorded so it is not re-litigated. |
| GATE #2 mechanism | Orchestrator-owned post-`/ship`/pre-`/land-and-deploy` gate via `scripts/cj-handoff-gate.sh` (in workbench-owned `/CJ_goal_run`) | Inject into upstream `/ship` | `/ship` is upstream, no-arg, owns its own non-interactive flow. No gate to "auto-answer" inside it. (Eng D9 → A.) |
| Gate logic | Deterministic exit-coded shell helper unit-tested in `scripts/test.sh` | LLM-judged gate via `eval.sh` harness | LLM judgment in the merge gate is the failure mode the design avoids. `eval.sh` is scoped to structured-report skills and can't drive Write/Edit/Skill/Agent. (Eng F5.) |
| Diff base | Frozen `git merge-base origin/main HEAD` (pinned, recorded in PR body) | `git diff origin/main` (recomputed each step) | Stable under main drift; counts are reproducible and auditable. (Eng F2.) |
| Denylist diff plumbing | `git diff --no-renames --raw -z $BASE HEAD` (rename/symlink-safe) | `git diff --name-only $BASE HEAD` | `--no-renames` makes a rename of a denylisted file surface as add+delete so the delete trips the denylist. Raw mode includes mode bits so symlinks (120000) are detectable. (Eng F4.) |
| QA predicate | Real Phase-2 markers (`PIPELINE_END_STATE=green` AND `SMOKE=pass` AND `E2E=pass` AND `PHASE2_GATES`) | Fictional "high/medium severity" scale | Phase 2 doesn't emit a severity scale; predicate must match what `/CJ_personal-pipeline` actually returns. (Codex finding.) |
| Test denylist | Include `tests/**`, `scripts/*test*.{sh,py}`, `*fixture*`, `*.golden` | Inherit validate.sh sensitive-surface list only | A ≤120-line diff weakening a test assertion while touching no other denylisted path would otherwise pass every GATE #2 condition. (CEO finding.) |
| Flag name | `--auto-merge-small-diffs` (literal) + `--handoff` deprecated alias | `--handoff` (original) | Both DX voices independently flagged "accidental consent" as biggest risk. Literal says what it does. (DX CRIT.) |
| Public shapes | Three explicit shapes (human-gated, auto-merge, dry-run) + resolved-mode echo | One shape with flag combinations | Safe on-ramp visible by default; operator can preview before committing. (DX.) |
| Halt contract | Structured stop block + `next_action=` + `resume_cmd=` + named tripped condition | Free-text halt message | Ports `CJ_goal_investigate`; operator can copy-paste resume. (DX HIGH.) |
| Audit receipt | Per-run jsonl at `~/.gstack/analytics/CJ_goal_auto.jsonl` + `--audit`/`--list-handoffs` | Thin 3-way cross-reference of git log + PR list + tracker journals | Receipt is the single source of truth; no post-hoc reconstruction. (DX HIGH.) |
| Retro cadence | Every-run for first 5 auto-merges, then every-5th | Always every-5th OR always every-run | Tightest trust margin while classifier has no data; relaxes as baseline accumulates. (DX.) |
| Concurrent-run handling | Advisory `check-version-queue.sh` preflight; documented as accepted v1 limitation | Atomic slot reservation | Single-developer personal tooling; concurrent unattended runs are not a v1 use case. v2 prerequisite if Approach C (scheduled drain) lands. (Eng F3.) |
| Single-story decomposition | One user-story (atomic, this is the entire v1 surface) | Multi-story split (Stage 0/0.5 vs Stage 2/3 vs gate) | Design doc constrains v1 to single-PR changes; cross-story coordination adds ceremony without leverage. (Design constraint.) |
