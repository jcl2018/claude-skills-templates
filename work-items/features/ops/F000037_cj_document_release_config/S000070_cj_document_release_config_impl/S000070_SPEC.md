## Requirements

### Functional

- **R1**: New file `cj-document-release.json` at repo ROOT (sibling of `skills-catalog.json`) declares the doc-sync whitelist + `--docs <token>` categories for `/CJ_document-release`.
- **R2**: Schema v1 (`schema_version: 1`) with three required top-level fields: `schema_version` (integer), `whitelist_patterns` (non-empty array of glob strings), `categories` (object mapping token names to non-empty glob arrays).
- **R3**: Globs use bash-globstar semantics (`**` for any-depth recursion). Helper script uses `shopt -s globstar nullglob`.
- **R4**: New helper script `scripts/cj-document-release-config.sh` (bash, executable) provides four subcommands:
  - `--parse` — echo the parsed JSON (pretty-printed via `jq '.'`).
  - `--expand-whitelist` — echo the expanded whitelist file list (globs resolved against working tree; sorted, unique).
  - `--resolve <token>` — echo file list for one category token; exit non-zero + halt-emit if token not declared.
  - `--validate` — exit 0 + print `OK schema_version=<n>` if JSON is valid + schema_version supported; exit 1 + halt-emit otherwise.
- **R5**: Strict-required posture — helper emits `[doc-sync-no-config] <reason>` and exits 1 when:
  - File missing at `$REPO_ROOT/cj-document-release.json`
  - File present but not valid JSON
  - `schema_version` field missing or value not in supported list (v1 supports `[1]`)
  - `whitelist_patterns` missing or not an array
  - `categories` missing or not an object

### Data flow

```
operator → /CJ_document-release [--docs <token>]
             │
             ├─ Step 0.5: Read config
             │    bash scripts/cj-document-release-config.sh --parse
             │      │
             │      ├─ jq empty cj-document-release.json
             │      ├─ jq -r '.schema_version' → assert in (1)
             │      ├─ jq -e '.whitelist_patterns | type == "array"'
             │      ├─ jq -e '.categories | type == "object"'
             │      └─ exit 0, echo parsed JSON
             │           OR exit 1, echo "[doc-sync-no-config] <reason>"
             │      ↓
             │    if exit non-zero → HALT [doc-sync-no-config]
             │
             ├─ Step 1: Parse arguments (--docs TOKEN raw)
             │
             ├─ Step 4: Build context block
             │    if --docs TOKEN:
             │      bash scripts/cj-document-release-config.sh --resolve TOKEN
             │        ├─ jq -r '.categories[TOKEN]' (assert non-empty)
             │        ├─ shopt -s globstar nullglob; expand globs
             │        └─ echo sorted file list
             │            OR exit 1 + halt-emit if token not declared
             │      ↓
             │    audit_files = <list>
             │
             ├─ Skill(/document-release with project-context block)
             │
             ├─ Step 6: Auto-commit doc-only
             │    whitelist_files = bash scripts/cj-document-release-config.sh --expand-whitelist
             │    changed_files = git status --porcelain --untracked-files=no
             │    diff = changed_files \ whitelist_files
             │    if diff non-empty → HALT [doc-sync-non-doc-write]
             │    else git commit -m "docs: post-build sync"
             │
             └─ success summary
```

```
/ship pre-commit hook → ./scripts/validate.sh
             │
             └─ Check 16: cj-document-release.json schema
                  if file exists:
                    ├─ jq empty (must succeed)
                    ├─ schema_version == 1 (must hold)
                    ├─ whitelist_patterns non-empty array (must hold)
                    ├─ categories non-empty object, each value an array (must hold)
                    ├─ bash scripts/cj-document-release-config.sh --validate (must exit 0)
                    └─ PASS or ERROR (increment ERRORS counter)
                  else: skip (non-adopting repos)
```

### Key interfaces

- **Helper CLI**: 4 subcommands (`--parse`, `--expand-whitelist`, `--resolve <token>`, `--validate`); exit 0 = green, exit 1 = red with `[doc-sync-no-config] <reason>` on stdout/stderr.
- **JSON schema (v1)**:
  ```json
  {
    "schema_version": 1,
    "whitelist_patterns": ["README.md", "doc/**/*.md", ...],
    "categories": {
      "readme": ["README.md"],
      "philosophy": ["doc/PHILOSOPHY.md"],
      ...
    }
  }
  ```
- **Halt marker**: `[doc-sync-no-config]` on stdout/stderr. Orchestrator HALT class: `halted_at_doc_sync_no_config`. Next-action / resume_cmd / pr_url fields per F000027 marker shape.

### Concurrency / failure modes

- **Helper script is stateless**: no temp files, no locks. Re-running it is safe.
- **JSON file missing** → helper exit 1, halt class `[doc-sync-no-config]`. Operator authors a JSON or uses workbench's bundled seed.
- **JSON file present but invalid** → same halt class; reason captures the specific validation failure.
- **`schema_version` unsupported** → halt with `schema_version=<n> unsupported (this helper supports <list>)`. Operator can downgrade JSON or upgrade workbench.
- **Category token not declared** → halt with `category '<token>' not declared in cj-document-release.json`. Operator adds to JSON or uses a different token.
- **Glob expansion empty** (`nullglob` makes globs that match nothing return empty) → helper returns empty file list; SKILL.md's caller treats empty list as green-noop (no files to audit; no halt).

## Tradeoffs Considered

### Approach Selected: B (Helper script + thin SKILL.md, mirroring F000029)

**Why**: Pattern-matches F000029's proven `scripts/skills-doc-sync-check + thin SKILL.md` shape. Parse logic isolated in one bash file → testable in isolation (`bash scripts/cj-document-release-config.sh --validate`). SKILL.md stays focused on orchestration.

### Approach A: Pure SKILL.md prose (inline bash + jq + shopt globstar)

**Rejected because**: SKILL.md prose balloons (~80 lines of new bash); bash globstar matching edge cases scattered through skill prose; hard to test parse logic in isolation (would test the whole skill invocation).

### Approach C: Node/Bun .mjs with minimatch

**Rejected because**: Introduces a Node/Bun runtime dependency to a skill that didn't need one; first `.mjs` in `scripts/` should be a separate decision, not folded into this PR; doesn't match other CJ_* skill helpers (all bash).

### Why strict-required, not backward-compat fallback (D2 option C)

**Decision**: Strict-required. No fallback to F000036's hardcoded defaults. `[doc-sync-no-config]` HALT fires when JSON missing/invalid/schema-unsupported.

**Why over fallback**: Matches memory `feedback_skill_contracts_strict`: convention-strict beats convenience-fallback. Every repo that adopts CJ_document-release must declare intent upfront; nothing ships under "I forgot to configure it" semantics. Cost (one extra file per repo to author) is bounded; benefit (no surprise behavior across repos) compounds.

**Bundling the workbench's own JSON in the same PR** = zero day-1 breakage. Future downstream repos pay the one-time cost when they adopt the skill.

### Why globs, not regex

**Decision**: Globs (`doc/**/*.md`).

**Why**: Operator-readable (matches .gitignore mental model); bash globstar-compatible (`shopt -s globstar nullglob`); JSON-ecosystem standard. Regex (`^doc/.+\.md$`) is what F000036 used inline; F000037 normalizes to globs for declarability.
