## Problem Statement

The operator (a workbench maintainer running the CJ_ skill family, on macOS and
Windows Git Bash) has a test CONTRACT that is statically audited but never
executed by any verb: `/CJ_test_audit` proves wiring, not passing. On their
Windows machine `validate.sh` was red (jq-CRLF defect) while every
audit-of-record was green — the exact false-confidence gap. They need one
standalone any-repo verb that runs the contract's declared runners and reports
evidence-derived pass/fail, without ever surprising them with model spend.

## Mental Model

Three layers, one direction of data flow: the CONTRACT declares (a `runners:`
overlay axis says HOW to run, alongside the existing `units:`/`gates:`/
`behaviors:` axes) → the ENGINE executes (`scripts/test-run.sh`: plan → tiered
execution → report + ledger, everything derived from captured evidence) → the
WRAPPER narrates (`/CJ_test_run`: deterministic Stage-1 audit pre-step, then the
engine, then the report + ledger paths). No layer infers: a registry with no
runners is an honest SKIP, never a guessed command.

## Acceptance Criteria

<!-- Given/When/Then. Each #n header matches the # column above. -->

### Story #1: runners: overlay axis grammar [core]

```
GIVEN a spec/test-spec-custom.md overlay with well-formed runners: rows
WHEN  test-spec.sh --validate runs
THEN  it passes; and a duplicate id, an unknown tier/platform value, an empty
      command, an unknown covers family, or an explicit ci/hook in covers each
      produce a named validation error; and a registry with NO runners: axis
      validates exactly as before (registry-gated, zero behavior change)
```

### Story #2: machine-readable listings [integration]

```
GIVEN a valid registry with runners: rows and units: rows
WHEN  test-spec.sh --list-runners and --list-units --with-family run
THEN  --list-runners emits the parsed runner rows machine-readably and
      --list-units --with-family emits id + family columns consumable by
      test-run.sh for per-family covered-unit counts
```

### Story #3: the plan (--dry-run) [usability]

```
GIVEN the workbench registry (or any valid registry with runners)
WHEN  test-run.sh --dry-run runs
THEN  it prints per-runner command/tier/platform-guard/covered-families/unit-
      count/will-run-or-skip(reason), lists ci as ci-only, hook as the
      installed check, any other uncovered family as
      skipped(no-covering-runner), and exits 0 having executed nothing
```

### Story #4: cost-tier gating [security]

```
GIVEN runners across free, paid, and local-only tiers
WHEN  test-run.sh runs with no flags / --evals / --e2e / --all
THEN  only free runs by default; paid joins under --evals; local-only joins
      under --e2e; --all selects everything; unselected runners are
      skipped(tier-not-selected); no default run ever invokes a model
```

### Story #5: report + ledger artifact [observability]

```
GIVEN a run against a registry with at least one runners: row
WHEN  execution completes
THEN  tests/test-run/reports/<UTC-ts>.md and its .json sibling exist with
      schema: 1, timestamp, HEAD SHA, repo root, flags, aggregate, and per-
      runner id/command/tier/rc/outcome/covered-families/unit-count/duration;
      skip reasons come only from the closed enums; failures carry verbatim
      FAIL lines; the reports dir is gitignored with EXAMPLE.md committed
```

### Story #6: evidence-derived aggregate [core]

```
GIVEN executed, failed, and skipped runners in one run
WHEN  the aggregate is computed
THEN  any executed failure => fail + exit 1; >=1 green and no failure => pass
      + exit 0; zero executed => all-skipped + exit 0 and NEVER rendered pass;
      and a runner with rc=0 whose FIRST output line matches ^SKIP: is
      skipped(self-gated) while a mid-output SKIP is not
```

### Story #7: registry edge paths [resilience]

```
GIVEN an absent registry / an invalid registry / a valid registry with zero
      runners: rows
WHEN  test-run.sh runs
THEN  absent => REGISTRY=absent + exit 0; invalid => [test-spec-no-config]
      passthrough + exit 1; zero-runners => "SKIP: no runners declared" +
      exit 0 with no report and no ledger written
```

### Story #8: workbench self-hosting rows [integration]

```
GIVEN the workbench overlay after this story
WHEN  test-run.sh --dry-run reads it
THEN  run-test-sh (free) covers validate+test+test-deploy+windows-smoke as ONE
      row, run-eval is paid, run-e2e-local is local-only, and ci/hook appear
      as family-level rows (ci-only; hook-check: pass|fail) outside the
      skipped(<reason>) enum
```

### Story #9: /CJ_test_run wrapper + paperwork [usability]

```
GIVEN the skills installed (workbench or consumer repo, incl. no repo-local
      scripts/)
WHEN  /CJ_test_run is invoked
THEN  the Stage-1 pre-step prints the four test-spec.sh engine calls verbatim
      (absent => named SKIP; invalid => halt with the passthrough marker;
      findings on a valid registry ride the report, not a block), the engine
      runs with forwarded flags, the report + ledger paths are narrated, and
      the engines resolved via sibling-in-scriptdir -> $REPO_ROOT/scripts/ ->
      _cj-shared; catalog/routing/roster/philosophy/regenerated docs all green
```

### Story #10: fixture tests + registration [resilience]

```
GIVEN tests/test-run.test.sh with temp-dir fixture registries
WHEN  scripts/test.sh runs
THEN  the fixtures (failing command, paid-tier row, platform-guarded row,
      absent/invalid/no-runners paths) all assert green, the real
      scripts/test.sh is never invoked by the engine under test, and the new
      units rows + regenerated catalog keep Checks 24/26 green
```

### Story #11: CR-safe jq [resilience]

```
GIVEN a jq build that emits CRLF (Windows Git Bash)
WHEN  any new test-spec.sh/test-run.sh jq consumption runs
THEN  CR is stripped from consumed output and all JSON string encoding goes
      through jq -R/-Rs, so no false halt or corrupt ledger occurs
```

## Architecture

```

  units: / gates: / behaviors: / behavior_coverage:
  runners:            <── NEW axis (id, command, tier, covers, platform, note)
        │ parsed by
        ▼

        │                 --list-runners            <── NEW
        │                 --list-units --with-family <── NEW
        │ consumed by
        ▼
scripts/test-run.sh (NEW deterministic engine)
  ├─ resolve registry (spec/-then-root; REPO_ROOT/TEST_SPEC_PATH overrides)
  ├─ PLAN (--dry-run: print + exit)
  ├─ EXECUTE (tier select: free | +paid(--evals) | +local-only(--e2e) | --all;
  │           platform guard; self-gate first-line ^SKIP: rule; rc + tail)
  └─ REPORT tests/test-run/reports/<UTC-ts>.md + .json ledger (schema: 1;
             aggregate {pass, fail, all-skipped}; jq -R/-Rs + CR strip)
        ▲
        │ invoked by (flags forwarded)
skills/CJ_test_run/SKILL.md (thin wrapper)
  └─ Stage-1 pre-step: test-spec.sh --validate / --check-coverage /
     --render-docs --check / --check-workflow-coverage (verbatim;
     absent=SKIP, invalid=HALT, valid-with-findings=report-and-proceed)
```

### Data Flow

<!-- Primary use case: operator runs /CJ_test_run (or test-run.sh) default. -->

1. Wrapper (or operator directly) resolves `test-spec.sh` + `test-run.sh` via sibling-in-scriptdir → `$REPO_ROOT/scripts/` → `_cj-shared`.
2. Stage-1 pre-step: the four `test-spec.sh` engine calls print verbatim; absent registry → named SKIP; invalid → `[test-spec-no-config]` HALT; valid-with-findings → recorded, run proceeds.
3. `test-run.sh` parses the merged registry (`test-spec.sh --list-runners`, `--list-units --with-family`), CR-stripping all jq reads.
4. Plan is computed: per-runner platform guard + tier selection + covered families/unit counts; `--dry-run` prints it and exits 0.
5. Execution: each selected runner runs once; rc + output tail captured; first-line `^SKIP:` with rc=0 → `skipped(self-gated)`.
6. Report + ledger written under `tests/test-run/reports/` (jq `-R`/`-Rs` encoding); aggregate derived from evidence; exit code mirrors the aggregate (`fail` → 1, else 0).
7. Wrapper narrates the report + ledger paths and the aggregate verdict.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Discovery mechanism | `runners:` overlay axis (contract declares HOW to run) | Hardcoded family→command mapping table (Approach C; also the cross-model recommendation) | Operator-driven D4: the contract is the source of truth, not the tool; general seed untouched; any-repo portability by data, not code |
| Story scope | Core only (Approach A) | Core + audit ledger-handshake + `--changed` (Approach B) | B = XL/High: three sensitive surfaces in one diff — one oversized hop to the same destination |
| Verdict granularity | Runner-granularity | Per-unit pass/fail attribution from monolith stdout | False precision from a 3,245-line monolith's output (cross-model input, adopted) |
| Workbench free-tier coverage | ONE `run-test-sh` row covering validate+test+test-deploy+windows-smoke | Separate rows per family | Verified: test.sh drives all four; separate rows double-execute on Windows and mis-report `skipped(platform)` on POSIX |
| No-runners handling | Honest `SKIP: no runners declared`, no report/ledger | Inferring commands, or emitting an `all-skipped` ledger | Never inference, never fake green; the aggregate enum applies only when ≥1 runner row exists |
| JSON encoding | jq `-R`/`-Rs` + CR strip | Hand-escaped sh strings | Output tails carry quotes/backslashes; CRLF jq builds corrupt naive reads (the live defect class) |
| Engine testing | Fixture repos via env overrides | Invoking the real `scripts/test.sh` | Recursion/runtime trap (cross-model input, adopted) |
| Hook-family check depth (v1) | Minimal: installed pre-commit hook present + grep for the validator reference | Full sentinel parsing | Deferred — decided in-session, recorded in the design doc's resolved items |
