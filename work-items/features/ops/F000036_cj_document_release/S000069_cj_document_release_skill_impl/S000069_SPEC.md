## Problem Statement

F000028 (post-merge hook + doc-sync-pending marker) + F000029 (`scripts/skills-doc-sync-check` + 3 cj_goal preamble AUQ) shipped a marker-based doc-sync loop. For cj_goal orchestrator-driven paths, that loop has two gaps: (1) the AUQ fires on next session, decoupled from the shipping cycle that caused the drift; (2) the marker-AUQ runs `/document-release` with no args — all-or-nothing, no per-doc parameterization. This story builds a new workbench skill `CJ_document-release` that wraps upstream `/document-release` with a `--docs <subset>` flag, halt-on-red contract, and auto-commit-doc-only behavior. All 3 cj_goal orchestrators auto-invoke it inline between QA pass and `/ship` so doc updates fold into the same code PR — drift closes to ~minutes, atomic with the code PR. F000029 BD#1 (rejected the new-skill-in-catalog shape) is explicitly superseded; F000029's marker-AUQ stays as fallback for non-orchestrator paths.

## Mental Model

Convention-stack reuse (F000028 → F000029 → F000032 → F000033 → F000034 → F000036):

```
F000028 post-merge hook drops marker at ~/.gstack/doc-sync-pending/<slug>.json
F000029 scripts/skills-doc-sync-check + 3 cj_goal preamble AUQ → fallback path
F000032 Check 13: USAGE.md presence + 5 H2 sections          → CJ_document-release subject
F000033 Check 14: USAGE.md freshness (%ct vs SKILL.md)       → CJ_document-release subject
F000034 doc/SKILL-CATALOG.md + Check 15: per-skill section   → CJ_document-release gets section
F000036 CJ_document-release skill + inline Step 5.5 wiring   ← THIS STORY
```

Wrapper-skill shape (workbench-only):

```
                 (orchestrator session)
                          |
                          v
            /CJ_goal_feature|defect|todo_fix
                          |
                  ... QA passes green ...
                          |
                          v
            Skill(CJ_document-release)
                          |
                          v
        ┌── arg parse: --docs <comma-list>
        ├── branch + clean-tree gate (refuse on main; refuse on dirty non-doc tree)
        ├── project-context block (doc-only, not enforced)
        ├── Skill(/document-release) ─→ upstream gstack
        ├── halt-on-red: RESULT=red → [doc-sync-red] HALT marker → orchestrator HALTs
        ├── auto-commit doc-only (whitelist gate): non-doc write → [doc-sync-non-doc-write] HALT
        └── success summary: green or green-noop
                          |
                          v
                       /ship (Step 6)
                          |
                          v
                   PR includes BOTH code + doc commits
```

Doc-only whitelist:

```
^(README|CHANGELOG|CLAUDE|ARCHITECTURE)\.md$     ← root-level convention docs
^doc/.+\.md$                                     ← doc/ folder
^templates/doc-.*\.md$                           ← F000032/F000033/F000034 template-doc convention
```

Halt classes:

```
[doc-sync-red]              ← /document-release failed (audit error, mid-write)
                              halt class: halted_at_doc_sync
                              resume_cmd: /CJ_goal_<verb> [args]
[doc-sync-non-doc-write]    ← /document-release wrote outside whitelist (upstream-misbehaved)
                              halt class: halted_at_doc_sync_non_doc_write
                              resume_cmd: /CJ_goal_<verb> [args]; manually inspect non-doc files first
```

F000029 coexistence at run-time (mutually exclusive paths):

```
orchestrator session (/CJ_goal_*)        non-orchestrator session
    |                                       (raw git push, manual /ship)
    v                                       |
F000036 Step 5.5 inline                     v
(this story)                            F000029 marker-AUQ on next session
                                        (preamble fires DOC_SYNC_PENDING)
```

### Data Flow

1. Operator invokes `/CJ_goal_feature "<topic>"` (or `/CJ_goal_defect` or `/CJ_goal_todo_fix`).
2. Orchestrator runs phases 1–5 (worktree → office-hours → design-summary gate → scaffold → implement → QA).
3. QA returns green (Phase 5 complete).
4. Orchestrator hits new Step 5.5 (Doc-sync) per the edited pipeline.md.
5. Orchestrator invokes `Skill(skill="CJ_document-release")` (no `--docs` flag in v1 orchestrator wiring; defaults to full audit).
6. CJ_document-release SKILL.md runs:
   a. Preamble (skills-update-check).
   b. Arg parse: `$ARGUMENTS` empty → full audit (no filter).
   c. Branch gate: `git symbolic-ref --short HEAD` — refuses on main/master/base; passes on feature branch.
   d. Clean-tree gate: `git status --porcelain | grep -vE '^.. (README\.md|CHANGELOG\.md|CLAUDE\.md|ARCHITECTURE\.md|doc/.*\.md|templates/doc-.*\.md)$'` — refuses if non-doc dirty.
   e. Project-context block: build a doc-only multiline string (documentation-only, not programmatic).
   f. Invokes `Skill(skill="document-release", args=<context-block>)`.
   g. Waits for completion; reads result.
   h. Halt-on-red branch: if upstream returned non-green → emit `[doc-sync-red]` marker + journal entry → exit RESULT=red.
   i. Auto-commit doc-only: `git status --porcelain` — for each line, check against whitelist. If any non-whitelist file is dirty → emit `[doc-sync-non-doc-write]` marker → exit RESULT=red. If all dirty files match whitelist → `git add <those files>` + `git commit -m "docs: post-build sync via CJ_document-release"`. If no dirty files → green-noop.
   j. Success summary: print `CJ_document-release: green / /document-release: green / commit: <sha-or-noop> / filtered: <subset-or-full>`. Exit RESULT=green or green-noop.
7. Orchestrator interprets RESULT:
   - red → HALT with the corresponding marker + journal entry; orchestrator exits with halt_class.
   - green or green-noop → continue to Step 6 (/ship).
8. /ship runs: Step 1 (clean-tree precondition NOW satisfied because doc commit was made), pre-landing review, VERSION bump, CHANGELOG, push, gh pr create.
9. /CJ_goal_feature halts at PR per design (no auto-merge).

### Halt-marker shape (machine-readable)

```
[doc-sync-red]
next_action=inspect /document-release output; fix doc errors; re-run /CJ_document-release
resume_cmd=/CJ_document-release [--docs <same-subset>]
pr_url=N/A
raw_output_path=<path-from-document-release-output-if-any>
```

```
[doc-sync-non-doc-write]
next_action=inspect uncommitted non-doc files; revert if unexpected; re-run /CJ_document-release
resume_cmd=/CJ_document-release [--docs <same-subset>]
pr_url=N/A
non_doc_files=<comma-separated-list-from-git-status>
```

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Skill shape | A: new SKILL.md skill | B: hybrid skill + bash helper script; C: inline triplicate in 3 pipeline.md | A reuses F000032+F000033+F000034 pattern; one folder, one evolution surface. B fragments discoverability (script `--help` vs SKILL.md). C is the exact anti-pattern F000029 BD#1 flagged. |
| F000029 BD#1 disposition | Supersede in-place; F000029 marker-AUQ stays as fallback | Deprecate F000029 entirely; or leave BD#1 unannotated | In-place annotation keeps audit trail navigable. Coexistence handles non-orchestrator paths (raw `git push`). |
| Auto-commit whitelist | Conservative: README\|CHANGELOG\|CLAUDE\|ARCHITECTURE.md + doc/.+\.md + templates/doc-.*\.md | Permissive: any `*.md` anywhere | Conservative prevents stealth code edits via doc-sync surface; false-positive surfaces in dogfood if needed. |
| Halt-on-red contract | Hard halt (build failure) | Soft warn (operator dismisses) | WARN gets ignored (memory `feedback_skill_contracts_strict`); ERROR + cheap override is load-bearing. |
| Halt class separation | Two classes: [doc-sync-red] + [doc-sync-non-doc-write] | One class for both | Different diagnostic surfaces. `[doc-sync-red]` = upstream failed; `[doc-sync-non-doc-write]` = upstream-misbehaved. Separation gives halt-journal clarity. |
| Step 5.5 uniformity | Identical across 3 orchestrators (modulo `<verb>` in resume_cmd) | Per-verb behavior variations | Uniform behavior; single evolution surface. Future divergence = add args to skill, not duplicate logic. |
| Phase-step section in SKILL-CATALOG.md | Tag `(phase-step in /CJ_goal_feature chain)`, no ASCII chart | Full ASCII chart | Per F000034 Open Question #1 resolution: phase-step skills get tag, not chart (chart would be one box). |
| portability field | workbench | standalone | Skill depends on workbench-specific conventions; explicit signal. |
| /document-release modification | None (wrap via Skill tool) | Modify upstream | Per memory `feedback_workbench_scope` + `project_workbench_auto_deploy_unsafe`. |
| Single user-story decomposition | Yes (atomic, 16 files in one commit) | Split into multiple stories | Same atomic-landing constraint as F000032+F000033+F000034. Splitting adds bookkeeping. |
