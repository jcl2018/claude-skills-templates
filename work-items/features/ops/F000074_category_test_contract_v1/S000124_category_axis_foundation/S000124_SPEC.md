## Problem Statement

The workbench operator maintaining `/CJ_test_audit` + `/CJ_test_run` wants them
finalized around a category-based test contract (`workflow` + `CI`) that any repo can
adopt, replacing the family/units/behaviors/runners mental model. Today the two skills
speak the older vocabulary and there is no category axis in the contract or the engines.
This story adds the category axis and re-points both skills — additively, so the
workbench's own CI gates (`validate.sh` Checks 24/26/28 over the existing grammar) stay
green while the new model lands.

## Mental Model

One noun — the test **category** — threads five surfaces: the folder a test lives in
(`tests/<category>/`), the contract section that declares it, the doc that describes it
(`docs/tests/<category>/<name>.md`), the index row that references it, and the
`/CJ_test_run` argument that runs it. V1 has exactly two categories: `workflow`
(deterministic end-to-end workflow tests — the `tests/eval/CJ_goal_*` cases + `e2e-local`)
and `CI` (what must be green to ship — the `*.test.sh` suites plus `validate`,
`test-deploy`, `windows-smoke`). This PR ships the FOUNDATION: the axis is added
backward-compatibly and the skills are re-pointed; the physical reorganization + grammar
removal + validate re-expression are DEFERRED.

## Acceptance Criteria

<!-- Given/When/Then. #n matches the Requirements # column. -->

### Story #1: Backward-compatible category axis [core]

```
GIVEN the two-tier test contract on the current overlay
WHEN  the category axis is added (a --seed portable category contract + new
      test-spec.sh category subcommands)
THEN  test-spec.sh --validate / --check-coverage / --render-docs --check /
      --check-workflow-coverage still pass unchanged AND the new category
      subcommands list/validate/render the workflow + CI categories
```

### Story #2: /CJ_test_audit five structural checks [usability]

```
GIVEN a repo with the category contract adopted
WHEN  /CJ_test_audit runs standalone
THEN  it reports checks (a) tests/ exists, (b) per-category subfolders,
      (c) category-scoped spec/test-spec.md, (d) one docs/tests/<category>/<name>.md
      per declared test, (e) a docs/tests/ index — each unmet check a FINDING:,
      and the audit never crashes
```

### Story #3: Idempotent doc-stub seeding, never move scripts [core]

```
GIVEN declared category tests missing their docs/index rows
WHEN  /CJ_test_audit runs (and is re-run)
THEN  it seeds the missing docs/tests/<category>/<name>.md stubs + index rows
      idempotently (present ⇒ skip on re-run) AND never moves or rewrites any
      test script
```

### Story #4: Category + single-name run selection [usability]

```
GIVEN the reshaped category contract mapping test-name -> command
WHEN  the operator runs /CJ_test_run --category workflow, --category CI, or a
      single test name, with or without --dry-run
THEN  the right tests are selected (reusing the docs/tests/ name), --dry-run
      prints the plan and executes nothing, and a default run touches no paid model
```

### Story #5: Skill + CLAUDE.md rewrite, additive + green [integration]

```
GIVEN the category axis and skills re-pointed at it
WHEN  the full checks run
THEN  both skills' SKILL.md + USAGE.md + CLAUDE.md describe the category model,
      new tests cover the behavior, validate.sh + test.sh + the doc/test audit
      are green, and the change is verified ADDITIVE (existing axes + family-doc
      render intact; no *.test.sh scripts moved)
```

## Architecture

<!-- High-level design: components affected, data flow. -->

```
                     spec/test-spec.md (general seed: portable category contract)
                     spec/test-spec-custom.md (overlay: repo category rows)
                                    |
                                    v
                        scripts/test-spec.sh  (NEW category subcommands:
                        --list-categories / --validate(+category) /
                        --check-structure / --render-docs(+category) / --seed)
                        [existing subcommands UNCHANGED -> stay green]
                        /                                   \
                       v                                     v
            /CJ_test_audit  (five checks a-e,         /CJ_test_run  (--category <cat>,
             report + seed doc stubs,                  single test name; maps
             never move scripts)                       name -> command via
                       |                               scripts/test-run.sh; cost-tiered)
                       v
            docs/tests/<category>/<name>.md (one per test) + docs/tests/ index table
```

| `spec/test-spec.md` | (this repo) | Modified | Add the portable category contract carried by `--seed` (backward-compatible). |
| `spec/test-spec-custom.md` | (this repo) | Modified | Add the repo's category rows (name/category/command/tier/doc/purpose) alongside the existing axes. |

| `skills/CJ_test_run/SKILL.md` + `USAGE.md` | (this repo) | Modified | Rewrite around category + single-name run. |
| `CLAUDE.md` | (this repo) | Modified | Update the scripts-reference + contract prose for the category surface. |
| `tests/` | (this repo) | New | Add coverage for the category subcommands + audit checks + run selection (assert additivity). |

### Data Flow

1. `test-spec.sh --seed` emits the portable general category contract; the repo's category tests live in the overlay.
2. `/CJ_test_audit` reads the merged registry, runs the five structural checks (a–e), prints `FINDING:` lines for gaps, and seeds missing `docs/tests/<category>/<name>.md` + index rows idempotently.
3. `/CJ_test_run` maps a `--category` or single test name to the declared command(s) via the reshaped contract, honoring cost tiering (default = free tier only).
4. The existing `test-spec.sh` subcommands continue to run over the untouched units/behaviors/runners grammar, so `validate.sh` Checks 24/26/28 stay green.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Increment sizing | Additive foundation first, defer the physical move | REPLACE + reorganize in one PR (Approach B end-state) | Keeps each PR reviewable and green; Checks 24/26/28 still validate the existing grammar this PR. |
| Audit posture | Report + seed doc stubs only | Audit physically reorganizes test scripts | "Runs standalone in ANY repo" is load-bearing — the audit must never mutate a foreign repo's code. |
| Category taxonomy | Closed V1 set `{workflow, CI}` | Open set incl. unit/integration/eval/property now | Mechanical assignment; the axis stays open-world for later growth without over-scoping V1. |
