## Problem Statement

The workbench has a defect taxonomy (`work-items/defects/<domain>/D000NNN_*/`) and per-defect artifacts (RCA, test-plan), but no one-keystroke path from "I have a defect work-item" to "fix is shipped and deployed." Operators currently chain `/investigate` → hand-write RCA → `/CJ_implement-from-spec` → `/ship` → `/land-and-deploy` manually. The pain: no automatic enforcement of the Iron-Law gate (no fixes without root cause), no audit trail bridge from `/investigate` output to RCA artifact, no idempotent re-entry.

As a workbench operator, I want `/CJ_goal_investigate D000NNN` to take a scaffolded defect work-item and ship a deployed fix end-to-end — with the Iron-Law gate enforced for free, RCA + test-plan populated from `/investigate`'s output, halt-on-red at every gate, and idempotent re-entry that resumes from the next pending step.

## Mental Model

```
defect work-item (D000NNN dir)
        │
        ▼  preflight: idempotency table (5 rows; RCA × fix × PR-open × PR-merged)
        │
        ▼  /investigate dispatch (Agent subagent, sentinel-wrapped JSON output)
        │
        ▼  parse JSON → halt-on-red taxonomy (9 end-states)
        │
        ▼  artifact writes (RCA template-mapped, test-plan row append)
        │
        ▼  /CJ_qa-work-item <defect-dir>
        │
        ▼  /ship Gate #2 (operator AUQ)
        │
        ▼  /land-and-deploy --suppress-readiness-gate
        │
        ▼  tracker journal: [investigate-shipped] D000NNN vX.Y.Z PR #NNN
        │
        ▼  telemetry line
```

## Acceptance Criteria

### Story #1: D-ID exact resolve [core]

```
GIVEN a scaffolded defect at work-items/defects/<domain>/D000019_<slug>/
WHEN  operator runs `/CJ_goal_investigate D000019`
THEN  the orchestrator finds the dir, prints "Resolved: D000019 at <path>"
```

### Story #2: Fragment fuzzy resolve [core]

```
GIVEN defects D000017_validate_section_anchors_test_clobber and D000019_step5_1_type_aware_input_artifact_drift
WHEN  operator runs `/CJ_goal_investigate "step5"`
THEN  the orchestrator finds D000019 unambiguously and proceeds

GIVEN two defects whose names both contain "validate"
WHEN  operator runs `/CJ_goal_investigate "validate"`
THEN  the orchestrator halts with a ranked candidate list and prints "Re-run: /CJ_goal_investigate <full-D-id>"
```

### Story #3: /investigate dispatch sentinel handoff [core]

```
GIVEN a resolved defect with empty RCA
WHEN  the orchestrator dispatches /investigate with the sentinel-wrapped JSON instruction
THEN  the subagent returns stdout containing `DEBUG_REPORT_BEGIN_JSON\n{...valid JSON...}\nDEBUG_REPORT_END_JSON`
```

### Story #4: Halt on missing/malformed/empty root cause [core]

```
GIVEN /investigate returned stdout without a DEBUG_REPORT_BEGIN_JSON block
WHEN  the orchestrator parses output
THEN  it writes [investigate-no-sentinel] with next_action= and raw_output_path= and exits with end_state=halted_at_gate

GIVEN JSON.root_cause is empty or matches /^\[.*\]$/
WHEN  the orchestrator validates
THEN  it writes [investigate-no-root-cause] and exits

GIVEN JSON cannot be parsed
WHEN  the orchestrator validates
THEN  it writes [investigate-parse-error] and exits
```

### Story #5: RCA template-heading mapping [core]

```
GIVEN a valid JSON DEBUG REPORT with keys: symptom, root_cause, fix (with file list), regression_test, evidence
WHEN  the orchestrator writes RCA.md
THEN  RCA contains:
  - ## Symptom (verbatim from JSON.symptom)
  - ## Reproduction Steps (synthesized; `<!-- TODO: operator fills repro steps -->` if absent)
  - ## Investigation Trail (3-line timestamped trail)
  - ## Root Cause (**Root cause:** ... + **Location:** file:line)
  - ## Affected Components (one row per file in JSON.fix)
  - ## Fix Description (prose from JSON.fix)
  - ## Regression Risk (with "Regression test added: <path>" + evidence)
```

### Story #6: test-plan row append [core]

```
GIVEN JSON.regression_test = "tests/test-d000019.sh"
WHEN  the orchestrator writes test-plan.md
THEN  test-plan.md contains a new row: `| tests/test-d000019.sh | regression test for D000019 root cause | smoke |`
```

### Story #7: 5-row idempotency resume [core]

```
GIVEN RCA empty, no fix in tree, no PR
WHEN  operator runs /CJ_goal_investigate D000019
THEN  resume from /investigate dispatch (fresh start)

GIVEN RCA populated, fix in tree, no PR
WHEN  operator re-runs
THEN  skip /investigate; jump to /CJ_qa-work-item

GIVEN RCA populated, fix in tree, PR merged
WHEN  operator re-runs
THEN  print one-line summary; no-op

GIVEN RCA empty, fix in tree (anomaly)
WHEN  operator re-runs
THEN  halt with [anomaly-rca-missing-with-fix]
```

### Story #8: --dry-run preview [core]

```
GIVEN any valid input
WHEN  operator runs /CJ_goal_investigate --dry-run D000019
THEN  the orchestrator prints:
  - resolved defect path
  - current idempotency state (which resume row matches)
  - expected /investigate dispatch plan
  - expected RCA + test-plan write paths
  - NO file written, NO subagent dispatched
```

### Story #10: DONE_WITH_CONCERNS Iron-Law halt [resilience]

```
GIVEN JSON.status = "DONE_WITH_CONCERNS"
WHEN  the orchestrator parses
THEN  it writes [investigate-unverified] and exits with end_state=halted_at_gate
  AND does NOT advance to /ship
```

### Story #11: Blast-radius pre-fix halt [resilience]

```
GIVEN /investigate emits FIX_PLAN_BEGIN_JSON ... FIX_PLAN_END_JSON with >5 files
WHEN  the orchestrator parses the plan BEFORE Phase 4 starts
THEN  it writes [investigate-blast-radius] and exits before any source edit
```

### Story #12: /ship Gate #2 fires [integration]

```
GIVEN a successful /investigate run with RCA + test-plan populated
WHEN  the chain reaches /ship
THEN  /ship Gate #2 (diff-review AUQ) surfaces the diff for operator approval
```

### Data Flow

1. Operator runs `/CJ_goal_investigate D000019` (or fragment).
2. Orchestrator resolves defect dir via `resolve_defect_dir()`; halts on ambiguity.
3. Orchestrator inspects RCA-populated, fix-in-tree, PR-open, PR-merged signals → picks resume row.
4. Orchestrator builds `/investigate` dispatch prompt with sentinel-wrapped JSON instruction + FIX_PLAN preamble request; spawns Agent subagent.
5. Subagent runs `/investigate` Phases 1-4 (Phase 4 writes fix); returns stdout containing FIX_PLAN_BEGIN_JSON + DEBUG_REPORT_BEGIN_JSON blocks.
6. Orchestrator extracts FIX_PLAN; halts on `[investigate-blast-radius]` if >5 files.
7. Orchestrator extracts DEBUG_REPORT; validates JSON; halts on missing/malformed/empty-root-cause/blocked/unverified/three-strike/no-sentinel.
8. On green: orchestrator writes RCA.md (heading mapping table) and test-plan.md (row append).
9. Orchestrator invokes `/CJ_qa-work-item <defect-dir>` via Skill.
10. Orchestrator invokes `/ship`; Gate #2 fires for operator approval; halt on `[ship-declined]`.
11. Orchestrator invokes `/land-and-deploy --suppress-readiness-gate`; halt on `[land-and-deploy-red]`.
12. Orchestrator writes `[investigate-shipped] D000NNN vX.Y.Z PR #NNN` to tracker journal; appends telemetry line.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Machine handoff format | Sentinel-wrapped JSON (`DEBUG_REPORT_BEGIN_JSON ... END_JSON`) | Free-text DEBUG REPORT regex-parsed | Reviewer-flagged critical brittleness in free-text parsing; sentinel is convention, not upstream feature, so we can ship without gstack change. |
| Blast-radius detection | FIX_PLAN preamble emitted by /investigate BEFORE Phase 4 (option c) | Reactive halt post-fix (option b) or parse pre-Phase-4 narration (option a) | Pre-write halt is the only one that prevents wasted source mutation. Option (a) is fragile narration parsing; option (b) destroys atomicity. |
| `/CJ_implement-from-spec` placement | Removed from chain | Kept after /investigate as separate impl step | `/investigate` Phase 4 already writes the fix; a second impl step would double-write or fight for ownership. RCA + test-plan are post-investigate audit, not inputs. |
| Iron-Law halt for DONE_WITH_CONCERNS | Distinct `[investigate-unverified]` halt | Treat as soft warning + auto-advance | Defeats the Iron-Law gate; reviewer-flagged. Operator can manually ship via `/ship` directly. |
| Defect dir layout in v1 | Legacy `work-items/defects/<domain>/D000NNN_<slug>/` only | Support both legacy + freestanding in v1 | Single helper swap in v1.1; v1 scope simplification. All D000001–D000018 use legacy. |
| Drain mode timing | v1.1 | v1.0 | Codex + Claude reviewers both flagged v1 as over-scoped; drain needs telemetry baseline + family-drain lock design. |
