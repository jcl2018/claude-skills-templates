## Problem Statement

The workbench maintainer wants the paid `run-eval` harness gone: `scripts/eval.sh`
(`tier: paid`) spawns a headless `claude --print` per eval case, which is metered
model spend. The replacement is in-session Claude verification (drive the
`tests/eval/<skill>/` cases and report pass/fail vs their expected outcomes) —
$0 marginal on the subscription. The constraint is that the
`tests/eval/<skill>/<case>/` dirs are the anchors `validate.sh` Check 28
(`--check-workflow-coverage`) greps, and `suite-eval`'s `source: scripts/eval.sh`
anchor is what Check 24's forward grep resolves — so the deletion cannot dangle
either gate.

## Mental Model

The subtraction preserves the free gates and drops only the paid runner:

```
Surface                                 Before                 After (this story)
------------------------------------    -------------------    ------------------
scripts/eval.sh (tier: paid runner)     present                DELETED
run-eval runners: row                   present                REMOVED
suite-eval unit source:                  scripts/eval.sh        tests/eval/ spec (live anchor)
run-test-sh covers: eval                 "run-eval owns exec"   "eval = specs-only, in-session"
goal-task-eval / goal-feature-eval       categories: workflow   REMOVED (+ front-door docs)
  categories: rows                        rows (topic:cj-goal-eval)
tests/eval/<skill>/<case>/ dirs          present (may leak)     KEPT + de-leaked (anchors preserved)
behaviors:/behavior_coverage: axis       present                UNCHANGED
validate.sh Check 28                     green                  green (4/4 orchestrators)
```

Check 28 is driven by the `behaviors:` axis (each `level: workflow` behavior →
its `workflow:` orchestrator + its `tests/eval/<skill>/<case>/prompt.md` anchor),
INDEPENDENT of the `categories:` rows — so the `categories:` removal is safe.

## Acceptance Criteria

<!-- Given/When/Then format. Each #n header matches the # column in the
     Requirements tables. -->

### Story #1: Paid runner deleted, no dangling reference [core]

```
GIVEN scripts/eval.sh is deleted and the run-eval runners: row is removed
WHEN  grep -rn "eval.sh" across scripts/ tests/ .github/workflows/ spec/ and the
      test-run.sh/test.sh engines runs
THEN  no dangling reference to eval.sh remains (only intended mentions in
      work-item docs, if any)
```

### Story #2: suite-eval re-anchored, family not orphaned [core]

```
GIVEN the suite-eval unit re-anchored onto a live tests/eval/ source + anchor
WHEN  bash scripts/test-spec.sh --check-coverage runs (Check 24 engine)
THEN  suite-eval's forward anchor-grep matches live; findings=0; the eval family
      remains declared (run-test-sh covers: eval + suite-eval family: eval)
```

### Story #3: run-eval row removed, covers note reframed [core]

```
GIVEN the run-eval runners: row removed and run-test-sh's covers: note reframed
WHEN  bash scripts/test-spec.sh --validate and --check-coverage run
THEN  the registry validates; eval stays in run-test-sh covers:; no row declares
      the deleted scripts/eval.sh
```

### Story #4: two categories rows + docs removed, Check 28 intact [core]

```
GIVEN the goal-task-eval + goal-feature-eval categories rows removed, cj-goal-eval
      dropped from the unenrolled-topics prose, the two doc-spec declaring rows
      removed, the two front-door docs deleted, and docs/tests/index.md reconciled
WHEN  bash scripts/validate.sh runs
THEN  Check 28 (workflow coverage) passes 4/4 orchestrators; Checks 15/15a/17/19
      pass (no orphaned doc, no missing declared doc); Check 30 exits 0 with only
      advisory notes (cj-goal-eval no longer labeled)
```

### Story #5: eval prompts honest + non-leaking, anchors preserved [usability]

```
GIVEN each tests/eval/<skill>/<case>/prompt.md de-leaked (the dry_run_preview
      expected-output removed from CJ_goal_feature/dry-run-plan/prompt.md)
WHEN  bash scripts/test-spec.sh --check-workflow-coverage runs (Check 28 engine)
THEN  every level: workflow behavior's tests/eval/<skill>/<case>/prompt.md anchor
      still matches live; no prompt.md states its expected output
```

### Story #6: full deterministic gates green [resilience]

```
GIVEN the regenerated catalogs and all edits applied
WHEN  bash scripts/validate.sh && bash scripts/test.sh && shellcheck run, plus
      bash scripts/test-spec.sh --check-structure and --check-topic-contract
THEN  all green — Checks 24/26/27/28/30 pass, --check-structure leaves no required
      tests/workflow/local-hook/ subfolder empty, --check-topic-contract exits 0
      with only advisory notes, the full suite passes, shellcheck clean
```

### Data Flow

<!-- Primary use case: the deterministic gates run after the subtraction. -->

1. `scripts/eval.sh` is deleted; a `grep -rn "eval.sh"` sweep confirms no dangling caller in scripts/tests/workflows/spec engines.
2. `test-spec.sh --check-coverage` (Check 24) forward-greps every unit anchor into its declared source — `suite-eval` now resolves against the live `tests/eval/` spec, not the deleted script → no dangling.
3. `test-spec.sh --check-workflow-coverage` (Check 28) resolves each `level: workflow` behavior to its `workflow:` orchestrator + its `tests/eval/<skill>/<case>/prompt.md` anchor → 4/4 orchestrators still wired (the `categories:` removal did not touch these).
4. `test-spec.sh --check-topic-contract` (Check 30) runs — `cj-goal-eval` no longer has labeled `categories:` rows, so it's absent from the unenrolled list; enrolled topics pass with only advisory agentic notes → exit 0.
5. `test-spec.sh --check-structure` confirms the `tests/workflow/local-hook/` category/layer pair is still satisfied by doc-sync + e2e-local (no empty required subfolder).
6. `--render-docs` regenerates the catalogs so Checks 26/27 diff clean; `validate.sh` + `test.sh` + shellcheck run fully green.

## Tradeoffs

<!-- Design decisions made in this spec, with rejected alternatives and rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Workflow-category rows | Remove the `goal-task-eval` + `goal-feature-eval` `categories:` rows (Approach C) | A: change their `command` to `echo "SKIP: drive tests/eval/… in-session"` + re-tier paid→local-only; B: a tiny `tests/eval/verify.sh` helper printing the SKIP + instructions | C is least surface, nothing to break, no doc-string-as-command awkwardness; A leaves a doc-string masquerading as an executable command; B adds new machinery, in tension with "retiring the harness is the feature." Operator-selected at the 2026-07-06 AUQ. |
| `suite-eval` anchor | Re-anchor onto a live `tests/eval/` `source` + `anchor` and reframe as the in-session verification specs | Delete the `suite-eval` unit entirely | Deleting it would orphan the `eval` family and drop a declared verification surface; re-anchoring keeps the family declared and the coverage cross-check honest. |
| Verification mechanism | In-session Claude ask (no wrapper) | A new `/CJ_verify` skill | The durable value lives in the specs, not a wrapper; adding a skill contradicts the retiring-machinery thesis. |
| Seed mirror | Mirror `spec/test-spec.md` ONLY if a general-file line changed (expected: none) | Blanket re-run the seed mirror regardless | The dual-write footgun is only tripped by a general-file edit; all edits here are overlay, so the seed should stay byte-identical untouched. |
