## Problem Statement

The maintainer of this doc-first workbench wants the `CI-nightly` verification layer
to run **deterministic-only for now** — no unattended model spend on a cron. Today
three agentic tests run unattended: `goal-task-eval` + `goal-feature-eval`
(`.github/workflows/eval-nightly.yml`) and `doc-sync` (`audit-nightly.yml`). The
agentic proofs should not disappear; they should re-layer to `local-hook` and run
on-demand. And because `audit-nightly.yml` is the job that auto-catches semantic prose
drift, the docs must be made truthful in the same change so nothing claims a nightly
job that no longer exists.

## Mental Model

The change is one coherent move across a fixed 7-part file set (from the design's
`## Implementation Plan`). Two orthogonal effects:

1. **Cadence deletion** — remove the two `.yml` cron wrappers so the `CI-nightly`
   cadence carries only the deterministic `windows-nightly.yml` (`portability-deploy`).
2. **Re-layer + truth** — move the 3 agentic tests' category rows / units / front-door
   docs / index rows / doc-spec declarations from `CI-nightly` to `local-hook`, then
   sweep every prose site that said "nightly in CI via audit-nightly.yml" to
   "on-demand locally." The `DEFER_AUDIT` behavior itself is unchanged — only the
   "where the deferred audit runs" clause changes.

## Acceptance Criteria

<!-- Given/When/Then. Each #n header matches the # column in Requirements. -->

### Story #1: Delete the two cron wrappers [core]

```
GIVEN the repo carries .github/workflows/eval-nightly.yml and audit-nightly.yml
WHEN  both files are deleted
THEN  neither file exists on disk
AND   `ls .github/workflows/*nightly*.yml` shows only windows-nightly.yml
```

### Story #2: Re-layer the 3 tests in the test contract [core]

```
GIVEN spec/test-spec-custom.md declares goal-task-eval / goal-feature-eval / doc-sync
      at layer: CI-nightly, plus ci-eval-nightly + ci-audit-nightly units
WHEN  the 3 categories: rows are changed to layer: local-hook with doc: paths under
      docs/tests/workflow/local-hook/ (mode: agentic + tier: paid retained), and the
      ci-eval-nightly + ci-audit-nightly units are removed
THEN  test-spec.sh --list-categories shows the 3 tests at local-hook
AND   test-spec.sh --validate + --check-coverage pass with no orphan-unit finding
```

### Story #3: Front-door docs + index + doc-spec agree with disk [usability]

```
GIVEN the 3 front-door docs live under docs/tests/workflow/CI-nightly/
WHEN  they are git mv'd to docs/tests/workflow/local-hook/, each ## How to run is
      reframed to on-demand/local, docs/tests/index.md's 3 rows move to local-hook,
      and spec/doc-spec-custom.md's 3 declaration rows point at the local-hook/ paths
THEN  every declared doc path equals an on-disk path (validate.sh Checks 15/15a pass)
AND   no moved doc's ## How to run says "runs nightly in CI"
```

### Story #4: Prose sweep to on-demand/local [observability]

```
GIVEN many doc/spec/skill sites say "nightly in CI via audit-nightly.yml"
WHEN  those sites are swept to on-demand/local (the DEFER_AUDIT behavior unchanged,
      only the "where it runs" clause) and docs/workflows/*.md are regenerated via
      workflow-spec.sh --render-docs
THEN  no doc/spec/skill text claims a nightly-CI audit or eval that no longer runs
AND   validate.sh Check 27 (generated workflow docs fresh) passes
```

### Story #5: Whole change stays green [resilience]

```
GIVEN the 7-part file set is edited
WHEN  test-spec.sh --render-docs + workflow-spec.sh --render-docs are run and the
      generated catalogs committed
THEN  ./scripts/validate.sh and ./scripts/test.sh both exit GREEN
AND   test-spec.sh --check-structure reports findings=0
AND   Check 28 passes (orchestrators=4, behaviors=4)
```

### Story #6: Scripts + regression test preserved [core]

```
GIVEN scripts/eval.sh, scripts/audit-nightly.sh, and tests/audit-nightly.test.sh exist
WHEN  the change lands
THEN  all three still exist (only the .yml cron wrappers were deleted)
AND   the Scripts-table When-to-run cells for eval.sh + audit-nightly.sh read
      on-demand/local, not "nightly CI"
```

## Architecture

```

    eval-nightly.yml   --cron--> scripts/eval.sh   [DELETE]   (gone)
    audit-nightly.yml  --cron--> scripts/audit-nightly.sh [DELETE]  (gone)
    windows-nightly.yml --cron--> portability-deploy [KEEP]   windows-nightly.yml --cron--> portability-deploy

  test contract (spec/test-spec-custom.md)
    categories: goal-task-eval   layer: CI-nightly  ---->   categories: goal-task-eval   layer: local-hook

                doc-sync          layer: CI-nightly  ---->               doc-sync          layer: local-hook
    units: ci-eval-nightly, ci-audit-nightly         ---->   (removed — deleted source .yml)
    units: ci-windows-nightly, test-audit-nightly, suite-eval  [KEEP]

  docs   docs/tests/workflow/CI-nightly/{3 docs}  --git mv-->  docs/tests/workflow/local-hook/{3 docs}
```

### Data Flow

<!-- How the change moves through the system for the primary use case. -->

1. Delete the two `.yml` cron wrappers (Part 1) → the `CI-nightly` cadence now carries only `windows-nightly.yml`.
2. Edit `spec/test-spec-custom.md` (Part 2): re-layer the 3 `categories:` rows + remove the 2 now-orphan units + soften prose.
3. `git mv` the 3 front-door docs to `local-hook/` + reframe each `## How to run` (Part 3); move the 3 `docs/tests/index.md` rows (Part 4); update `spec/doc-spec-custom.md` paths (Part 5).
4. Sweep the remaining prose sites + edit `spec/workflow-spec.md` (Part 6).
5. Regenerate the catalogs (`test-spec.sh --render-docs` + `workflow-spec.sh --render-docs`), run `--check-structure`, then `validate.sh` + `test.sh` to green (Part 7).

## Tradeoffs

<!-- Design decisions, rejected alternatives, rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| How to stop the agentic cron | Delete both `.yml` wrappers (Approach A) | Neuter: keep `workflow_dispatch`, drop `schedule:` (Approach B) | A is the most truthful end-state; a dormant-but-referenced workflow is the exact half-truth this change removes the auto-catch for. Reversibility = git revert. |
| Replace the nightly drift catch | Manual-local safety net (hand-run script / audit verbs) | Add a deterministic model-free nightly heartbeat (Approach C) | C is highest ongoing surface for lowest value — the checks already gate every PR and main only advances via passing PRs. |
| Scope of prose changes | Full sweep in the same PR | Defer the prose sweep to a follow-up | The deleted `audit-nightly.yml` was the auto-catch for prose drift; deferring would leave docs claiming a job that no longer exists with nothing to catch it. |
| The re-layered rows' cost tier | Keep `mode: agentic` + `tier: paid` | Downgrade to free | They still spend tokens when run on-demand; the tier must stay honest. |
