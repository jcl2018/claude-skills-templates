## Problem Statement

The workbench developer wants to retire the deprecated `CJ_company-workflow` skill. Today, `scripts/validate.sh` Error check 10 enforces byte-identity between `deprecated/CJ_company-workflow/` and `work-copilot/` via a MIRROR_SPECS loop with 4 shape handlers. That enforcement is the only thing forcing `deprecated/CJ_company-workflow/` to stay in-tree as the source-of-truth. The workbench operator (chjiang) needs validate.sh to enforce existence of the canonical bundle (`work-copilot/`) without referencing the deprecated upstream — so the deprecated dir can be deleted in S000053 without leaving the bundle unguarded.

## Mental Model

Today: `deprecated/CJ_company-workflow/` (truth) → byte-mirror → `work-copilot/` (consumed). validate.sh Error check 10 enforces the byte-identity. Error check 10b separately enforces existence of bundle-only files (F000015 prompts, domain templates) that have no upstream counterpart.

After: `work-copilot/` (truth) → consumed. Error check 10 is gone. Error check 10b enforces existence of all the files that were previously covered by check 10 (templates, WORKFLOW.md, reference/, philosophy/, examples/, fixtures/, copilot-artifact-manifests.json) plus the existing bundle-only files. One flat existence check, no shape distinction.

## Acceptance Criteria

<!-- Given/When/Then format. One block per requirement or logical group. -->

### Story #1: validate.sh PASSes after Error check 10 removal [core]

```
GIVEN  scripts/validate.sh has been edited to delete Error check 10's MIRROR_SPECS array, loop, and shape handlers
WHEN   the operator runs `./scripts/validate.sh`
THEN   the script exits 0 (PASS)
AND    no MIRROR_SPECS-related output appears in stdout
```

### Story #2: All 7 previously-mirrored paths are covered by EXPECTED_BUNDLE_FILES [core]

```
GIVEN  the rewritten scripts/validate.sh Error check 10b
WHEN   the operator deletes any file under work-copilot/templates/, work-copilot/WORKFLOW.md,
       work-copilot/reference/, work-copilot/philosophy/, work-copilot/examples/,
       work-copilot/fixtures/, or work-copilot/copilot-artifact-manifests.json (one at a time)
       and runs `./scripts/validate.sh`
THEN   the script fails with a clear "missing expected bundle file" error
AND    the failure names the missing path
```

### Story #3: F000015 bundle-only files remain covered [core]

```
GIVEN  the rewritten scripts/validate.sh Error check 10b
WHEN   the operator deletes any work-copilot/prompts/*.prompt.md or work-copilot/domain/*.template.md
       and runs `./scripts/validate.sh`
THEN   the script fails with a clear "missing expected bundle file" error (the same as for previously-mirrored files — Story #2 — distinguishable in the SPEC but identical in shape at runtime)
AND    these files appear in the EXPECTED_BUNDLE_FILES enumeration alongside the previously-mirrored paths
```

### Story #4: Byte-identity baseline verified before edit [resilience]

```
GIVEN  scripts/validate.sh is unmodified
WHEN   the operator runs `./scripts/validate.sh`
THEN   the script exits 0 (PASS) — i.e., MIRROR_SPECS byte-identity holds today
AND    the operator only proceeds to the Story #1 edit after this PASS is confirmed
```

### Story #5: EXPECTED_BUNDLE_FILES shape is legible [usability]

```
GIVEN  the rewritten Error check 10b in scripts/validate.sh
WHEN   a future maintainer opens the section
THEN   the listing is either a flat array of leaf paths OR a small directory-shape enumerator
AND    the choice is consistent within the section (no half-flat-half-enumerator hybrid)
```

### Data Flow

1. Operator runs `./scripts/validate.sh` before any edit; confirms PASS (byte-identity baseline holds).
2. Operator opens `scripts/validate.sh`, locates Error check 10 (~lines 255–446 per the design's autoplan note).
3. Operator deletes the MIRROR_SPECS array, the loop, and the four shape handlers; deletes orphan-policy handlers.
4. Operator extends `EXPECTED_BUNDLE_FILES` (Error check 10b) — either flat extension or via a small enumerator helper.
5. Operator adjusts any section header / numbered comment that referenced "Error check 10".
6. Operator runs `./scripts/validate.sh`; expects PASS.
7. (Optional manual smoke) Operator temporarily renames one file under `work-copilot/templates/`, re-runs validate.sh, expects RED with a clear missing-file message; restores the file.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| When to drop the byte-identity cross-check | Delete it as part of this story | Keep it until S000053 deletes the deprecated source | Without deleting check 10 now, S000053's `rm -rf deprecated/CJ_company-workflow/` would break the validator. The two changes are causally linked; doing both at the boundary between S000052 and S000053 keeps each diff minimal and reviewable. |
| `EXPECTED_BUNDLE_FILES` shape | Implementer's call at write time | Mandate flat-only OR enumerator-only | Both shapes are valid; the choice is purely a legibility judgment over the resulting `validate.sh`. Mandating either now would over-constrain the implementer. |
| Renumbering Error checks 11+ to fill the gap left by deleting 10 | Don't renumber; keep gap-numbering with adjusted comments | Renumber 11→10, 12→11, etc. | Renumbering churns more lines for cosmetic gain; preserving numbers keeps git blame stable for unchanged checks. The downside (a missing number in the sequence) is acceptable; a comment near the deletion site can note "Error check 10 retired in F000023/S000052". |
| File scope of this story | `scripts/validate.sh` only | Also touch a README pointer or CHANGELOG note | Single-file diff is the smallest reviewable unit. CHANGELOG entry rides on `/ship` automation later. |
