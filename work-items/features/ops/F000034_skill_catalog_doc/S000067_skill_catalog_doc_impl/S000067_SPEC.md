## Problem Statement

F000030 (PR #180, v5.0.11) added `doc/PHILOSOPHY.md` + `doc/ARCHITECTURE.md` and a `## /document-release workbench audit conventions` section in CLAUDE.md, but provides no convention for what happens when a third `doc/*.md` file is added. F000032 + F000033 ship per-skill `USAGE.md` + Check 13 (presence) + Check 14 (freshness), but there is no consolidated workbench-level surface where a reader can scan ALL skills at a glance with workflow charts. This story builds both: a hand-written `doc/SKILL-CATALOG.md` with one section per routable non-deprecated skill (4 orchestrators with ASCII workflow charts + 7 single-step skills with explicit tags), plus a tracked-doc/ manifest in CLAUDE.md registering every doc/*.md with an `audit_class`, plus a new `scripts/validate.sh` Check 15 enforcing orphan-detection + per-skill section completeness. CLAUDE.md + `## Creating a new skill` document the convention.

## Mental Model

Convention-stack reuse (F000030 → F000032 → F000033 → F000034):

```
F000030 doc/ folder      ("PHILOSOPHY.md + ARCHITECTURE.md, named audits")
F000032 Check 13: USAGE.md presence + 5 H2 sections
F000033 Check 14: USAGE.md freshness (%ct vs SKILL.md)
F000034 Check 15: doc/ manifest orphan + SKILL-CATALOG.md per-skill chart-or-tag
```

Audit set derived at run time from `skills-catalog.json` (matches Check 13 + 14 exactly):

```
status != "deprecated" AND (files | length) > 0
```

= 11 routable non-deprecated skills as of 2026-06-01.

Per-section requirement (Check 15b):

```
HAS_CHART = (grep -cE '^```' SECTION) >= 2     # fenced block open + close
HAS_TAG   = (grep -cE '^\((single-step utility|validator|phase-step in /CJ_goal_feature chain)\)' SECTION) >= 1
PASS      = HAS_CHART OR HAS_TAG
```

Tracked-doc/ manifest (inline YAML block in CLAUDE.md, awk-range parsed):

```yaml
- path: doc/PHILOSOPHY.md
  audit_class: skill-routing-drift
  owner: F000030 — workbench-level overview + decision tree
- path: doc/ARCHITECTURE.md
  audit_class: skill-routing-drift
  owner: F000030 — mechanism reference
- path: doc/SKILL-CATALOG.md
  audit_class: skill-catalog-completeness
  owner: F000034 — consolidated per-skill catalog with ASCII workflow charts
```

`audit_class` enum (closed):

- `skill-routing-drift` — F000030 retired-skill + new-skills check
- `skill-catalog-completeness` — every active routable skill has a section AND every section has chart-or-tag (Check 15)
- `static-reference` — file exists; no drift check beyond existence (reserved; v1 has no entries)
- `auto-generated` — file regenerated by a script; audit checks script-output == on-disk (reserved; v1 has no entries)

## Architecture

### Check 15 placement

After Check 14 (F000033). Same audit predicate. Two-half structure:

```bash
echo ""
echo "=== Check 15: doc/ manifest + SKILL-CATALOG.md completeness ==="

# 15a: manifest parse + orphan/missing-from-disk
MANIFEST_PATHS=$(awk '/^### Tracked doc\/ files manifest$/,/^### /{if($0 ~ /^- path:/) print $3}' CLAUDE.md)
DOC_FILES_ON_DISK=$(find doc -maxdepth 1 -type f -name '*.md' 2>/dev/null | sort)

for f in $DOC_FILES_ON_DISK; do
  if ! echo "$MANIFEST_PATHS" | grep -qFx "$f"; then
    echo "  ERROR: $f is in doc/ but not registered in CLAUDE.md tracked-doc/ manifest"
    ERRORS=$((ERRORS+1))
  fi
done

for p in $MANIFEST_PATHS; do
  if [ ! -f "$p" ]; then
    echo "  ERROR: $p is in CLAUDE.md manifest but missing from disk"
    ERRORS=$((ERRORS+1))
  fi
done

# 15b: SKILL-CATALOG.md per-skill section completeness (gated)
CATALOG_FILE="doc/SKILL-CATALOG.md"
if [ -f "$CATALOG_FILE" ]; then
  TAG_RE='^\((single-step utility|validator|phase-step in /CJ_goal_feature chain)\)'
  while IFS= read -r SKILL_NAME; do
    if ! grep -qE "^### ${SKILL_NAME}$" "$CATALOG_FILE"; then
      echo "  ERROR: $CATALOG_FILE missing section: ### $SKILL_NAME"
      ERRORS=$((ERRORS+1))
      continue
    fi
    SECTION=$(awk "/^### ${SKILL_NAME}$/,/^### /" "$CATALOG_FILE" | sed '/^### /d')
    HAS_CHART=$(echo "$SECTION" | grep -cE '^```' || true)
    HAS_TAG=$(echo "$SECTION"   | grep -cE "$TAG_RE" || true)
    if [ "$HAS_CHART" -lt 2 ] && [ "$HAS_TAG" -lt 1 ]; then
      echo "  ERROR: $CATALOG_FILE section '$SKILL_NAME' has neither ASCII chart (fenced block) nor a tag line; one is required"
      ERRORS=$((ERRORS+1))
    else
      echo "  PASS: $CATALOG_FILE has section for $SKILL_NAME"
    fi
  done < <(jq -r '.[] | select(.status != "deprecated") | select((.files | length) > 0) | .name' skills-catalog.json)
fi
```

Markdown skeleton with placeholders + inline instructions per field. Each placeholder uses `{name}`, `{status}`, etc. with HTML-comment instructions inline.

### Catalog (`doc/SKILL-CATALOG.md`)

- Header: title + one-paragraph description ("This is the consolidated index of every routable non-deprecated skill in this workbench. Sections are hand-written and audited by `scripts/validate.sh` Check 15.")
- 3 H2 groupings: Orchestrators / Phase-step skills / Validators + Utilities.
- 11 H3 sections (one per skill), each ~25-40 lines.
- Footer: links to PHILOSOPHY.md `## Decision tree` + per-skill USAGE.md.

## Tradeoffs

- **Inline YAML manifest vs separate JSON:** chose inline for v1 simplicity. Loses tool-friendly structured parsing; gains single-file context for `/document-release`. Hoist when manifest exceeds awk-range comfort (~5 entries) OR when a future tool needs to query it programmatically.
- **Hand-written sections vs auto-generation:** chose hand-written (Approach A). Loses zero-cost backfill of new skills; gains ASCII charts that are actually useful (not derivable from prose). Friction is intentional — Check 15 forces explicit declaration.
- **Closed `audit_class` enum:** chose closed for v1 to prevent free-text drift. Loses flexibility to invent new classes per-doc; gains audit-shape predictability. Adding a new class is a one-line enum bump + the Check 15 / `/document-release` code path; v1 only consumes 2 of 4 values.
- **ERROR severity:** chose ERROR + cheap-override per F000030/32/33 precedent. WARN replays F000030's 1/13 DESIGN.md adoption decay; ERROR forces operator action. The "override" here is just "add the section / add the manifest entry" — there's no special-case suppress flag.

## Out of Scope

- README.md per-skill workflow-chart column.
- work-copilot/ analog catalog/manifest.
- Upstream `/document-release` modification.
- Auto-generated catalog.
- Per-skill snooze of Check 15.
- Backfilling `auto-generated` or `static-reference` audit_class entries.
- ARCHITECTURE.md per-skill audit beyond F000030's existing skill-routing-drift.
