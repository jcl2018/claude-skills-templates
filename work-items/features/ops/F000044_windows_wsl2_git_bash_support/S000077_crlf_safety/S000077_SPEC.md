## Problem Statement

A fresh clone on Git-for-Windows (default `core.autocrlf=true`) rewrites all 42
tracked `*.sh` files to CRLF on checkout, so `#!/usr/bin/env bash\r` becomes a
bad interpreter and bash hits parse errors on every line. The repo has no
`.gitattributes` today, so Windows devs cannot run any script. (Note:
`scripts/lib.sh` already has a `jq()` CRLF shim for runtime jq OUTPUT тАФ that is a
different layer; this story fixes checked-out SOURCE line endings. Build on it,
don't duplicate.)

## Mental Model

One config file (`.gitattributes`) controls checkout line endings repo-wide.
`git checkout` reads `.gitattributes` and normalizes text files to LF on the way
to the working tree, regardless of the local `core.autocrlf` setting; binaries
declared `binary` are left byte-for-byte untouched.

```
git checkout тФАтФАтЦ║ reads .gitattributes тФАтФАтЦ║ applies eol=lf to text files
                                     тФФтФАтФАтЦ║ leaves binaries untouched
```

## Acceptance Criteria

<!-- Given/When/Then. One block per requirement. -->

### Story #1: Shell scripts checked out with LF [core]

```
GIVEN a fresh clone on Git-for-Windows with core.autocrlf=true
WHEN  the repo is checked out
THEN  every tracked *.sh has LF (git ls-files --eol shows i/lf w/lf)
```

### Story #2: Extensionless entrypoints forced to LF [core]

```
GIVEN .gitattributes
WHEN  `git check-attr eol -- scripts/skills-deploy` runs
THEN  it reports eol: lf
```

### Story #3: Binaries protected from EOL normalization [core]

```
GIVEN .gitattributes
WHEN  a *.png is committed
THEN  no EOL normalization is applied
```

## Architecture

A single `.gitattributes` at the repo root. Intended content sketch:

```
* text=auto eol=lf
scripts/skills-deploy        text eol=lf
scripts/skills-update-check  text eol=lf
*.png binary
*.jpg binary
*.ico binary
```

The two extensionless entrypoints get explicit `text eol=lf` lines because they
have no extension to match a `*.ext` pattern. On adoption, run a one-time
`git add --renormalize .` to apply the policy to already-tracked files (document
this in the implement step).

```
git checkout тФАтФАтЦ║ reads .gitattributes тФАтФАтЦ║ applies eol=lf to text files (*.sh, *.py, entrypoints)
                                     тФФтФАтФАтЦ║ leaves *.png / *.jpg / *.ico untouched (binary)
```

| Component | Repo | Change Type | Description |

### Data Flow

1. `git checkout` reads `.gitattributes`.
2. Text files are normalized to LF.
3. Binaries are left untouched.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| EOL policy | `* text=auto eol=lf` (force LF for all text) | per-extension allowlist | repo is bash-heavy with no CRLF-required files; force-LF is simplest correct |
