## Problem Statement

`/CJ_suggest` and `/CJ_improve-queue` hard-refuse off macOS (`[ "$(uname -s)" !=
"Darwin" ]` at `skills/CJ_suggest/scripts/suggest.sh:7` and
`skills/CJ_improve-queue/scripts/improve_queue.sh:76`) and use BSD-only `date -j
-f` (`suggest.sh:284`, `improve_queue.sh:676`). On WSL2 (uname=Linux) they refuse
outright; even past the gate, GNU date has no `-j`. A WSL2 / Git Bash user cannot
rank TODOs or run the improvement queue at all, and `/CJ_suggest`'s refusal
cascades into `/CJ_goal_todo_fix` ranking.

## Mental Model

Two edits per script — a portable date helper plus a widened OS allowlist. The
helper is a feature-probe (`date --version`), not a uname branch, so Git Bash
(GNU coreutils) routes to the GNU path correctly.

```
skill start ─► OS allowlist gate ─► date_to_epoch(updated) ─► age_days ─► ranking
              (Darwin|Linux|         (GNU date -d  /
               MINGW*|MSYS*;          BSD date -j -f)
               refuse unknown)
```

## Architecture

Inline a `date_to_epoch()` helper into BOTH scripts (feature-probe, NOT
uname-based):

```
date_to_epoch() {  # $1 = date string; $2 = BSD strptime format (ignored by GNU)
  if date --version >/dev/null 2>&1; then date -d "$1" +%s 2>/dev/null   # GNU (Linux/WSL2/Git Bash)
  else date -j -f "$2" "$1" +%s 2>/dev/null; fi                          # BSD (macOS)
}
```

Replace the `date -j -f` calls (`suggest.sh:284`, `improve_queue.sh:676`) with
`date_to_epoch`. Widen the gate from `[ "$(uname -s)" != "Darwin" ]` to a `case`
allowlist (`Darwin|Linux|MINGW*|MSYS*`) with a loud refuse for unknown. Inlined
(NOT shared in `scripts/lib.sh`) because deployed skill scripts under
`~/.claude/skills/` cannot source the repo's `scripts/`. Note: the bare-date form
diverges harmlessly (GNU fills local midnight, BSD fills current wall-clock) —
within a day, below the `age_days = epoch/86400` resolution.

```
skill start
   │
   ▼
OS allowlist gate  (case "$(uname -s)" in Darwin|Linux|MINGW*|MSYS*) proceed ;; *) refuse loudly ;; esac)
   │
   ▼
date_to_epoch(updated)   (probe: date --version → GNU date -d  |  else BSD date -j -f)
   │
   ▼
age_days = epoch / 86400
   │
   ▼
ranking / score
```

3. date_to_epoch parses `updated:`
4. age_days computed
5. ranking/score proceeds

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| date helper location | inline in each skill script | shared scripts/lib.sh | deployed skill scripts can't source repo scripts/ at runtime |
| OS detection | feature-probe `date --version` | uname branch | probe is robust: Git Bash ships GNU coreutils so it selects the GNU branch correctly |
