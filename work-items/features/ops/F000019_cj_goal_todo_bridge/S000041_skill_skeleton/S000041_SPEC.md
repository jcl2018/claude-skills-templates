## Problem Statement

Build the /CJ_goal v1 skill end-to-end per the /office-hours design doc.
Mostly orchestration over shipped pieces; genuinely new code is the pre-flight
gate stack, TODOS.md parser, T-task scaffold writes, direct-dispatch chain,
per-session skip-list mechanic, hash-verify TODOS.md DONE-mark, and telemetry.

User persona: workbench owner who routinely drops TODOs into TODOS.md and
wants a single-keystroke path to close them via the existing pipeline.

## Mental Model

```
TODOS.md row → /CJ_goal preflight → T-task scaffold → /CJ_personal-pipeline
   → /ship Gate #2 → /land-and-deploy → TODOS.md DONE-mark → telemetry line
```

Halt-on-red at every gate; preflight halts skip-and-continue under `/loop`;
substantive halts stop the loop entirely.

## Acceptance Criteria

### Story #1: /CJ_goal exists [core]

```
GIVEN claude-skills-templates with skills-deploy install run
WHEN  user invokes /CJ_goal from Claude Code
THEN  the skill loads, preamble runs, no error
```

### Story #2: No-args picks /CJ_suggest top-1 [core]

```
GIVEN TODOS.md has actionable rows and /CJ_suggest returns top-1
WHEN  user invokes /CJ_goal with no args
THEN  scripts/goal.sh resolves the top-1 heading and proceeds to preflight
```

### Story #3: T-ID dispatches existing tracker [core]

```
GIVEN T000022 tracker exists at work-items/tasks/skills/T000022_*/T000022_TRACKER.md
WHEN  user invokes /CJ_goal T000022
THEN  scripts/goal.sh sets EXISTING_WORK_ITEM_DIR, sets idempotent_skip flag, jumps to step 4 dispatch
```

### Story #4: Fragment match with multi-match AUQ [core]

```
GIVEN multiple TODO headings match "fragment"
WHEN  user invokes /CJ_goal "fragment"
THEN  AUQ surfaces with up to 4 matches; user picks; flow continues with chosen heading
```

### Story #5: Preflight halts [resilience]

```
GIVEN TODO heading has (P1, S) suffix
WHEN  user invokes /CJ_goal on it
THEN  exit code != 0; halt class halted_at_preflight; no work-items/ writes; under /loop, written to skip-list and next candidate tried
```

### Story #6: Scaffolded T-task is valid [core]

```
GIVEN preflight passes
WHEN  scripts/goal.sh writes TRACKER + test-plan
THEN  /CJ_personal-workflow check on the scaffolded dir returns clean
```

### Story #7: Dispatch chain works [core]

```
GIVEN valid scaffold
WHEN  /CJ_personal-pipeline --work-item-dir --suppress-final-gate runs to green
AND   /ship completes (Gate #2 approved by user)
AND   /land-and-deploy --suppress-readiness-gate runs to green
THEN  PR is merged; deployment verified; end_state=green
```

### Story #8: Hash-verify protects TODOS.md [resilience]

```
GIVEN PRE_HASH captured at Step 0
WHEN  Step 5 fires and NOW_HASH differs
THEN  halt_at_todos_md; no write; user notified PR is merged but TODOS.md needs manual reconcile
```

### Story #9: Telemetry written [observability]

```
GIVEN /CJ_goal completes any end_state
WHEN  Step 6 fires
THEN  one JSONL line appended to ~/.gstack/analytics/CJ_goal.jsonl
```

### Story #10: Per-session skip-list [resilience]

```
GIVEN /loop /CJ_goal session with RUN_ID
WHEN  preflight halts on TODO X
THEN  X is appended to /tmp/cj-goal-skip-${RUN_ID}.txt
AND   next iteration's no-args /CJ_suggest output is filtered via grep -vFxf to skip X
AND   when all candidates are skipped, halt_at_resolve terminates the loop cleanly
```

### Data Flow

1. User invokes `/CJ_goal [arg]`
2. SKILL.md loads, runs preamble, dispatches to `scripts/goal.sh "$@"`
3. goal.sh parses input (no-args → /CJ_suggest top-1; T-ID → tracker lookup; fragment → grep + AUQ)
4. goal.sh runs preflight gates sequentially on resolved TODO; halt on first fail
5. goal.sh scaffolds T-task (TRACKER + test-plan) under `work-items/tasks/{domain}/T{ID}_<slug>/`
6. goal.sh invokes `/CJ_personal-workflow check` for boundary verification
7. goal.sh dispatches `/CJ_personal-pipeline --work-item-dir ... --suppress-final-gate` (Skill or Agent — implementation chooses based on shipped infra)
8. On pipeline green, goal.sh invokes `/ship` (user gate)
9. On /ship success, goal.sh parses PR_NUM and invokes `/land-and-deploy --suppress-readiness-gate #PR_NUM`
10. On deploy green, goal.sh hash-verifies TODOS.md and atomically marks heading DONE
11. goal.sh appends one JSONL line to `~/.gstack/analytics/CJ_goal.jsonl`
12. Exit with end_state code

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Skill home | Top-level /CJ_goal | Extend /CJ_run with Branch(h) | Separation of concerns; `/loop /CJ_goal` is cleaner primitive |
| Hand-off | Direct chain /CJ_personal-pipeline + /ship + /land-and-deploy | Via /CJ_run | /CJ_run Branch(f) rejects type: task; direct chain matches Branch(f)'s own error-message guidance |
| Preflight scope | Refuse P1 OR size L/XL | --force-design-skip flag | Size is load-bearing risk control; explicit refusal keeps gate aggressive |
| Skip-list scope | Per-session at /tmp/cj-goal-skip-${RUN_ID}.txt | Persistent across sessions | Per-session lets next /loop retry previously-skipped TODOs after user fixes metadata |
| TODOS.md write | Hash-verify + atomic awk + mv | sed -i (in-place) | macOS sed requires '' arg; awk + mv is portable + race-aware |
| ID picker | Verbatim copy-paste from /CJ_scaffold-work-item Step 5 | Extract to shared script | v1 simplicity; v1.1 refactor noted in Open Q |
| Eval coverage | Preflight halts only | Green-path eval | Green-path blows $0.50/case budget; matches /CJ_personal-pipeline precedent |
| Domain inference | Silent default to ops on miss | AUQ for top-2 | v1 simplicity; v1.1 adds AUQ if telemetry shows mis-routing |
