## Problem Statement

The operator (chjiang, running this workbench AND consumer repos like the
portfolio repo) has no single verb to ask "do this repo's docs follow its doc
contract, and do its tests follow its test contract?" — today the doc
contract's enforcement is scattered across CI checks that fire post-PR, the
test contract is a 66-row workbench-only registry no consumer repo can carry,
and a cj_goal run never SHOWS the operator either answer before spending ship
budget. The general half of the doc contract cannot be delivered as a seed
because custom rows are fused into the same file.

## Mental Model

Two symmetric contracts, each split into two tiers, each fronted by one skill:

- **Tier 1 (general, skill-delivered seed):** `spec/doc-spec.md` ==
  `doc-spec.sh --seed` byte-for-byte; `spec/test-spec.md` ==
  `test-spec.sh --seed`. Portable; identical in every repo.
- **Tier 2 (custom, optional overlay):** `spec/doc-spec-custom.md` /
  `spec/test-spec-custom.md` — same fenced-yaml grammar; the parser merges
  general + overlay internally so consumers see ONE registry. Overlay-absent
  repos: nothing to merge, no finding.
- **Skills:** `/CJ_doc_audit` / `/CJ_test_audit` — seed-deliver if missing,
  validate the merge, run deterministic conformance, layer agent-judged
  alignment, emit a findings report. Standalone in any repo; INLINE (read,
  not spawned) when executed inside QA (the nested-subagent wall).
- **Pipeline moment:** QA Step 8.6 refreshes both custom overlays FIRST, then
  runs both audits; findings ride a GREEN RESULT; the orchestrator checkpoint
  AUQ owns the Continue/Halt decision.
- **Demolition:** the old one-tier test-pipeline machinery is REMOVED in the
  same PR — its deterministic coverage engine is PORTED into `test-spec.sh`,
  not dropped.

### Data Flow

<!-- How does data move through the system for the primary use case? -->

1. Operator (or QA Step 8.6c/d) invokes an audit skill in some repo.
2. The skill resolves its engine (repo-local `scripts/{doc,test}-spec.sh`, else `~/.claude/_cj-shared/scripts/`), checks `spec/<contract>.md` exists; if missing, creates `spec/` and writes `--seed` output (`seeded: yes`).
3. The engine parses general + overlay-if-present into ONE merged registry (`--validate`: schema, closed enums, duplicate guard).
4. Deterministic pass: doc conformance (declared-exists / orphans / root-declared / no work-item IDs / front_table / views-in-sync) or test coverage (`--check-coverage`: forward anchor-grep per unit, reverse sweep of live surfaces, floor — units-gated).
5. Agent-judged pass: per-doc `requirement:` verdicts (T000038 shape) / rules `suite-green` + `new-code-tested`.
6. The skill emits `DOC_AUDIT:`/`TEST_AUDIT:` + `FINDINGS=` + per-finding lines; QA lifts the four step reports into the RESULT's `AUDITS=` field + AUDIT_FINDINGS block.
7. The orchestrator surfaces the checkpoint AUQ; Continue proceeds to doc-sync, Halt journals `[qa-audit-declined]` and ends `halted_at_qa_audit`; `--quiet` auto-decides.

## Tradeoffs

<!-- Design decisions made in this spec, with rejected alternatives and rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Parser depth (D6) | Full parser parity — ported deterministic coverage engine | Thin parser (schema-only) + agent-judged alignment; pure prose specs | Agent verdicts vary run-to-run; the demolition must not net-reduce determinism; a malformed spec must hard-fail, not silently degrade |
| Old machinery (D5.2) | Demolish + sweep in the same PR | Keep test-pipeline as a compatibility layer | One format fully enforced beats two formats half enforced; the 66-row registry is undeliverable to consumers |
| Custom rows (D5.3) | Separate overlay FILES, merged inside the parser | Keep `section: custom` rows in one file; rewrite all call sites | Seed byte-identity requires a general-only file; internal merge keeps every consumer call site unchanged |
| front_table collision | Promote into the portable seed schema (optional field) | Keep it workbench-local (duplicate row / fork the seed) | Byte-identity + duplicate-path-is-an-error leave no legal home for workbench-only fields on common docs |
| Unit row schema | Old row shape kept VERBATIM in the overlay | Redesign the unit schema for the new format | Literal row copy (no mapping, no synthesized values); coverage engine ports with extraction grammar + reverse-sweep id conventions intact |
| QA verdict semantics | Audit findings ride a GREEN RESULT (`AUDITS=` field); checkpoint owns the decision | Audit findings flip QA red | A red RESULT halts at the existing qa gate — the operator would never see the findings prompt; waivers stay auditable via journal lines |
| Checkpoint cadence (D5.4) | ALWAYS prompt in interactive runs | Prompt only on findings | Operator explicitly asked to SEE the four steps' outcomes; green-skip hides the new surface; --quiet covers cron |
| Absent-registry contract | Distinct `REGISTRY=absent` + exit 0 | Old ambiguous exit-1 halt text | Callers (Check 24, the skills) classify skip-vs-findings without parsing prose |
| Reverse-floor scope | Applied ONLY when `units:` rows exist; named note otherwise | Always-on floor | A seeded-general-only consumer repo would drown in misleading extraction-grammar findings |
