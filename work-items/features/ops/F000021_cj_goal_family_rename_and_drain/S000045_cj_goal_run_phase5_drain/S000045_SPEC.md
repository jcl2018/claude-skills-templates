## Problem Statement

Post-S000044 rename, /CJ_goal_run still calls a feature "complete" when the feature PR merges, even though it routinely manufactures TODO debt in the same run. Operator must invoke /loop /CJ_goal manually afterward. This story adds Phase 5 — a post-/land-and-deploy drain phase that closes the new-debt loop in the same pipeline invocation. Persona: workbench operator running feature pipelines (jcl2018), plus AI agents running /CJ_goal_run in autonomous flows.

## Mental Model

```
Phase 1: /autoplan                 [GATE #1: final-approval AUQ]
Phase 2: /CJ_personal-pipeline     [Agent subagent, --suppress-final-gate]
Phase 3: /ship                     [GATE #2: diff-review AUQ — feature PR]
Phase 4: /land-and-deploy          [auto on green; halt on red]
Phase 5: TODO drain phase          [NEW]
  ├── Diff TODOS.md additions in merged PR
  ├── Count new ^### headings → N
  ├── If N == 0: skip phase, emit green with new_todos_count: 0
  ├── If N > 0: AUQ "Drain N new TODOs?"
  │              [recommended yes if N ≤ cap=5, no otherwise]
  └── On yes: per-TODO loop (cap=5) → /CJ_goal_todo_fix per child
              On red: STOP, drained_partial
              On all green: drained_complete
Final: telemetry line with new_todos_count, drained_count, drained_pr_urls
```

## Acceptance Criteria

### Story #1: TODOS.md diff parsing [core]

```
GIVEN /CJ_goal_run has finished Phase 4 (/land-and-deploy) with end_state=green
WHEN  Phase 5 runs `git diff <parent>..HEAD -- TODOS.md | grep -cE '^\+### '`
THEN  it captures the count of newly-added top-level TODO headings as N
AND   N is an integer >= 0
```

### Story #2: N==0 silent skip [core]

```
GIVEN Phase 5 has computed N=0
WHEN  Phase 5 evaluates the threshold
THEN  it skips drain silently
AND   emits end_state=green with new_todos_count=0 in telemetry
AND   does NOT surface an AUQ
```

### Story #3: N>0 AUQ with recommendation [core]

```
GIVEN Phase 5 has computed N=3
WHEN  Phase 5 evaluates the threshold
THEN  it AUQs "Drain 3 new TODOs?"
AND   recommendation is "yes" because 3 ≤ cap=5
AND   options are [Yes, drain | No, skip drain | Show list, then decide]

GIVEN Phase 5 has computed N=7
WHEN  Phase 5 evaluates the threshold
THEN  it AUQs "Drain 7 new TODOs?"
AND   recommendation is "no" because 7 > cap=5
AND   options include "Drain top 5, defer rest"
```

### Story #4: Per-TODO drain loop [core]

```
GIVEN operator chose "yes, drain"
WHEN  Phase 5 enters the drain loop
THEN  for each of the N TODOs (up to cap=5):
        invoke /CJ_goal_todo_fix <heading-or-T-ID>
        which runs: preflight → scaffold T-task → /CJ_personal-pipeline → /ship Gate #2 → /land-and-deploy → TODOS.md DONE-mark
AND   each child invocation is sequential (not parallel)
```

### Story #5: Halt-on-red [resilience]

```
GIVEN drain loop is mid-iteration with 2 children green
WHEN  the 3rd child halts red (e.g., /ship Gate #2 rejected)
THEN  the loop STOPS immediately
AND   end_state=drained_partial in telemetry
AND   drained_pr_urls array contains the 2 successful PRs
AND   drained_count=2 in telemetry
```

### Story #6: --no-drain flag [usability]

```
GIVEN operator invokes `/CJ_goal_run <design> --no-drain`
WHEN  the pipeline completes Phases 1-4
THEN  Phase 5 is skipped entirely
AND   end_state=green
AND   telemetry includes `no_drain_flag: true`
```

### Story #7: Telemetry schema [observability]

```
GIVEN Phase 5 has completed with drained_count=3
WHEN  the telemetry line is written
THEN  it includes: {end_state, mode, run_id, new_todos_count: 3, drained_count: 3, drained_pr_urls: ["url1", "url2", "url3"], no_drain_flag: false, ts}
```

### Data Flow

1. Operator invokes `/CJ_goal_run <design-doc>`.
2. Phases 1-4 run as today (autoplan, pipeline, ship, deploy).
3. After /land-and-deploy emits green, Phase 5 fires.
4. Phase 5 reads the merged PR's diff: `git diff <parent>..HEAD -- TODOS.md`. `<parent>` resolved from gh PR view.
5. Phase 5 greps `^\+### ` → captures new TODO heading lines.
6. Phase 5 computes N = line count.
7. If N == 0: emit green telemetry with `new_todos_count: 0`; exit.
8. If `--no-drain` flag set: emit green with `no_drain_flag: true`; exit.
9. Otherwise: AUQ "Drain N new TODOs?" with options Yes/No/Show-list.
10. On Yes: enter drain loop (cap=5).
11. Per iteration: invoke `/CJ_goal_todo_fix <heading>`. Reads shared lockfile (`/tmp/cj-goal-active-headings-$(date +%Y%m%d).txt`) before scaffolding; skips heading if already locked.
12. Halt-on-red after each child: stop loop, emit `drained_partial`, capture progress.
13. All green: emit `drained_complete` with `drained_pr_urls` array.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Drain cap | Hardcoded 5 | Config file / `--max-drain` flag | Greppable in SKILL.md (P5 explicit). Phase 5 scope is naturally smaller than backlog. Operator-tunable via `--no-drain` covers escape valve. |
| Per-child gate | Sequential /ship Gate #2 | Batched gate after all 5 | Targeted review > 5-at-once review. Same as /CJ_run Branch (b) today. |
| AUQ default for N > 5 | Recommend "no" | Recommend "yes" with cap=5 truncation | Partial drain isn't completeness; operator should explicitly choose. |
| N == 0 behavior | Silent skip + green | Always-AUQ "no debt found" | Quiet happy path > noisy validation. Telemetry distinguishes via `new_todos_count: 0`. |
| Telemetry fields | New: `new_todos_count`, `drained_count`, `drained_pr_urls`, `no_drain_flag` | Backfill via prior-run inspection | Forward-looking fields; backfill is lossy and the trip-wire only reads prior-5. |
| `--no-drain` semantics | Per-run flag (not persisted) | Sticky setting / persistent state | Per-run is explicit; persistent state surprises operators on re-invocation. |
