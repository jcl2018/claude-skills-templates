## Problem Statement

After S000046, /CJ_goal_todo_fix natively drains backlog — but Phase 3 still emits an interactive summary AUQ. Wiring this into a `/schedule` cron creates noise in cron output and operator notifications. The user wants a `--quiet` mode that suppresses the summary AUQ (writes to journal instead) while preserving /ship Gate #2 (the autonomy ceiling). Persona: workbench operator running scheduled drains via `/schedule` (e.g., daily 9am cron preparing 3 PRs for the operator's review window).

## Mental Model

```
Without --quiet (interactive):
  Phase 3: AUQ "Drained N of M attempted. Continue (y/n)?"
           → operator sees + answers

With --quiet (scheduled):
  Phase 3: [scheduled-drain-summary] N drained, M attempted
           written to work-item tracker journal + telemetry
           → no AUQ surfaces; cron output stays clean
  /ship Gate #2 still surfaces per child — operator reviews on their cadence
  Halt-on-red entries still written
```

## Acceptance Criteria

### Story #1: --quiet suppresses summary AUQ [core]

```
GIVEN /CJ_goal_todo_fix is run with `--quiet --max-drain 3`
WHEN  Phase 3 fires
THEN  no interactive AUQ appears
AND   stdout is minimal (perhaps just a one-line "[quiet drain complete]")
```

### Story #2: --quiet writes journal entry [core]

```
GIVEN /CJ_goal_todo_fix ran with `--quiet` and drained 2 of 3 attempted
WHEN  Phase 3 completes
THEN  `[scheduled-drain-summary]` line written to a journal
AND   line content: "[scheduled-drain-summary] drained=2, attempted=3, halt=red-on-3rd, PRs=[u1, u2]"
```

### Story #3: /ship Gate #2 unaffected by --quiet [core]

```
GIVEN /CJ_goal_todo_fix is run with `--quiet`
WHEN  any child invokes /ship
THEN  /ship Gate #2 (diff-review AUQ) STILL surfaces to the operator
AND   the operator must approve before each child PR opens
```

### Story #4: scheduled_run telemetry field [observability]

```
GIVEN /CJ_goal_todo_fix ran with `--quiet`
WHEN  telemetry is written
THEN  the line includes `"scheduled_run": true`

GIVEN /CJ_goal_todo_fix ran without `--quiet`
WHEN  telemetry is written
THEN  the line includes `"scheduled_run": false`
```

### Story #5: Halt-on-red preserved with --quiet [resilience]

```
GIVEN /CJ_goal_todo_fix is running with `--quiet`
WHEN  any child halts red
THEN  halt entry is written to the work-item tracker journal as today
AND   `drained_partial` telemetry is written
AND   the loop STOPS (no continue-after-red even in quiet mode)
```

### Story #6: CLAUDE.md cron-pattern example [usability]

```
GIVEN the workbench CLAUDE.md
WHEN  operator searches for "/schedule" or "cron"
THEN  finds an example: `/schedule create "/CJ_goal_todo_fix --max-drain 3 --quiet" daily 9am`
AND   a one-paragraph explanation of expected behavior (PRs queue for review at cron cadence)
```

### Story #7: SKILL.md --quiet doc [usability]


## Architecture

```
                                       /CJ_goal_todo_fix --quiet --max-drain 3
                                                 │
                                                 ▼
                                       Phase 1: enumerate (same as S000046)
                                                 │
                                                 ▼
                                       Phase 2: drain loop (same)
                                              ┌─ each child
                                              │   /ship Gate #2 STILL surfaces
                                              │   (operator reviews per child)
                                              ▼
                                       Phase 3: summary
                                                 │
                                          if $QUIET:
                                                 │
                                                 ▼
                                       ┌────────────────────────────────────┐
                                       │ Write [scheduled-drain-summary]    │
                                       │ to drained-children's tracker      │
                                       │ journals + telemetry line          │
                                       │ NO interactive AUQ                 │
                                       └────────────────────────────────────┘
                                          else:
                                                 │
                                                 ▼
                                       AUQ "Drained N of M. Continue?" (today's behavior)

   Telemetry line (with --quiet):
     {run_id, end_state, mode, scheduled_run: true, drained_count, ...}

   Cron pattern (documented):
     /schedule create "/CJ_goal_todo_fix --max-drain 3 --quiet" daily 9am
       → at 9am: dispatches the skill
       → Phase 2 drains up to 3 (with /ship Gate #2 per child)
       → Phase 3 logs to journal (no AUQ)
       → operator reviews PRs at their cadence via `gh pr list`
```

### Data Flow

1. `/schedule` (or operator) invokes `/CJ_goal_todo_fix --max-drain 3 --quiet`.
2. Phases 1-2 run as S000046 (enumerate, drain loop with /ship Gate #2 per child).
3. After cap or halt-on-red: Phase 3 fires.
4. Phase 3 reads `$QUIET`:
   - If set: write `[scheduled-drain-summary]` journal entry + telemetry with `scheduled_run: true`; exit.
   - If unset: AUQ "Drained N of M. Continue?" as today's behavior + telemetry with `scheduled_run: false`.
5. Operator's review queue: `gh pr list --author @me --state open` shows the drained PRs whenever they next check.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Scope of suppression | Only Phase 3 summary AUQ | All AUQs including /ship Gate #2 | /ship Gate #2 is the autonomy ceiling. F000021 explicit constraint. |
| Journal entry location | Drained children's tracker journals + dedicated session log | Single skill-level session log | Per-tracker entries are searchable from work-item context; session log is auditable. Both. |
| Telemetry attribution field | `scheduled_run: bool` | Inferred from `--quiet` presence in args (already captured) | Explicit bool is more grep-friendly; redundant with args but cheap. |
| /schedule integration shape | Doc-only example | Hard binding via /schedule plugin call | Smaller blast radius. /schedule binding is a separate concern; documenting the pattern is sufficient for v4.x. |
| When `--quiet` + `nothing_to_drain` | Silent (no stdout, write journal) | Print one-line "no work today" | Cron-friendly: empty stdout when there's nothing to do. Journal entry preserves the fact. |
