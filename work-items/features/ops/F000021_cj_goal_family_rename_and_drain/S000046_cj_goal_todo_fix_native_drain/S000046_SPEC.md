## Problem Statement

Post-rename, /CJ_goal_todo_fix retains the v1.1 single-shot shape (fix one TODO, exit). Backlog drain requires `/loop /CJ_goal_todo_fix` and operator babysitting. The user wants `/CJ_goal_todo_fix` (no args) to drain up to N TODOs natively, no /loop needed, cron-eligible. Plus S000045's Phase 5 drain and this story's Phase 2 drain share ~150 LOC of per-TODO inner loop — DRY violation if both inline-duplicate. Persona: workbench operator (jcl2018) running backlog drain at end-of-week + automated cron flows.

## Mental Model

```
Phase 1: Enumerate easy-fix TODOs
  ├── Read TODOS.md (delegate to /CJ_suggest --for-skill /CJ_goal_todo_fix)
  ├── Filter: passes preflight (inherited from /CJ_goal v1.1):
  │            P2/P3, size S, body ≥50, not sensitive-surface, not design-keyword,
  │            not already-tracked, not in this-session skip-list
  ├── Returns ordered list
  └── If empty: print "No easy-fix TODOs available." → end_state=nothing_to_drain
Phase 2: Drain loop (cap = --max-drain N, default 10)
  ├── For each TODO in list, up to cap:
  │     ┌──────────────────────────────────────┐
  │     │ scripts/drain-one-todo.sh <heading>  │  ← SHARED HELPER
  │     │   preflight → scaffold T-task →      │
  │     │   /CJ_personal-pipeline → /ship      │
  │     │   Gate #2 → /land-and-deploy →       │
  │     │   TODOS.md DONE-mark → PR URL        │
  │     └──────────────────────────────────────┘
  │     Halt-on-red → STOP, drained_partial
  └── Cap reached → STOP cleanly, drained_partial (or _complete if entire list done)
Phase 3: Summary + telemetry
  ├── Print: "Drained N of M attempted. PRs: [...]. Remaining easy-fix: K."
  └── Append to ~/.gstack/analytics/CJ_goal_todo_fix.jsonl
```

```
Concurrency safety:
  Both /CJ_goal_run Phase 5 (S000045) and /CJ_goal_todo_fix Phase 2 (this story)
  check the shared lockfile at /tmp/cj-goal-active-headings-$(date +%Y%m%d).txt
  before scaffolding. Loser of race skips that TODO this session.
  Per-day TTL → no GC; file replaces itself daily.
```

## Acceptance Criteria

### Story #1: Default drain mode [core]

```
GIVEN TODOS.md has 7 easy-fix rows passing preflight
WHEN  operator invokes `/CJ_goal_todo_fix` (no args)
THEN  Phase 1 returns the 7 rows
AND   Phase 2 drains up to cap=10 (so all 7)
AND   Phase 3 prints "Drained 7 of 7 attempted. PRs: [u1...u7]. Remaining easy-fix: 0."
AND   end_state=green in telemetry
```

### Story #2: --max-drain flag [core]

```
GIVEN TODOS.md has 12 easy-fix rows
WHEN  operator invokes `/CJ_goal_todo_fix --max-drain 3`
THEN  Phase 2 drains 3 rows
AND   end_state=drained_partial in telemetry
AND   summary prints "Drained 3 of 3 attempted. PRs: [...]. Remaining easy-fix: 9."
```

### Story #3: Single-TODO mode preserved [core]

```
GIVEN TODOS.md has a row at heading "T000099 — fix typo"
WHEN  operator invokes `/CJ_goal_todo_fix T000099`
THEN  Phase 1 returns just that one row
AND   Phase 2 drains the one row
AND   behavior matches /CJ_goal v1.1 exactly (smoke + E2E behavior identical)
```

### Story #4: Fuzzy-match single-TODO [core]

```
GIVEN TODOS.md has a row at heading "T000099 — fix typo in README"
WHEN  operator invokes `/CJ_goal_todo_fix "fix typo"`
THEN  Phase 1 fuzzy-matches and returns the row
AND   Phase 2 drains it
```

### Story #5: --dry-run flag [core]

```
GIVEN TODOS.md has 5 easy-fix rows
WHEN  operator invokes `/CJ_goal_todo_fix --dry-run`
THEN  Phase 1 enumerates the 5 rows
AND   Phase 2 is skipped (no scaffolds, no /ship, no /land-and-deploy)
AND   Phase 3 prints "Would drain 5: [T1, T2, ...]. Run without --dry-run to execute."
AND   no writes occur to disk
```

### Story #6: drain-one-todo.sh extraction [core]

```
GIVEN /CJ_goal_run Phase 5 (S000045) and /CJ_goal_todo_fix Phase 2 both need to drain a single TODO
WHEN  this story ships
THEN  `scripts/drain-one-todo.sh` exists with the shared inner loop
AND   `skills/CJ_goal_run/run.md` Phase 5 calls `drain-one-todo.sh` (no inline duplicate)
AND   `skills/CJ_goal_todo_fix/scripts/todo_fix.sh` Phase 2 calls `drain-one-todo.sh`
```

### Story #7: Shared lockfile concurrency [resilience]

```
GIVEN /CJ_goal_run Phase 5 has acquired lockfile entry for heading H
WHEN  /CJ_goal_todo_fix Phase 2 (running concurrently) processes the same H
THEN  the second skill detects the lock and skips H this session
AND   logs "[lock-skip] H locked by RUN_ID=..." to its own session log
AND   continues with the next eligible TODO
```

### Story #8: Phase 3 summary line [observability]

```
GIVEN Phase 2 has drained 4 TODOs (3 green, 1 halt-red)
WHEN  Phase 3 prints the summary
THEN  output includes: "Drained 3 of 4 attempted. PRs: [u1, u2, u3]. Remaining easy-fix: K."
AND   "Halted on T0000NN: <reason>" line precedes summary if applicable
```

### Story #9: nothing_to_drain end_state [observability]

```
GIVEN TODOS.md has 0 easy-fix rows passing preflight (or all already locked)
WHEN  Phase 1 returns empty
THEN  skill prints "No easy-fix TODOs available."
AND   end_state=nothing_to_drain in telemetry
AND   exit code is 0 (not an error — "no work today")
```

### Story #10: Telemetry path migration [observability]

```
GIVEN /CJ_goal.jsonl has 4 invocation lines (pre-rename)
AND   /CJ_goal_todo_fix.jsonl has 2 invocation lines (post-rename)
WHEN  sunset-trip-wire reads prior 5 invocations
THEN  it merges both paths and sees 5 of the 6
AND   halt-count calculation is unaffected by the rename
```

### Data Flow

1. Operator invokes `/CJ_goal_todo_fix` (no args; drain mode).
2. Phase 1: delegate to `/CJ_suggest --for-skill /CJ_goal_todo_fix --limit 20`; receive preflight-filtered, ranked list.
3. If list empty → print "No easy-fix TODOs available." + telemetry `nothing_to_drain` + exit 0.
4. Phase 2: for each row up to `--max-drain` (default 10):
   - Invoke `scripts/drain-one-todo.sh <heading>`.
   - `drain-one-todo.sh` acquires lockfile entry, runs preflight, scaffolds T-task, invokes /CJ_personal-pipeline, invokes /ship (Gate #2 AUQ surfaces per child), invokes /land-and-deploy, marks DONE.
   - On green: capture PR URL, continue.
   - On red: stop loop, capture progress, emit `drained_partial`.
5. Phase 3: print summary line + write telemetry line.
6. Telemetry write: `~/.gstack/analytics/CJ_goal_todo_fix.jsonl`. Future read paths fallback-merge with `CJ_goal.jsonl` for sunset-trip-wire history.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Default cap | 10 | 5 | TODOS.md typically has 5-15 easy-fix; 10 clears most of backlog. /CJ_goal_run cap=5 is smaller because Phase 5 scope is naturally smaller. |
| Default mode | drain (no args) | preserve single-shot default (`/CJ_goal_todo_fix` shows top-1) | User's explicit pivot intent. Single-TODO mode kept via positional arg form. |
| Helper extraction order | This story extracts canonically; S000045 ships Phase 5 inline first, then refactors here | S000045 extracts first | S000045 has urgency (it's the user-facing v4.1.0 win); this story (v4.2.0) is the right place for the helper. |
| Lockfile granularity | Per-day TTL via filename date | Persistent lockfile with manual cleanup | Self-cleaning eliminates GC; per-day is long enough to catch real races. |
| `nothing_to_drain` exit code | 0 (success — "no work today") | non-zero (error) | Cron-friendly: zero exit code = success even when nothing to do. Distinguishes via telemetry end_state. |
| /CJ_suggest delegation for Phase 1 | Yes (it's already preflight-aware as of v3.6.0) | Re-implement enumeration in /CJ_goal_todo_fix | DRY. /CJ_suggest is the right home for "rank TODOs". |
