## Problem Statement

The current workbench has two adjacent skills (`/CJ_run` feature-ship pipeline, `/CJ_goal` single-TODO bridge) named inconsistently. The pivot reshapes them as a `_goal_*` family. Renaming is the prerequisite mechanical step before semantic changes in S000045–S000047. The operator persona is the workbench dogfooder (jcl2018) plus any AI agent running these slash commands.

## Mental Model

Pure mechanical refactor with alias-preserved backward compatibility. Two `git mv` operations + reference cascade + thin SKILL.md alias files. Single PR; major version bump; v4.x retains aliases; v5.0.0 removes them.

```
Before:                            After (v4.0.0):
  skills/CJ_run/SKILL.md       →   skills/CJ_goal_run/SKILL.md (renamed)
  skills/CJ_goal/SKILL.md      →   skills/CJ_goal_todo_fix/SKILL.md (renamed)
                                   skills/CJ_run/SKILL.md (NEW thin alias)
                                   skills/CJ_goal/SKILL.md (NEW thin alias)
  skills-catalog.json: 2 entries → both entries updated
  rules/skill-routing.md: 12 refs → all updated to new names
  CLAUDE.md routing block       → updated
  ~/.gstack/analytics/CJ_run.jsonl → fallback-read; writes go to CJ_goal_run.jsonl
```

### Data Flow

1. Operator invokes `/CJ_run <design>` (muscle-memory).
2. Claude's slash-command router resolves to `skills/CJ_run/SKILL.md` (the new thin alias).
3. Alias SKILL.md prints "renamed to /CJ_goal_run; will be removed in v5.0.0".
4. Alias dispatches to `/CJ_goal_run` with the same args.
5. `skills/CJ_goal_run/SKILL.md` executes the (renamed-but-otherwise-unchanged) feature-ship pipeline.
6. Telemetry: writes to `~/.gstack/analytics/CJ_goal_run.jsonl`; sunset trip-wire fallback-reads `CJ_run.jsonl` too.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Batch PR 1 + PR 2 | Single chore PR (~800 LOC mechanical diff) | Two separate PRs in sequence | Batching avoids "PR 1 shipped, PR 2 didn't" intermediate state. Reviewable in one diff. Autoplan-gate confirmed (auto-approved per P3 pragmatic). |
| Alias delegation pattern | Thin SKILL.md wrapper that prints banner + delegates | Hard cutover at v4.0.0 (no aliases) | Protects operator muscle memory. Hard cutover would break every operator's first post-upgrade invocation. Soft cutover with deprecation period costs ~80 LOC of alias code. |
| Telemetry path migration | Fallback-read both old + new during v4.x; write new only | Migrate old file on first read; delete old | Migration tool is more code + risk. Fallback-read is 5 lines of additional logic per skill. |
| Version bump magnitude | Major (3.6.5 → 4.0.0) | Minor (3.7.0) | Slash-command surface is a public contract. Renaming = breaking by name; semver compliance requires major bump. |
| `goal.sh` → `todo_fix.sh` cosmetic rename | Included in this PR | Defer to S000046 | Internal consistency keeps the file name aligned with the skill name; tiny add to the diff; doing it later costs another `git mv` + reference cascade. |
