## Problem Statement

The workbench maintainer (the operator of this repo) cannot enroll
load-bearing test topics (`validator`, `full-suite`) into the three-layer
topic contract because Check 30 HARD-requires a local-hook+agentic test per
enrolled topic, and agentic tests are not buildable/runnable on this machine
(no local claude CLI; agentic proofs run on-demand or via another agent). The
result: the can't-rot layer verification covers exactly one topic
(`portability`), and 7 labeled topics sit unenrolled on the advisory matrix.
Additionally, `/CJ_test_audit` Stage 1 has claimed since F000082 to surface
`--check-topic-contract` / `--check-topic-docs` but never invokes either — an
inherited drift this story also fixes.

## Mental Model

The topic contract's coverage matrix per enrolled topic goes from four HARD
points to three HARD + one ADVISORY:

```
Coverage point                     Before (F000082)   After (this story)
--------------------------------   ----------------   ------------------
CI-push        (deterministic)     HARD FINDING       HARD FINDING
CI-nightly     (deterministic)     HARD FINDING       HARD FINDING
local-hook     (deterministic)     HARD FINDING       HARD FINDING
local-hook     (agentic)           HARD FINDING       ADVISORY note
front-door doc per row             HARD FINDING       HARD FINDING
```

The demotion is the agentic MODE point, never a layer. Enrollment grows from
`[portability]` to `[portability, validator, full-suite]` — the two new topics
covered entirely by rows describing surfaces that already run today.

### Data Flow

<!-- Primary use case: an operator (or CI) runs the topic-contract check. -->

1. `test-spec.sh --check-topic-contract` parses the merged registry: `topic_contracts:` enrollment + `categories:` rows (with `topic:` fields).
2. For each enrolled topic, `_run_topic_contract` checks the three deterministic coverage points (CI-push, CI-nightly, local-hook+deterministic) → each missing one emits `FINDING:` + increments `_TC_FINDINGS`.
3. The agentic point check runs NEXT but on the advisory path: missing → print `note: topic-contract — enrolled topic '<t>' has no local-hook+agentic test (advisory — agentic proofs run on-demand, not required)`; no counter increment.
4. Each topic-attributed row's front-door doc is checked (HARD, unchanged).
5. The tail line `topic contract: enrolled=N findings=M` prints; exit 1 iff findings>0.
6. `validate.sh` Check 30 invokes the engine; its green path echoes only the tail line (notes surface where the contract is READ — direct engine runs, audit reports).
7. `/CJ_test_audit` Stage 1 (NEW) invokes `--check-topic-contract` + `--check-topic-docs` verbatim; advisory notes ride the Stage-1 report; findings are prefixed `stage1/`.
8. `/CJ_test_run --topic validator|full-suite` selects the topic-attributed rows; free-tier commands execute; agentic rows (none for the new topics) would require `--e2e`.

## Tradeoffs

<!-- Design decisions made in this spec, with rejected alternatives and rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Demotion scope | Global advisory demotion for ALL enrolled topics (Approach C) | A: parallel `topic_contracts_deterministic:` list (session's recommendation); B: structured enrollment entries (`modes:`, `agentic: external`) | Operator's D4 informed reversal: agentic proofs are never a requirement in this environment; A/B preserve machinery for a distinction the operator does not want to maintain; C is one loop change, no new syntax, no parser change |
| Portability's agentic proof | Stays present, declared, runnable via `/CJ_test_run --topic portability --e2e`; no longer REQUIRED | Keep it enforced just for portability (Approach A) | Accepted trade named on the D4 option label; the drill's arm (b) pins the advisory note so the demotion itself can't rot |
| deploy-harness | Stays UNENROLLED, documented (overlay comment + TODOS note) | Claim windows-smoke as its CI-push row | windows-smoke is labeled `portability`; double-counting would be a dishonest row — honest coverage beats complete-looking coverage (same instinct as the F000081 decision that created the gap) |
| Consumer seeded prose | Accepted prose-vs-engine drift for EXISTING consumers (idempotent `seed-contracts` skips present seeds) | Force-reseed consumers | The engine, not the seeded prose, is what runs; only fresh seeds carry the new prose — documented in the Distribution Plan |
| Rewrite unit for the spec prose | The WHOLE `## The topic axis` section (incl. mechanization + Stage-2 sentences) | Minimal per-sentence patches | Scopes the byte-identical seed-mirror delta once instead of scattering it |
| CI-push rows for the new topics | Reuse the EXISTING `validate`/`suite` rows (already `topic:`-attributed) | Add duplicate CI-push rows | No new per-PR workload; same-command dual-row precedent (test-deploy/portability-deploy) covers the other layers only |
