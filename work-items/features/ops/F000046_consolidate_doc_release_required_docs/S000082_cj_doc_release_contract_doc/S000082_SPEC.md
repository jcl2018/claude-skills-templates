## Problem Statement

The workbench maintainer (and any agent running `/CJ_document-release`) has no single
canonical doc describing what the doc-release machinery requires. The contract is spread
across `cj-document-release.json` (machine config), three CLAUDE.md convention sections,
the per-doc `requirement:` strings in the tracked-doc manifest, and the per-skill
`doc_requirement` field in `skills-catalog.json`. A repo adopting the CJ_ family must
reconstruct the contract from five sites. This story produces one canonical root contract
doc (`CJ-DOC-RELEASE.md`) and makes `/CJ_repo-init` enforce its presence as a 4th
prerequisite — so "is this repo ready for doc-release?" becomes a single presence check.

## Mental Model

Two artifacts, two audiences, one contract:

```
   HUMAN / AGENT read                 MACHINE read
  ┌─────────────────────┐          ┌──────────────────────────┐
  │  CJ-DOC-RELEASE.md   │  explains │  cj-document-release.json │
  │  (NEW prose contract)│ ───────▶  │  (parsed config — UNCHANGED)│
  │  · wrapper flow      │          │  whitelist_patterns       │
  │  · halt-on-red       │          │  categories               │
  │  · schema reference  │          └──────────────────────────┘
  │  · registered-doc    │                    ▲
  │    audit             │                    │ parsed by
  │  · declaration index ├──┐                 │ cj-document-release-config.sh
  └─────────────────────┘  │ indexes          │ + validate.sh Check 16 (UNCHANGED)
            ▲              │                  │
            │ presence-    │   declaration sites (stay co-located):
            │ checked by   │   · skills-catalog.json  doc_requirement
            │              │   · CLAUDE.md tracked-doc  requirement:
  ┌─────────┴──────────┐   │
  │  /CJ_repo-init     │◀──┘ (4th prereq, mirrors docrel: present/missing/invalid + --fix seed)
  │  scripts/cj-repo-init.sh│
  └────────────────────┘
```

CARVE-OUT (load-bearing): the CLAUDE.md `### Tracked doc/ files manifest` block + its
per-entry `requirement:` strings + `### Reporting` + the `## Registered-doc requirements
audit` and `## cj-document-release.json convention` headings are consumed by runnable
parsers / SKILL.md anchors and MUST stay verbatim/in-place. The Step 6.7 awk keeps
reading CLAUDE.md — no `skills/CJ_document-release/SKILL.md` Step-6.7 edit.

## Acceptance Criteria

<!-- Given/When/Then. Each #n matches the Requirements # column. -->

### Story #1: Canonical contract doc exists [core]

```
GIVEN the repo root
WHEN  I open CJ-DOC-RELEASE.md
THEN  it documents the /CJ_document-release wrapper flow (halt-on-red, doc-only
      auto-commit whitelist gate), references the cj-document-release.json schema,
      describes the registered-doc audit (registered set, verdict taxonomy,
      surfacing, posture), and indexes where each requirement is declared
      (catalog doc_requirement + manifest requirement:).
```

### Story #2: CLAUDE.md points at the doc, CARVE-OUT preserved [usability]

```
GIVEN the three CLAUDE.md convention sections
WHEN  I read them after the change
THEN  their narrative prose points at CJ-DOC-RELEASE.md as the canonical read,
AND   `### Tracked doc/ files manifest`, the per-entry `requirement:` strings,
      `### Reporting`, and the `## Registered-doc requirements audit` +
      `## cj-document-release.json convention` headings are byte-for-byte unchanged
      and in their original positions.
```

### Story #3: /CJ_repo-init verifies + seeds the 4th prereq [integration]

```
GIVEN a scratch repo with CJ_ skills deployed but no CJ-DOC-RELEASE.md
WHEN  I run ./scripts/cj-repo-init.sh
THEN  the health table shows CJ-DOC-RELEASE.md as a missing prerequisite (REPO_GAP),
WHEN  I run ./scripts/cj-repo-init.sh --fix
THEN  a generic portable CJ-DOC-RELEASE.md starter is written and a re-run reports ok;
GIVEN a present-but-headingless CJ-DOC-RELEASE.md
WHEN  I run --fix
THEN  it reports invalid, prints a NOTE:, and does NOT overwrite the file.
```

### Story #4: All 8 mirror sites wired [core]

```
GIVEN scripts/cj-repo-init.sh
WHEN  I diff it against the docrel prereq
THEN  the new prereq appears in all 8 sites: NEED_DOCGUIDE decl, the
      DETECT_SOURCE==none branch, the per-skill trigger loop, a *_PATH var,
      verify_docguide(), seed_docguide() (portable heredoc, no workbench paths),
      the collect() stanza (add_row + REPO_GAP), and the --fix ladder.
```

### Story #5: Skill-facing surfaces updated 3→4 [usability]

```
GIVEN skills/CJ_repo-init/SKILL.md, skills-catalog.json, doc/ARCHITECTURE.md
WHEN  I read the prereq enumerations
THEN  each reads "4" prerequisites (cj-document-release.json, TODOS.md, work-items/,
      CJ-DOC-RELEASE.md), the SKILL.md frontmatter description + Overview + health
      table agree, the catalog description mirrors the frontmatter, and
      skills/CJ_repo-init/USAGE.md last-updated is bumped (Check 14 satisfied).
```

### Story #6: Check 17 allowlist accepts the root doc [core]

```
GIVEN CLAUDE.md `### Tracked root docs allowlist`
WHEN  validate.sh Check 17 runs
THEN  CJ-DOC-RELEASE.md is allowlisted (a `- path:` / `  reason:` pair, with no
      `#`-leading line introduced into the block) and Check 17 passes (no orphan).
```

### Story #7: CARVE-OUT regression guard [resilience]

```
GIVEN the slimmed CLAUDE.md
WHEN  the Step 6.7 awk (from skills/CJ_document-release/SKILL.md) parses CLAUDE.md's
      `### Tracked doc/ files manifest`
THEN  it yields exactly 3 tracked-doc/ entries (PHILOSOPHY.md, ARCHITECTURE.md,
      WORKFLOWS.md), proving the CARVE-OUT held and no verdict silently vanished.
```

### Story #8: /CJ_document-release canonical-home pointer [usability]

```
GIVEN skills/CJ_document-release/SKILL.md
WHEN  the optional pointer is added
THEN  it is a thin prose line naming CJ-DOC-RELEASE.md as the canonical convention
      home, with NO change to the Step 6.7 awk or its CLAUDE.md anchors; USAGE.md is
      bumped only if the SKILL text actually changed.
```

### Story #9: skill-routing.md prereq enumeration [integration]

```
GIVEN rules/skill-routing.md
WHEN  I check whether it enumerates the /CJ_repo-init prerequisites
THEN  if it does, CJ-DOC-RELEASE.md is added to that enumeration; if it does not,
      no change is made (verify-first).
```

### Data Flow

The 8 mirror sites in `scripts/cj-repo-init.sh` for the new `docguide` prereq (mirror
the existing `docrel` prereq verbatim in shape):

1. `NEED_DOCGUIDE` declaration (alongside `NEED_DOCREL` etc.).
2. The `DETECT_SOURCE==none` branch — include the new prereq when no skill source is detected.
3. The per-skill trigger loop — reuse the `CJ_document-release` / `CJ_goal_*` skill set that already triggers `docrel`.
4. A `*_PATH` var — `DOCGUIDE_PATH="$REPO_ROOT/CJ-DOC-RELEASE.md"` (or equivalent).
5. `verify_docguide()` — presence + required-headings check → `ok` | `missing` | `invalid` (headings set: H1 title + a `## ` schema-reference heading + the registered-doc section heading).
6. `seed_docguide()` — portable heredoc starter, NO workbench-specific paths (generic, like `seed_docrel`).
7. The `collect()` stanza — `add_row` for the health table + `REPO_GAP` accounting.
8. The separate `--fix` ladder — invoke `seed_docguide` on `missing`; on `invalid` print a `NOTE:` and do NOT overwrite.

Verification flow: `cj-repo-init.sh` (default) → `collect()` runs `verify_docguide` →
`add_row` to the health table + bump `REPO_GAP` if not `ok` → exit code reflects gap count.
`--fix` → for each gap, run the matching seed fn (`seed_docguide` on `missing`; `invalid`
is a no-overwrite `NOTE:`).

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Doc vs single-file | Approach A — doc + adjacent JSON | B (embed JSON in one file) | B rewrites `cj-document-release-config.sh` + Check 16, migrates the JSON, and re-tests the whole F000037 path — too much risk for "one file instead of two." |
| Doc role | Documents + indexes declarations | C (JSON source-of-truth, doc generated) | C makes the doc derived, adds a generator + drift-check surface, and pulls declarations out of their co-located homes (violates Premise 1). |
| CLAUDE.md treatment | Slim narrative prose only; preserve CARVE-OUT | Move the convention blocks into the new doc | Moving `### Tracked doc/ files manifest` empties the Step 6.7 awk parse → every tracked-doc/ verdict silently vanishes. Keeping anchors in-place keeps the change low-risk. |
| Doc enforcement | `/CJ_repo-init` presence check (no new validate.sh check) | A new hard `validate.sh` check for the doc | Presence is a per-repo prerequisite like `TODOS.md`; a root `.md` is structurally outside the registered-doc set. Avoids new CI surface + a parallel `zzz-test-scaffold` fixture edit. |
| Seed location | Inline portable heredoc in `seed_docguide()` | A `templates/CJ_repo-init/` template file | Mirrors the existing `seed_docrel` inline-heredoc pattern; avoids a new templates deployment + manifest surface for one tiny generic starter. |
