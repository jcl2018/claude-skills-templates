## Problem Statement

F000032 ships per-skill `USAGE.md` (required for routable non-deprecated skills) + `scripts/validate.sh` Check 13 (presence + the five required H2 sections). What Check 13 does NOT catch: USAGE.md going stale when SKILL.md changes. A SKILL.md edit that renames a flag, adds a feature, or changes behavior can leave USAGE.md silently out of date. This story builds the cheap, deterministic, repo-internal drift check: a new validate.sh Check 14 that compares git-log commit timestamps (`%ct`) of SKILL.md vs USAGE.md per audited skill, ERRORs on `SKILL > USAGE`, and embeds the documented operator override (a `last-updated:` frontmatter bump) in the ERROR message. CLAUDE.md + PHILOSOPHY.md document the override. scripts/test.sh smokes the full drift → override → green cycle.

## Mental Model

Symmetric pair with F000032:

```
F000032 Check 13: presence + structure  ("USAGE.md exists with five H2 sections")
F000033 Check 14: freshness             ("USAGE.md %ct >= SKILL.md %ct")
```

Audit set derived at run time from `skills-catalog.json` (matches Check 13 exactly):

```
status != "deprecated" AND (files | length) > 0
```

= 11 routable non-deprecated skills as of 2026-06-01.

Operator override:

```
sed -i.bak 's/^last-updated:.*/last-updated: "'"$(date +%Y-%m-%d)"'"/' skills/{name}/USAGE.md && rm skills/{name}/USAGE.md.bak
git add skills/{name}/USAGE.md && git commit -m "docs: verify USAGE.md current for {name}"
```

A one-line content change to `last-updated:` (the field exists in F000032's template + every USAGE.md backfill) produces a real commit, advances USAGE.md's `%ct` past SKILL.md's, and leaves a human-readable audit trail (the date).

### Data Flow

1. Operator (or pre-commit hook, or CI) runs `./scripts/validate.sh`.
2. Check 14 invokes `jq -r '.[] | select(.status != "deprecated") | select((.files | length) > 0) | .name' skills-catalog.json` to derive the audit set.
3. For each name: read `SKILL_PATH=skills/$name/SKILL.md`, `USAGE_PATH=skills/$name/USAGE.md`.
4. If either path missing: continue (Check 12/13 handle presence).
5. Run `git log -1 --format=%ct -- $SKILL_PATH` → `$SKILL_CT`; same on `$USAGE_PATH` → `$USAGE_CT`.
6. If either `%ct` empty (file untracked / staged-only): print one-line SKIP note; continue.
7. If `$SKILL_CT > $USAGE_CT`: print ERROR with the override one-liner; increment `$ERRORS`.
8. Else: print PASS line.
9. After loop, validate.sh exits non-zero if `$ERRORS > 0`.

### Override execution path

```
operator sees the ERROR with override commands embedded
        ↓
copy-paste sed: `sed -i.bak 's/^last-updated:.*/last-updated: "<today>"/' USAGE.md && rm USAGE.md.bak`
        ↓
USAGE.md last-updated: field updated to today's date (one-line content change)
        ↓
git add + git commit  → real commit, touches USAGE.md
        ↓
next `git log -1 --format=%ct -- USAGE.md` returns the new commit's ct
        ↓
USAGE.md %ct > SKILL.md %ct
        ↓
validate.sh Check 14 → PASS
```

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Drift detection primitive | `git log -1 --format=%ct` (committer Unix timestamp) | Filesystem mtime (`stat -f %m`) | Commit timestamps are deterministic across worktrees, fresh clones, CI runners. Filesystem mtimes diverge. |
| Comparison strictness | `>` | `>=` | Atomic SKILL.md+USAGE.md commits share `%ct`; `>=` would false-fire on every F000032-backfilled USAGE.md the day this PR ships. |
| Severity | ERROR (with documented cheap override) | WARN | F000030 + F000032 already establish ERROR + override as the workbench's load-bearing pattern. WARN replays F000030's 1/13 adoption decay. |
| Override mechanism | `sed` on `last-updated:` frontmatter + commit | `git commit --allow-empty` | Empty commits don't advance `%ct` for any specific path; the frontmatter bump produces a real commit + a human-readable audit trail. |
| Audit set source | Computed via jq from catalog | Hard-coded name list | Future-proof; matches Check 13's predicate exactly. |
| Empty `git log` response | SKIP (visible note) | ERROR | Check 13 owns presence; Check 14 owns freshness; freshness of an uncommitted file is meaningless. Visible note ≠ silent pass. |
| Story decomposition | Single user-story | Multiple stories (Check 14 / CLAUDE.md / PHILOSOPHY.md / test.sh as separate) | Atomic under the pre-commit hook. Same shape as F000032. |
| PR target | `cj-feat-20260601-152835-3769` (PR #186's branch) | `main` | Check 14 references USAGE.md paths only present in PR #186; against main, the audit would fail to find them. Stacked PR is correct. |
