## Problem Statement

The workbench operator maintains the portable doc/test contract files that
`/CJ_doc_audit` and `/CJ_test_audit` seed into arbitrary repos. Today those
files are heavy and redundant: `doc-spec.md` keeps the same list in three forms
(prose, a fenced-YAML registry, and two generated views), and the verification
story is split across `test-spec.md` (abstract rules) and `gate-spec.md`
(the layer/when map). The operator wants ONE table-shaped source per contract and
ONE verification contract — lightweight enough to drop into a bare repo, with no
second copy to drift.

## Mental Model

Two internal phases inside one PR, each verified green before the next:

- **Phase 1 (doc-spec):** the file IS the table. `| Doc | Purpose | Requirement |`
  is parsed directly; the YAML registry, the generated views, the generator, and
  the `--render`/`--list-front-table-docs` surfaces are gone. `audit_class` is no
  longer a column — it is re-derived inside the parser from path convention so
  Check 19 (no work-item IDs in human-docs) is unchanged.
- **Phase 2 (test-spec/gate-spec merge):** the test-spec family absorbs gate-spec
  whole. The general `test-spec.md` carries the `layers[]` four-layer map; the
  `test-spec-custom.md` overlay grows a new top-level `gates:` array for the
  per-mode pipeline-gate rows (kept out of `units:` because the `units:` `layer`
  enum would reject `pipeline-gate`). `gate-spec.sh` is absorbed into
  `test-spec.sh` and deleted; `validate.sh` Check 22 folds into Check 24 with its
  marker-drift portion kept advisory.

## Acceptance Criteria

<!-- Given/When/Then. One block per requirement. -->

### Story #1: doc-spec is a 3-column table source [core]

```
GIVEN spec/doc-spec.md and spec/doc-spec-custom.md
WHEN  the build rewrites them to the | Doc | Purpose | Requirement | shape
THEN  there is no fenced-YAML block and no section/audit_class/front_table field,
      AND spec/doc-spec.md is byte-identical to `doc-spec.sh --seed` and to
      templates/doc-spec-common.md (3-way identity).
```

### Story #2: --check-on-disk runs 4 checks, human-doc-ids intact [core]

```
GIVEN the rewritten scripts/doc-spec.sh table parser
WHEN  `doc-spec.sh --check-on-disk` runs against the repo
THEN  it reports CHECKS_RUN=4 (declared-exists, orphans, root-declared,
      human-doc-ids) with the front-table and views-render arms removed,
      AND the human-doc-ids check still fires via a path-derived audit_class
      (path under docs/ or root README.md ⇒ human-doc),
      AND --render and --list-front-table-docs no longer exist.
```

### Story #3: retired files are deleted and grep-clean [integration]

```
GIVEN docs/doc-general.md, docs/doc-custom.md, scripts/generate-doc-views.sh,
      scripts/gate-spec.sh, spec/gate-spec.md
WHEN  the build removes them
THEN  none of the files exist on disk
      AND grep across scripts/, skills/, docs/, tests/ finds no live reference
      to any of them.
```

### Story #4: test-spec family carries layers[] + gates[] [core]

```
GIVEN the merged test-spec family
WHEN  the build folds gate-spec.md's layers[] into general spec/test-spec.md and
      its per-mode pipeline-gate gates[] into spec/test-spec-custom.md as a NEW
      top-level gates: array (NOT a units: row)
THEN  `test-spec.sh --validate` accepts both the units: rows and the gates: array,
      AND the general test-spec.md stays byte-identical to `test-spec.sh --seed`.
```

### Story #5: Check 22 folds into Check 24, advisory preserved [resilience]

```
GIVEN validate.sh Check 22 (advisory gate-marker drift) and Check 24 (hard
      test-spec coverage)
WHEN  the build folds 22 into 24
THEN  the merged check keeps the coverage portion hard (exit 1 on a coverage
      finding) AND the per-mode marker-drift portion advisory (exit 0 on a
      marker-drift finding),
      AND Check 20 and Check 23 no longer exist.
```

### Story #6: all four cj_goal pipelines cite test-spec.md, suite green [integration]

```
GIVEN the four cj_goal {pipeline,SKILL}.md files citing "Canonical gate
      sequence: gate-spec.md"
WHEN  a grep-driven sweep re-points every reference to spec/test-spec.md
THEN  no live reference to gate-spec remains in any pipeline,
      AND ./scripts/validate.sh and ./scripts/test.sh both exit green.
```

### Story #7: audit skills re-enumerate + run clean standalone [usability]

```
GIVEN skills/CJ_doc_audit/SKILL.md (enumerating six checks + --render) and the
      test/doc audit skills
WHEN  the build re-enumerates CJ_doc_audit to the 4 checks and drops --render
THEN  both /CJ_doc_audit and /CJ_test_audit seed their contract and run clean in
      a bare repo AND in this workbench.
```

### Data Flow

<!-- How data moves for the primary use case (an audit run). -->

1. `/CJ_doc_audit` (or `validate.sh`) invokes `doc-spec.sh --check-on-disk`.
2. `doc-spec.sh` parses the merged 3-column table (general + custom overlay) into the internal TSV, deriving `audit_class` for `$3` from each path (under `docs/` or root `README.md` ⇒ `human-doc`, else `operational`).
3. The engine runs 4 deterministic checks (declared-exists, orphans, root-declared, human-doc-ids) and prints `CHECKS_RUN=4`.
4. `/CJ_test_audit` (or `validate.sh` Check 24) invokes `test-spec.sh --validate`, which parses the general `layers[]` + the custom `units:` and `gates:` arrays.
5. Check 24 runs the coverage cross-check (hard) and the per-mode marker-drift cross-check over the `gates:` rows (advisory).

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Consolidation scope | Full merge: retire gate-spec into test-spec | Keep gate-spec separate (status quo / F000060 split) | Unified verification-contract framing worth more than test-vs-process-gate purity (operator decision) |
| doc-spec representation | One 3-column table as source | Keep prose + YAML + generated views | One artifact, not three — no second copy to drift; lighter as a portable seed |
| `audit_class` after dropping the column | Re-derive in the parser from path convention | Leave TSV `$3` empty / keep the field | Empty `$3` silently voids the human-doc-id check; path heuristic matches today's data exactly |
| `pipeline-gate` rows home | New top-level `gates:` array | A `units:` row | The `units:` `layer` enum `{local-hook, ci}` would reject `pipeline-gate` |
| Check 22→24 disposition | Keep marker-drift advisory inside Check 24 | Let it inherit Check 24's hard exit | Silently promoting an advisory finding to a hard fail would change gating behavior |
| Sequencing | One PR, two internal phases (Approach B) | All-at-once (A) / two PRs (C) | B de-risks the rewrite by ordering while keeping one review/ship chain |
