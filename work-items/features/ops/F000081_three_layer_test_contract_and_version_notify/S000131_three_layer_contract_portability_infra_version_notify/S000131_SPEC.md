## Problem Statement

The workbench maintainer (and every consumer/remote install) is served by an F000078
two-axis test contract that is half-populated with no gap report, two portability harness
tests mislabeled `workflow`, a version-notification blind on any machine without a live
workbench `.git` checkout, and a redundant `/CJ_portability-audit` verb. This story closes
all four in one PR: it completes the per-category three-level contract with an advisory
matrix, reclassifies portability to `infra` and backfills its local-hook level, makes the
version-notification checkout-independent via `git ls-remote`, adds safe-additive nightly
CI + a targeted-negative-test refactor, and retires the redundant verb while keeping its
engine.

## Mental Model

The change has five sequential layers, each unblocking the next:

```
WS1 contract + seed + advisory matrix (foundation)
  в”њв”Ђ WS3 skills-update-check git-ls-remote + root unit test  (the local-hook proof)
  в”‚    в””в”Ђ WS2 portabilityв†’infra reclass + doc moves + local-hook command-row (points at WS3)
  в”‚         в””в”Ђ WS5 retire /CJ_portability-audit (engine + Check 18 kept)
  в””в”Ђ WS4 nightly.yml + ci unit + targeted-negative-test refactor  (defer the validate.yml trim)
```

The contract axes: **category** `{workflow, regression, infra}` Г— **layer**
`{CI-push, CI-nightly, pipeline-gate, local-hook}`; the three per-category TEST levels are
`{CI-push, CI-nightly, local-hook}` (pipeline-gate is not a per-category test level).
`test`-family units live at repo ROOT; category cells are command-only `categories:` rows.

### Data Flow

<!-- How does data move through the system for the primary use case (the version-check). -->

1. A skill preamble on any machine invokes `skills-update-check`.
2. The script reads local version = `collection_version` from `~/.claude/.skills-templates.json` (no `.source`/`.git` requirement).
3. It resolves `upstream_url` from the manifest, normalizes ssh-form в†’ https, and runs `git ls-remote --tags <upstream_url> 'v*'` (or the `SKILLS_UPDATE_REMOTE_URL` / PATH-stub in tests).
4. It parses the max `v<X.Y.Z>` tag as the remote version; unreachable / no tags в†’ fail-soft silent.
5. It compares remote vs local against the existing cache/snooze/skip state (`SKILLS_UPDATE_STATE_DIR` override in tests).
6. If remote > local and not snoozed/skipped, it emits `SKILLS_UPGRADE_AVAILABLE <local> <remote>` + a context hint (`.source` present в†’ `git pull --ff-only && skills-deploy install`; else `skills-deploy install --bundle` / `setup.sh`).

## Tradeoffs

<!-- Design decisions made in this spec, with rejected alternatives and rationale. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Remote version read | `git ls-remote --tags <upstream_url>` | curl-raw against the GitHub API/raw; gh-api-embed | git is the repo's vetted hard dependency; handles ssh + non-GitHub URLs; zero curl precedent (`grep curl scripts/` = 0); gh-api adds state + auth |
| CI scope this feature | Safe-additive: add nightly workflow + negative-test refactor now | All-now (also trim `validate.yml`); all-deferred | An autonomous PR-stop can't verify a trimmed gate; all-deferred loses the free, in-PR-verifiable OOM fix + nightly infra |
| Coverage matrix posture | Advisory `NOTE:`s, exit 0 always | Hard-fail on empty cells | `--check-structure` is "findings are the product, exit 0"; an intentionally-empty cell must not gate |
| Portability category cells | Command-only `categories:` rows (no folder move) | Move test files into `tests/infra/<layer>/` | `test`-family units live at root; the reverse sweep globs `tests/*.test.sh` at root; command-only rows are exempt from `--check-structure` (b) |
| Portability local-hook fill | Deterministic (stubbed-remote local sandbox) | A truly agentic `claude --print` stale-sandbox variant | The advisory matrix permits a deterministic fill; the agentic variant is deferred (Q1) |
| `/CJ_portability-audit` | Retire the verb, keep engine + Check 18 | Keep the verb; delete the engine too | The automatic per-PR lint + reclassified infra tests make the manual verb redundant; the engine is what proves portability |
| Story decomposition | One atomic user-story, no task children | Split WS1вЂ“WS5 into task children | The workstreams are one cohesive sequential change in one PR; splitting fragments a coherent diff |
