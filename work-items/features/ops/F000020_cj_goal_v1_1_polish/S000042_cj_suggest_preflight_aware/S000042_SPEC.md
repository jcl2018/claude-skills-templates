## Problem Statement

`/CJ_goal` no-args path currently chains: `/CJ_suggest` (top-5 hard cap) → /CJ_goal post-filter via skip-list → preflight → halt-or-continue. Two compounding problems make this unreliable under `/loop /CJ_goal`:

1. /CJ_suggest hard-caps at top-5. When 5+ top-ranked rows are skip-listable, post-filter returns empty and /CJ_goal halts.
2. /CJ_suggest is /CJ_goal-blind — its top-5 routinely contains rows /CJ_goal will reject (P1 size-cap, size L|XL, sensitive surfaces, design-needed keywords).

The user (operator running /loop /CJ_goal as an unattended-grind session) hits these compounded as `halted_at_resolve` even when TODOS.md has plenty of eligible-but-lower-ranked rows.

## Mental Model

Today's flow:

```
TODOS.md → /CJ_suggest (top-5 cap, generic ranking)
  → /CJ_goal post-filter (skip-list)
  → /CJ_goal preflight (P1, size, sensitive, design-keyword)
  → /CJ_run dispatch OR halt
```

Proposed flow:

```
TODOS.md → /CJ_suggest --for-skill cj-goal --limit 15
            (apply /CJ_goal preflight predicates AT RANKING TIME, return up to 15 rows)
  → /CJ_goal post-filter (skip-list)  [unchanged]
  → /CJ_goal preflight (P1, size, sensitive, design-keyword)  [unchanged; defense-in-depth]
  → /CJ_run dispatch
```

Pre-filter at ranking time means rows that will preflight-reject never enter the candidate window in the first place. The preflight check itself stays as defense-in-depth (catches drift between /CJ_suggest's predicate set and /CJ_goal's actual preflight logic).

## Acceptance Criteria

### Story #1: /CJ_suggest accepts --for-skill flag [core]

```
GIVEN  /CJ_suggest is invoked with --for-skill cj-goal
WHEN   the script processes its arguments
THEN   the named-skill predicate block (cj-goal preflight criteria) is applied to ranking
AND    no row matching any preflight criterion appears in output
```

### Story #2: /CJ_suggest accepts --limit flag [core]

```
GIVEN  /CJ_suggest is invoked with --limit 15
WHEN   the script renders its ranked output
THEN   up to 15 rows are returned (instead of the default 5)
AND    `--limit 5` (or no flag) returns up to 5 rows (regression compatible)
```

### Story #3: --for-skill cj-goal excludes 4 row classes [core]

```
GIVEN  TODOS.md contains rows of each class: P1 size-cap, size L|XL, sensitive-surface, design-needed keyword
WHEN   /CJ_suggest --for-skill cj-goal --limit 15 runs
THEN   no row from any of the 4 classes appears in the output
AND    rows that pass all 4 predicates appear in ranked order up to limit
```

### Story #4: /CJ_goal no-args path uses new flags [core]

```
GIVEN  /CJ_goal is invoked with no arguments (no T-ID, no fragment)
WHEN   goal.sh's resolve step runs
THEN   it shells out to `/CJ_suggest --for-skill cj-goal --limit 15` (or equivalent argv)
AND    the resulting candidate set is the top of that filtered list
```

### Story #5: /CJ_suggest with no flags returns top-5 unchanged [resilience]

```
GIVEN  /CJ_suggest is invoked with no flags
WHEN   it renders output
THEN   the result is byte-identical to v3.5.5 (or whatever the immediate-pre-WI-A version is)
AND    behavior for all existing interactive callers is unchanged
```

## Architecture

```
┌────────────────────────────────────────────────────────────────────┐
│  TODOS.md                                                          │
└──────────────────────────────┬─────────────────────────────────────┘
                               │
                               ▼
┌────────────────────────────────────────────────────────────────────┐
│  /CJ_suggest scripts/suggest.sh                                    │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │  parse_args:                                                │  │
│  │    --for-skill <name>  → SKILL_PREDICATES=<predicate-set>   │  │
│  │    --limit N           → LIMIT=N                            │  │
│  │  rank_rows:                                                 │  │
│  │    apply existing scoring                                   │  │
│  │  apply_skill_predicates:  (NEW)                             │  │
│  │    if SKILL_PREDICATES set:                                 │  │
│  │      for each row: if any predicate matches → exclude       │  │
│  │      log [CJ_suggest] excluded: <T-ID> reason=<criterion>   │  │
│  │  render:                                                    │  │
│  │    take top LIMIT rows (default 5)                          │  │
│  └─────────────────────────────────────────────────────────────┘  │
└──────────────────────────────┬─────────────────────────────────────┘
                               │
                               ▼
┌────────────────────────────────────────────────────────────────────┐
│  /CJ_goal scripts/goal.sh (no-args path)                           │
│  ┌─────────────────────────────────────────────────────────────┐  │
│  │  resolve:                                                   │  │
│  │    suggest_output=$(./CJ_suggest --for-skill cj-goal        │  │
│  │                                   --limit 15)               │  │
│  │    candidate=$(post-filter $suggest_output via skip-list)   │  │
│  │  preflight: (unchanged — defense in depth)                  │  │
│  └─────────────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────────────┘
```

### Data Flow

1. Operator runs `/CJ_goal` (no args), or `/loop /CJ_goal` schedules invocations.
2. goal.sh resolve step shells out: `/CJ_suggest --for-skill cj-goal --limit 15`.
3. suggest.sh parses --for-skill, looks up the cj-goal predicate block (4 awk one-liners checking priority, size, sensitive-surface regex, design-keyword presence).
4. suggest.sh ranks all TODOS.md rows by existing scoring.
5. suggest.sh filters: any row where any predicate fires is excluded; log line emitted per exclusion.
6. suggest.sh renders top-LIMIT (15) of remaining rows.
7. goal.sh consumes the filtered list, applies skip-list post-filter (unchanged), then preflight (unchanged; defense-in-depth catches predicate drift), then dispatches /CJ_run on the first survivor.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Predicate location | Inline in suggest.sh per-skill block | Shared predicate file imported by both scripts | Single consumer today; YAGNI on a shared file. Revisit if drift becomes painful. |
| --limit default | 5 (unchanged) | 10 or 15 globally | Existing interactive /suggest user sees no change; /CJ_goal callers opt in explicitly. Smallest blast radius. |
| --for-skill granularity | Named skill (e.g., `cj-goal`) | Generic --predicate "p1=excluded,size=excluded" API | Per-name block is more discoverable + lower coupling surface. Future consumers add named blocks. |
| Defense-in-depth (keep /CJ_goal's own preflight) | Keep preflight as backup catch | Remove preflight (single source of truth in /CJ_suggest) | If predicate sets drift between scripts, /CJ_goal's own preflight is the safety net. Cheap to keep. |
