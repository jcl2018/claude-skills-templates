## Problem Statement

A developer running any `cj_goal` orchestrator (`/CJ_goal_feature`,
`/CJ_goal_defect`, `/CJ_goal_todo_fix`) on a machine that hasn't pulled recently
starts the build on stale `main` (the worktree branches off current local HEAD
with no fetch/ff) and with stale installed skills (`~/.claude/skills/` lags
`origin/main` until a manual `git pull` + `skills-deploy install`). The result is
cross-machine merge pain at ship/land and silent drift between installed skills
and trunk. The developer wants every build to start from a known-fresh base +
current skills with zero extra ceremony.

## Mental Model

Two fail-soft halves run at build start, in this order:

```
operator invokes /CJ_goal_*
  → preamble: skills-update-check (advisory, unchanged)
  → cj-goal-common.sh --phase sync        (Fork 2: guarded pull .source + install from .source; --no-sync skips)
  → cj-goal-common.sh --phase worktree → cj-worktree-init.sh   (Fork 1: fetch + ff local main, THEN git worktree add off fresh main)
  → isolation gate → office-hours / pipeline → build
```

Fork 2 (heavy, global-state, opt-out-able via `--no-sync`) runs first so the
install is based on fresh `.source`; Fork 1 (cheap, local-only, always runs)
then fast-forwards local `main` so the new worktree branches off the fresh tip.
In the common self-dev case where `.source` == repo root, Fork 2 already pulled
`main`, so Fork 1's ff is a no-op.

## Acceptance Criteria

<!-- Given/When/Then. Each #n matches the Requirements # column. -->

### Story #1: Fast-forward local main when behind [core]

```
GIVEN the current branch is main (or master) and origin/main exists
  AND local main is behind origin/main and fast-forwardable (HEAD is ancestor of origin/main)
WHEN  cj-worktree-init.sh runs its freshness step before `git worktree add`
THEN  it runs `git merge --ff-only origin/$BRANCH`, records `ff'd N commits` in the JSON `note`,
  AND the new worktree base satisfies `git merge-base --is-ancestor origin/main <worktree-base>` (true)
```

### Story #2: Diverged local main warns and proceeds [resilience]

```
GIVEN the current branch is main and local main has a commit origin/main does not (HEAD is NOT an ancestor of origin/main)
WHEN  the freshness step runs
THEN  it does NOT ff, records `local main diverged from origin; building on local main` in the `note`,
  AND the build proceeds (no halt), branching the worktree off local main
```

### Story #3: Offline proceeds on local main [resilience]

```
GIVEN `git fetch` fails (no network / no remote / no origin/$BRANCH ref)
WHEN  the freshness step runs
THEN  it silently proceeds on local main, records `freshness skipped (offline)` in the `note`,
  AND the overall exit code is 0 with no operator-facing error
```

### Story #4: Sync phase installs from .source and reports versions [core]

```
GIVEN `.source` resolves from ~/.claude/.skills-templates.json to a clean on-main git repo
WHEN  `cj-goal-common.sh --phase sync` runs
THEN  it `git -C "$_SRC" pull --ff-only`, runs `skills-deploy install` from `.source` (never the worktree),
  AND prints VERSION_BEFORE and VERSION_AFTER (collection_version) and PHASE_RESULT
```

### Story #5: Sync phase is fail-soft [resilience]

```
GIVEN `.source` is missing, not a git repo, not on main, or has a dirty tracked tree (guard refusal), OR the pull is offline
WHEN  `--phase sync` runs
THEN  it emits `PHASE_RESULT=skipped` (not `failed`) and exits 0, never blocking the build
```

### Story #6: --no-sync opts out of the install while ff still runs [usability]

```
GIVEN `--no-sync` is passed
WHEN  the orchestrator preamble runs the sync phase
THEN  the phase is skipped entirely (`PHASE_RESULT=skipped`, no `skills-deploy install` invoked)
  AND Fork-1's ff in the worktree phase still runs
```

### Story #7: All three orchestrators wired from one implementation [integration]

```
GIVEN the shared `--phase sync` in cj-goal-common.sh and the in-helper ff in cj-worktree-init.sh
WHEN  /CJ_goal_feature, /CJ_goal_defect, and /CJ_goal_todo_fix each run their preamble
THEN  each calls `--phase sync` before its Default-worktree block, forwards `--no-sync`,
  AND CJ_goal_todo_fix additionally runs the `skills-update-check` snippet (which it previously lacked)
```

### Story #8: Stable phase output schema [observability]

```
GIVEN `--phase sync` runs in any mode (ran / skipped / dry-run)
WHEN  it completes
THEN  stdout contains the KEY=VALUE keys SYNC_RAN, VERSION_BEFORE, VERSION_AFTER, PHASE_RESULT,
  AND the schema is consistent with the other cj-goal-common.sh phases
```

## Architecture

<!-- High-level design. -->

```
                        /CJ_goal_{feature,defect,todo_fix}  (preamble)
                                        |
                 skills-update-check (advisory; F000009; todo_fix newly gains it)
                                        |
                 cj-goal-common.sh --phase sync   ── Fork 2 ──┐
                   resolve .source (~/.claude/.skills-templates.json)
                   guard: exists / git repo / on main / clean tracked tree
                   git -C $_SRC pull --ff-only       (offline → skipped)
                   skills-deploy install  FROM .source (not worktree)
                   emit SYNC_RAN / VERSION_BEFORE / VERSION_AFTER / PHASE_RESULT
                   --dry-run → preview; --no-sync → PHASE_RESULT=skipped
                                        |
                 cj-goal-common.sh --phase worktree
                                        |
                 cj-worktree-init.sh  ── Fork 1 ──┐
                   (between dirty-state check and name composition,
                    before `git worktree add`; only when BRANCH ∈ {main,master}
                    and origin/$BRANCH exists)
                   git fetch origin $BRANCH 2>/dev/null || true
                   if HEAD ancestor-of origin/$BRANCH and behind → git merge --ff-only ; note="ff'd N commits"
                   elif HEAD not-ancestor (diverged)            → note="local main diverged from origin; building on local main" (no ff)
                   else offline/no-origin                       → note="freshness skipped (offline)"
                   git worktree add "$WT_PATH" -b "$NAME"   (off now-fresh main)
                                        |
                            isolation gate → pipeline → build
```

### Data Flow

<!-- Primary use case: build start on a behind-origin machine. -->

1. Operator invokes `/CJ_goal_*` from `main` with arguments.
2. Preamble runs `skills-update-check` (advisory banner only).
3. Preamble calls `cj-goal-common.sh --phase sync` (Fork 2): resolves `.source`, guards it, `git pull --ff-only` on `.source`, `skills-deploy install` from `.source`, emits version before→after + `PHASE_RESULT`. `--no-sync` short-circuits to `PHASE_RESULT=skipped`.
4. Preamble calls `cj-goal-common.sh --phase worktree` → `cj-worktree-init.sh`: fetches `origin/$BRANCH`, fast-forwards local `main` (or warns on divergence / skips offline), then `git worktree add` off the now-fresh main; the freshness outcome rides the JSON `note`.
5. Isolation gate confirms the worktree, then office-hours / pipeline / build proceed on fresh base + synced skills.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Base-freshness mechanism | Fast-forward local `main` (Fork 1), warn on divergence | Branch worktree off `origin/main` directly (Approach C) | Preserves "branch off main" semantics + keeps local main current; branching off origin/main would silently skip rare unpushed local main commits. |
| Skills-sync trigger | Unconditional install from `.source` at every build start (Fork 2) | Warn-only base + tighten skills-update-check (Approach B) | Approach B doesn't deliver a fresh base — the build still proceeds on stale main. Operator chose the strong guarantee. |
| Install source | From `.source` (resolved from manifest) | From the worktree | A worktree-invoked install skips foreign-owned skills (collection-version-stuck bug); `.source` is what `post-land-sync.sh` already uses. |
| Pull/install machinery | Reuse `post-land-sync.sh`'s guarded core | New machinery inside cj-goal-common.sh | The vetted helper already guards (`.source` exists / on-main / clean tracked tree) and installs from `.source`. |
| `--no-sync` scope | Suppresses only the heavy install half | Also suppress Fork-1 ff | Fork-1 ff is cheap/local and core to the value; `--no-sync` is about the heavy global-state install + latency. |
| Sync-phase failure mode | `PHASE_RESULT=skipped` on guard refusal / offline | `PHASE_RESULT=failed` (blocking) | Same fail-soft posture as the `pr-check` phase — must never block a build. |
