## Problem Statement

The `/CJ_goal_investigate` developer (workbench owner) has a bug fresh in their
head and wants `/CJ_goal_investigate "discord bot crashed on stale Yahoo
response"` to ship a fix in one command. Today v1.0's `pipeline.md` Step 2
`case "$MATCH_COUNT" in 0)` branch halts on any fragment that doesn't match an
existing canonical `work-items/defects/<domain>/D000NNN_<slug>/` dir, forcing a
two-command flow (`/CJ_scaffold-work-item --type defect …` then
`/CJ_goal_investigate D000NNN`). The friction breaks the muscle-memory loop
between observation and action.

## Mental Model

Two-phase lifecycle for a zero-match fragment:

```
INTAKE (moment of least clarity)            PROMOTION (moment of clarity)
  zero canonical match                        Iron-Law gate passed
  → .inbox/<slug>/DRAFT.md                     (STATUS=DONE + ROOT_CAUSE set)
  → NO D-ID                                   → allocate D-ID (mkdir-locked)
  → IS_DRAFT=1                                → write canonical TRACKER (commit point)
  → pipeline runs /investigate                → rebind $DEFECT_* to canonical
     against the draft (Row 1)                → rm -rf the consumed draft
                                              → Steps 7.5-12 unchanged
```

The canonical Step 2 resolver (`if`/`MATCHES`/`MATCH_COUNT` block) is unchanged.
Drafts are invisible to it by *construction*, not by an added filter: a draft
dir has no `D[0-9]{6}_` basename (excluded from BASENAME_HITS's
`grep -E '/D[0-9]{6}_'`) and contains `DRAFT.md` not `*_TRACKER.md` (excluded
from NAME_HITS's `--include="*_TRACKER.md"` grep). Only the `0)` case body
changes.

## Acceptance Criteria

<!-- Given/When/Then format. One block per requirement or logical group.

### Story #1: Zero-match captures a non-canonical draft [core]

```
GIVEN a fragment "freshly observed bug" with no matching canonical defect dir
WHEN  /CJ_goal_investigate "freshly observed bug" runs Step 2's 0) branch
THEN  work-items/defects/.inbox/freshly_observed_bug/DRAFT.md is created,
      no D-ID is allocated, IS_DRAFT=1 is set, and the pipeline continues
      into Step 3 resolving to idempotency Row 1
```

### Story #2: Promotion only after the Iron-Law gate [core]

```
GIVEN a draft that /investigate has carried to STATUS=DONE with ROOT_CAUSE populated
WHEN  control reaches new Step 7.4 with IS_DRAFT=1
THEN  a D-ID is allocated under the mkdir-lock, a canonical TRACKER is written,
      $DEFECT_DIR/$DEFECT_ID/$TRACKER/$RCA_PATH/$TEST_PLAN_PATH are rebound to
      work-items/defects/uncategorized/D000NNN_freshly_observed_bug/, and the
      .inbox draft is removed
AND   GIVEN /investigate fails the Iron-Law gate (ROOT_CAUSE empty)
      WHEN  Step 7 Halt 3 [investigate-no-root-cause] fires
      THEN  no promotion runs, no D-ID is consumed, the draft remains in .inbox/,
            and a re-invocation resumes it
```

### Story #3: C1 — Step 2 var-clobber guard [resilience]

```
GIVEN IS_DRAFT=1 set by the 0) branch
WHEN  control passes pipeline.md lines 105-107 (the post-case
      TRACKER/RCA_PATH/TEST_PLAN_PATH recompute)
THEN  the recompute is skipped (wrapped in if [ "${IS_DRAFT:-0}" != "1" ])
      and TRACKER still equals "$DRAFT_DIR/DRAFT.md", RCA_PATH="", TEST_PLAN_PATH=""
```

### Story #4: C2 — blank $DEFECT_ID never leaks [resilience]

```
GIVEN IS_DRAFT=1
WHEN  Step 5 dispatches /investigate AND any Step 7 halt branch fires
THEN  the /investigate prompt's DEFECT_ID line reads
      "(draft — none yet; working dir is the draft)" with WORK_ITEM_DIR=draft dir,
      and every Step 7 halt emits resume_cmd=/CJ_goal_investigate "$DRAFT_FRAGMENT"
      (never an empty-$DEFECT_ID command)
```

### Story #5: C3 — atomic promotion, no duplicate D-ID [resilience]

```
GIVEN promotion is running inside the mkdir-lock
WHEN  the ordered protocol executes
THEN  DRAFT_OLD is captured before any rebind, the canonical TRACKER (containing
      name: $DRAFT_FRAGMENT) is written BEFORE rm -rf "$DRAFT_OLD", and the saved
      absolute DRAFT_OLD path (never reconstructed from $INBOX/$DRAFT_SLUG) is removed last
AND   GIVEN a crash injected after mkdir "$CANON_DIR" but before the TRACKER write
      WHEN  /CJ_goal_investigate is re-invoked on the same fragment
      THEN  it does NOT allocate a second D-ID (the empty orphan dir is counted by
            the highest-N scan; re-invocation resumes the draft, next promotion gets N+1)
```

### Story #6: C4 — lock-timeout halt full bookkeeping [observability]

```
GIVEN the D-ID allocation lock ($DEFECTS_ROOT/.scaffold.lock.d) is held by a stale dir
WHEN  the mkdir-lock acquisition loop exceeds its >10s / 50-iteration cap
THEN  a [promote-lock-timeout] journal entry is appended to the draft's DRAFT.md
      (with next_action= and resume_cmd=/CJ_goal_investigate "$DRAFT_FRAGMENT"),
      a telemetry line with end_state=halted_at_promote_lock_timeout is written,
      and SKILL.md's halt-taxonomy table lists that as the 13th end-state
```

### Story #7: C5 — resumed draft echoes stored fragment [usability]

```
GIVEN an existing draft at .inbox/<slug>/ with fragment: stored in DRAFT.md
WHEN  /CJ_goal_investigate is re-invoked and resolves to that draft
THEN  the stored fragment: value is echoed to the terminal so an operator can
      spot a wrong-bug slug collision, and a v1.2 open question for draft-level
      partial-fix detection is recorded in the spec/design
```

### Story #8: C6 — slug lowercasing load-bearing comment [resilience]

```
GIVEN the Step 2 0) case body's slugifier
WHEN  the implementer writes the slug derivation
THEN  a one-line comment states the lowercasing is load-bearing for
      resolver-isolation (a future preserve-case change would let
      .inbox/D000099_* collide with the case-sensitive find -name "D000099_*"
      at pipeline.md:62)
```

### Story #9: C7 — plain-English terminal narration [usability]

```
GIVEN any non-happy transition (new-draft capture, draft resume, each Step 7
      Halt 1-5, the Step 7.4 lock-timeout, promotion success, --dry-run zero-match)
WHEN  that transition occurs
THEN  the pinned plain-English message for it is printed to the terminal
      (not only journaled), using the verbatim copy in the Tradeoffs/Data Flow
      sections below, with copy-pasteable resume commands
```

### Story #10: --dry-run on a zero-match fragment [core]

```
GIVEN DRY_RUN=1 and a zero-match fragment
WHEN  Step 2's 0) branch runs
THEN  the would-create OR would-resume line + the would-promote line + the
      pinned dry-run message are printed, NO draft dir is created, and the
      process exits 0 with telemetry end_state=dry_run_preview
```

### Story #11: promoted-run telemetry [observability]

```
GIVEN a run that promoted a draft (IS_DRAFT=1 reached Step 7.4 successfully)
WHEN  Step 11 writes the telemetry line
THEN  the JSON includes auto_scaffolded: true (additive key; existing consumers
      reading named fields see no schema break); a non-promoting run emits
      auto_scaffolded: false
```

### Story #12: validator frontmatter compat [integration]

```
GIVEN the canonical TRACKER adds frontmatter keys auto_scaffolded + promoted_from_draft
WHEN  the implementer runs the pre-implementation prereq check
THEN  skills/CJ_personal-workflow/ + personal-artifact-manifests.json are
      inspected; if a strict key-allowlist validator exists it is extended in
      the SAME PR; if pass-through, no extra work and the first promoted defect
      passes /CJ_qa-work-item
```

### Story #13: SKILL.md documentation update [usability]

```
GIVEN SKILL.md currently defers ad-hoc bugs to "Not in scope"
WHEN  this story ships
THEN  that line is moved to a v1.1 feature line describing the draft→promote
      flow, frontmatter version is 1.0.0 → 1.1.0, and the description's
      "deferred to v1.1" sentence no longer claims this is deferred
```

### Data Flow

<!-- How does data move through the system for the primary use case?
     Step-by-step, component to component. -->

1. Operator runs `/CJ_goal_investigate "discord bot crashed on stale Yahoo response"`.
2. Step 2's unchanged resolver computes `MATCH_COUNT=0` (no canonical dir for that fragment).
3. The rewritten `0)` body: `INBOX="$DEFECTS_ROOT/.inbox"`, `mkdir -p "$INBOX"`; slugify the fragment (lowercase, `[^a-z0-9]+`→`_`, collapse, cap 50, trim trailing `_`; empty→`untitled`) with the **C6 load-bearing-lowercasing comment**; `DRAFT_DIR="$INBOX/$SLUG"`.
4. If `DRY_RUN=1`: print would-resume (if `-d "$DRAFT_DIR"`) or would-create line + the would-promote line + the **C7 dry-run message** `DRY RUN: writes nothing. Re-running the same phrase later would resume this draft; reworded text would create a different draft.`; telemetry `end_state=dry_run_preview`; exit 0. No draft created.
5. Else if `-d "$DRAFT_DIR"`: print the **C7 draft-resume message** `Resuming the temporary draft at $DRAFT_DIR (originally: "<fragment: from DRAFT.md>"). Still no D-ID until the root cause is found.` (echoes stored fragment per C5).
6. Else: `mkdir -p "$DRAFT_DIR"`; write `DRAFT.md` (frontmatter `kind: defect-draft`, `created`, `fragment: $ARG`; body notes no D-ID allocated); print the **C7 draft-capture message** `No existing defect matched "$ARG", so I created a temporary draft at $DRAFT_DIR (no D-ID yet). Re-run this exact phrase to resume it; it becomes a real D-ID only after the root cause is found. Stale drafts are safe to rm -rf — they are not canonical.`
7. Set `IS_DRAFT=1`, `DEFECT_DIR="$DRAFT_DIR"`, `DEFECT_ID=""`, `DRAFT_SLUG="$SLUG"`, `DRAFT_FRAGMENT="$ARG"`, `TRACKER="$DRAFT_DIR/DRAFT.md"`, `RCA_PATH=""`, `TEST_PLAN_PATH=""`. (`IS_DRAFT=0` is set once near the top of Step 2 so `1)`/`*)` branches leave it zero.)
8. **C1 guard:** lines 105-107's TRACKER/RCA_PATH/TEST_PLAN_PATH recompute is wrapped `if [ "${IS_DRAFT:-0}" != "1" ]; then … fi` — skipped for the draft; draft vars survive.
9. Step 3: `IS_DRAFT=1` → `R=0;F=0;P=0;M=0; RESUME_ROW=1`; echo `Idempotency: draft (IS_DRAFT=1) → Row 1 (fresh) by construction`.
10. Step 5 dispatches `/investigate`; **C2:** when `IS_DRAFT=1`, the prompt's `DEFECT_ID:` line = `(draft — none yet; working dir is the draft)` and `WORK_ITEM_DIR` = the draft dir.
11. `/investigate` runs against the draft. If it fails the Iron-Law gate, Step 7 Halt 3 `[investigate-no-root-cause]` fires; **C2:** `resume_cmd=/CJ_goal_investigate "$DRAFT_FRAGMENT"`; **C7:** a 3-line terminal block (`Why it stopped:` / `State preserved: draft retained at $DEFECT_DIR, no D-ID consumed` / `Next: /CJ_goal_investigate "$DRAFT_FRAGMENT"`). No promotion; no D-ID consumed; draft remains.
12. On Iron-Law pass (STATUS=DONE + `$ROOT_CAUSE` populated), **new Step 7.4** runs (only when `IS_DRAFT=1`): `LOCK_DIR="$DEFECTS_ROOT/.scaffold.lock.d"`; **C3 step 1** `DRAFT_OLD="$DEFECT_DIR"`; acquire mkdir-lock (loop; `>50` iterations → **C4** `[promote-lock-timeout]` journal entry on `DRAFT_OLD/DRAFT.md` + telemetry `end_state=halted_at_promote_lock_timeout` + the SKILL.md 13th-end-state row; exit 1).
13. Inside the lock: `HIGHEST=find "$DEFECTS_ROOT" -maxdepth 2 -type d -name 'D[0-9][0-9][0-9][0-9][0-9][0-9]_*' | …max`; `NEXT_N=$((HIGHEST+1))`; `DEFECT_ID=$(printf "D%06d" "$NEXT_N")`; `DOMAIN="uncategorized"`; `CANON_DIR="$DEFECTS_ROOT/$DOMAIN/${DEFECT_ID}_${DRAFT_SLUG}"`; `mkdir -p "$CANON_DIR"`.
14. **C3 durable commit point:** write `$CANON_DIR/${DEFECT_ID}_TRACKER.md` with frontmatter `type: defect`, `id: $DEFECT_ID`, `name: $DRAFT_FRAGMENT`, `status: phase-1-investigating`, `created`, `auto_scaffolded: true`, `promoted_from_draft: .inbox/$DRAFT_SLUG`, plus a `## Journal` `[auto-scaffolded]` line. Once this file exists, NAME_HITS (`grep -rli --include="*_TRACKER.md" "$ARG"`) resolves the canonical dir by fragment, so re-invocation resumes correctly with no second D-ID.
15. Rebind `DEFECT_DIR="$CANON_DIR"`, `TRACKER="$CANON_DIR/${DEFECT_ID}_TRACKER.md"`, `RCA_PATH="$CANON_DIR/${DEFECT_ID}_RCA.md"`, `TEST_PLAN_PATH="$CANON_DIR/${DEFECT_ID}_test-plan.md"`.
16. **C3 last:** `rm -rf "$DRAFT_OLD"` (the saved absolute path — never reconstructed from `$INBOX/$DRAFT_SLUG`). Release lock (`rmdir "$LOCK_DIR"`; `trap - EXIT`). **C7:** print `Root cause found, so I converted the draft into defect $DEFECT_ID at $CANON_DIR (was: "$DRAFT_FRAGMENT"). The .inbox draft is now gone.`
17. Steps 7.5-12 run unchanged on the rebound canonical vars (artifact writes, QA, ship, deploy).
18. Step 11 telemetry: `--argjson auto_scaffolded "$([ "${IS_DRAFT:-0}" = "1" ] && echo true || echo false)"` added to the existing `jq -nc` line (additive key).

## Tradeoffs

<!-- Design decisions made in this spec, with rejected alternatives and rationale.
     Future readers need to know why this path over the others. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Where the auto-capture lands | Non-canonical `work-items/defects/.inbox/<slug>/DRAFT.md`, D-ID minted only at promotion | Approach D: zero-match directly creates `work-items/defects/uncategorized/D000NNN_<slug>/` with full stubs | Approach D mints a canonical D-ID for every typo'd/re-worded fragment → permanent resolver disambiguation entropy that `rm -rf` does not fix (recall trust degrades). The draft model bounds entropy to the explicitly non-canonical inbox and strengthens Iron-Law. USER DECISION at the /autoplan premise gate. |
| Resolver Step 2 block | `if`/`MATCHES`/`MATCH_COUNT` byte-for-byte unchanged; only the `0)` case body changes | Add a draft-aware filter into the resolver | Drafts are invisible by construction (no `D######_` basename, `DRAFT.md` not `*_TRACKER.md`). Touching the resolver risks a regression in the load-bearing canonical path for zero behavioral gain. |
| Promotion commit point | Canonical TRACKER write (C3 step 3) is the single durable commit point | Treat `mkdir $CANON_DIR` as the commit point | A bare `mkdir`'d dir has no `*_TRACKER.md`, so NAME_HITS can't resolve it and BASENAME_HITS needs `*${fragment}*` (spaces vs underscores) — re-invocation would resume the draft and allocate a SECOND D-ID. The TRACKER is the first artifact the resolver can find. |
| D-ID allocation lock | POSIX `mkdir`-based lock (`.scaffold.lock.d`) around highest-N scan + canonical mkdir | `flock` | Stock macOS has no `flock`; `mkdir` is POSIX-atomic and universally available. |
