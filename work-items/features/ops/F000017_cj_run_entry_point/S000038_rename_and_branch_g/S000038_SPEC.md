## Problem Statement

A user on the workbench wants one command to "run the pipeline." Today they
must remember the difference between `/CJ_ship-feature` (full pipeline) and
`/CJ_personal-pipeline` (no-PR variant), and both require a design doc as input.
They cannot say "I'm partway through, finish this." They have to look up the
exact sub-skill (`/CJ_implement-from-spec`, `/CJ_qa-work-item`, `/ship`,
`/land-and-deploy`) and run them in order by hand.

This story renames `/CJ_ship-feature` to `/CJ_run` and adds a no-arg mode that
detects in-progress work on the current branch and resumes automatically.

## Acceptance Criteria

### Story #1: Rename mechanics [core]

```
GIVEN the workbench at the current state
WHEN  I run `git mv skills/CJ_ship-feature skills/CJ_run` and update SKILL.md,
      catalog, routing
THEN  `cat rules/skill-routing.md | grep CJ_ship-feature` returns no matches
AND   `cat rules/skill-routing.md | grep CJ_run` shows /CJ_run routing entries
AND   `validate.sh` passes
```

### Story #2: Branch(g) single-candidate resume [core]

```
GIVEN the current branch has exactly one work-item with Phase 1 all-checked
      AND Phase 2 impl-gate unchecked AND no merged PR
WHEN  I run `/CJ_run` with no arguments
THEN  the skill detects the candidate
AND   dispatches to work-item-dir mode (Branch(f) — delivered by S000039)
AND   resumes the pipeline from the correct phase
```

### Story #3: Branch(g) empty case [core]

```
GIVEN the current branch has no work-items/ directory OR no in-progress items
WHEN  I run `/CJ_run` with no arguments
THEN  the skill prints "No work-items/ found. Run /office-hours or
      /CJ_scaffold-work-item first." (or equivalent guidance)
AND   exits 0 without raising an error
```

### Story #4: Branch(g) multi-candidate AUQ [core]

```
GIVEN the current branch has two or more in-progress work-items
WHEN  I run `/CJ_run` with no arguments
THEN  the skill enumerates the candidates
AND   AskUserQuestion presents a picker
AND   dispatches to work-item-dir mode for the picked one
```

### Story #7: SKILL.md description prefix [usability]

## Architecture

```
   /CJ_run (no args)
        │
        v
   ┌──────────────────────────────────────────┐
   │ Step 1 of run.md — Input Detection       │
   │                                          │
   │   if arg && is *.md && APPROVED          │
   │       → Branch (a/b/c/d) [existing]      │
   │   elif arg && is dir && has TRACKER.md   │
   │       → Branch (f) [S000039]             │
   │   elif no arg                            │
   │       → Branch (g) [this story]          │
   └──────────────────────────────────────────┘
        │ (Branch g)
        v
   ┌──────────────────────────────────────────┐
   │ Scan work-items/ for user-story TRACKERs │
   │  - Phase 1 all checked                   │
   │  - Phase 2 impl gate NOT checked         │
   │  - No PR or PR not MERGED                │
   └──────────────────────────────────────────┘
        │
   ┌────┴──────┬────────────┐
  N=0         N=1          N>1
   │           │            │
   v           v            v
print     dispatch       AUQ pick
guidance  to Branch(f)   then dispatch
exit 0    [S000039]      to Branch(f)
```

### Data Flow

1. User invokes `/CJ_run` with no arguments.
2. Step 1 of `run.md` checks `$#` — zero args → dispatch Branch(g).
3. Branch(g) checks `$REPO_ROOT/work-items` exists; if not → print guidance, exit 0.
4. Branch(g) loops over `find work-items -name '*_TRACKER.md'` (user-story matched via gate strings).
5. For each candidate, check: Phase 1 all `[x]`, Phase 2 impl gate `[ ]`, no merged PR.
6. Collect matching paths in `CANDIDATES` (bash 3.2 array).
7. Switch on `${#CANDIDATES[@]}`:
   - 0 → "Nothing to resume" message, exit 0.
   - 1 → set `WORK_ITEM_DIR=${CANDIDATES[0]%/*}`; jump into Branch(f) dispatch.
   - >1 → AUQ with candidate list; on selection, jump into Branch(f) dispatch.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Loop construct | `while IFS= read -r` | `mapfile -t` (cleaner) | bash 3.2 (macOS default) lacks mapfile |
| TRACKER scope | user-story only | user-story + defect + task | Gate strings differ per type; v0.3 can extend |
| Empty case behavior | exit 0 with guidance | exit 1 (treat as error) | Empty isn't a failure mode; it's a normal "no work to resume" state |
| Multi-candidate behavior | AUQ to pick one | auto-pick most-recently-modified | User intent is ambiguous; explicit pick is safer |
| Branch(g) sequence vs Branch(f) | Branch(g) dispatches into Branch(f) | Branch(g) duplicates Branch(f) logic | DRY; Branch(f) is already implementing phase detection |
