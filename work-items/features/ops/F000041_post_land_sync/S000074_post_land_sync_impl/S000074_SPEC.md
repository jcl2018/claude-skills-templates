## Problem Statement

After a skill PR is merged with `gh pr merge` (a remote merge), the operator's
local `~/.claude/skills/` and the manifest `collection_version` are NOT refreshed,
because the local post-merge auto-sync hook only fires on a local `git pull`/`merge`.
The operator — the workbench maintainer running the land convention — is left with a
machine where the just-merged skill is on `main` but not invocable, and a
`collection_version` that lags `.source` (observed: 6.0.8 vs 6.0.10). Today the only
fix is to remember to run `git pull && skills-deploy install` by hand, with the right
checkout and the right install source.

## Mental Model

One helper script, three phases: **resolve** `.source` from the manifest → **guard**
(exists, on `main`, clean tree; else warn + exit non-zero) → **act** (`git pull
--ff-only` + `skills-deploy install`, with a `collection_version` before→after
report). `--dry-run` runs resolve + guard and prints what *would* run, mutating
nothing. A test drives all three phases against a temp fixture / `--dry-run` so the
real `~/.claude` is never touched, and the test is wired into the suite so it
actually runs.

## Acceptance Criteria

<!-- Given/When/Then format. One block per requirement. -->

### Story #1: --dry-run previews without mutation [core]

```
GIVEN a resolvable `.source` in ~/.claude/.skills-templates.json
WHEN  the operator runs `scripts/post-land-sync.sh --dry-run`
THEN  the script prints the resolved `.source`, the would-run `git pull --ff-only`
      and `skills-deploy install` commands, and the current collection_version,
      AND makes no change to `.source` or to ~/.claude (no pull, no install)
```

### Story #2: real run reconciles + reports before→after [core]

```
GIVEN a resolvable `.source` that is on `main` with a clean tree
WHEN  the operator runs `scripts/post-land-sync.sh` (no --dry-run)
THEN  the script runs `git -C <.source> pull --ff-only` then
      `<.source>/scripts/skills-deploy install`,
      AND prints collection_version before→after
```

### Story #3: guards refuse a bad `.source` [resilience]

```
GIVEN `.source` is missing, not on `main`, or has a dirty working tree
WHEN  the operator runs `scripts/post-land-sync.sh`
THEN  the script prints a clear warning naming the failed guard
      AND exits non-zero WITHOUT pulling or installing
```

### Story #4: land convention documents the step [observability]

```
GIVEN a reader of CLAUDE.md "CI/CD merge convention"
WHEN  they read the section
THEN  they find (a) the post-merge `./scripts/post-land-sync.sh` step after
      `gh pr merge` + verify MERGED, (b) a subsection on why `gh pr merge`
      bypasses the local post-merge hook, and (c) a note that the step also
      reconciles collection_version drift
```

### Story #5: test exists and is wired in [integration]

```
GIVEN the test suite is run via `./scripts/test.sh`
WHEN  the suite executes
THEN  `tests/post-land-sync.test.sh` runs (it is wired into scripts/test.sh)
      AND it exercises the helper's resolution + guards + --dry-run output
      using a temp fixture / --dry-run so the real ~/.claude is never mutated
```

## Architecture

<!-- High-level system design for this story. -->

```
                 gh pr merge (REMOTE)  ── lands skill PR on main, but
                                          local post-merge hook does NOT fire
                                                       │
operator runs:  ./scripts/post-land-sync.sh [--dry-run]
                                                       │
        ┌──────────────────────────────────────────────────────────────┐
        │ post-land-sync.sh                                              │
        │   1. resolve  _SRC = jq .source ~/.claude/.skills-templates.json
        │   2. guard    _SRC exists?  on main?  clean tree?             │
        │                 └─ no → warn + exit non-zero (no mutation)    │
        │   3. report   collection_version (before)                    │
        │   4. act      git -C "$_SRC" pull --ff-only                  │
        │               "$_SRC/scripts/skills-deploy" install          │
        │   5. report   collection_version (after)                     │
        │   (--dry-run: do 1–3 + echo would-run 4; skip real 4, skip 5)│
        └──────────────────────────────────────────────────────────────┘
                                                       │
                             ~/.claude refreshed + collection_version reconciled

  tests/post-land-sync.test.sh  ── drives 1–3 + --dry-run via temp fixture,
                                    asserts no real ~/.claude mutation
                                          │ wired into
                                    scripts/test.sh
```

### Data Flow

<!-- How data moves through the system for the primary use case. -->

1. Operator merges a skill PR via `gh pr merge` (remote) — local hook does not fire.
2. Operator runs `./scripts/post-land-sync.sh` (optionally `--dry-run`).
3. Helper reads `~/.claude/.skills-templates.json` and resolves `.source` via `jq -r .source`.
4. Helper guards `.source`: directory exists, current branch is `main`, working tree clean. On any failure → warn naming the guard + exit non-zero.
5. Helper records `collection_version` (before) from the manifest.
6. Helper runs `git -C "$_SRC" pull --ff-only`, then `"$_SRC/scripts/skills-deploy" install` (skipped under `--dry-run`, which echoes the would-run commands instead).
7. Helper records `collection_version` (after) and prints the before→after delta.

## Tradeoffs

<!-- Design decisions made in this spec. -->

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| How to reconcile after a remote merge | A `post-land-sync.sh` helper (pull + install + version report in one command) | Doc-only "run `git pull && skills-deploy install`" reminder line in CLAUDE.md | The helper encodes the correct pull-checkout + install-from-where once; it's testable (`--dry-run`) and dogfoodable against the live drift. The reminder is two commands the operator must remember + get right. |
| `.source` pull safety | `git pull --ff-only` behind exists/on-main/clean-tree guards; refuse otherwise | Best-effort `git pull` (possibly with merge/rebase) | Constraint: never force, never clobber local `.source` state. `--ff-only` + guards fail safely with a clear message instead of producing a surprise merge commit or a dirty-tree conflict. |
| Trigger model | Explicit operator command (or documented step) | Automatic hook on `gh pr merge` | Constraint: must NOT auto-mutate `~/.claude`. An explicit command keeps the operator in control. |
| Test isolation | Drive resolution + guards + `--dry-run` via a temp fixture; assert no real `~/.claude` mutation | Run the real install in the test | The test must never mutate the operator's real `~/.claude`; `--dry-run` + fixtures give coverage without side effects. |
