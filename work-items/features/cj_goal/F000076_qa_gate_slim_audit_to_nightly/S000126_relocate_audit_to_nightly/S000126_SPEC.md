## Problem Statement

Every orchestrated `CJ_goal_*` build (feature, task, defect, todo_fix) runs an
inline agent-judged post-sync audit (Step 5.6 â€” a combined `/CJ_doc_audit` +
`/CJ_test_audit` subagent) plus a QA-audit checkpoint AUQ (Step 3.4/4.5/8.5),
gated by the `DEFER_AUDIT: true` handoff and carrying the `halted_at_qa_audit`
halt + the `[qa-audit-declined]` / `[qa-audit-waived]` markers. This costs ~5-8
min on every build's critical path. It is removable because its Stage 1 is
deterministic and already runs per-PR in CI (the engine calls ARE `validate.sh`
Checks 15-19/24/26/27/28) and its Stage 2/3 is already advisory (never flips QA
red). The operator (a workbench maintainer) wants the advisory audit OFF the hot
path â€” relocated to a CI-nightly Claude sweep of `main` that files findings â€”
while the deterministic per-PR gate stays unchanged.

## Mental Model

Audit enforcement has two halves today: (1) Stage 1 â€” DETERMINISTIC, already the
per-PR hard gate (`validate.sh` + `validate.yml` + pre-commit), kept exactly; and
(2) Stage 2/3 â€” AGENT-JUDGED + ADVISORY, currently run inline at Step 5.6 behind a
checkpoint, moved by this story to a nightly `claude --print` job. `DEFER_AUDIT:
true` is the switch that makes QA skip its inline 8.6c/8.6d audit; it STAYS, its
meaning shifting from "orchestrator re-runs post-sync" to "nightly CI covers it."
Removing half (2) from the four orchestrator paths leaves half (1) intact, so
nothing structural can slip a per-PR gate; a doc/test drift is instead caught by
the next nightly run (within-24h). The nightly job copies the `eval-nightly.yml`
shape (cron + `workflow_dispatch`, secret pre-check, budget cap, `SKIP` without a
key), so it never spends model budget on a normal `test.sh` / a secret-less fork.

### Story #1: Inline audit + checkpoint removed from the four paths [core]

```

      QA-audit checkpoint (feature 3.4 / task 4.5 / defect 8.5 / todo_fix
```

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Fate of the agent-judged audit | Relocate to CI-nightly | Delete it entirely | Deleting loses the advisory drift-catch that has real value on a doc-first workbench; nightly keeps it while taking it off the hot path. |
| Fate of the agent-judged audit | Relocate to CI-nightly | Keep it inline (status quo) | The status quo is the ~5-8 min + machinery being removed; keeping it inline defeats the goal. |
| `DEFER_AUDIT: true` | Keep + repurpose as the skip-inline switch | Remove it and re-enable QA's inline 8.6c/8.6d for orchestrator runs | Re-enabling QA's inline audit moves the ~5-8 min into QA rather than removing it from the build; keeping DEFER_AUDIT is minimal-surface and preserves the QA RESULT contract. |
| Nightly findings surface | One create-or-update `audit-drift` GitHub issue | A blocking gate / a failing check | An issue matches the advisory posture â€” the maintainer watches it (surfaced in the job summary); a blocking gate would re-introduce the hard-gate coupling this change removes. |
| Decomposition | Single atomic user-story | Multi-story split | The file inventory must land together to keep the pre-commit + CI gates green; intermediate states would be red. |
