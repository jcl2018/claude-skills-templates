## Problem Statement

Each cj_goal orchestrator (feature/defect/task/todo) runs QA → checkpoint → doc-sync. The
checkpoint must instead decide on post-sync docs. This requires moving doc-sync ahead of the
audit + checkpoint, adding an automated pre-doc-sync commit (so doc-sync's `[doc-sync-red]`
uncommitted-non-doc refusal never fires during an autonomous build — the F000038 gotcha),
adding an orchestrator-level post-sync read-only audit, and re-pointing the checkpoint — all
applied per file against each pipeline's distinct commit topology.

## Mental Model

New sequence (replaces QA → checkpoint → doc-sync):

```
implement (no commit) ▸ QA (8.6a/8.6b inline; audit DEFERRED) ▸ pre-doc-sync COMMIT (idempotent)
   ▸ doc-sync (/CJ_document-release) ▸ post-sync AUDIT (one combined read-only subagent)
   ▸ QA-audit checkpoint (operator Continue/Halt on POST-sync audit) ▸ portability ▸ /ship ▸ PR
```

The audit moves from inside the QA subagent (qa.md 8.6c/8.6d) to the orchestrator level
(depth ≤ 2: orchestrator → audit subagent), enabled by S000106's `DEFER_AUDIT: true` signal.

## Acceptance Criteria

### Story #1: Pre-doc-sync commit per pipeline [core]

```
GIVEN QA is green and the tree has uncommitted code + 8.6a/8.6b overlay writes
WHEN  the orchestrator reaches the new pre-doc-sync commit step
THEN  it commits the changes (idempotent: skips when the tree is already clean at HEAD), placed against that pipeline's existing commit topology
```

### Story #2: doc-sync before audit + checkpoint [core]

```
GIVEN the reordered pipeline
WHEN  the orchestrator runs
THEN  /CJ_document-release (doc-sync) executes BEFORE the doc/test audit and the QA-audit checkpoint
```

### Story #3: Post-sync audit once, read-only [core]

```
GIVEN doc-sync has completed
WHEN  the orchestrator runs the post-sync audit step
THEN  it dispatches /CJ_doc_audit + /CJ_test_audit as ONE combined depth-2 fresh-context subagent, READ-ONLY (writes no overlay/doc fixes)
```

### Story #4: Checkpoint consumes the post-sync report [core]

```
GIVEN the post-sync audit has produced a report
WHEN  the operator reaches the QA-audit checkpoint
THEN  the checkpoint surfaces the POST-sync audit; the AUQ text, [qa-audit-waived]/[qa-audit-declined] journal lines, and halted_at_qa_audit end-state are unchanged in meaning
```

### Story #5: QA does not also audit pre-sync [integration]

```
GIVEN the orchestrator dispatches /CJ_qa-work-item
WHEN  the dispatch prompt is composed
THEN  it embeds the literal `DEFER_AUDIT: true` directive, so QA defers 8.6c/8.6d and the audit runs only once, post-sync
```

### Story #6: Resume is double-commit-safe [resilience]

```
GIVEN a resume after the pre-doc-sync commit already ran
WHEN  the orchestrator re-enters the commit step on a clean tree
THEN  it skips the commit (idempotent), and re-runs doc-sync → audit → checkpoint without skipping
```

## Architecture

```
orchestrator (depth 1)
  ├── dispatch /CJ_qa-work-item  [prompt carries DEFER_AUDIT: true]   (depth 2 subagent — QA)
  ├── pre-doc-sync COMMIT  (idempotent: git diff --quiet → skip)
  ├── doc-sync  /CJ_document-release  (inline)
  ├── post-sync AUDIT  → ONE combined subagent: /CJ_doc_audit + /CJ_test_audit  (depth 2, read-only)
  └── QA-audit checkpoint  (operator Continue/Halt on post-sync report)
        → /ship → PR
```

### Data Flow

1. Orchestrator dispatches QA with `DEFER_AUDIT: true`; QA runs smoke/E2E + 8.6a/8.6b, defers 8.6c/8.6d, returns `AUDITS=deferred`.
2. Orchestrator runs the idempotent pre-doc-sync commit (skips on a clean tree).
3. Orchestrator runs doc-sync (`/CJ_document-release`), folding doc updates into the same commit/PR.
4. Orchestrator dispatches ONE combined read-only audit subagent (`/CJ_doc_audit` + `/CJ_test_audit`) over the post-sync tree.
5. The QA-audit checkpoint surfaces the post-sync report; operator Continue/Halt.
6. portability gate → `/ship` (commits the tracker journal lines, opens PR) → PR.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Audit location | Orchestrator level (depth ≤ 2) | Keep inside the QA subagent | A subagent cannot spawn subagents; orchestrator-level enables fresh-context audit dispatch and consumes post-sync state |
| Audit subagents | One combined (doc + test) | Two separate | Two would double the cost C-i exists to avoid |
| Commit placement | Per-file against each pipeline's topology | One symmetric "move the audit" edit across four files | Each pipeline's existing commit points differ |
