## Problem Statement

The cj_goal orchestrators need QA to run its spec-overlay writes (8.6a/8.6b) but DEFER its
three-stage doc/test audits (8.6c/8.6d) so the audit can run later at the post-sync point.
QA must detect this defer intent from the dispatch prompt and skip the audits cleanly,
while standalone `/CJ_qa-work-item` (which has no later doc-sync) keeps the inline audit.

## Mental Model

Step 8.6 has two halves: WRITES (8.6a refresh `spec/test-spec-custom.md`, 8.6b refresh
`spec/doc-spec-custom.md`) and AUDITS (8.6c `/CJ_doc_audit` inline, 8.6d `/CJ_test_audit`
inline). The split makes the WRITES always-inline and the AUDITS conditionally-deferred on
a `DEFER_AUDIT: true` directive in the dispatch prompt.

## Acceptance Criteria

### Story #1: Overlay writes always inline [core]

```
GIVEN a green QA path (orchestrator-driven OR standalone)
WHEN  QA reaches Step 8.6
THEN  8.6a (refresh spec/test-spec-custom.md) and 8.6b (refresh spec/doc-spec-custom.md) run inline regardless of any defer directive
```

### Story #2: Audits defer on directive [core]

```
GIVEN the QA dispatch prompt contains the literal string `DEFER_AUDIT: true`
WHEN  QA reaches Step 8.6c/8.6d
THEN  QA skips 8.6c/8.6d, emits NO AUDIT_FINDINGS block, and the RESULT's AUDITS= field reads `deferred`
```

### Story #3: Standalone keeps inline audit [core]

```
GIVEN a standalone /CJ_qa-work-item run (no `DEFER_AUDIT: true` directive)
WHEN  QA reaches Step 8.6c/8.6d
THEN  QA runs /CJ_doc_audit + /CJ_test_audit inline and emits the AUDIT_FINDINGS fenced block, exactly as today
```

### Story #4: Deferral is observable [observability]

```
GIVEN an orchestrator-driven deferred QA run
WHEN  the orchestrator parses the QA RESULT
THEN  the AUDITS= field reads `deferred` so the orchestrator knows it owns the post-sync audit
```

## Architecture

```
QA Step 8.6 (every green path)
  ├── 8.6a refresh spec/test-spec-custom.md   ── ALWAYS inline
  ├── 8.6b refresh spec/doc-spec-custom.md     ── ALWAYS inline
  └── 8.6c/8.6d doc/test audits
        ├── prompt has `DEFER_AUDIT: true` ─── SKIP → RESULT AUDITS=deferred, no AUDIT_FINDINGS
        └── no directive (standalone) ──────── RUN inline → emit AUDIT_FINDINGS, AUDITS=<doc:..,test:..>
```

### Data Flow

1. Orchestrator dispatches `/CJ_qa-work-item` as an Agent-tool subagent with `DEFER_AUDIT: true` in the prompt.
2. QA runs smoke/E2E, then Step 8.6a/8.6b overlay writes inline.
3. QA checks the dispatch prompt for the literal `DEFER_AUDIT: true`; present → skip 8.6c/8.6d, set `AUDITS=deferred`.
4. QA RESULT carries `AUDITS=deferred` and no `AUDIT_FINDINGS` block; the orchestrator runs the post-sync audit itself (S000107).
5. Standalone path: no directive → QA runs 8.6c/8.6d inline + emits `AUDIT_FINDINGS`.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Defer signal carrier | Literal `DEFER_AUDIT: true` in the dispatch prompt | An argv `--defer-audit` flag | QA is dispatched as a subagent prompt, not a CLI with argv; a literal string is greppable in the pipeline.md templates |
| Granularity of the defer | Only 8.6c/8.6d defer | Defer all of 8.6 | Overlay writes must stay pre-sync so doc-sync folds them into the PR |
