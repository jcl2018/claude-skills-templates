## Problem Statement

A headless cj_goal run blocks at the first human-gate `AskUserQuestion` — the
autonomy ceiling — so the autonomous BUILD cannot be driven unattended even
locally. The operator (and any later harness) needs a way to auto-answer ONLY the
cj_goal *build* gates (design-gate, qa-audit) under a hard guard, dormant by
default, fully deterministic and unit-testable, that can NEVER become an auto-ship
path. This story builds that seam and its non-activation proof.

## Mental Model

The seam has two halves because the gates are agent-emitted `AskUserQuestion`
calls (prose in pipeline.md), not shell — a shell helper cannot suppress them:

```
            cj_goal pipeline reaches a build gate (design-gate / qa-audit)
                                   |
                                   v
   (a) run scripts/cj-e2e-gate.sh --gate <id> [--digest <...>]   <- pure, deterministic
                                   |
              prints exactly: AUTO=continue | halt | inactive
                                   |
   (b) agent-prose branches on the verdict:
        continue -> skip the AUQ + print [E2E-AUTO] banner + proceed
        halt     -> [qa-audit-declined]
        inactive -> fire the AUQ unchanged  (the default / normal-run path)
```

The helper is active ONLY when the double guard holds (`CJ_GOAL_E2E_AUTO=1` AND
`.cj-e2e-sandbox` present at repo root); otherwise every gate returns `inactive`
and the pipeline behaves exactly as today.

## Acceptance Criteria

<!-- Given/When/Then. One block per requirement or logical group. -->

### Story #1: Deterministic verdict helper [core]

```
GIVEN scripts/cj-e2e-gate.sh exists and is executable
WHEN  it is invoked with --gate qa-audit (and a --digest)
THEN  it prints exactly one line of the form AUTO=continue|halt|inactive and exits 0
```

### Story #2: Dormant under an incomplete guard [security]

```
GIVEN the helper is invoked for an allowlisted gate
WHEN  only CJ_GOAL_E2E_AUTO=1 is set (no marker), OR only .cj-e2e-sandbox exists (no flag)
THEN  the helper prints AUTO=inactive (a normal run is behavior-unchanged)
```

### Story #3: Build-gates-only allowlist [security]

```
GIVEN BOTH CJ_GOAL_E2E_AUTO=1 AND .cj-e2e-sandbox are present
WHEN  the helper is invoked with a gate id outside {design-gate, qa-audit} (e.g. a ship/merge/deploy id)
THEN  the helper prints AUTO=inactive (it never matches a ship/merge/deploy marker)
```

### Story #4: Green-only continue, halt on findings [core]

```
GIVEN BOTH guards hold and --gate qa-audit
WHEN  the digest is green (doc:ok,test:ok)            THEN AUTO=continue
WHEN  the digest carries findings                     THEN AUTO=halt  (never auto-waive)
```

### Story #5: Four pipelines branch on the helper [integration]

```
GIVEN the four cj_goal pipelines
WHEN  a pipeline reaches its qa-audit AUQ step (and CJ_goal_feature its design-gate step)
THEN  it runs the helper first and branches: continue -> skip the AUQ + print [E2E-AUTO] banner + proceed;
      halt -> [qa-audit-declined]; inactive -> fire the AUQ unchanged
```

### Story #6: Marker cannot ship [security]

```
GIVEN .cj-e2e-sandbox is .gitignore'd
WHEN  validate.sh runs and .cj-e2e-sandbox is in the tracked tree
THEN  validate.sh HARD-fails (the marker-absence check)
```

### Story #7: Verdict matrix asserted (no Claude) [observability]

```
GIVEN tests/cj-e2e-gate.test.sh
WHEN  scripts/test.sh runs it
THEN  it asserts flag-only->inactive, marker-only->inactive, both+green->continue,
      both+findings->halt, non-allowlisted-gate->inactive — all deterministically, no Claude
```

## Architecture

<!-- High-level system design for this story. -->

```
  scripts/cj-e2e-gate.sh  (pure verdict function)
     guard:     CJ_GOAL_E2E_AUTO=1  AND  -f <repo-root>/.cj-e2e-sandbox
     allowlist: gate in {design-gate, qa-audit}  (else -> inactive)
     digest:    green (doc:ok,test:ok) -> continue ; findings -> halt
     output:    one line  AUTO=continue|halt|inactive
        ^
        | (run first, branch on stdout)
        |
  skills/CJ_goal_{feature,task,defect,todo_fix}/pipeline.md
     qa-audit AUQ step  (+ design-gate step in CJ_goal_feature only)
        continue -> skip AUQ + print [E2E-AUTO] banner + proceed
        halt     -> [qa-audit-declined]
        inactive -> fire AUQ unchanged

  scripts/validate.sh   -> marker-absence check (HARD-fail if .cj-e2e-sandbox tracked)
  scripts/test.sh       -> runner block for tests/cj-e2e-gate.test.sh + zzz-test-scaffold fixture row
  .gitignore            -> .cj-e2e-sandbox
  spec/test-spec-custom.md -> family: test units row for the new test
  docs/tests/test-hierarchy.md -> one-line seam-test note
```

### Data Flow

<!-- How data moves through the system for the primary use case. -->

1. A cj_goal pipeline reaches a build gate (design-gate or qa-audit).
2. The pipeline prose runs `scripts/cj-e2e-gate.sh --gate <id> [--digest <...>]`.
3. The helper checks the double guard (`CJ_GOAL_E2E_AUTO=1` AND `.cj-e2e-sandbox`); if incomplete → prints `AUTO=inactive`.
4. With the guard satisfied, it checks the allowlist; a non-`{design-gate, qa-audit}` id → `AUTO=inactive`.
5. For an allowlisted gate, it applies the green-only predicate to the digest: green → `AUTO=continue`; findings → `AUTO=halt`.
6. The pipeline branches on the printed verdict: `continue` → skip AUQ + `[E2E-AUTO]` banner + proceed; `halt` → `[qa-audit-declined]`; `inactive` → fire the AUQ unchanged.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| How to suppress an agent AUQ | Verdict helper (shell) + agent-prose branch in pipeline.md | A pure shell suppressor | The gates are agent-emitted AUQ calls; shell cannot suppress them, so the logic lives in a unit-testable helper and the prose branches on it. |
| Auto-continue predicate | Reuse `todo_fix --quiet`'s green-only predicate, widened to two triggers | A new bespoke predicate | One path, two triggers preserves the autonomy ceiling (never auto-waive findings) and reduces surface area. |
| Helper home | Standalone `scripts/cj-e2e-gate.sh` | A `cj-goal-common.sh` phase | Simplest to unit-test as a pure deterministic verdict function. |
| Guard shape | Double hard guard (`CJ_GOAL_E2E_AUTO=1` AND `.cj-e2e-sandbox`) + gitignore + validate marker-absence check | A single env flag | Two independent conditions + a marker that cannot ship makes accidental activation in a real repo impossible. |
