## Problem Statement

Part A shipped the dormant build-gate auto-answer seam but changes no real run's
behavior. To actually PROVE the autonomous cj_goal BUILD runs end to end — and to
make that proof legible rather than a bare checkmark — the workbench needs a LOCAL
harness that drives a real `/CJ_goal_task` build through the seam in a sandbox,
stops at the `/ship` boundary, and writes a materialized report separating the
DETERMINISTIC guarantees from the `claude --print` model run. The harness must be
discoverable as workbench machinery (workflow docs), and it must SKIP cleanly in CI
(no gstack, no key, no budget there).

## Mental Model

```
  scripts/e2e-local.sh   (gated on CJ_E2E_LOCAL=1; SKIP-with-reason otherwise)
     |
     |-- pre-flight: CJ_E2E_LOCAL=1 + gstack + ANTHROPIC_API_KEY + claude + gh   (else SKIP)
     |-- lib/sandbox.sh: mktemp clone + .cj-e2e-sandbox marker + LOCAL bare origin
     |-- run: claude --print "/CJ_goal_task <tiny non-sensitive non-doc topic>"
     |         (broad tools Write,Edit,Agent,Skill; CJ_GOAL_E2E_AUTO=1; budget+timeout)
     |-- collect post-run EVIDENCE (grep the sandbox): work-items/tasks/T*/, diff, end_state
     |-- assert /ship-boundary pair: bare origin got the branch AND gh pr create blocked
     |-- lib/report.sh: write reports/<verb>-<ts>.md + .json  (DETERMINISTIC vs claude-print)
     `-- teardown: rm -rf the tmpdir; echo the report path
```

The report's value: the operator sees EXACTLY which guarantees are deterministic
(the seam logic + the boundary checks, asserted in shell) vs which depend on a real
model run (the build itself), the exact task tested, and where the run stopped —
never a bare ✓.

## Acceptance Criteria

<!-- Given/When/Then. One block per requirement or logical group. -->

### Story #1: The harness SKIPs cleanly with no prerequisites [resilience]

```
GIVEN scripts/e2e-local.sh exists and is executable
WHEN  it runs with CJ_E2E_LOCAL unset (as in CI / a normal test.sh)
THEN  it prints a one-line SKIP reason, exits 0, and never invokes claude
```

### Story #2: Sandbox provisions + tears down deterministically [core]

```
GIVEN tests/e2e-local/lib/sandbox.sh is sourced
WHEN  e2e_sandbox_provision runs
THEN  a mktemp clone exists with a .cj-e2e-sandbox marker at its root AND a LOCAL
      bare origin repointed as 'origin'
WHEN  e2e_sandbox_teardown runs
THEN  the tmpdir is removed
```

### Story #3: The report generator labels rows by real evidence [observability]

```
GIVEN the report generator is fed a run's evidence (topic, sandbox path, end_state,
      whether a work-items/tasks/T*/ dir appeared, whether the diff is non-empty,
      whether the bare origin received the branch, whether gh pr create was blocked)
WHEN  it renders the report
THEN  a reports/<verb>-<ts>.md is written with a coverage table whose rows are each
      labelled DETERMINISTIC or `claude --print`, a legend, and a cost/time line,
      AND a sibling reports/<verb>-<ts>.json carries the same data
```

### Story #4: Report artifacts are gitignored except the sample [security]

```
GIVEN tests/e2e-local/reports/ is gitignored with an un-ignored !EXAMPLE.md
WHEN  git status runs after a harness run
THEN  the generated reports/<verb>-<ts>.md/.json are ignored AND reports/EXAMPLE.md
      is a tracked, committed file
```

### Story #5: Deterministic smoke asserts the harness's non-model half [observability]

```
GIVEN tests/e2e-local.test.sh
WHEN  scripts/test.sh runs it
THEN  it asserts (no Claude): the SKIP path (flag unset -> exit 0, no claude call),
      sandbox provision/teardown, the report generator on synthetic evidence, and
      EXAMPLE-tracked/reports-ignored — wired as a units: row + a test.sh runner block
```

### Story #6: The harness is discoverable in the workflow docs [usability]

```
GIVEN spec/workflow-spec.md's utilities-and-phase-steps roster
WHEN  a ### scripts/e2e-local.sh subsection is added and workflow-spec.sh --render-docs runs
THEN  docs/workflows/utilities-and-phase-steps.md contains the harness entry AND
      validate.sh Check 27 (workflow-docs freshness) is green
```

### Story #7: The test-hierarchy explainer is updated [usability]

```
GIVEN docs/tests/test-hierarchy.md
WHEN  the full-happy-path-E2E layer is described
THEN  it reads as a local-only real run via the seam that emits a materialized
      report (deterministic vs claude-print), replacing the "blocked/deferred" framing
```

### Story #8: Whole repo stays green [core]

```
GIVEN the Part B/C changes are in the tree
WHEN  scripts/validate.sh and scripts/test.sh run
THEN  validate.sh reports 0 errors AND test.sh is green (the e2e-local family SKIPs)
```

### Data Flow

<!-- How data moves through the system for the primary use case. -->

1. `scripts/e2e-local.sh` pre-flight: unless `CJ_E2E_LOCAL=1` AND gstack + `ANTHROPIC_API_KEY` + `claude` + `gh` are present, it SKIPs with a one-line reason (exit 0).
2. It provisions a sandbox via `lib/sandbox.sh`: a `mktemp` clone of the repo, a `.cj-e2e-sandbox` marker at the clone root, and a LOCAL bare origin repointed as `origin`.
3. It runs `claude --print "/CJ_goal_task <topic>"` in the sandbox with a broad tool grant, `CJ_GOAL_E2E_AUTO=1`, `--plugin-dir <sandbox>/skills`, and a budget + timeout.
4. It collects post-run evidence by grepping the sandbox: did a `work-items/tasks/T*/` dir appear, is the diff non-empty, what is the run's `end_state`, did the bare origin receive the branch, was `gh pr create` blocked.
5. It asserts the `/ship`-boundary pair (bare origin got the branch AND no real PR) and treats `halted_at_ship` as the sandbox SUCCESS terminal.
6. `lib/report.sh` renders `reports/<verb>-<ts>.md` + `.json`, labelling each coverage row DETERMINISTIC (asserted in shell) or `claude --print` (from the real run) by the collected evidence.
7. The sandbox is torn down and the report path is echoed.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Where the "does it really work" proof lives | Local-only real run (flag-gated), CI SKIPs | CI-automated E2E | CI is blocked four ways (gstack, read-only tools, budget, AUQs); the honest proof is a local real run. |
| Report row trust model | Grep-backed real post-run evidence per row | A hand-assigned pass template | A templated ✓ proves nothing; only rows verified against the actual sandbox state are "verified." |
| Auto-ship backstop | LOCAL bare origin (accepts push, defeats `gh pr create`) | Relying on an AUQ to refuse | `task`'s `/ship` diff-review AUQ is already suppressed, so the no-remote origin is the sole stop and is a tested invariant. |
| Workflow-docs placement | Extend the existing `utilities-and-phase-steps` roster body | A new roster section | A new section orphans an undeclared `docs/workflows/*.md` (Check 15a ERROR) and breaks the "six docs/workflows" header prose. |
| Report artifacts | Gitignored, with one committed `EXAMPLE.md` | Commit every run report | Reports are per-run artifacts; a single sample keeps the format reviewable without churn. |
