## Problem Statement

The four cj_goal orchestrators (`/CJ_goal_feature`, `/CJ_goal_task`,
`/CJ_goal_defect`, `/CJ_goal_todo_fix`) each emit a land/PR recap as hand-authored
prose, so the recap shape drifts between verbs and runs. The operator — who reads
the recap to know what shipped, how to confirm it, and what to do next — gets an
inconsistent surface. The orchestrators need a single shared formatter that
renders a uniform 3-part block, so the only thing that varies is the
change-specific content (which the agent authors), not the shape.

## Mental Model

The `recap` phase is a **pure formatter** on the existing `cj-goal-common.sh`
dispatcher — the same script that already hosts `--phase telemetry`,
`--phase sync`, `--phase portability-audit`, etc. It has two structural inputs:

- `--when {before|after}` selects the header line.
- three `--field` values (`delivered`, `e2e`, `next`) supply the body sections.

```
agent (authors content) ──▶ cj-goal-common.sh --phase recap --when X --field … ──▶ 3-part block on stdout
                                          (no state, no telemetry, no mutation, exit 0)
```

## Acceptance Criteria

### Story #1: 3-part labelled block [core]

```
GIVEN cj-goal-common.sh with the recap phase implemented
WHEN  invoked as `--phase recap --when before --field delivered=D --field e2e=E --field next=N`
THEN  stdout contains a header line, then the labelled sections Delivered (with D), How to E2E-test it (with E), and Next step (with N)
AND   the tail emits PHASE=recap and PHASE_RESULT=ok with exit code 0
```

### Story #2: --when header switch [core]

```
GIVEN the recap phase
WHEN  invoked with --when before vs --when after
THEN  the header differs ("About to land …" for before; "Landed …" / "PR opened …" for after)
AND   the three labelled body sections are present in both cases
```

### Story #3: telemetry --field reuse [integration]

```
GIVEN the recap phase reuses the telemetry repeatable --field KEY=VALUE parsing
WHEN  a --field value contains spaces / special characters
THEN  the value is rendered verbatim in its section (no eval, no truncation)
```

### Story #4: fail-soft on missing field [resilience]

```
GIVEN the recap phase
WHEN  invoked with one or more of delivered/e2e/next omitted
THEN  the corresponding section renders empty (no error)
AND   PHASE_RESULT=ok and exit code 0 (the phase never halts, mutates nothing, writes no telemetry)
```

### Story #5: test registered [observability]

```
GIVEN tests/cj-goal-common-recap.test.sh exists
WHEN  scripts/test-spec.sh --validate and --check-coverage run
THEN  the test resolves to exactly one spec/test-spec-custom.md units row (forward + reverse sweep pass)
AND   scripts/test.sh runs the new test and it is green
```

### Story #6: usage block updated [usability]

```
GIVEN the cj-goal-common.sh header-comment usage block
WHEN  a reader inspects the script
THEN  the --phase {…} list includes recap and --when is documented
```

## Architecture

```
                 cj-goal-common.sh  (existing shared dispatcher)
                 ┌───────────────────────────────────────────────┐
  --phase recap  │  arg parse: --when, --mode, repeatable --field │
  --when X  ────▶│  (--field parser REUSED from telemetry phase)  │
  --field …      │                                                │
                 │  recap) case:                                  │
                 │    print header(--when)                        │
                 │    print "Delivered:"        + field[delivered]│──▶ stdout (3-part block)
                 │    print "How to E2E-test it:"+ field[e2e]     │
                 │    print "Next step:"        + field[next]     │
                 │    print PHASE=recap / PHASE_RESULT=ok ; exit 0│
                 └───────────────────────────────────────────────┘
                 (no state, no telemetry write, no mutation)
```

### Data Flow

1. An orchestrator (the agent) authors `delivered`, `e2e`, `next` content for THIS change.
2. It invokes `cj-goal-common.sh --phase recap --when {before|after} --mode <verb> --field delivered=… --field e2e=… --field next=…`.
3. The recap case parses `--when`, `--mode`, and the repeatable `--field` values.
4. It prints the header + the three labelled sections to stdout, then `PHASE=recap` / `PHASE_RESULT=ok`, and exits 0.
5. The orchestrator relays the block to the operator. No state is written.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Where the content comes from | Agent authors content, passes via `--field` | Helper computes content (parses git/PR state) | The helper cannot know the change-specific E2E commands; keeping it a pure formatter avoids brittle git/PR parsing and keeps the shape uniform. |
| `--field` parsing | Reuse the telemetry phase's parser | Write a fresh parser | Reuse is less code and inherits the safe verbatim (no-eval) printing already proven. |
| Failure mode | Fail-soft (empty section, exit 0) | Error on missing field | Advisory posture — a recap must never block a land. |
