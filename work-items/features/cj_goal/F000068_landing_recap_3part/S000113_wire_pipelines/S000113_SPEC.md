## Problem Statement

The `--phase recap` formatter (S000112) is inert until each of the four cj_goal
pipelines invokes it at its land/PR-stop step, and until the `CLAUDE.md`
`## Post-land recap` convention is reframed to describe the new before+after
3-part shape and name the helper. Without this story, the recap shape stays
hand-authored per orchestrator (the original drift). The operator who reads the
recap to know what shipped, how to confirm it, and what to do next needs the
uniform block wired in at every cj_goal land/PR point.

## Mental Model

Two classes of cj_goal verb, with different recap placement:

```
LANDING verbs (land in-pipeline)            PR-STOP verbs (stop at the PR)
  defect   : before-recap ──▶ land ──▶ after   feature : one at-PR recap (reshape Step 6.5 terminal block)
  todo_fix : before-recap ──▶ land ──▶ after   task    : one at-PR recap (reshape Step 7 terminal block)
             (around /ship → /land-and-deploy,
              per drained TODO)
```

Each recap call is a thin invocation of `cj-goal-common.sh --phase recap` with a
documented prose fallback for an absent helper. Upstream `/land-and-deploy` is
never edited — the before/after calls bracket it in this repo's pipeline.md.

## Acceptance Criteria

### Story #1: PR-stop verbs at-PR recap [core]

```
GIVEN skills/CJ_goal_feature/pipeline.md and skills/CJ_goal_task/pipeline.md
WHEN  the terminal STOP-at-PR block (feature Step 6.5, task Step 7) is reshaped
THEN  each calls cj-goal-common.sh --phase recap (single at-PR recap) rendering Delivered / How to E2E-test it / Next step
AND   grep finds the recap pointer at the terminal step in both files
```

### Story #2: landing verbs before+after recap [core]

```
GIVEN skills/CJ_goal_defect/pipeline.md and the todo_fix verb's pipeline.md/SKILL.md
WHEN  the land step is wired (defect around Step 10; todo_fix around the /ship → /land-and-deploy tail, per drained TODO)
THEN  a before-recap precedes the land and an after-recap follows it in both
AND   grep finds both recap pointers around the land step
```

### Story #3: CLAUDE.md convention reframed [usability]

```
GIVEN CLAUDE.md ## Post-land recap
WHEN  reframed
THEN  it describes the 3-part before+after land/PR recap, names cj-goal-common.sh --phase recap as producer,
      states the agent authors delivered/e2e/next, and keeps "advisory, never blocks, no validate.sh check asserts it fired"
AND   the cj-goal-common.sh Scripts-reference row mentions the new recap phase
```

### Story #4: prose fallback documented [resilience]

```
GIVEN each recap call site in the four pipelines
WHEN  the helper is absent/unreachable
THEN  the pipeline prose documents falling back to emitting the 3-part block as prose (no halt)
```

### Story #5: upstream untouched [integration]

```
GIVEN the wiring
WHEN  the diff is inspected
THEN  no change is made to the upstream /land-and-deploy skill; the recap calls live only in this repo's pipeline.md files
```

### Story #6: docs in sync [observability]

```
GIVEN the cj-goal-common.sh Scripts-reference row and any docs/workflow(s) Touches block enumerating cj-goal-common phases
WHEN  this story lands
THEN  the recap phase is listed in the Scripts-reference row and in any such Touches block
AND   scripts/validate.sh stays green
```

## Architecture

```
  CJ_goal_feature/pipeline.md  (Step 6.5 terminal) ──┐
  CJ_goal_task/pipeline.md     (Step 7 terminal)   ──┤  one at-PR recap each
                                                     │       ┌──────────────────────────────┐
  CJ_goal_defect/pipeline.md   (around Step 10)    ──┼─────▶ │ cj-goal-common.sh --phase     │──▶ stdout
  CJ_goal_todo_fix pipeline    (around ship/land)  ──┘       │   recap --when before|after … │   (3-part block)
                                                     before+after pair  └──────────────────────────────┘
                                                                          (absent ⇒ prose fallback)
  CLAUDE.md ## Post-land recap  ──▶ reframed convention + Scripts-reference row
  docs/workflow(s) Touches      ──▶ + recap (conditional)
```

### Data Flow

1. A cj_goal orchestrator reaches its land/PR-stop step.
2. The agent authors `delivered`/`e2e`/`next` for THIS change.
3. The pipeline invokes `cj-goal-common.sh --phase recap --when {before|after} --mode <verb> --field …` (or, if the helper is absent, emits the same 3-part block as prose).
4. The block prints to the operator. Landing verbs do this twice (around the land); PR-stop verbs do it once (at the PR).

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Recap count per verb class | before+after for landing verbs; one at-PR for PR-stop verbs | before+after for all four | PR-stop verbs never land in-pipeline; a before/after pair around a land that doesn't happen is meaningless. |
| Where the calls live | This repo's pipeline.md, bracketing the land | Inside `/land-and-deploy` | `/land-and-deploy` is untouchable upstream gstack. |
| Posture | Advisory + prose fallback | A validate.sh presence-check | Parent decision: the recap is a courtesy, not a correctness gate. |
