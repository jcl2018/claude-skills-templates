## Problem Statement

The workbench's workflow documentation — the `docs/workflow.md` index plus the six
`docs/workflows/*.md` files (the four `CJ_goal_*` orchestrator pages + the two
prose rosters `utilities-and-phase-steps.md` and `utility-audits.md`) — is
hand-authored. The four orchestrator pages each carry an ASCII workflow chart and
a four-bullet **Touches** block (skills / steps / scripts / docs); those Touches
blocks drift on every workflow change because they are hand-edited and easy to
forget. `validate.sh` Check 15b (each orchestrator doc has a chart + 4 anchored
Touches bullets) + Check 15c (the index links each) police only the *shape* of
this surface, never its *truth/freshness*: a Touches bullet can be stale and a
chart can be wrong while both checks stay green.

Story 1 (S000114, already shipped) built the generate→freshness→audit primitive
for the test catalog (`spec/test-spec.md` → `test-spec.sh --render-docs` →
`docs/tests/` → Check 26). This story is the THIRD instance of that primitive
(after `README.md` ↔ `generate-readme.sh` ↔ Check 25), applied to the workflow
docs: a new `spec/workflow-spec.md` registry becomes the single source of truth, a
`scripts/workflow-spec.sh` engine renders the entire workflow surface, a
`validate.sh` Check 27 freshness gate regenerates→diffs, and `/CJ_doc_audit` runs
the same freshness check standalone in any repo. Checks 15b/15c retire, their
no-vanish intent folded into the engine's `--validate` registry-completeness + Check 27.

## Mental Model

One primitive, a third time, with a TWO-shape registry:

```
spec/workflow-spec.md (source of truth)
   header block: docs/workflow.md index prose preamble
   ## <name> sections, each one of two kinds:
     orchestrator (4): kind/status/category/source/invoke_when + ```chart``` (verbatim)
                       + 4 Touches axes (skills/steps/scripts/docs) + "In words" summary
     roster (2):       kind: roster + ```body``` (verbatim curated prose)
        │
        ▼  workflow-spec.sh --render-docs   (normalized template, deterministic)
   docs/workflow.md (index over all entries) + docs/workflows/<name>.md (×6)
        │
        ▼  workflow-spec.sh --render-docs --check   (regenerate→temp→diff→exit code)
             ↑ called by BOTH validate.sh Check 27 AND /CJ_doc_audit Stage 1
```

The renderer is PURE: registry in → deterministic docs out. The `--check` mode is
the single freshness owner. Charts, roster bodies, and the index preamble are
stored + emitted verbatim; only structural bits (headers, Touches ordering,
whitespace) may reformat (a normalized template, NOT a strict byte round-trip).
The rendered output must be work-item-ID-free so the human-docs pass Check 19.

## Acceptance Criteria

### Story #1: The two-shape registry exists [core]

```
GIVEN the 6 existing docs/workflows/*.md bodies + the docs/workflow.md intro
WHEN  spec/workflow-spec.md is authored
THEN  it is a bash-parseable structured-Markdown registry (NOT YAML block scalars)
AND   it has a header block holding the docs/workflow.md index prose preamble
AND   it has one `## <name>` section per workflow: 4 orchestrator entries (kind: orchestrator, status, category, source, invoke_when, a verbatim ```chart``` block, the four Touches axes skills/steps/scripts/docs, and an "In words" summary) and 2 roster entries (kind: roster + a verbatim ```body``` block)
AND   the 6 doc bodies + the index intro are migrated into it
```

### Story #2: Engine validate / list / classify / seed [core]

```
GIVEN spec/workflow-spec.md
WHEN  I run scripts/workflow-spec.sh --validate
THEN  it asserts each entry's required fields by kind + a closed `kind` enum and exits 0 on the valid registry
GIVEN a present-but-invalid registry
WHEN  I run --validate
THEN  it emits [workflow-spec-no-config] and exits non-zero
AND   --list-workflows emits every entry name
AND   --classify emits GENERATION=absent|canonical|malformed (+ DUPLICATE) mirroring the other engines
AND   --seed emits a minimal valid skeleton registry (header + contract prose, zero workflow sections)
AND   the engine resolves the registry via git rev-parse --show-toplevel and honors REPO_ROOT / WORKFLOW_SPEC_PATH env overrides
```

### Story #3: Registry-completeness = the no-vanish guarantee [core]

```
GIVEN the registry has an orchestrator entry for every routable CJ_goal_* skill
WHEN  I run --validate
THEN  it exits 0 (completeness satisfied)
GIVEN an orchestrator entry is removed (or a routable CJ_goal_* lacks an entry)
WHEN  I run --validate
THEN  the registry-completeness check fails closed (non-zero) naming the missing workflow
```

### Story #4: Deterministic render of the whole surface [core]

```
GIVEN the valid registry
WHEN  I run scripts/workflow-spec.sh --render-docs
THEN  docs/workflow.md (preamble + an index table over all entries) is written
AND   one docs/workflows/<name>.md is written per entry, to a normalized template
AND   charts + roster bodies + the index preamble are reproduced verbatim
AND   a second --render-docs is byte-identical (stable order, fixed headers)
```

### Story #5: Self-check freshness [observability]

```
GIVEN the on-disk docs/workflow.md + docs/workflows/*.md match a fresh render
WHEN  I run scripts/workflow-spec.sh --render-docs --check
THEN  it renders to a temp dir, diffs vs on-disk, and exits 0
GIVEN a hand-edited (or missing) generated doc
WHEN  I run --render-docs --check
THEN  it exits 1 and names the mismatched/missing file
```

### Story #6: Regenerated, committed, ID-free [integration]

```
GIVEN spec/workflow-spec.md
WHEN  all 6 docs/workflows/*.md + docs/workflow.md are regenerated and committed
THEN  the one-time normalized reformat diff is reviewed in the PR (expected, not a regression)
AND   no rendered file contains a work-item ID matching [FSTD][0-9]{6} (Check 19 green)
AND   the index lists all 6 entries with links and reproduces the preamble verbatim
```

### Story #7: Hard workbench gate + its test fixture; 15b/15c retired [integration]

```
GIVEN the generated workflow surface is on disk
WHEN  scripts/validate.sh runs
THEN  Check 27 regenerates to a temp dir, diffs vs on-disk, and ERRORs on any mismatch/missing file (registry-gated; mirror of Check 26)
AND   scripts/test.sh contains the parallel Check-27 integration fixture (positive PASS + negative drift-fires-ERROR + regenerate-green), mirroring the Check-26 fixture
AND   Checks 15b + 15c are retired, with a one-line pointer comment at the old 15b/15c site pointing to Check 27 + workflow-spec --validate
```

### Story #8: Doc audit owns workflow freshness + generated-surface recognition [integration]

```
GIVEN /CJ_doc_audit runs in any repo carrying the engine
WHEN  Stage 1 executes
THEN  it also runs workflow-spec.sh --render-docs --check and reports a freshness finding if the workflow surface is stale
AND   Stage 3 ground-truth enumeration recognizes docs/workflow.md + docs/workflows/ as a GENERATED surface (sourced from spec/workflow-spec.md), never an orphan/drift
```

### Story #9: Declared in the contract registries [integration]

```
GIVEN the registry + new tests exist
WHEN  the contract registries are updated
THEN  spec/doc-spec-custom.md declares spec/workflow-spec.md as an operational registry doc
AND   docs/workflow.md + docs/workflows/*.md remain declared human-docs (now generated)
AND   spec/test-spec-custom.md has units rows for validate Check 27 + the workflow-spec-render test
AND   doc-spec.sh --check-on-disk reports no orphan/human-doc-id finding and test-spec.sh --check-coverage reverse-sweep resolves the new test(s)
```

### Story #10: Hermetic test incl. the no-vanish drill [core]

```
GIVEN a temp-dir registry fixture
WHEN  tests/workflow-spec-render.test.sh runs
THEN  it asserts --render-docs is deterministic (render→render byte-stable) and ID-free
AND   it asserts --render-docs --check passes on fresh, fails on a hand-edit, and fails on a missing file
AND   a remove-an-entry drill proves --validate registry-completeness fails closed (the retired-15c replacement)
```

### Data Flow

1. The maintainer authors `spec/workflow-spec.md` by migrating the 6 `docs/workflows/*.md` bodies + the `docs/workflow.md` index intro into the two-shape registry (4 orchestrator sections, 2 roster sections, 1 header block).
2. `scripts/workflow-spec.sh --validate` parses the registry, asserts per-kind required fields + a closed `kind` enum, and asserts registry-completeness — every routable `CJ_goal_*` skill (`jq` on `skills-catalog.json`, `!= deprecated`, non-empty `files`) has an `orchestrator` entry.
3. `scripts/workflow-spec.sh --render-docs` renders `docs/workflow.md` (preamble + index table over all entries) + one `docs/workflows/<name>.md` per entry to a normalized template, deterministically (charts + roster bodies + preamble verbatim).
4. The maintainer commits the regenerated docs (one-time normalized reformat reviewed in the PR).
5. `scripts/validate.sh` Check 27 (and `/CJ_doc_audit` Stage 1) call `--render-docs --check`: render to a temp dir, diff vs on-disk; non-zero exit → ERROR / finding. Checks 15b/15c no longer run (retired).
6. `spec/doc-spec-custom.md` declares `spec/workflow-spec.md` (no orphan); `spec/test-spec-custom.md` declares the new test units (reverse-sweep resolves them).
7. `tests/workflow-spec-render.test.sh` proves determinism / ID-freeness / `--check` pass-fail / the no-vanish remove-an-entry drill in CI via `scripts/test.sh`.

## Tradeoffs

| Decision | Chosen | Rejected Alternative | Why |
|----------|--------|---------------------|-----|
| Scope of generation | All 6 docs + the index, truly symmetric (two entry shapes) | Generate only the 4 orchestrator pages; leave the 2 prose rosters + index hand-authored | The operator chose full symmetry so the ENTIRE workflow surface is un-rottable; a partially-generated surface would still drift in the hand-authored half |
| Round-trip fidelity | Normalized template, one-time reviewed reformat | Strict byte round-trip of the existing 6 docs | A strict round-trip would force the registry + template to reproduce every incidental whitespace/heading choice; the operator chose a one-time normalized reformat (charts/rosters/preamble verbatim, structure may shift), reviewed in the PR |
| Registry format | Bash-parseable structured Markdown (sections + fenced blocks) | YAML block scalars | The engine is bash, like `doc-spec.sh`/`test-spec.sh`; structured Markdown + awk section/fenced-block extraction reuses the proven parsing idiom with no new deps |
| No-vanish guarantee | `--validate` registry-completeness (every `CJ_goal_*` has an entry) + Check 27 freshness | Keep Checks 15b/15c (shape-only) | Completeness checks the SOURCE OF TRUTH (stronger than 15c's index-link grep); freshness makes a generated doc un-missable its chart/Touches; 15b/15c only assert shape, never truth |
| Freshness owner | One `--render-docs --check` entry point shared by Check 27 + audit Stage 1 | Separate diff logic in `validate.sh` and in the audit | A single owner means the workbench gate and the portable standalone audit can't disagree (same pattern as Story 1's Check 26) |
| Test fixture timing | Add the `scripts/test.sh` Check-27 fixture in THIS story | Defer the fixture to a follow-up | A new validate check without its parallel test.sh fixture is the recurring implement-subagent blind spot (F000032/34/35); pinning it here closes it |
