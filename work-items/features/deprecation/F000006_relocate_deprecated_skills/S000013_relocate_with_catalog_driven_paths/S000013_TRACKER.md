## Insights

<!-- Non-obvious findings worth remembering. -->

- The catalog-driven refactor is principled (works for any path) but only `company-workflow` exercises the new behavior today. `personal-workflow/` and `system-health/` continue to live at `skills/{name}/`; the resolver returns the same path it always did for them.
- F000005's design rationale on single-story decomposition applies again here: all script edits hit the same files (`skills-catalog.json`, `scripts/skills-deploy`, `scripts/validate.sh`, `scripts/test.sh`), so two stories would create artificial seams.
- Once `skills-deploy` derives the source root from catalog `files[0]` parent, future skill relocations (if any) become a one-line catalog change instead of a multi-script refactor.
