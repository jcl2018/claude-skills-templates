## Problem Statement

The company-workflow skill has templates and validation (S000003) but no way to
create new work items from them. Engineers must manually copy templates and fill
placeholders. The skill needs a `create` subcommand that scaffolds a complete
work item directory with the correct tracker and companion artifacts for the type.

## Mental Model

```
company-workflow create --type feature --name "google-search-clone"
  │
  ├── Reads templates/company-workflow/ (via fallback chain)
  ├── Determines required artifacts for type (feature = 5)
  ├── Generates next ID (scan work-items/ for highest F-prefixed ID)
  ├── Creates work-items/F{next}_{slug}/
  │     ├── F{next}_TRACKER.md   (from tracker-feature.md, placeholders filled)
  │     ├── F{next}_PRD.md       (from doc-PRD.md)
  │     ├── F{next}_ARCHITECTURE.md
  │     ├── F{next}_TEST-SPEC.md
  │     └── F{next}_milestones.md
  └── Validates the tracker against contract.json
```

## Acceptance Criteria

### Story #1-5: Scaffold each type [core]

```
GIVEN the company templates exist at templates/company-workflow/
WHEN  I run company-workflow create --type {type} --name {name}
THEN  a new directory is created under work-items/ with the correct slug
AND   the tracker has all company-required frontmatter fields filled
AND   all required companion artifacts for the type are created
AND   artifact count matches: feature=5, defect=3, task=2, userstory=5, review=2
```

### Story #6: Placeholder substitution [core]

```
GIVEN I scaffold a work item with --name "google-search-clone"
WHEN  I read the tracker frontmatter
THEN  name is "google-search-clone"
AND   id is the next available ID for that type
AND   created and updated are today's date
AND   branch is the current git branch
AND   repo is the current repo name
```

### Story #7: Scaffolded output validates [resilience]

```
GIVEN I scaffold a complete feature work item
WHEN  I run company-workflow validate on the tracker
THEN  exit code is 0
```

## Out of Scope

- Interactive prompting for work item details (batch mode only)
- Scaffolding with pre-filled content from user description (just placeholders)
- Updating existing work items

## Assumptions

- S000003 is complete (templates, registry, SKILL.md with validate exist)
- ID generation scans work-items/ in the current repo
