## Symptom

Two related defects in the `/personal-workflow` skill, surfaced via `portfolio/work-items/features/discord-v1/`:

1. **Unprefixed work-item subdirectories.** The scaffolder names files like `{ID}_TRACKER.md` but names the containing directory just `{slug}/`. Inside `discord-v1/`, all 5 user-story dirs (accounts, market-regime, portfolio, reference, trading-desk) lack the `S00000X_` prefix. The convention is inconsistent at the directory boundary.

2. **Missing hierarchy levels.** `personal-artifact-manifests.json` declares `hierarchy.user-story = {required_child: task, min: 1}`, yet none of the discord-v1 user-stories contain any task subdirectories. Initial scaffolding produced the user-story level and stopped. `check` Tier 2 Step 19b documents an `[INCOMPLETE]` rule for this case — verify whether it actually fires.

Both issues went unnoticed during the discord-v1 work because they were not surfaced by `check`.

**Frequency:** always reproducible on any new scaffold producing nested children.

## Reproduction Steps

1. In a clean repo, scaffold a feature with user-story children via the `/personal-workflow` workflow described in `WORKFLOW.md`
2. List the resulting `work-items/features/{slug}/` tree
3. **Observe:** child directories are bare slugs, not `{ID}_{slug}`
4. **Observe:** no task subdirectories under any user-story, despite manifest declaring `hierarchy.user-story.required_child = task, min = 1`
5. Run `/personal-workflow check` on the directory
6. **Observe:** neither violation is flagged

**Environment:** macOS Darwin 25.3.0, claude-skills-templates branch `claude/nostalgic-volhard`, deployed via symlinks to `~/.claude/skills/personal-workflow/`.

## Root Cause

**Root cause (two related):**

1. **Spec gap (Bug 1).** `WORKFLOW.md` Directory Layout was written to require ID prefixes on **files** but not on **directories**. The AI scaffolder follows the spec literally and produces unprefixed dirs. There is no validator rule to catch the inconsistency, so it propagates.

2. **Validator coverage (Bug 2).** Step 19b of `check.md` declares an `[INCOMPLETE]` rule for missing required children, but the rule applies only when `check` is run. The discord-v1 scaffold predates running `check` against it, so the rule never fired in practice. Whether the rule is also genuinely buggy (silent on count=0) needs confirmation during implementation.

**Locations:**
- `skills/personal-workflow/WORKFLOW.md` § Directory Layout, § Placeholder Replacement
- `skills/personal-workflow/check.md` Step 19 (Structural Completeness)

## Fix Description

1. **Spec update.** Amend `WORKFLOW.md` Directory Layout to require `{ID}_{slug}/` for every work-item directory. Add a `{SLUG_DIR}` placeholder (or repurpose existing) so the convention is explicit. Update the example in the same section.

2. **Validator: subdir-name rule.** Add to `check.md` Step 19 (after 19d Stray Detection) a rule: each work-item subdirectory's name must match `^{ID_PREFIX}\d+_[a-z0-9_-]+$` AND the embedded ID must equal the TRACKER.md frontmatter `id`. Flag mismatches as `[MISFORMATTED]`.

3. **Validator: hierarchy.min hardening.** Re-read Step 19b. If the `[INCOMPLETE]` rule is silent for count=0, fix the logic. If it correctly fires, document that the discord-v1 case will surface the moment `check` is run on it.

4. **Fixtures.** Add three fixtures under `skills/personal-workflow/fixtures/`:
   - `invalid-unprefixed-subdir/` — feature `F000099_demo/` with child `bare_slug/` (no `S00X_` prefix). `check` must emit `[MISFORMATTED]`.
   - `invalid-missing-required-child/` — user-story `S000099_demo/` with TRACKER + 3 docs but no task children. `check` must emit `[INCOMPLETE]`.
   - `valid-nested-feature/` — positive control: `F000098_demo/` → `S000098_thing/` → `T000098_subthing/`. `check` must pass.

5. **No manifest change.** `personal-artifact-manifests.json` already has `hierarchy` and `placement`. The fix is in the consumers (spec + validator), not in the contract.
