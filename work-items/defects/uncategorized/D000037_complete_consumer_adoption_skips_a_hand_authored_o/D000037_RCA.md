# D000037 — Root Cause Analysis

## Symptom

A consumer repo that hand-authored its `spec/doc-spec-custom.md` BEFORE turnkey-adoption existed gets the `cj-contract-gate` pre-commit hook installed by `skills-deploy install` but NOT the surface refresh / orphan declaration, so its very next commit is BLOCKED by the gate — violating the "never bricks a fresh adopter" guarantee. Confirmed live on a knowledge-base clone: 3 hard findings (undeclared `spec/workflow-spec.md` orphan, missing `docs/test-catalog.md`, stale `docs/workflow.md`).

## Reproduction Steps

Clone a consumer with a hand-authored `spec/doc-spec-custom.md` (no auto-marker) + seeded registries where `docs/` surfaces are missing/stale, run `skills-deploy install` from inside it, then run `cj-contract-gate.sh --repo <clone>`: pre-fix it VERDICT: BLOCKs.

## Investigation Trail

- 2026-06-29: Confirmed the early-return in `complete_consumer_adoption()` — a hand-authored overlay (one WITHOUT the `# doc-spec-custom.md — auto-generated by skills-deploy (S000117 adoption)` marker) hit `echo ...; return 0` and skipped both step (b) surface-refresh and step (a) orphan-declare.
- 2026-06-29: Verified the kb repo is a genuine consumer (hand-authored overlay, no `skills-catalog.json`, `spec/workflow-spec.md` absent) — so `is_workbench_self_repo` correctly returns "no" (D000036 fix) and the repo reaches `complete_consumer_adoption`, where the early-return is the bug.
- 2026-06-29: Reproduced the 3 hard findings live on a kb clone with the OLD path (gate VERDICT: BLOCK).
- 2026-06-29: Discovered the doc-spec parser reads "the LAST canonical Markdown table; rows must be contiguous under the `| Doc | Purpose | Requirement |` header" — a naive EOF-append (after the overlay's trailing `<!-- ...END -->` comment) is parsed as a separate header-less fragment and DROPPED. Fixed by splicing new rows immediately AFTER the last table-data row (before the END comment).

## Root Cause

**Root cause:** `complete_consumer_adoption()` early-returned (no-op) for a hand-authored overlay, treating it as "already adopted" and skipping the surface render + orphan declaration that make a freshly-seeded repo gate-clean. The hand-authored repo therefore got the gate hook but no contract cleanup.

**Location:** `scripts/skills-deploy` — the hand-authored early-return in `complete_consumer_adoption()`.

## Affected Components

| File | Change type |
|------|-------------|
| scripts/skills-deploy | fix — hand-authored overlay now completed append-only (render + splice-declare); shared `_adopt_refresh_surfaces` / `_adopt_purpose_for` helpers |
| scripts/test-deploy.sh | regression test — Test S000117b |

## Fix Description

Replaced the hand-authored early-return with a new `complete_consumer_adoption_handauthored()` branch that (b) re-renders the generated surfaces via a shared `_adopt_refresh_surfaces()` helper (test-spec + workflow-spec `--render-docs` — generated files are not curation) and (a) APPEND-ONLY-declares only the NEW undeclared orphans, SPLICING each declaring row immediately after the last table-data line so it stays contiguous under the curated header (never after the trailing END comment, where the parser would drop it). It backs up the file, validates the merged registry, and ROLLS BACK the append (byte-identical restore) if invalid — never regenerating the overlay, never touching curated rows/prose. The per-path purpose mapping is extracted into a shared `_adopt_purpose_for()` helper reused by both the owned-regenerate and hand-authored-append paths. The existing auto-marked/absent-overlay regenerate path is unchanged.

## Regression Risk

Regression test added: `scripts/test-deploy.sh` Test S000117b — asserts the gate BLOCKs pre-adoption, the hand-authored adoption branch fires, the gate EXITS 0 post-adoption, the user's original header+prose+curated row survive verbatim, NO auto-marker is added (not wholesale-regenerated), a new declaring row is appended, and a second install is idempotent (no double-append, gate stays clean).
