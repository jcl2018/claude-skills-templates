## Symptom

In drain mode, when `cj-worktree-init.sh` is unavailable on the machine,
`skills/CJ_goal_todo_fix/scripts/drain-one-todo.sh` silently dispatched the
first scaffold into the current — possibly dirty, possibly unrelated — branch
instead of halting the drain iteration. The operator hit this in production: a
drained-TODO scaffold was dispatched into uncommitted WIP on an unrelated
branch (`fix/option-cost-per-share`, a downstream consumer repo; cleaned up by
the operator, out of scope for this workbench defect). This destroys the
F000025/S000054 per-TODO worktree isolation and the `/ship` Gate #2
one-PR-per-TODO contract. The `[ -x ]` guard turned the unreachable helper into
a no-op, so the failure was **silent** — drain appeared to proceed normally.

## Reproduction Steps

Simulated deployed / helper-unavailable layout: `~/.claude/.skills-templates.json`
with no usable `.source`, `cj-worktree-init.sh` unreachable everywhere, and the
run dir not a git repo (so the `BASH_SOURCE`-relative in-repo fallback also
fails) — `$_WT_HELPER` resolves empty. Invoke
`drain-one-todo.sh dispatch <heading> <session>`. Pre-fix: exits 0, silently
delegates to `todo_fix.sh` (tripwire stub fires), scaffolds in-place.
Deterministic via `tests/drain-one-todo-helper-unavailable.test.sh` Case 2
(FAILS pre-fix, PASSES post-fix).

## Root Cause

**Root cause:** `drain-one-todo.sh`'s dispatch path treated an unreachable
`cj-worktree-init.sh` as a silent no-op: when `$_WT_HELPER` is empty (manifest
`.source` missing/empty/non-executable AND the `BASH_SOURCE`-relative in-repo
fallback also not executable), the `if [ -x "$_WT_HELPER" ]` guard at line 226
is false and lines 246-248 are pure comments with no code, so execution
silently falls through to the `todo_fix.sh` delegation and scaffolds the
drained TODO into the current (possibly dirty / unrelated) branch. Per-TODO
worktree isolation is load-bearing in drain context (F000025/S000054 / `/ship`
Gate #2 one-PR-per-TODO), so the unavailable-helper path must fail loud and
halt the drain iteration — not degrade silently. D000021 fixed only the
path-resolution half and its RCA explicitly scoped this silent-fallthrough
out, making it a distinct root cause.

**Location:** `skills/CJ_goal_todo_fix/scripts/drain-one-todo.sh:246`
(pre-fix line; the silent comment-only fallthrough).

## Fix Description

Replaced the silent comment-only fallthrough at `drain-one-todo.sh:246-248`
with a fail-loud guard `if [ ! -x "$_WT_HELPER" ]`: release the lock, print a
clear stderr diagnostic, emit
`RESULT: STATUS=halted; STAGE=preflight; HEADING=...; REASON=worktree-helper-unavailable`,
and `exit 2` — structurally consistent with the adjacent `worktree-cd-failed`
(:234-239) and `todo_fix.sh-not-found` (:260-266) halt exits already in this
dispatch path. The orchestrator treats `exit 2` as a halt and STOPS the drain
loop (no in-place scaffold, operator's WIP untouched). Reachable-but-non-created
states (`failed/detected/skipped/opted_out`) still degrade gracefully — they
are handled inside the `if [ -x ]` block because the helper actually ran and
made a deliberate call (`detected` = already inside an isolating Conductor
worktree; `opted_out` = `--no-worktree`), which is distinct from "helper
unreachable". The fail-loud is DRAIN-context only: `dispatch` is invoked solely
by the drain loop; single-TODO mode has its own SKILL.md worktree preamble and
never reaches this block, so single-TODO graceful degradation is unaffected.
The in-repo / deployed happy path is unchanged (D000021 sibling test still
green).
