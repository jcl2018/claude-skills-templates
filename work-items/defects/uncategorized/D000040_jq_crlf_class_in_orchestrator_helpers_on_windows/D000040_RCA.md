## Symptom

On Windows Git Bash (MINGW64) with a CRLF-emitting jq build (jq 1.7.1), the
`CJ_goal_*` / `check-*` orchestrator helpers consume `$(jq -r ...)` without
stripping CR, so every captured value carries a trailing `\r`. The proven taint:
`scripts/cj-goal-common.sh` resolves `src=$(jq -r '.source // empty'
~/.claude/.skills-templates.json)` → `src='E:/projects/claude-skills-templates\r'`,
so `[ -d "$src" ]` is false and the Fork-2 pre-build skills-sync + pr-check phases
silently degrade to `PHASE_RESULT=skipped` (fail-soft hides the breakage — no error
surfaces). This is the orchestrator-helper sibling of D000038 (which fixed the same
class in the standalone spec engines). Linux CI cannot see it (Linux jq emits LF).

## Reproduction Steps

1. On Windows Git Bash with jq 1.7.1 (Windows console binary), run:
   `jq -r '.source // empty' ~/.claude/.skills-templates.json | cat -A`
2. **Observe:** `E:/projects/claude-skills-templates^M$` — the `^M` is a trailing CR.
3. Generic proof jq emits CR on this box: `printf '{"k":"v"}' | jq -r '.k' | od -c`
   → `v  \r  \n`.
4. Consequence: in `cj-goal-common.sh` the CR-tainted `src` fails `[ -d "$src" ]`,
   so the sync / pr-check phases skip on Windows while reporting success.

**Environment:** Windows 10 Home 10.0.19045, Git Bash (MINGW64_NT-10.0-19045),
jq-1.7.1 (CRLF-emitting), repo v6.0.113.

## Investigation Trail

- Confirmed the jq CRLF taint live: `jq -r '.source' ~/.claude/.skills-templates.json | cat -A` → trailing `^M`; the generic `printf '{"k":"v"}' | jq -r '.k' | od -c` → `v \r \n`.
- Enumerated the jq call sites: all five named helpers call jq (cj-goal-common.sh 15, check-gates-update.sh 9, check-version-queue.sh 6, cj-worktree-init.sh 5, cj-worktree-cleanup.sh 1) and NONE defined a CR-stripping wrapper or sourced `scripts/lib.sh` (whose `jq()` wrapper at lib.sh:24 already solves this for validate.sh/test.sh).
- Classified each jq call for the wrapper choice: most are stdout captures (`src=$(jq -r ... || echo "")`), but two use jq's EXIT STATUS — `cj-goal-common.sh:346` (`if jq -nc ... >> "$RCPT"`) and `check-version-queue.sh:39` (`jq -e 'type == "array"'`). Two helpers deliberately omit `set -o pipefail` (cj-goal-common.sh, cj-worktree-cleanup.sh, both `set -u` with an explicit "do NOT set -e" comment), so the plain lib.sh wrapper (`command jq "$@" | tr -d '\r'`) — which relies on pipefail to propagate jq's exit status — would MASK those two exit-status checks.
- Chose the pipefail-independent form `jq() { command jq "$@" | tr -d '\r'; return "${PIPESTATUS[0]}"; }` — one uniform wrapper that is correct with OR without pipefail, so it is safe in all five helpers without the risk of adding a global `set -o pipefail` to a script not written for it.
- Verified live on the affected box (all with real CRLF jq first in PATH): the wrapper strips CR (`command jq` → `...templates^M$`, wrapper → `...templates$`); preserves jq's non-zero exit status without pipefail (`jq -e` on a non-array → rc=1, on an array → rc=0); and the proven-taint scenario now passes (`[ -d "$src" ]` true).
- Red-proof of the regression drill: a PATH-prepended CRLF-emitting jq shim (T7 style) shows raw `command jq` under the shim emits CR (1 byte) while the wrapped path emits 0 CR bytes — the drill genuinely distinguishes fixed from unfixed.

## Root Cause

**Root cause:** the `CJ_goal_*` / `check-*` orchestrator helpers parse
`$(jq -r ...)` directly and never strip CR from jq's output. On a Windows jq build
(CRLF), every captured value keeps a trailing `\r`; in `cj-goal-common.sh` this
makes `[ -d "$src" ]` false, so the pre-build sync / pr-check phases silently
degrade to `skipped`. Like the D000038 spec engines, these helpers do not source
`scripts/lib.sh`, so they never inherit its `jq()` CR-stripping wrapper (lib.sh:24)
— the standalone-script asymmetry. Two of the helpers additionally omit
`set -o pipefail`, so the fix must preserve jq's exit status independently of it.

**Location:** scripts/cj-goal-common.sh:164 (the proven-taint capture); the same
class across scripts/cj-worktree-init.sh, scripts/cj-worktree-cleanup.sh,
scripts/check-version-queue.sh, scripts/check-gates-update.sh.

## Fix Description

Added the canonical CR-stripping wrapper — in the pipefail-independent form
`jq() { command jq "$@" | tr -d '\r'; return "${PIPESTATUS[0]}"; }` — right after
the `set` line in each of the five orchestrator helpers. The form mirrors
`scripts/lib.sh:24` but uses `return "${PIPESTATUS[0]}"` so it preserves jq's exit
status with OR without `set -o pipefail`, keeping the two exit-status call sites
(`cj-goal-common.sh`'s `if jq -nc`, `check-version-queue.sh`'s `jq -e`) correct in
the two helpers that omit pipefail — without adding a risky global `set -o pipefail`
to a script not written for it. It covers every current and future jq call site,
preserving each helper's standalone (no-lib.sh) posture.

`tests/cj-goal-jq-crlf.test.sh` is the regression drill: T1 structurally asserts
the wrapper is present in all five helpers (guards removal, including helpers whose
jq output feeds internal logic not observable stdout); T2 exercises the wrapper
under a faithful CRLF-emitting jq shim (real jq piped through awk that appends
`\r\n`, exit status forwarded via `PIPESTATUS`) and asserts CR is stripped AND jq's
non-zero exit status is preserved with pipefail OFF; T3 runs
`cj-goal-common.sh --phase worktree --dry-run` under the shim and asserts non-empty,
CR-free jq-derived output. Registered via a hand-wired runner block in
`scripts/test.sh` + a `units:` row in `spec/test-spec-custom.md` (Check 24), and the
generated test catalog regenerated (Check 26).
