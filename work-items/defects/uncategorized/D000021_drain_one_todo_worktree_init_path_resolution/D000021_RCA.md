## Symptom

`scripts/cj-worktree-init.sh` is unreachable from `drain-one-todo.sh` when
`/CJ_goal_todo_fix` runs from the deployed `~/.claude/` location, so drain mode
silently skips per-iteration worktree creation and runs every drained TODO
in-place — defeating the F000025/S000054 collision-avoidance and causing
`/ship` Gate #2 branch collisions across drained TODOs. The `[ -x ]` guard
turns the missing helper into a no-op, so the failure is **silent**: drain
appears to work and only surfaces downstream as a branch collision.

Reproduction frequency: always, whenever drain mode runs from the deployed
`~/.claude/skills/CJ_goal_todo_fix/scripts/` location (i.e. every real
`/CJ_goal_todo_fix --max-drain N` invocation post-`skills-deploy install`).

## Reproduction Steps

1. Deploy skills via `scripts/skills-deploy install` (puts `drain-one-todo.sh`
   at `~/.claude/skills/CJ_goal_todo_fix/scripts/`).
2. Invoke `/CJ_goal_todo_fix` drain mode so the orchestrator runs the deployed
   `~/.claude/skills/CJ_goal_todo_fix/scripts/drain-one-todo.sh dispatch <heading>`.
3. **Observe:** the pre-fix
   `$(cd "$(dirname "${BASH_SOURCE[0]}")/../../.." && pwd)/scripts/cj-worktree-init.sh`
   resolves to `~/.claude/scripts/cj-worktree-init.sh` (does not exist —
   `skills-deploy` never deploys repo-root `scripts/` there); the
   `[ -x "$_WT_HELPER" ]` guard is false; no per-TODO worktree is created;
   drain runs every drained TODO in-place on the current branch.

**Environment:** macOS (darwin), zsh; any machine where the workbench skills
are deployed via `skills-deploy install` (the normal install path).
Deterministic repro: `tests/drain-one-todo-worktree-resolve.test.sh` Case 2
(simulated deployed layout — FAILS pre-fix: no `cj-todo-*` worktree created).

## Root Cause

**Root cause:** `drain-one-todo.sh` resolved `scripts/cj-worktree-init.sh` via a
`BASH_SOURCE`-relative path
`$(cd "$(dirname "${BASH_SOURCE[0]}")/../../.." && pwd)/scripts/cj-worktree-init.sh`,
which only yields the workbench repo root for the in-repo checkout. From the
deployed location `~/.claude/skills/CJ_goal_todo_fix/scripts/` that path
resolves to `~/.claude/scripts/cj-worktree-init.sh`, but `skills-deploy` only
deploys per-skill files into `~/.claude/skills/<name>/` and never copies
repo-root `scripts/` to `~/.claude/scripts/`. The helper was therefore
unreachable post-deploy, the `[ -x ]` guard fell through, and drain silently
ran TODOs in-place. The established workbench convention (used by
`todo_fix.sh`, the single-TODO `SKILL.md` preamble, and the F000009
update-check preamble) is to resolve repo-root scripts via the `.source` field
of `~/.claude/.skills-templates.json`; `drain-one-todo.sh` was the only site
that diverged from it.

**Location:** `skills/CJ_goal_todo_fix/scripts/drain-one-todo.sh:204`
(pre-fix line; the buggy `_WT_HELPER` `BASH_SOURCE`-relative resolution).

## Fix Description

`drain-one-todo.sh` now resolves `cj-worktree-init.sh` primarily via
`jq -r '.source' ~/.claude/.skills-templates.json` + `/scripts/cj-worktree-init.sh`
(the same convention `todo_fix.sh` and the single-TODO `SKILL.md` preamble
use), falling back to the original `BASH_SOURCE`-relative path only for the
in-repo / no-manifest case so consumer repos without workbench source still
degrade gracefully (helper simply unreachable — today's behavior, no regression).
Option (a) — teaching `skills-deploy` to deploy repo-root `scripts/` — was
rejected because it introduces a new deployment surface that contradicts the
documented "scripts stay in the clone; resolve via `.source`" workbench
convention. Added `tests/drain-one-todo-worktree-resolve.test.sh` (Case 1:
static assertion the manifest-`.source` resolution is present; Case 2:
behavioral — simulated deployed layout asserts a real per-iteration `cj-todo-*`
worktree is created). Wired the new test into `scripts/test.sh` after the
F000025 regression block.
