## Symptom

`CJ_repo-init` is declared `portability: standalone`, but it is not genuinely
standalone: `/CJ_portability-audit --no-adjudication` reports `FINDINGS=1`
(`CJ_repo-init declared standalone but depends on scripts/cj-repo-init.sh`). It
passed the default audit only via a `portability_requires` adjudication
(documented debt, v6.0.36 / T000042). Bundling the engine to fix that surfaced a
second, deeper issue: the audit's `is_exec()` false-flags config filenames that
appear as **seed-data string literals** inside a bundled `.sh`.

## Reproduction Steps

1. `bash scripts/cj-portability-audit.sh --no-adjudication` → `CJ_repo-init`
   flagged: `declared standalone but depends on scripts/cj-repo-init.sh`.
2. Bundle the engine under `skills/CJ_repo-init/scripts/` + add it to the catalog
   `files`, then re-run `--no-adjudication`.
3. **Observe:** the script finding is gone (carve-out), but `CLAUDE.md` and
   `skills-catalog.json` are now flagged — both are seed-data string literals the
   engine *writes* (`"CLAUDE.md"` in a `whitelist_patterns` array; a
   `skills-catalog.json` mention in a printed markdown table), never reads.

**Environment:** macOS (workbench-only); `/CJ_repo-init` + `/CJ_portability-audit` v0.1.0.

## Root Cause

**Root cause:** Two coupled causes —
1. `CJ_repo-init`'s engine lived at the workbench root (`scripts/cj-repo-init.sh`),
   a helper that does not exist in a fresh consumer repo. Since `/CJ_repo-init`
   bootstraps a repo that has never seen the workbench, it was never genuinely
   standalone — only adjudicated.
2. The portability audit's `is_exec()` 3rd clause matched a **quoted string
   literal at line-start**, so once the bundled engine was scanned, its seed-data
   config filenames (`"CLAUDE.md"`, `skills-catalog.json`) were mis-classified as
   executed reads.

**Location:** `scripts/cj-repo-init.sh` (now `skills/CJ_repo-init/scripts/`);
`scripts/cj-portability-audit.sh:305` (`is_exec` clause 3).

## Fix Description

Bundle the engine under `skills/CJ_repo-init/scripts/cj-repo-init.sh` (`git mv`)
so it deploys with the skill into `~/.claude/skills/CJ_repo-init/scripts/`;
`SKILL.md` resolves it repo-local-first then via the deployed skill dir (the
`CJ_qa-work-item` sibling-asset idiom) — no root `scripts/` or `.source`
reach-back. The catalog adds the bundled file to `files` and drops the
`portability_requires` adjudication. Separately, tighten the audit's `is_exec()`
3rd clause from `[^ \t]*$` to `$` so a quoted/indented string-literal at
line-start is no longer mis-read as an executed read (real bare-command execution
still matches). `scripts/test.sh` gains **S000083i** guarding the precision fix;
`tests/cj-repo-init.test.sh` `ENGINE=` repointed.
