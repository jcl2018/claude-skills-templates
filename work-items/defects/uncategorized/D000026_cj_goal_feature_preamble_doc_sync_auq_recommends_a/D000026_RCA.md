## Symptom

`/CJ_goal_feature` preamble's doc-sync AUQ labelled option A ("Run `/document-release` now") as "recommended on main", but on main A always aborts because upstream gstack `/document-release` Step 1 hard-aborts with "You're on the base branch. Run from a feature branch." Same wording shipped in `/CJ_goal_defect` + `/CJ_goal_investigate`. Empirical proof: 2026-05-31 `/CJ_goal_feature` run for F000031 casing-fix (PR #181) — AUQ fired, A was recommended, `/document-release` aborted with the base-branch error, fallback to snooze 1h saved the pipeline.

## Reproduction Steps

1. On a repo with the F000028 doc-sync hook installed, merge a non-trivial PR to main so the hook drops a marker.
2. Invoke `/CJ_goal_feature "<topic>"` on main.
3. Preamble emits `DOC_SYNC_PENDING`.
4. AUQ surfaces with A labelled "recommended on main".
5. **Observe:** picking A invokes `/document-release` which aborts at Step 1 with "You're on the base branch. Run from a feature branch."

**Environment:** macOS APFS, claude-skills-templates workbench, gstack `/document-release` SKILL.md upstream (unmodified).

## Root Cause

**Root cause:** F000029 (PR #178) shipped the doc-sync AUQ template with the branch-aware recommendation polarity inverted. The author's mental model was "doc-sync runs on main after a merge," but upstream `/document-release` is designed for feature branches that need to update their docs before shipping a PR — it explicitly rejects the base branch at Step 1. The inverted polarity was duplicated identically across all 3 `cj_goal` SKILL.md preambles AND documented (still inverted) in the CLAUDE.md mechanism note at line 274.

**Location:** `skills/CJ_goal_feature/SKILL.md:38-60` (and identical blocks in `skills/CJ_goal_defect/SKILL.md:38-60`, `skills/CJ_goal_investigate/SKILL.md:38-60`); plus `CLAUDE.md:274`.

## Fix Description

Flipped the branch-aware recommendation in all 3 `cj_goal` SKILL.md preamble AUQ templates: on main, B (snooze) is now recommended and A is flagged "WILL ABORT on main" with the verbatim upstream abort message; on a feature branch, A is recommended (where `/document-release` is designed to run). Also flipped the per-branch follow-through blocks: "On A (when on a feature branch)" now invokes `/document-release` and auto-commits doc updates; "On A (when on main)" now warns and falls back to snooze. Corrected the CLAUDE.md F000029 mechanism note (line 274) to match. Wrote a NEW regression test that asserts both POSITIVE (corrected wording present in all 3 SKILL.md + CLAUDE.md) and NEGATIVE (pre-fix wording absent) — 17/17 assertions pass. Wired the new test into `scripts/test.sh` after the existing `cj-goal-investigate-did-allocator` test.
