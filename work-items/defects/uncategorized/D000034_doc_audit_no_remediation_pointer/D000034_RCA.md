# D000034 — RCA

## Symptom

Running `/CJ_doc_audit` (Stage 1 = `doc-spec.sh --check-on-disk`) in a repo where
declared docs are missing — e.g. a consumer repo where `docs/workflow.md` does
not exist — produces a bare list of `FINDING: stage1/declared-exists — declared
doc missing on disk: …` lines and then stops. The operator is given the gap but
no remedy: nothing in the report names the tool that creates the missing docs.

## Root Cause

**Root cause:** An asymmetry in the doc-audit's self-bootstrap. `/CJ_doc_audit`
Step 2 *does* force-regenerate the missing contract registry itself
(`spec/doc-spec.md` via `doc-spec.sh --seed`, `seeded: yes`), but the docs that
contract then *declares* were surfaced as `declared-exists` findings only —
report-only, by the audit's deliberate read-mostly design — with **no pointer**
to `/CJ_document-release`, which is the documented verb that stub-scaffolds
exactly those declared-but-missing docs. So "ensure the contract exists"
half-did its job from the operator's chair: it created the registry but left
every entry the registry requires as a dead-end finding.

**Location:** `scripts/doc-spec.sh` — `_check_on_disk()` (the `declared-exists`
check emitted findings with no trailing remediation guidance).

## Fix Description

The engine (`_check_on_disk`) now captures the count of declared-but-missing
docs and, when that count is > 0, emits one trailing advisory line:

```
REMEDIATION: stage1/declared-exists — <N> required doc(s) declared but missing;
run /CJ_document-release to stub-scaffold them (it reads this same doc-spec
registry). This audit reports, it does not scaffold.
```

The line is **advisory, not a finding** — it does not change `FINDINGS=` or the
skill's `STAGE1_FINDINGS`, so the audit's read-mostly posture and its
report-grammar contract are preserved. Because Stage 1 prints the engine output
verbatim, the pointer appears uniformly in every audit context (standalone, the
in-QA inline path at `qa.md` Step 8.6c, and a direct manual engine call) and is
deterministic rather than agent-dependent. The audit still never scaffolds the
docs itself — it names the remedy. (Operator-chosen disposition: "remediation
pointer only".)
