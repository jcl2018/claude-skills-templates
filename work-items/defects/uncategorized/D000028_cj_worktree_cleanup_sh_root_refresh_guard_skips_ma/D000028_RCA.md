## Symptom

`scripts/cj-worktree-cleanup.sh`'s guarded root-main refresh emits
`ROOT_REFRESH=skipped` + `[cleanup-note] root checkout is dirty; skipped main refresh`
whenever the root checkout has ANY untracked files. In this workbench the root
always carries untracked `.gstack/*.md` design docs, so the
`git checkout main && git pull --ff-only` half of the janitor effectively never
runs — even when the *tracked* tree is clean (`git status --porcelain --untracked-files=no`
is empty). The worktree-removal half works correctly; only the "switch back to
main + pull" half mis-fires. Reproduction: always, in any repo that keeps
untracked scratch at the root.

## Reproduction Steps

1. From a `cj-*` worktree whose root checkout has only untracked files at top
   level (e.g. this repo's `.gstack/*.md` design docs), with a clean *tracked* tree.
2. Run `scripts/cj-worktree-cleanup.sh` (or the orchestrator post-land terminal).
3. **Observe:** `ROOT_REFRESH=skipped` despite the tracked tree being clean; the
   root is never switched to `main` / pulled.

**Environment:** macOS, this workbench; `cj-worktree-cleanup.sh` (T000036, shipped v6.0.15).

## Root Cause

**Root cause:** The root-refresh guard used `git status --porcelain` with no flags,
which counts untracked files. Because `git checkout main` and `git pull --ff-only`
are non-destructive to untracked files, untracked-only content was wrongly treated
as a dirty root and permanently skipped the refresh in any repo that keeps
untracked scratch at the root (this workbench: `.gstack/*.md`).

**Location:** `scripts/cj-worktree-cleanup.sh:309` (the root-refresh guard).

## Fix Description

Changed ONLY the root-refresh guard (line 309) to
`git -C "$_ROOT" status --porcelain --untracked-files=no`, so it skips the refresh
only on a dirty *tracked* tree; untracked-only roots now refresh. Updated the block
comment to explain the tracked-only rationale and the asymmetry with the
per-worktree rail, and the `[cleanup-note]` message now reads "root checkout has a
dirty tracked tree; skipped main refresh". The per-worktree dirty rail at line 185
(bare `--porcelain`) is deliberately left unchanged — removing a worktree with
untracked scratch would lose it.

## Regression Risk

Regression test added: `tests/cj-worktree-cleanup.test.sh` — Case 12 converted to
dirty a genuinely *tracked* file (still asserts `ROOT_REFRESH=skipped` + the new
"dirty tracked tree" message); Case 12b proves an untracked-only root now yields
`ROOT_REFRESH=ok`.
