## Symptom

`scripts/test-spec.sh --check-coverage` (consumed by `validate.sh` Check 24,
`/CJ_test_audit` Stage 1, and `/CJ_qa-work-item` Step 8.6d) flips to a permanent
false-red in a **consumer** repo that adopts the two-tier test contract against
its own surface ‚Äî e.g. a TypeScript project with vitest `*.test.ts` and a GitHub
Actions workflow, but **no** `scripts/validate.sh`, **no** `tests/*.test.sh`, and
**no** `scripts/setup-hooks.sh`. The reverse-sweep zero-token floors fire on the
legitimately-absent shell namespaces, surfacing as `TEST_AUDIT: findings`. The
workbench itself (all four surfaces present) is unaffected.

## Reproduction Steps

1. Build a temp consumer fixture: `spec/test-spec.md` = `test-spec.sh --seed`;
   `spec/test-spec-custom.md` = a `units:` overlay declaring rows against a
   `tests/foo.test.ts` and a `.github/workflows/ci.yml`; create those two files.
2. Do **not** create `scripts/validate.sh`, `tests/*.test.sh`, or
   `scripts/setup-hooks.sh`.
3. Run `REPO_ROOT=<fixture> bash scripts/test-spec.sh --check-coverage`.
4. **Observe:** `COVERAGE: findings=4` and exit 1 ‚Äî a global `<20`-token floor
   finding plus `ZERO live tokens` findings for `validate`, `test-files`, and
   `hooks`, even though those three surfaces do not exist in a consumer repo.

**Environment:** any POSIX shell (bash/dash), macOS/Linux/Git-Bash; the
deployed copy is `~/.claude/_cj-shared/scripts/test-spec.sh`.

## Root Cause

**Root cause:** The per-namespace zero-token floors and the global `<20`-token
floor in `_run_coverage()` fired without **surface-existence gating**. A
consumer repo that adopts the contract against its own surface legitimately has
zero tokens in the absent shell namespaces (`validate` / `test-files` /
`hooks`), so the floors ‚Äî calibrated to the full workbench shape where all four
surfaces exist ‚Äî misclassified that legitimate absence as extraction-grammar rot
and flipped `--check-coverage` to a permanent false-red. The forward anchor-grep
and the reverse single-owner sweep were correct; only the floor assertion was wrong.

**Location:** `scripts/test-spec.sh:711-760` (the floor-assert block inside `_run_coverage()`)

## Fix Description

Gate the reverse-sweep floors on **effective surface presence** = the surface is
part of THIS repo's verification contract. A namespace counts as present for the
floors only when BOTH hold:

1. **Path present** ‚Äî the surface file/dir exists at the workbench's reserved
   path (`scripts/validate.sh`; ‚â•1 `tests/*.test.sh`; ‚â•1
   `.github/workflows/*.yml|*.yaml`; `scripts/setup-hooks.sh`), using the same
   POSIX/BSD-portable existence idioms the reverse sweep already uses; AND
2. **Family rows present** ‚Äî the merged registry declares ‚â•1 unit row in that
   namespace's family (`validate` / `test` / `ci` / `hook`), counted with the same
   `awk -F'\t' '$2==<family>'` shape the reverse sweep uses. The rows are what make
   us *expect* live tokens, so they are the direct signal the repo contracts to
   verify the namespace.

`_SURFACES_PRESENT` is the sum of the four effective-present booleans.

- The **global** `<20`-token floor fires **only when `_SURFACES_PRESENT -eq 4`**
  (the floor value is calibrated to the full workbench shape; a partial/consumer
  set legitimately yields few tokens and relies on the per-namespace floors).
- Each **per-namespace** floor fires **only when that namespace is effectively
  present AND its token count is 0** ‚Äî a namespace the contract does not claim
  (surface absent, OR a reserved-path file with no rows in that family) is N/A.

**Why family-rows, not path alone (the strengthening).** Path-existence alone is
too coarse: a non-workbench consumer that declares units AND happens to have a
file at a reserved path in a *different* grammar ‚Äî a husky-style
`scripts/setup-hooks.sh` with no `if install_hook` lines, or its own
`scripts/validate.sh` with no `=== Check N:` banners ‚Äî would otherwise false-fire
a zero-token floor it could only escape by renaming its file. That is the exact
"misfire in a consumer repo" class this defect closes. Composing path-presence
with family-row-presence closes that residual **without** weakening the workbench
(which declares rows in all four families, so a present-yet-empty surface the
registry DOES claim ‚Äî genuine grammar rot ‚Äî still fires).

The forward anchor-grep, the reverse single-owner sweep, the units-gating, and
the `--seed` heredoc are untouched (the fix is confined to the floor block).
This is surface gating, **not** floor removal: a contracted-but-zero-token
namespace still fires.
