## Symptom

`scripts/test-spec.sh --check-coverage` (consumed by `validate.sh` Check 24,
`/CJ_test_audit` Stage 1, and `/CJ_qa-work-item` Step 8.6d) flips to a permanent
false-red in a **consumer** repo that adopts the two-tier test contract against
its own surface — e.g. a TypeScript project with vitest `*.test.ts` and a GitHub
Actions workflow, but **no** `scripts/validate.sh`, **no** `tests/*.test.sh`, and
**no** `scripts/setup-hooks.sh`. The reverse-sweep zero-token floors fire on the
legitimately-absent shell namespaces, surfacing as `TEST_AUDIT: findings`. The
workbench itself (all four surfaces present) is unaffected.

## Reproduction Steps

1. Build a temp consumer fixture: `spec/test-spec.md` = `test-spec.sh --seed`;
   `spec/test-spec-custom.md` = a `units:` overlay declaring rows against a
   `tests/foo.test.ts` and a `.github/workflows/ci.yml`; create those two files.
2. Do **not** create `scripts/validate.sh`, `tests/*.test.sh`, or
   `scripts/setup-hooks.sh`.
3. Run `REPO_ROOT=<fixture> bash scripts/test-spec.sh --check-coverage`.
4. **Observe:** `COVERAGE: findings=4` and exit 1 — a global `<20`-token floor
   finding plus `ZERO live tokens` findings for `validate`, `test-files`, and
   `hooks`, even though those three surfaces do not exist in a consumer repo.

**Environment:** any POSIX shell (bash/dash), macOS/Linux/Git-Bash; the
deployed copy is `~/.claude/_cj-shared/scripts/test-spec.sh`.

## Root Cause

**Root cause:** The per-namespace zero-token floors and the global `<20`-token
floor in `_run_coverage()` fired without **surface-existence gating**. A
consumer repo that adopts the contract against its own surface legitimately has
zero tokens in the absent shell namespaces (`validate` / `test-files` /
`hooks`), so the floors — calibrated to the full workbench shape where all four
surfaces exist — misclassified that legitimate absence as extraction-grammar rot
and flipped `--check-coverage` to a permanent false-red. The forward anchor-grep
and the reverse single-owner sweep were correct; only the floor assertion was wrong.

**Location:** `scripts/test-spec.sh:711-760` (the floor-assert block inside `_run_coverage()`)

## Fix Description

Surface-existence-gate the reverse-sweep floors. Compute four surface-present
booleans using the same POSIX/BSD-portable existence idioms the reverse sweep
already uses (`validate` iff `scripts/validate.sh` exists; `test-files` iff ≥1
`tests/*.test.sh`; `workflows` iff ≥1 `.github/workflows/*.yml|*.yaml`; `hooks`
iff `scripts/setup-hooks.sh` exists) and `_SURFACES_PRESENT` as their sum.

- The **global** `<20`-token floor fires **only when `_SURFACES_PRESENT -eq 4`**
  (the floor value is calibrated to the full workbench shape; a partial/consumer
  set legitimately yields few tokens and relies on the per-namespace floors).
- Each **per-namespace** floor fires **only when that namespace's surface is
  present AND its token count is 0** — an absent surface is N/A, never a finding.

The forward anchor-grep, the reverse single-owner sweep, the units-gating, and
the `--seed` heredoc are untouched (the fix is confined to the floor block).
This is surface-existence gating, **not** floor removal: a present-but-zero-token
namespace (genuine grammar rot in the workbench) still fires.
