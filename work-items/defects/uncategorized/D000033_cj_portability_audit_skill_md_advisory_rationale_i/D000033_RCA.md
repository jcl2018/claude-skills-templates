## Symptom

`skills/CJ_portability-audit/SKILL.md` (`## Overview`, ~lines 39-43) claimed the
audit is "advisory-first" BECAUSE "the current workbench HAS real
declared-vs-actual mismatches, so v1 surfaces findings WITHOUT hard-failing,"
with the strict path framed as a future ("once declarations are reconciled").
Both claims are now factually wrong: the catalog is clean (no mismatches) and the
audit is a hard gate on the cj_goal orchestrated path. Documentation-accuracy
defect (no runtime symptom); always reproducible by reading the file vs the
current engine/orchestrator state.

## Reproduction Steps

1. `bash scripts/cj-portability-audit.sh` → `FINDINGS=0 / SKILLS_AUDITED=12` (also `FINDINGS=0` raw via `--no-adjudication`).
2. `grep -n "HAS real declared-vs-actual" skills/CJ_portability-audit/SKILL.md` → matched the stale claim.
3. `grep -n "portability-audit" scripts/cj-goal-common.sh` → the F000051 `--phase portability-audit` step runs the engine under `PORTABILITY_STRICT=1` and `exit 2`s on findings; all 3 cj_goal orchestrators HALT `[portability-red]` before `/ship`.
4. **Observe:** the SKILL.md prose contradicts both (1) the clean-catalog reality and (3) the live hard-gate posture.

**Environment:** claude-skills-templates workbench @ v6.0.50; macOS / POSIX shell.

## Root Cause

**Root cause:** The advisory-rationale prose was authored when the workbench
genuinely carried declared-vs-actual mismatches and the strict path was only a
future env-flag. It was never updated when (a) the mismatches were reconciled to
`FINDINGS=0` and (b) F000051 (v6.0.49) promoted the audit to a hard gate on the
cj_goal orchestrated path (`cj-goal-common.sh --phase portability-audit` under
`PORTABILITY_STRICT=1`, halt-on-red before `/ship`), while `validate.sh` Check 18
stays advisory globally. Pure documentation drift; no code defect.

**Location:** `skills/CJ_portability-audit/SKILL.md:39`

## Fix Description

Prose-only correction (2 files). Rewrote the SKILL.md `## Overview` advisory
rationale to state the current posture accurately: the catalog is clean
(`FINDINGS=0`, even raw); the audit stays advisory in `validate.sh` Check 18
globally (`PORTABILITY_STRICT=1` flips it) but is a hard gate on the cj_goal
orchestrated path as of F000051 (`cj-goal-common.sh --phase portability-audit`,
halt `[portability-red]` before `/ship`). Corrected the Usage-section "documented
future hard-fail path" line (the orchestrators already use it). Mirrored the fix
in USAGE.md and bumped `last-updated`. Left genuinely-accurate lines untouched
(frontmatter `description`; the engine-default "exits 0 even with findings"
pitfalls, which still describe default engine behavior).
