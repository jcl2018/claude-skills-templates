## Symptom

Every `/CJ_goal_defect` run halts at **Step 8** (`/CJ_qa-work-item`): the
promoted defect tracker fails the `/CJ_personal-workflow check` boundary gate QA
runs before testing, so the run cannot advance to doc-sync / ship without the
operator manually fleshing the tracker to template compliance and committing the
fix. Reproduction frequency: always (every defect run). Surfaced live on the
D000030 run, which required a hand-built tracker to get past QA.

## Reproduction Steps

1. Run `/CJ_goal_defect "<any bug>"`.
2. At **Step 7.4** the pipeline writes a minimal `D000NNN_TRACKER.md`
   (frontmatter + `## Bug Report` + `## Journal` only).
3. **Step 8** invokes `/CJ_qa-work-item`; its boundary check runs
   `/CJ_personal-workflow check` (Directory Mode), validating the tracker against
   `tracker-defect.md`.
4. **Observe:** the check flags missing frontmatter (`updated`/`repo`/`branch`/
   `blocked_by`) + missing `## Lifecycle` (3 phases, 11 checkboxes) /
   Reproduction Steps / Todos / Log / PRs / Files / Insights sections; and the
   Phase-2 `Fix committed` implementer gate is unchecked (no commit step before
   QA) → QA refuses **"Phase 2 incomplete"**.

**Environment:** macOS (workbench-only); `/CJ_goal_defect` v0.1.0 (experimental).

## Root Cause

**Root cause:** `/CJ_goal_defect` `pipeline.md` Step 7.4 promotes a
structurally-minimal tracker that does not satisfy the `tracker-defect.md`
structure `/CJ_personal-workflow check` validates, and there is no commit step
before Step 8, so the Phase-2 `Fix committed` implementer gate is unchecked. The
QA boundary check therefore refuses every run unless the operator hand-fleshes
the tracker and commits.

**Location:** `skills/CJ_goal_defect/pipeline.md:583` (Step 7.4 `TRK` heredoc) +
the missing commit step before Step 8.

## Fix Description

Step 7.4's `TRK` heredoc now emits a fully `tracker-defect.md`-compliant
tracker: all 9 required frontmatter fields
(`name/type/id/status/created/updated/repo/branch/blocked_by`) + `## Lifecycle`
(3 phases, 11 gate checkboxes, Phase 1 + Phase 2 marked checked) + Reproduction
Steps / Todos / Log / PRs / Files / Insights / Journal. The old `## Bug Report`
section is dropped (its content moves to `## Reproduction Steps`). New
`_TRACKER_DATE`/`_TRACKER_REPO`/`_TRACKER_BRANCH` vars feed the frontmatter
(`repo` strips the worktree suffix so the landed tracker shows the main repo).
A new **Step 7.6** commits the `/investigate`-written fix + the work-item
artifacts BEFORE Step 8, so the `Fix committed` gate is honest AND doc-sync's
clean-tree gate (Step 5.5) sees a clean non-doc tree; Step 8 notes committing the
post-QA `[qa-pass]` tracker update before doc-sync. `scripts/test.sh` gains a
D000031 assertion that extracts the heredoc and checks the fields/phases/
checkboxes/sections + the Step 7.6 step.
