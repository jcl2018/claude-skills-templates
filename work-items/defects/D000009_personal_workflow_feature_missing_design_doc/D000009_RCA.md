## Symptom

Feature work items pass `/personal-workflow check` (and `/company-workflow validate`) with only the existing required artifacts (personal: TRACKER + milestones; company: TRACKER + feature-summary + milestones). No enforcement that a feature carries a design document capturing cross-story engineering intent. As features grow to span multiple user-stories, design rationale has no canonical home — it scatters across TRACKER journal entries and per-user-story ARCHITECTURE.md files.

Reproduction frequency: **always**. Every feature scaffolded from F000001 through F000004.

## Reproduction Steps

1. `jq '.types.feature.required' skills/personal-workflow/personal-artifact-manifests.json`
2. **Observe:** 2 required artifacts — `tracker` + `milestones`. No `design`.
3. Scaffold a new feature directory with only those 2 files.
4. Run `/personal-workflow check <dir>`.
5. **Observe:** validation passes (0 violations) despite no design artifact.

**Environment:** workbench repo; `personal-artifact-manifests.json` v2.0.0; `company-artifact-manifests.json` v1.0.0; fix/feature-requires-design-doc branch.

## Root Cause

**Root cause:** `personal-artifact-manifests.json` and `company-artifact-manifests.json` never declared a `design` artifact for the feature type. The feature type-to-artifact mapping was scoped under the implicit assumption that architectural decisions would live in per-user-story `ARCHITECTURE.md` files. That works for single-user-story features but loses cross-story design coherence for multi-story features.

The author already recognized the gap in practice — F000005 has a rich DESIGN.md written by hand — but the manifest did not enforce the pattern, so it didn't propagate.

**Location:**
- `skills/personal-workflow/personal-artifact-manifests.json` — `feature.required` array
- `skills/company-workflow/company-artifact-manifests.json` — same
- `templates/personal-workflow/` — no `doc-DESIGN.md`
- `templates/company-workflow/` — no `doc-DESIGN.md`

## Fix Description

Narrow-scope fix:

1. Author `templates/personal-workflow/doc-DESIGN.md` with 7 sections mirroring F000005's existing DESIGN.md: Problem, Shape of the solution, Big decisions, Risks & open questions, Definition of done, Not in scope, Pointers.
2. Author `templates/company-workflow/doc-DESIGN.md` with 6 sections (same as personal minus "Not in scope", since company's `feature-summary.md` already owns Out-of-Scope).
3. Add `{"artifact": "design", "template": "doc-DESIGN.md", "filename": "DESIGN.md"}` to `feature.required` in both manifests.
4. Add `doc-DESIGN.md` entries to `skills-catalog.json` template lists (personal + company).
5. Backfill minimal `DESIGN.md` for F000001–F000004 (closed features); align F000005's existing DESIGN.md to the template (rename "Big decisions (already made)" → "Big decisions"; remove Sequencing section duplicating milestones.md).
6. Add D000009 regression block to `scripts/test.sh` to prevent the manifest from silently losing the design entry in a future refactor.

**Out of scope** (doc drift, not validator-breaking; file a follow-up if still wanted):

- WORKFLOW.md type-to-artifact tables
- tracker-feature.md template step/gate text
- company fixtures / examples / SKILL.md / philosophy reference updates
- work-copilot bundle sync (F000005 owns the bundle)
