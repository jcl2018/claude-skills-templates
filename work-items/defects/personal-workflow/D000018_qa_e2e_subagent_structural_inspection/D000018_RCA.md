## Symptom

`/CJ_qa-work-item` dispatches a QA engineer subagent for E2E verification (Step 7), and the subagent reports `ambiguous` for E2E rows that should produce a concrete `green` or `red` verdict. The journal entry describes structural source-code inspection ("verified file X exists at path Y", "checked function Z is exported") rather than the actual end-user behavior the E2E row was supposed to verify.

The skill silently degrades: E2E rows that should verify user-facing flows become code-structure lints. The Phase 2 qa-owned gates then remain unchecked because the aggregate E2E verdict is `ambiguous` (or `mixed`).

Reproduces deterministically — observed in 5+ recent runs across S000027, S000028, S000022, S000020, S000030.

## Reproduction Steps

1. Take any user-story with TEST-SPEC E2E rows that describe user-facing flows (not just code structure verification).
2. Run `/CJ_qa-work-item <work-item-dir>`.
3. **Observe:** journal contains `[qa-e2e] E# (...): ambiguous — <structural inspection of source>` and `[qa-e2e-summary] ambiguous` (or `mixed`).

**Environment:** Claude Code 2.1.91; current `main` of `jcl2018/claude-skills-templates`.

## Root Cause

Three issues, in order of severity:

**Root cause 1 (largest — prompt blind spot):** `skills/CJ_qa-work-item/qa.md` Step 7's stable preamble (lines 299-301) explicitly lists only Read, Bash, Grep, and Glob as the subagent's available tools. The subagent **does** have the Skill tool at top level (re-probed 2026-05-11; confirmed for both `subagent_type` values), but obediently doesn't invoke it because the prompt never grants permission. Any E2E row that requires running a `/skill` command degrades to structural source inspection.

**Root cause 2 (architectural — AUQ absence):** AskUserQuestion is absent from the subagent's deferred-tools list. E2E rows requiring user interaction cannot be executed inside the subagent at all and must be handled by the parent orchestrator.

**Root cause 3 (architectural — Agent absence):** The Agent tool is absent from the subagent's deferred-tools list. Subagents cannot recurse. E2E rows that need to dispatch a skill which itself dispatches subagents (e.g., /CJ_personal-pipeline → Phase 1/2/3 subagents) fail from inside the QA subagent.

**Root cause:** qa.md Step 7's prompt omits Skill (causing #1) and the subagent context lacks AUQ + Agent (causing #2, #3); the skill has no compensating parent-inline execution path for E2E rows that need these tools.

**Location:**
- Primary: `skills/CJ_qa-work-item/qa.md` lines 299-301 (Step 7 stable preamble)
- Secondary: `skills/CJ_qa-work-item/qa.md` Step 4 (E2E row planning — no tool-need classifier today)
- Secondary: `skills/CJ_qa-work-item/qa.md` Step 8 (verdict aggregator — single-source today)

## Fix Description

Approach B from the design doc (chosen): Approach A's minimal prompt fix + parent-inline execution path for AUQ/recursion rows.

Concretely:

1. **Prompt fix (Approach A core):** Edit `qa.md` Step 7 preamble to add `Skill` to the subagent's tool list. Add a sentence: "When an E2E row requires running a /skill command, invoke it directly via the Skill tool."
2. **Row classifier (Step 4 / new Step 4.5):** Add a per-E2E-row tool-need classifier with four categories: `read-only`, `skill-invoking`, `interactive`, `recursive`. Unclassifiable rows default to `interactive` (parent-inline) as the safer fallback.
3. **Parent-inline execution (new Step 7.5):** Parent orchestrator runs `interactive` and `recursive` rows inline using its full toolbelt (Skill + AUQ + Agent). Each row generates a `[qa-e2e]` journal entry in the same shape the subagent emits.
4. **Verdict aggregator update (Step 8):** Merge `[qa-e2e]` entries from both sources by row number. Compute the aggregate `E2E_VERDICT` from the union.
5. **Documentation:** Update `SKILL.md` Overview + per-type table to describe the classifier and parent-inline path. Append a 2026-05-11 re-probe note to `tests/spike/subagent-capabilities/findings.md`.

Smoke-red short-circuit (Step 7 today) preserves current invariant: smoke red still blocks all E2E (parent-inline included). Idempotency contract preserved (Step 3).

## Regression Risk

| Area | Risk Level | Why | Mitigation |
|------|-----------|-----|------------|
| Verdict aggregation (Step 8) | Med | Two-source aggregation could double-count or skip rows if the parent-inline path writes `[qa-e2e]` entries with a different shape | Use identical journal-entry template in both paths; row-number is the join key; add a unit-level smoke test asserting aggregation correctness |
| Row classifier accuracy (Step 4) | Med | A row mis-classified as `read-only` that actually needs Skill silently degrades to ambiguous | Classifier defaults to parent-inline on ambiguity (safer); document the heuristic; provide explicit `Tag: e2e-parent` override marker in TEST-SPEC |
| Parent context bloat | Low-Med | Many `interactive`/`recursive` rows push parent past comfortable token limits | Cap parent-inline to 5 rows per run; surplus rows mark ambiguous with "deferred to manual" |
| Pre-fix journal entries in old trackers | Low | Old `[qa-e2e]` entries (ambiguous-via-structural) remain in trackers; confusing baseline for re-runs | Document in CHANGELOG that pre-fix entries are historic; users can clean manually; do nothing automatically |
