## Symptom

Two confirmed substrate bugs in `skills/CJ_personal-pipeline/pipeline.md` block downstream pipeline automation:

**Bug 1 — Step 7 strict halt-on-ambiguous blocks defects and tasks.**
For `defect` and `task` work-item types, `/CJ_qa-work-item` structurally cannot return `E2E=green` — the inner E2E subagent only dispatches for user-stories (per qa.md line 179: defect/task have no E2E; line 643: smoke-based `[qa-pass]` only). So `E2E=ambiguous` from a defect/task QA always means "n/a for this type", not "uncertain test result". But Step 7's halt rule treats `E2E=ambiguous` uniformly as `User Challenge — Halt-at-Gate`. Strictly applied, the pipeline can never reach `end_state=green` on a defect or task. Reproduction frequency: 100% for defect/task types (structural).

**Bug 2 — sensitive-surface regex misses `skills/*/scripts/`.**
The Step 5.1 pre-scan regex matches `skills-catalog.json`, `*-artifact-manifests.json`, `templates/{personal,company}-workflow/`, `scripts/(validate|test|test-deploy).sh`, and `.git/hooks/`. New shell-script files added under `skills/*/scripts/` (e.g. D000017's new `skills/CJ_suggest/scripts/suggest.sh`) are NOT flagged. The implement subagent ships them auto-approved without surfacing to the user at Step 8.5. Reproduction frequency: 100% (whenever a new file under `skills/*/scripts/` is introduced).

## Root Cause

**Root cause:** Pipeline gate semantics in Step 5.1 (pre-scan input selection + regex) and Step 7 (halt rule) are not type-aware. Both were originally designed for user-story work-items (the only type when F000014's pipeline shipped) and were never generalized when F000012/v1.11.0 added defect and task types. The shared cause manifests as two visible bugs:
- Bug 1: Step 7's halt rule treats `E2E=ambiguous` uniformly, ignoring that the structural contract for defect/task makes ambiguous the canonical N/A marker.
- Bug 2: Step 5.1's pre-scan (a) skips when SPEC is empty (defects/tasks have no SPEC) and (b) doesn't match the `skills/*/scripts/` trust-boundary surface even when called.

**Location:**
- Bug 1: `skills/CJ_personal-pipeline/pipeline.md` Step 7 line 522
- Bug 2: `skills/CJ_personal-pipeline/pipeline.md` Step 5.1 lines 398-419

## Fix Description

Five surgical edits to one file (`skills/CJ_personal-pipeline/pipeline.md`):

- **Edit #0 (orchestrator prelude):** Add a two-line load (`TRACKER` from work-item dir via find; `WORK_ITEM_TYPE` from tracker frontmatter via frontmatter-anchored awk) to each consuming bash block (or as a Step 4 sub-step 2 trailer). The frontmatter anchor `awk '/^---$/{n++; next} n==1 && /^type:/'` restricts the match to the YAML frontmatter section between the first two `---` lines (avoids false matches on `type:` in prose / code blocks). Trailing `sed`+`awk` strip CRLF + comments + whitespace. This is the load-bearing prerequisite: Edit #1 and Edit #2b both read these variables.

- **Edit #1 (Step 7 type-aware halt):** Above line 522, insert a type-aware branch — when `$WORK_ITEM_TYPE in {defect, task}` AND `SMOKE=green` AND `PHASE2_GATES=green` AND `E2E=ambiguous`, treat as green silently and continue to Step 8 (same green path as user-story-green branch at line 521). Inline comment documents the rationale referencing qa.md line 179.

- **Edit #2a (Step 5.1 regex broaden):** Append `|skills/[^/]+/scripts/[^/]+` to the alternation at line 410. Matches any file under `skills/*/scripts/`, not just `.sh`. Trust boundary is the directory, not the extension. Acceptable over-match: README.md under scripts/ would trip — false positive ack'd as security-gate trade-off.

- **Edit #2b (Step 5.1 type-aware input selection):** Replace the "skip if SPEC empty" block (lines 401-403) with type-aware input selection. User-stories scan SPEC (current behavior). Defects scan RCA + test-plan. Tasks scan TRACKER + test-plan. `$SCAN_INPUTS` aggregator passed unquoted to `grep -E`. Requires Edit #0 to have loaded `$TRACKER` + `$WORK_ITEM_TYPE` first.

- **Edit #3 (Step 7 dispatch prompt — documentation only):** In the prompt block at lines 503-512, tighten the RETURN CONTRACT prose to say "for type=defect|task, emit `E2E=ambiguous` (smoke is the verification layer; `ambiguous` means N/A for this type — `/CJ_qa-work-item`'s inner E2E subagent only dispatches for user-stories, per qa.md line 179). The orchestrator's Step 7 type-aware halt rule (Edit #1) interprets `ambiguous` correctly for defect/task." Does NOT rewrite `ambiguous` as `green` (preserves qa.md contract). Edit #1 carries the orchestrator-side interpretation.

- **Edit #4 (reference table row):** At the table near lines 811-822, add: `| Skill scripts | skills/*/scripts/* | Any new file under a skill's scripts/ dir — executable code dispatched by skill routing; trust-boundary surface. Includes .sh, .bash, .py, extensionless executables, helpers, READMEs (over-match acceptable for security gate). |`.
