## Symptom

Running `/CJ_goal_investigate --dry-run D000017` against an already-shipped defect (PR merged, RCA file populated with 110 lines of prose) reports:

- `Idempotency state: R=0 F=1 P=0 M=1`
- `Resume row: 5 (anomaly: RCA missing but fix in tree → halt for manual review)`

Expected: `R=1`, `Resume row: 4 (no-op: already shipped, print summary, exit 0)`.

Frequency: deterministic. Reproduces on every dry-run invocation against any
already-shipped defect.

## Reproduction Steps

1. Pick any defect with a populated RCA file and a merged PR (e.g. `D000017_cj_suggest_zsh_crash` — RCA has 110 lines, PR #114 MERGED).
2. Run `/CJ_goal_investigate --dry-run D000017`.
3. **Observe:** `R=0`, dispatched to Row 5 (anomaly) instead of Row 4 (no-op).

**Environment:** `bash 3.2` (macOS default), git worktree, repo at HEAD with D000017 directory present.

## Root Cause

**Root cause:** Two independent edge-case failures in `Step 3` of `skills/CJ_goal_investigate/pipeline.md`, both tracing to the same architectural premise — "happy path covers the common case" — leaving terminal-state and parser-edge cases exposed.

**Bug A — `R` detection awk range is degenerate.** The pattern `/^## Root Cause/,/^## /` is a range expression where the start AND end patterns both match the literal line `## Root Cause`. awk's range semantics include both endpoints, so the range captures exactly the start line. The downstream `sed '1d;$d'` then strips it, leaving empty content. Result: `R=0` regardless of how much prose the `## Root Cause` section contains.

**Bug B — anomaly check fires before terminal-state check.** The resume-row dispatch evaluates `R=0 && F=1` (Row 5: anomaly) before `R=1 && F=1 && M=1` (Row 4: no-op, terminal). A fully-shipped defect with under-detected RCA — exactly the state Bug A produces — falls into the anomaly row instead of the terminal no-op. Even after Bug A is fixed, the dispatch order leaves a hole for any future RCA-detection edge case (e.g. RCA file with no `## Root Cause` heading at all, RCA file with placeholder-only prose, etc.).

**Location:**
- `skills/CJ_goal_investigate/pipeline.md` Step 3, line 118 (Bug A — awk range)
- `skills/CJ_goal_investigate/pipeline.md` Step 3, lines 144–158 (Bug B — dispatch order)

## Fix Description

**Bug A fix — stateful flag instead of awk range** (~8 lines):

```bash
ROOT_CAUSE_BODY=$(awk '
  /^## Root Cause/ { in_rc=1; next }
  in_rc && /^## / { in_rc=0 }
  in_rc { print }
' "$RCA_PATH" | grep -v '^[[:space:]]*$' | grep -v '<!-- TODO' || true)
```

The stateful flag enters the block at `## Root Cause` (skipping the heading line via `next`), exits on the next `## ` heading (any heading), and prints everything in between. No range-pattern degenerate-overlap issue.

**Bug B fix — hoist `M=1` terminal-state check** (~5 lines):

```bash
if [ "$M" = 1 ]; then
  RESUME_ROW=4  # no-op: PR merged (terminal — wins over any other signal)
elif [ "$R" = 0 ] && [ "$F" = 1 ]; then
  RESUME_ROW=5  # anomaly: only fires when NOT merged
elif ...
```

`M=1` is a terminal "this defect is shipped" signal and should always win over any other interpretation. Defense-in-depth: even if `R` detection has future edge-case bugs, the merge-state check prevents false anomaly halts on shipped defects.
