## Symptom

Five format inconsistencies in the work item system:
1. All work items flat under `work-items/` with no type grouping
2. D000001 artifacts lack ID prefix (`TRACKER.md` vs `F000001_TRACKER.md`)
3. F000001 `milestones.md` also lacks ID prefix
4. Defect template Phase 2 gate says "Fix committed with regression test" (too prescriptive)
5. D000001 tracker and test-plan left open after fix shipped in #28

Reproduction frequency: always (structural/convention-level defect).

## Root Cause

**Root cause:** D000001 was the first defect ever scaffolded in this repo. The conventions that evolved organically for features (ID-prefixed filenames, structured nesting) were never codified as rules for all work item types. The defect template was written with assumptions about regression testing that don't apply to all fixes. And the defect lifecycle close-out was never performed because there was no prior practice for it.

**Location:** `rules/work-items.md` (directory structure), `artifact-manifests.json` (placement values), `templates/tracker-defect.md` (gate text)

## Fix Description

1. Close D000001: set tracker to closed, test-plan to complete, mark all gates
2. Update rules to document type subfolders and ID-prefix convention
3. Update artifact-manifests.json placement from "root" to "features"/"defects"
4. Fix defect template gate text: "Fix committed" (remove "with regression test")
5. Migrate all existing work items into type subfolders via git mv
6. Rename all unprefixed artifacts with their item ID
7. Update /docs check placement validation to handle type subfolders
